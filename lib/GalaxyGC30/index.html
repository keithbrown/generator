Welcome to the Galaxy Application Environment 3.0 (R) release. Galaxy allows you to quickly and easily build powerful cross platform applications.

Overview of Galaxy

Galaxy is a development environment with editing tools, interface construction tools, and runtime classes that combine into a comprehensive environment. Galaxy can be used to deploy business solutions ranging from stand-alone departmental applications to sophisticated distributed applications designed to address enterprise-wide problems. Galaxy-built applications look and act like native applications built with C or C++.

Galaxy's principle advantage is that it provides a powerful class hierarchy for application development. Galaxy's class libraries implement a rich user interface environment.

Galaxy provides the following:

  • Cross platform development
    Galaxy is available across multiple platforms. Projects developed on one platform can be easily run on other platforms without making changes to the source code. User Interface items are always drawn according to the look-and-feel of the target platform.

    Source code for Galaxy applications is portable and can be compiled on other platforms supported by Galaxy.

  • Robust business development
    Using the Galaxy API and the Visual Resource BuilderTM you can develop fast, full-featured, scaleable applications. Galaxy provides a rich user-interface toolkit containing items such as lists, textboxes, combo boxes, notebooks, and choosers all of which can be easily modified through the Visual Resource Builder.

  • Extensive API
    The Galaxy API provides access to an extensive set of classes that can be modified according to your needs. It includes full-featured support for lists, text, drawing, printing, and other essential classes for developing sophisticated applications.

  • Rapid user-interface prototyping
    Using the Visual Resource Builder, you can quickly layout and test dialogs, menus, and their contents. Interactive geometry management using Springs and Struts TM connection devices allows you to easily and precisely position user-interface items. This relative positioning model seamlessly handles the display of items across multiple platforms.

  • Persistent objects
    Objects are stored in resource files, allowing for the separation of user-interface specifications from the application code.

  • Comprehensive drawing model
    Galaxy uses an object-based drawing model; the drawing model uses paths, transformations, and enhanced image manipulation, allowing for real time display of graphics.

  • Support for component software
    Galaxy currently provides ActiveX support on Windows platforms.

First Things First

Before using Galaxy 3.0, you should do the following:

  1. Read this document and the Release Notes in their entirety.
  2. Review the Getting Started document for tips on how to get up and running with Galaxy.
  3. Consult the Bugs List for information on known problems with Galaxy.
You may also want to do the following before using Galaxy:

  1. Review the online documentation.
  2. Load and run some of the samples shipped with this release.
  3. Read the tutorial; it contains lessons showing how to build and run applications.
  4. Explore any of the options available from the Help menu on the main Visual Resource Builder menubar.
