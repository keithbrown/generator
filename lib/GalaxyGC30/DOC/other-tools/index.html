Resource Utilities and Command Line Options

When using any of the various utilities that accompany the Galaxy distribution, supplying command line options is often necessary. Methods for supplying command line options differ slightly from platform to platform. On UNIX systems, as well as under Windows, Windows NT, OS/2, and Macintosh, an option is usually supplied with a leading dash (-), so that is how they are presented in this documentation. To supply command line options under the Macintosh operating system, hold down the Control key immediately after launching the application. Under VMS, options are preceded with a slash (/). Options with parameters under VMS must be supplied with an equal sign (=) between the option and the parameter (for example, VRX2VR /VR=RESOURCES.VR). Be aware of these syntax differences as you read this chapter.

On Macintosh systems, launch the vr2vrx, vrx2vr, vrescopy, and vhdrdef applications supplied with the Galaxy distribution in the Galaxy Tools folder. You are prompted with the standard file choosers to select an input file and an output file. If you want to supply options, you must hold down the Control key while launching.

Help Server and Compiler

The Galaxy Help Server and Compiler, along with the .vhs help language has been deprecated. Galaxy 2.x customers should use the Help Authoring Guide from that release. The Help Server and Help Compiler documentation is provided here, but not the language reference. Customers new to Galaxy should not try to use any of the Galaxy 2.x help system, as it will not be supported beyond Galaxy 3.

Native Windows Help

Previous releases of Galaxy contained a form of support for the Microsoft Windows Help System using the Galaxy 2.x Help System and the .vhs language. The Galaxy 2.x Help System has been deprecated, and Visix will not support it after Galaxy 3. Therefore, Visix will not support this method of using the Microsoft Windows Help System in Galaxy applications beyond Galaxy 3.
