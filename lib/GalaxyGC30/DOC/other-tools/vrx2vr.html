vrx2vr



Usage

 Command line: vrx2vr [options] ASCII_VRX_FILE [RESOURCE_FILE]

The utility vrx2vr reads in an ASCII representation of a Galaxy resource file specified by the pathname ASCII_VRX_FILE and writes it to a binary resource file. By default, an entire resource file is transcribed unless the -add, -replace, or -translate options are specified. The ASCII representation may have been manipulated with generic utilities that work on ASCII files or may have been read and edited directly with any text editor. However, it must still be in the Galaxy resource text language.

By default, the output resource file is placed in the current directory with the same name as the source file, except that the extension (if any) is replaced with .vr. However, you may directly specify an output file with the -vr option or by specifying the pathname RESOURCE_FILE on the command line. If a file already exists with the same name as the output resource file and the -force, -add, and -replace flags are not used, the program fails.

The -translate option replaces the strings within all or part of the resource file. The input file should have an ASCII representation of two root level string arrays. By default, the tags of the two arrays need to be "OriginalStrings" and "TranslatedStrings," but you may specify different tags by using the -original and -translated arguments, respectively. If a string in the target resource file (or part of resource file as specified by the -dest argument) matches one of the strings in the original array, then it is replaced with the corresponding string from the translated array. Non-ASCII characters can be specified by octal value (for example, 276) or by hexadecimal value (for example, x03AE) in the ASCII strings.

Options

-add
Specifies that, if the resource file already exists, the resources from the input file should be added into the existing file. Unless the -replace option is also used, resources with the specified tags should not already exist.

-compact
Specifies that the output resource file is compacted if possible. This compaction is not a maximal compression, but a reduction that may sacrifice writing speed for either size or reading speed. Any operation by the utility may be slowed down in the presence of this option.

-dest internalResPath
Specifies a subresource within the resource file to add or replace. Acceptable path strings start with a separator character. This character separates the tags of successively deeper dictionary resources and the tag of the ultimate target resource. The ultimate resource does not need to be a dictionary resource.
Elements within an array resource can be selected by an index specifier delimited by hard brackets, [ ], immediately after the tag of the array.

-force
First removes any file with the same path name as the output file, if one exists. Normally, the program fails if there is already a file with that path name.

-help
Displays a usage guide. No other action is taken, regardless of any other command line arguments. The usage guide is also displayed if no resource file is provided or if the -vrx option (which really belongs to the companion utility vr2vrx) is used.

-original internalResPath
Specifies a resource path for the array of original strings in the input file. This value only makes sense in the presence of the -translate flag. By default, this value is :OriginalStrings.

-replace
Specifies that, if the resource file already exists, the resources from the input file should be swapped into the existing file. Unless the -add option is also used, resources with the specified tags should already exist.

-translate
Specifies that instead of adding or replacing a resource file or a single part of one, the utility should perform a string translation operation. The translation requires two arrays of strings¯one with all of the strings that are to be substituted and the other, of the same size, with the strings that are the substitutes. By default, these two arrays are found in the root of the input file with the tags "OriginalStrings" and "TranslatedStrings," but you may use the - original and -translated arguments to specify different arrays.

An original string must match a string in the target resource file exactly, character for character, for it to be replaced. The order of the original strings does not matter, but the translated strings must correspond to the same order. The companion vr2vrx utility dumps out all of the strings in a resource file appropriately, if you call it with the -dumpstrings flag.

-translated internalResPath
Specifies a resource path for the array of translated strings in the input file. This value only makes sense in the presence of the -translate flag. By default, this value is :TranslatedStrings.

-vr resourcefile
Specifies the output file name and path. No extension is added on top of the provided name, so you must add one, if you want an extension. (By convention, resource files have a .vr extension.)
