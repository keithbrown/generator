vrescopy



Usage

 Command line: vrescopy inputfile outputfile [-new] [-old] [-c varname]

vrescopy converts a resource file from the old resource file format to the new format. It is also used to compact resource files.

vrescopy copies the resources in the resource file inputfile to a newly created file specified by outputfile. inputfile can be either an old format or a new format resource file. Depending on the options specified, outputfile is either a new format resource file (the default), an old format resource file, or a C declaration for an embedded resource file. vrescopy does not preserve resource IDs.

outputfile is generally not bitwise identical to inputfile even if both files are in the same format. outputfile may be smaller, especially if the source file has been edited. This is because most of the empty space in the resource file is compacted and unnecessary resource IDs are removed.

Options

-c varname
Specifies outputfile to be a C declaration for an imbedded resource file. varname is the name of the variable which is declared to contain the resource file data. The newly created file is suitable for use with vresourceOpenInternal.

-new
Specifies outputfile to be created in the new format (default).

-old
Specifies outputfile to be created in the old format.
