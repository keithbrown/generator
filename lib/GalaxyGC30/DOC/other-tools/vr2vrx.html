vr2vrx



Usage

 Command line: vr2vrx [options] RESOURCE_FILE [ASCII_VRX_FILE]

vr2vrx writes out an ASCII representation of the Galaxy resource file specified with the pathname RESOURCE_FILE. By default, the entire resource file is transcribed. With the -dumpstrings and -source options, vr2vrx only transcribes part of a resource file.

The ASCII representation can be used with other utilities that work on ASCII files or can be read and edited directly with any text editor. By default, vr2vrx generates identical output for functionally identical Galaxy resource files. This is because dictionary tags in a resource file are sorted into a canonical order. An ASCII representation of resource files is useful when using a source code control system.

By default, the ASCII output is placed in the current directory. The name of the output file is the same as RESOURCE_FILE, except any extension is replaced with .vrx or .vrs. The .vrs extension is used when the -dumpstrings flag is specified. In either case, you can directly specify an output file and path with the -vrx option. If a file already exists with the same name as the output file and the -force flag is not used, the program fails.

If the pathname ASCII_VRX_FILE is specified, the ASCII output is placed here. No extension is added on top of the provided name¯you must provide any extensions you want. This is an alternate form of the -vrx option.

The -nosort flag can be used to speed up the transcription when there are dictionaries with many tags and no need to have the dictionary elements in any particular order. However, with this option the output of vr2vrx could vary each time it is run, even when using the same resource file.

The -dumpstrings option collects all of the string resources from within all or part of the resource file and writes them to the output files as two identical arrays. One array is tagged "OriginalStrings" and the other is tagged "TranslatedStrings." By default, the strings are sorted into a canonical order and duplicates are removed. The -nounique flag leaves the duplicates, and the -nosort flag, which implies the -nounique flag, leaves the strings unsorted.

Options

vr2vrx recognizes the following options (user-supplied variables are indicated in italics).

-addindent spacecount
Specifies the number of additional spaces with which to start each begin line in the output file based on the depth of the resource. Unless -notabs is specified, every 8 indent spaces is replaced with a tab. However, opaque data and wrapped strings always start at the extreme left.
By default, spacecount is 8.

-dumpstrings
Specifies that instead of transcribing the entire resource file or part of it, the program dumps out two identical arrays with all of the string resources. The arrays are sorted and uniqued unless the -nosort and/or -nounique flags are used.
These arrays are intended to be used to help translate resource files from one natural language to another. The TranslatedStrings array can be modified in place by any text editor, and the modified file can be fed into the vrx2vr utility with the -translate option and the original resource file. As long as the original resource file and the OriginalStrings array have not been changed, all of the string resources in the resource file (or part of the resource file) are translated.

-firstindent spacecount
Specifies the number of spaces with which to start each begin line in the output file. However, opaque data and wrapped strings always start at the extreme left.
By default, spacecount is zero.

-force
First removes any file with the same path name as the output file, if one exists. Normally, the program fails if a file with that path name already exists.

-help
Displays a usage guide. No other action is taken, regardless of any other command line arguments. The usage guide is also displayed if no resource file is provided or if the -vr option (which really belongs to the companion utility vrx2vr) is used.

-hexopaque
Specifies that a hexadecimal encoding should be used for opaque resources rather than the default uuencoding. The hexadecimal encoding represents 36 bytes per line versus 45 bytes per line for the uuencoding.

-nosort
If the -dumpstrings option is used, this flag turns off the sorting and uniquing of the string resources. If the -dumpstrings option is not used, this flag turns off the sorting of dictionary tags, which for some resource files can speed up the transcription. Note that dictionary tags are sorted first if -nosort and - dumpstrings are provided. This guarantees the order of the strings (whether or not that order is actually meaningful).

-notabs
Specifies that tabs are not to be used in the indents. By default, every 8 indent spaces is replaced with a tab.

-nounique
If the -dumpstrings option is used, this flag turns off the uniquing of the string resources. This means that a particular string is dumped as many times as it appears in the resource file (or part of the resource file).

-shortstring length
Specifies the maximum length of a string (not including quotes) on the indented begin line. Strings longer than this length start at the left of a fresh line and wrap as needed.
By default, length is 36.

-source internalrespath
Specifies a subresource within the resource file to write out. Acceptable path strings start with a separator character, and that character separates the tags of successively deeper dictionary resources and the tag of the ultimate target resource. The ultimate resource does not need to be a dictionary resource.
Elements within an array resource can be selected by an index specifier delimited by hard brackets ([]) immediately after the tag of the array. If all index specifiers in a resource path are single element specifiers, then the ultimate resource is not written as an array. If more than one index specifier is a multiple element specifier, then one combined array is written. The following are examples of valid index specifiers:
    []all elements, in order
    [12] element 12 (the 13th)
    [0...] all elements, in order
    [2...] all elements except the first two, in order
    [.] the last element
    [.1] the second to last element
    [...0] all elements, in reverse order
    [0,12,9] elements 0, 12, and 9, in that order
    [1..5] elements 1 through 5, inclusive and in order
    [5..1] elements 1 through 5, inclusive and in reverse order
    [0.2..] all even elements, in order
    [..2.0] all even elements, in reverse order
    [..2..] all even or odd elements depending on whether the array's size is, respectively, odd or even, in reverse order
    [0,0,9.2.2,0]element 0 twice, elements 9, 7, 5, 3, and element 0 again

-vrx asciivrxfile
Specifies the output file name and path. No extension is added on top of the provided name¯you must provide any extensions you want. By convention, transcriptions of full or partial resource files have a .vrx extension and dumps of resource strings have a .vrs extension. However, both files are in ASCII and use the same language.

-wraplength length
Specifies the maximum length of a string (not including quotes) on a single line. Strings longer than this length wrap. Wraps that occur on spaces (which is attempted first) replace the space with a newline (real newlines are represented as n) unless the space is immediately preceded by some other white space character. Wraps that occur after white space other than a single space character just add a newline after the white space. Wraps that occur within a word add an escaped newline.
By default, length is 72.

Examples

    vr2vrx abc.vr
vr2vrx abc.vr -force
vr2vrx abc.vr -dumpstrings
vr2vrx abc.vr xyz.vrx
vr2vrx abc.vr -source ":Dialog:ItemList:Objects[0]:Rect"
vr2vrx abc.vr -source ":Dialog:ItemList:Objects[]:Rect[0]"
