Menus provide a way for users to issue commands in an application. Each command in a menu is represented on the menu as a menu element. Menu elements can be any of the following:
  • MenuElement
  • SeparatorMenuElement
  • SubmenuMenuElement
  • ToggleMenuElement

Menus are usually built upon a menu bar. However, a pop-up menu can be attached to any dialog item (using its Menu property). A pop-up menu is opened with the menu button of the mouse. Additionally, an option menu is a specially defined pop-up menu that provides a convenient way to place a menu within a dialog.

The Title property of a menu specifies the name of a menu. On some platforms, menus can be "torn off" the application, and remain open (pinned) on the screen. Use the Pinnable property to specify this behavior.

Menu Editor

Use the menu editor to add menu elements to a menu. There are various ways to invoke the menu editor, depending on the type of menu you are editing.
  • Menu
    Drag a Menu from the Component Toolbar to the Project Navigator. Double-click on the menu in the Project Navigator to open the Menu Editor.

  • Menu bar
    In the Menu Bar Editor, double-click on a menu in the list of menus to invoke the Menu Editor.

  • Pop-up menu
    In the Property page for a dialog item, select the Menu property, and specify Custom Menu. Then click on the ellipsis button to open the Menu Editor.

  • Option menu item
    An Option menu (OptionItem) is a dialog item with its Menu property set to Custom Menu. Select the Menu property, and then click on the ellipsis button to open the Menu Editor.

Mnemonic Nothing Defines a mnemonic key for selecting the menu from the keyboard. This is a one character field. The character entered must match a character in the title name. All mnemonic keys defined for menus in a set of menus must be unique.
