Combo chapter
vcombo Prefix
A combo (ComboItem) is a dialog item subclass that enables a user to pick a selection from a list of choices. A combo is also available as a PopdownComboItem.

A combo has three parts:

  • List of values
    The available choices.

  • Choice box
    Displays the list of values.

  • Entry box
    Receives user input.

Combo Behavior

A combo has specific behavior for each of the following situations:
  • Accept
  • Select
  • Scroll
The behavior of a combo affects how the combo maps user input (typing in the entry box or selecting from the choice box) to values from its list of values. Note that the definition of these behaviors may vary according to look-and-feel.

Accept Behavior

An accept situation is when the combo makes the selected (or entered) value the current value. This occurs when the user presses Return or Enter, or when the focus leaves the combo.

Select Behavior

A select situation is when the combo tries to map user input to a combo value. This occurs when the user selects a value from the choice box or enters text into the entry box. Every keystroke in the entry box (with the exception of Return and Enter) triggers a select situation.

Note that the entry box is not affected if it is blocked (blockEntrybox). Otherwise it is set to the value selected in the choice box, or the empty value if there is no choice box value selected.

Scroll Behavior

A scroll situation is when the list in the choice box is scrolled. As the user types in the entry box, for every keystroke (except Return and Enter), a scroll situation is triggered as the combo tries to match a choice box value to the entry box value. The scroll behavior does not affect the entry box in the default class.

For more information on combos, refer to the following:

  • Combo chapter in the Programmer's Guide.
  • vcombo class in the Reference Manual.

Key Properties

Property Defaults Description
Accept Behavior Default Possible Values

  • Default - LAF defined
  • Do Nothing
  • Match or Add
  • Match or Closest
  • Match or First
  • Match or Nothing
  • Match or Partial
  • Match or Revert

Default on Windows and CUA is Match or Partial. Default on Macintosh and Unix is Match or Closest.

You can override Do Nothing to implement your own behavior.

Auto-sort False Automatically sorts the list of values.
Modifiable True Allows new selections from the user to be added to the value list.
Scroll Behavior Default Possible Values

  • Default - LAF defined
  • Do Nothing
  • Match or Add
  • Match or Closest
  • Match or First
  • Match or Nothing
  • Match or Partial
  • Match or Revert

Default on Windows, Macintosh, and CUA is Match or Partial. Default on Unix is Match or Closest.

You can override Do Nothing to implement your own behavior.

Select Behavior Default Possible Values

  • Default - LAF defined
  • Do Nothing
  • Match or Add
  • Match or Closest
  • Match or First
  • Match or Nothing
  • Match or Partial
  • Match or Revert

Default on Windows and CUA is Match or Partial. Default on Macintosh and Unix is Match or Closest.

You can override Do Nothing to implement your own behavior.

Tag ComboItem Tag name of the combo, for referencing in application code.
Title Nothing The title of the combo, often not displayed.
Visible Value 5 The number of entries in the list of values that are visible.
Wrap False Indicates whether scrolling through the list of values wraps from the end of the list back to the beginning.
