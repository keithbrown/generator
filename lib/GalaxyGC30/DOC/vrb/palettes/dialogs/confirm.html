A confirm dialog is a dialog subclass that enables a user to confirm (or reject) the settings of properties before an application continues to execute. Various confirmation buttons define the available actions for the user in a confirm dialog. The action, behavior, and existence of these buttons vary, depending on look-and-feel.

Use the Confirm Dialog Editor to build confirm dialogs for your projects.

For more information on confirm dialogs, refer to the following:

  • Confirm chapter in the Programmer's Guide.
  • ConfirmDialog class in the Reference Manual.

ConfirmDialog Subclasses

All of the choosers are subclasses of confirm dialogs. These include, among other subclasses, the following:

  • ColorChooser
  • FileChooser
  • FontChooser
  • ImageEditor
  • TextStyleChooser

Key Properties

Property Defaults Description
Apply False Determines if the confirm dialog has an Apply button.
Buttons Vertical False Specifies whether the confirm dialog's buttons should be placed vertically or horizontally.
Defaults False Determines if the confirm dialog has a Defaults button.
Help False Determines if the confirm dialog has a Help button.
Reset False Determines if the confirm dialog has a Reset button.
Tag ConfirmDialog Tag name of the confirm dialog, for referencing in application code.
Title Confirm Dialog The title of the confirm dialog. On most platforms, the title is displayed in the title bar of the confirm dialog.
Consult the Dialog Key Properties table for additional properties that are inherited by the confirm dialog.
