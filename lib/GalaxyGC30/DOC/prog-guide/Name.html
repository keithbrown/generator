Name



The Name class creates unique, atomic identifiers for arbitrary character strings. A name is a pointer to a character string that uniquely identifies that character string in the context of a given name table. A name table is a collection of zero or more unique names. You can pass a character string to a name table to retrieve the unique name for that character string.

Given two names in the same name table, you can compare their pointer values to determine equality without comparing the characters of the names themselves. The fact that names can be compared quickly makes them particularly well suited as keys for arbitrary values. Names are often used as dictionary keys (refer to the Dictionary chapter) and resource tags (refer to the Resource chapter).

A name is represented as a pointer to its character string contents so its printable representation can be easily recovered. Note that you can pass a const vname* in place of a const vchar*, but not vice-versa. The type vname is compatible with vchar in the same way that vstr is compatible with vchar. Under no circumstances can a const vname* be passed in place of a (non-const) vchar* or vstr*. Refer to the String chapter for more information about characters and strings.

Virtually all Galaxy modules with functions that have name arguments expect global names. The global name table is provided as a convenience by the Name class for applications that do not wish to manage their own names. You will only want to manage your own names if you create a large number of names that can be destroyed once your application is finished with them (a language compiler is an example of this kind of application). Note that you cannot pass a name in a local name table to a function that expects a name in the global name table. Refer to the documentation for the module you are using to determine whether it expects global names or not.

Galaxy global names are modeled on name objects in the PostScript language. Galaxy global names also correspond to atoms in the Lisp language and the class Symbol in the Smalltalk language.

Key Functionality

Use InternGlobalLiteral to obtain a global name for a literal character string. Use InternGlobal to obtain a global name for a character string in the vchar character set. Use InternGlobalScribed to obtain names for strings in other character sets. The following code fragment illustrates the differences between these functions:

       vwindow     *window;                /* contains a window */
   const vname *name1, *name2, *name3; /* will put names here */

   name1 = vnameInternGlobalLiteral("Spam");
   name2 = vnameInternGlobal(vwindowGetTitle(window));
   name3 = vnameInternGlobalScribed(vnumScribeInt(5));
Pass vnameHash to vdictCreate or vdictInit when you wish to create a dictionary with names as its keys. Most Galaxy functions with dictionary arguments expect dictionaries of this type. For example, the following code stores a window pointer in a dictionary:

       vdict   *dict;
   vwindow *window; /* contains a window */

   dict = vdictCreate(vnameHash);

   vdictStore(dict, vnameInternGlobalLiteral("Spam"), window);
