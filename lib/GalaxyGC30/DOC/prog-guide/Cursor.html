Cursor



The Cursor class provides facilities for handling cursors in a device-independent fashion. A cursor is a subclass of an image and consists primarily of an image, a mask, and a hot-spot. The cursor's hot-spot is the point of the image that actually triggers an activity when, for instance, the user clicks a mouse button.

Typically, one designs a cursor with the Visual Resource Builder and store it in the application's resource file with other resources, such as dialogs and icons. Then the cursor is loaded from a resource file and associated with an on-screen element like a window or dialog item. The code below illustrates how this is done.

    void SetDialogCursor (vresource rRoot)
{
   // Locating a dialog and cursor located on the
   // root level of the resource (rRoot).
   vdialog *dialog = vdialogLoad
     (vresourceGet(rRoot, vnameInternGlobalLiteral("Dialog")));
   vcursor *cursor = vcursorLoad
     (vresourceGet(rRoot, vnameInternGlobalLiteral("My Cursor")));

   // Assign the cursor to the dialog's window
   vwindowSetCursor(vdialogGetWindow(dialog), cursor);
}

Relationship to Other Classes

 While cursor methods manipulate and store cursors, using them with windows and dialog items is handled at a higher level by the Window and Dialog classes, respectively.

For information on how to manipulate the image or mask underlying the cursor, see the Image chapter.

Blocking Cursors

A common action is to change an application's cursor to a "blocking" or wait cursor while the application is performing a long operation. This can be accomplished easily by using the Block and Unblock methods in the Window class. Call the Block method to make a window or dialog use the system's default wait cursor. When the operation has completed, call the Unblock method to restore the previous cursor to the window or dialog.
