Dictionary



The Dictionary class maintains associations between arbitrary pairs of objects. A pointer or scalar value can be stored in a dictionary as the value of another pointer, called its key. The value can be quickly recovered from the dictionary through the key. Each distinct key must have a distinct address, because the values of the pointers are compared to determine equality. The Name class is useful for obtaining key pointers to character strings. Each dictionary has a hash function, that is used to distribute its keys into a hash table. Dictionaries are commonly used to bind names to various kinds of objects such as commands, classes, and generic attributes.

Dictionary key pointers are typically obtained from the Name class methods vnameInternGlobal and vnameIntern. These functions return pointers to name objects for character strings. A name in this sense is a pointer to a character string that is unique based on the contents of the string. The Name class returns the same string pointer for each string with the same contents¯name pointers can thus be compared as if they were atomic values. You do not need to compare the characters of a name with the characters of another name to determine equality.

Every name pointer is unique within the context of a single name table; you will typically use the global name table to store your names. The global name table is provided as a convenience by the Name class for applications that do not wish to manage their own names. You will want to manage your own names only if you create a large number of names that can be destroyed once your application is finished with them (a language compiler is an example of this kind of application). Virtually all Galaxy modules with functions that have name arguments expect global names. Refer to the Name chapter for more detailed information about names.

Galaxy dictionaries are modeled on dictionary objects in the PostScript language, though different verbs are used by the Galaxy Dictionary class. The Galaxy dictionary type also corresponds to the IdentityDictionary class in the Smalltalk language and the finite function type included in many functional programming languages.

Key Functionality

Note that the Dictionary class uses the verbs Load and Store to read and write dictionary keys and values, not to read and write dictionary resources. This usage is inconsistent with the use of Load and Store in other parts of the Galaxy API.

Sample Code

Complete source code demonstrating some of the concepts presented in this chapter is available in your Galaxy distribution in the samples/docsamps directory.

The program listing in dictionary contains sample code to help you get started using the Dictionary class in an application. Rather than exploiting all of the features of the Dictionary class, this sample demonstrates the use of the common routines you will most likely need to use the Dictionary class.

The sample program allows a user to store key/value string pairs in a dictionary and query its contents. It uses the Name class's global name table to obtain unique pointers to character strings. This program uses the C stdio libraries since its purpose is to demonstrate use of the Dictionary and Name classes, not user interface components. The command-line syntax is <key>=<value> to store and <key> to load. The contents of the dictionary can be dumped with "?". The program exits on "!".
