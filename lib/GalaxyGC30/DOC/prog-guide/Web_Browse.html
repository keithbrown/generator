Web Browse



To facilitate access to HTML browsers, Galaxy 3.0 introduces a new class called a browser (vbrowser). The base class of browser handles basic functionality, such as causing a browser to display the information at a given URL or to raise its window (or to raise itself each time it is told to change URL's).

A browser is usually used in conjunction with the Galaxy Help system. Refer to the Help chapter for more information.

Subclasses of vbrowser include:

    vbrowserNetscape controls a Netscape Navigator

    vbrowserExplorer controls an Internet Explorer (Windows only)

    vbrowserSystem controls the system default browser (either Netscape or Explorer)

Browser Process

 The browser class also provides support for spawning a new external browser if one is not already in existence. This is done via the browser process (vbrowserProcess) object. Subclasses of vbrowserprocess perform browser specific operations to

  1. test to see if a given type of browser is already running and
  2. start the browser if necessary.
The browser class may also be used to control browsers internal to a given application.

Browser Commands

The browser encapsulates changes in the browser state via browser commands (vbrowserCommand). A browser command can contain various types of information but currently only holds a type enumerator and a URL.

A browser handle (vbrowserHandle) is mostly an internal utility but it is exposed so that developers can use it in their own subclasses of vbrowser. A browser handle is used strictly to convey information from the vbrowserProcess back to the vbrowser. Often, process creation results in a handle to the process. In that case, vbrowserProcess creates a vbrowserHandle to which the vbrowser can hold a reference.
