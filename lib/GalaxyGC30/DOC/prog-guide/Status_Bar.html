Status Bar



A statusbar (vstatusbar) is a subclass of container designed to display concise and up-to-date status information. Status bars are typically located along the bottom edge of application dialogs. Usually they display short informational messages pertaining to the application state, or information about the dialog item over which the pointer is currently located.

Status bars can be used in conjunction with status help to display help messages about the items located under the pointer. When motion events are enabled, help requests are generated bearing the name vname_Status. The application will search for a help object by that name to handle this help request. Refer to the Help chapter for more information on these topics.

The default status help object, activated by vhelpStartupStatus, uses vhelptextlocator and vhelpstatus to display help text in a statusbar. By default, status bars are receptive to help text from the default status help object in the global help registry. Use SetObserveStatusHelp to disable this feature.

Key Functionality

Status bars have a default item. The default item should bear the tag vstatusbarDefaultItem. This item is responsible for displaying the default text messages. By default, this item is of class vstatusbarItem. To change the class of this item, set the class variable vstatusbarDEFAULT_ITEM_CLASS to the appropriate class. Note that this must be done prior to the instantiation of a statusbar.

Applications can control what messages the status bar displays by SetDefaultItemText and SetStatusbarText. SetDefaultItemText, or SetDefaultItemTextScribed, is used to set the text of the default text display item in a status bar. SetStatusbarText (or SetStatusbarTextScribed) is meant to be called as a static function (without necessarily instantiating a vstatusbar). It is used to set the default text display item of a status bar located above a given dialog item in the UI hierarchy.
