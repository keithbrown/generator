Mainview



A mainview, a special subclass of viconview, implements the document interfaces peculiar to certain window systems. Under the Microsoft Windows look-and-feel, the document interface is the Multiple Document Interface (MDI). When running with the Windows look-and-feel, a main view will provide native Windows MDI functionality. Currently, this functionality is only supported when running the Windows look-and-feel under Windows itself. No other combination of platform and look-and-feel supports native main view functionality.

An application uses this subclass if it wants normal icon view functionality under X, OS2/PM, and Macintosh, and native functionality under Windows.

Key Functionality

There are a couple of restrictions on using main views. The dialog associated with a main view must have vwindowSTYLE_MAIN set. For example:

       vwindowAddStyle(vdialogGetWindow(myDlg), vwindowSTYLE_MAIN);
Also, there can only be one main view attached to any dialog. This is a fundamental limitation of Microsoft Windows MDI.

Preferences

The preference mdi is turned on when native functionality is desired and turned off when normal icon view functionality is desired. By default this preference is On when running the Window look-and-feel under Windows, and by default Off in all other cases. To turn off native behavior, the preference nomdi should be turned On.
