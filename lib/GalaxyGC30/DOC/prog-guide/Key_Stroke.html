Key Stroke



A key stroke (KeyStroke) is composed of an integer value representing a key and an input modifier. An input modifier is a set of flags that indicate what mouse buttons or modifier keys (such as Shift or Control) were held down during the keystroke.

Determining Keys and Their Modifiers

If you want to know what key was pressed, use GetStrokeKey, which returns an integer value representing the key. Similarly, if you want to know what modifiers were pressed, you can use GetStrokeModifiers. Example 78 illustrates this concept.

Example 78. Examining keys and modifiers 

    vbool IsKeyACtrlReturn (vkeyStroke stroke)
{
   if (vkeyGetStrokeKey(stroke) != vkeyRETURN)
     return vFALSE;
   if (vkeyGetStrokeModifiers(stroke) != vkeyMODIFIER_KEY_CTRL)
     return vFALSE;
   return vTRUE;
}
Note that this example does not return vTRUE if additional modifier keys (such as the Shift key modifier) is pressed in addition to the Control key modifier and Return key. Refer to the Galaxy Application Environment Reference Manual for a complete list of possible key constants and key modifier constants.

Note that not all key values can be generated on any given keyboard.

On some platforms, the Control key modifier is bound by default to the Command key modifier. This is dependent on the commandkey value in the preference dictionary, and is done for reasons of backward compatibility. To make certain that the Control key only generates a Control key modifier, set the commandkey preference back to "command." Note that many keyboards do not have an actual Command key, and hence some functionality may be lost. Refer to the Preference class for details on what preferences can be set for keys and key modifiers.
