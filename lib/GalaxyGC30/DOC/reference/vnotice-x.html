Types [3] Functions [78] Messages [10] Constants [1] Macros [1]

Types

vnotice

        typedef ... vnotice;
Represents a notice, a modal confirmation dialog with standard buttons and a text item. Notices provide a means for forcing the user to respond immediately to a situation.

vnoticeClass

        typedef ... vnoticeClass;
Describes the behavior of a class of notices. The notice class is referenced by a pointer to this type. The default notice class is a subclass of the default confirmation class.

vnoticeHookProc

        typedef ... vnoticeHookProc;
Denotes a hook function that is called when a designated notice handles an event of importance. A hook function, unlike a notification function, returns a value that may alter the behavior of the notice. notice is the notice handling the event, and event is the event being handled. Notices use pointers of this type to represent hook functions that are called when a notice's button has been pressed.

vnoticePlaceWindow

        void vnoticePlaceWindow(
                vnotice * notice,
                vwindow * base,
                int modeX,
                int modeY);
Positions the notice notice relative to the window base. base may be the result of vwindowGetRoot, in which case the bounds of the display are used. modex and modey specify the relation of window to base on each coordinate axis; appropriate flags are defined by the Plane Geometry class function vrectPlace. The new position and size of the window is pinned to the dimensions of the display. This function, unlike vwindowMove and vwindowSetRect, may take the window's frame into account to obtain correct relative positioning. This function does not affect the size of the window unless it is larger than the display.

This function is equivalent to vwindowPlace(vdialogGetWindow(dialog), base, modeX, modeY).

vnoticeProcess

        const vname * vnoticeProcess(
                vnotice * notice);
Opens a blocking notice notice and waits until a confirmation button is activated, and then closes the notice. This single call is equivalent to calling:
  • vnoticeProcessStart
  • vnoticeProcessGet
  • vnoticeProcessStop

This routine is useful when the application does not want to use the standard notice hook functions and simply get the tag of the confirmation button activated. vnoticeProcess could be used as follows: 


/*
 * assume notice is created and 
 * initialized. 
 * 
 * open notice and wait for button 
 */

tag = vnoticeProcess(notice);

/* 
 * flow will not return here until 
 * a confirmation button is pressed. 
 */ 

if (tag == vname_Ok) /* do OK code */ 
else if (tag == vname_No) /* do No code */ 
else if (tag == vname_Cancel)  /* do Cancel code */

It is important to note that when vnoticeProcess is used the hook function will not be called. The underlying dialog's close hook will also not be called when the dialog closes (since the close always happens on a button press and the confirm will be closed when control returns). Valid tags consist of:

Warning Notice:

vname_Ok
for ok button
vname_Cancel
for cancel button
vname_Help
for help button

Caution Notice:

vname_Ok
for yes button
vname_No
for no button
vname_Cancel
for cancel button
vname_Help
for help button

vnoticeProcessGet

        const vname * vnoticeProcessGet(

Returns the tag of a confirmation button pressed within a blocking notice notice. The notice must have been opened using vnoticeProcessStart and subsequently closed with vnoticeProcessStop. All calls to vnoticeProcessGet must be enclosed by calls to vnoticeProcessStart and vnoticeProcessStop.

vnoticeProcessStart

        void vnoticeProcessStart(
                vnotice * notice);
Opens a blocking notice notice and returns. A call must be made to vnoticeProcessGet which will block until a confirmation button is activated. To close this notice a call to vnoticeProcessStop must be made. Each vnoticeProcessStart must have a matching vnoticeProcessStop.

vnoticeProcessStop

        void vnoticeProcessStop(
                vnotice * notice);
Closes a blocking notice notice opened with vnoticeProcessStart. Each vnoticeProcessStop call must have a matching vnoticeProcessStart.

vnoticeSetIcon

        void vnoticeSetIcon(
                vnotice * notice,
                vimage * icon);
Controls whether the notice notice should have an icon. Some applications may wish to decorate notices with icons. The icon icon is shown in the notice in a position adhering to look-and-feel specification. By default, the Motif, Windows, and CUA look-and-feel have icons as specified by their respective style guides. This function will override that icon and add an icon in the OPEN LOOK look-and-feel. NULL can be passed for the icon parameter which will reset the notices' icon to its default.

This message is sent when a Cancel button is to be added or deleted from a notice. If flag is TRUE the button is added, if it is FALSE, the button is removed.

By default this method uses the underlying Confirmation class' Cancel button and forces it on for all look-and-feels.

vnoticeSET_CAUTION

        <vnoticeSET_CAUTION>
Controls whether the notice notice is a warning notice or caution notice. If flag is TRUE, it is a caution notice, otherwise it is a warning notice.

The default method uses the underlying Confirmation class OK button for both the OK button of a warning notice and the Yes button of a caution notice. Titles are set to ``OK'' or ``Yes'' depending on flag. Caution notices use a generic confirmation button with a tag vname_No for the No button. This button is removed or does not exist for warning notices.

vnoticeSET_ICON

        <vnoticeSET_ICON>
This message sets the icon for notice notice. Using NULL will reset the icon to its default if one exists for the current look and feel. The default method simply sets the icon. If the icon is NULL the default icons are retrieved from the system resource file.
