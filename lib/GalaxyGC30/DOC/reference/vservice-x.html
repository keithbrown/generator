Types

vservice

        typedef ... vservice;
Denotes a service that an application is providing to other applications. The service contains primitives and attributes. This type is specific to service providers. Consumers do not talk to the vservice directly, they talk to a vsession. An active session has a vsession in both the service consumer and the service provider. The vservice is used by the service provider to generate a vsession with each service consumer, and to contain any centralized information for the service, such as its primitives. The session is used for communication; the service is used for storing primitives and generating new sessions.

Services are created with vserviceCreate or initialized with vserviceInit. All services should be destroyed with vserviceDestroy when they are no longer used by the application. An application can destroy a service at any time, but typical applications do not destroy their services until they are ready to exit.

Services are enabled with vserviceEnable and disabled with vserviceDisable. Enabling a service allows the Service Manager to generate sessions with new service consumers. Disabling a service does not affect any existing sessions, but prevents the Service class from generating new sessions.

The Service class creates a vsession for each session that is generated by the service provider. These sessions are initialized from the template session returned by vserviceGetTemplateSession. The application providing the service can set the vsessionNoteProc so it can initialize or monitor each vsession instance that is created.

vservicePrimitive

        typedef ... vservicePrimitive;
Denotes a function entry point within a service. A primitive is created with vserviceCreatePrimitive or initialized with vserviceInitPrimitive. Each primitive is added to a service with vserviceAddPrimitiveOwned. Primitives are destroyed with vserviceDestroyPrimitive, but since ownership is typically passed back to the Service class (by adding the primitive to a service), applications usually do not destroy primitives directly.

Primitives are assigned a signature with vserviceSetPrimitiveSignature, and a handler function with vserviceSetPrimitiveHandlerProc.

vservicePrimitiveClass

        typedef ... vservicePrimitiveClass;
Contains method and variable bindings for objects of type vsessionPrimitive. Classes declared with this type contain the same set of methods and variables as the default primitive class, but the behavior of methods and the contents of variables can vary. This class type is derived from vloadableClass.

vservicePrimitiveHandlerProc

        typedef ... vservicePrimitiveHandlerProc;
Denotes the function that is called when a primitive is invoked. A function of this type is assigned to a primitive with vserviceSetPrimitiveHandlerProc. primitive is the primitive being invoked. session is the session invoking the primitive. scrapArgs are the arguments to the primitive, as specified by the primitive's signature. The actual argument values are retrieved by passing scrapArgs to vserviceGetPrimitiveArgs.

The handler should return a scrap, created with vserviceMakePrimitiveReturnScrap. This scrap contains the return result as specified by the primitive's signature. If the primitive does not specify a return result, then the handler should return NULL.

vservicePrimitiveIterator

        typedef ... vservicePrimitiveIterator;
Denotes an iterator that sequences over the primitives that have been added to a service. Instances of this type are typically automatic variables. These iterators are initialized with vserviceInitPrimitiveIterator and destroyed with vserviceDestroyPrimitiveIterator.

vserviceSessionIterator

        typedef ... vserviceSessionIterator;
Denotes an iterator that sequences over the sessions using a service. Instances of this type are typically automatic variables. These iterators are initialized with vserviceInitSessionIterator and destroyed with vserviceDestroySessionIterator. Sessions are added to a service by setting the session service with vsessionSetService. Sessions generated by the Service class on behalf of a vservice are automatically added.

Functions

vserviceAddObject

        void vserviceAddObject(
                vservice * service,
                const vname * name,
                vobject * object);
Gives the service service access to the object object, using the name name. This function is only intended for service providers that are using the default handler for the primitive returned by vserviceCreateIssueCommandPrimitive, however the application can create other primitives that make use of this facility. Objects that are private to a particular session can be added to the session itself with vsessionAddObject.

Ownership of the object is still retained by the application. The object should be removed with vserviceRemoveObject before the object is destroyed.

vserviceAddPrimitiveOwned

        void vserviceAddPrimitiveOwned(
                vservice * service,
                vservicePrimitive * primitive);
Adds the primitive primitive to the service service making the primitive accessible by all sessions that are using the service. The primitive is destroyed when the service is destroyed. Removing the primitive with vserviceRemovePrimitive passes ownership back to the application.

The primitive can be retrieved later using vserviceFindPrimitive. The tag for the primitive is the tag for the signature that the primitive uses.

Ownership of the primitive is passed to the Service class. If an application is providing multiple services that want to use the same primitive, the vservicePrimitive should be cloned by calling vserviceClonePrimitive.

vserviceAddSignatureGroup

        void vserviceAddSignatureGroup(
                vservice * service,
                vsignatureGroup * group);
Iterates through the vsignatureGroup. For each vsignature a new primitive is created and added to the vservice. The primitives created will be destroyed when the service is destroyed.

The vsignatureGroup will not be destroyed when the service is destroyed. The application must destroy the vsignatureGroup. The primitives created can be removed from the vservice with vserviceRemoveSignatureGroup.

vserviceClone

        vservice * vserviceClone(
                vservice * service);
Creates a copy of the service service and returns a pointer to it. The copy is initialized with clones of the attributes and primitives that were contained within service. The copy is not registered or enabled, nor does it have any sessions. Any attributes in the template session are also cloned into the new service's template session. Any added attributes are cloned; any added object references are copied.

The caller is responsible for eventually calling vserviceDestroy to reclaim memory when the clone is no longer needed.

vserviceClonePrimitive

        vservicePrimitive * vserviceClonePrimitive(
                vservicePrimitive * primitive);
Creates a copy of the primitive primitive and returns a pointer to it. The copy is not be added to a service. This function essentially allocates a new primitive and calls vserviceCopyInitPrimitive. The caller is responsible for eventually calling vserviceDestroyPrimitive to reclaim memory when the clone is no longer needed.

vserviceCopyInitPrimitive

        void vserviceCopyInitPrimitive(
                vservicePrimitive * primitiveSource,
                vservicePrimitive * primitiveDest);
Initializes the primitive primitiveDest with a copy of primitive primitiveSource. The primitiveprimitiveDest is not added to any services. The caller is responsible for eventually calling vserviceDestroyPrimitive to reclaim memory when primitiveDest is no longer needed. You should never need to call this function directly.

vserviceCreate

        vservice * vserviceCreate();
Allocates and initializes a new instance of a service and returns a pointer to it. The service has only the default attributes and default primitives, and is not enabled or registered. The caller is responsible for eventually calling vserviceDestroy to reclaim memory when the service is no longer needed.

vserviceCreateIssueCommandPrimitive

        vservicePrimitive * vserviceCreateIssueCommandPrimitive();
Creates a primitive that implements the default issue command signature returned by vsessionGetIssueCommandSignature and returns a pointer to it. The result can be passed to vserviceAddPrimitiveOwned to provide the default Command class service. Service consumers can use the statement returned by vsessionMakeIssueCommandStatement to invoke this primitive.

The default handler function uses the objects that were added to the service and session with vserviceAddObject and vsessionAddObject when issuing a command. The application can replace this handler function by calling vserviceSetPrimitiveHandler, or a generic primitive can be created and assigned the signature returned by vsessionGetIssueCommandSignature along with the primitive handler.

If the default handler is used, the application can add the application object to the service, vapplicationGetCurrent using the name vnameNULL as a convention. When using a custom implemented primitive handler, this convention can still be followed, if appropriate.

Ownership of the resulting primitive is passed to the application, and should be treated like any other primitive it might create.

Creates a new service of class clas and returns a pointer to the new session. Memory is allocated by the Service class. The new service is created with a set of default attributes, determined by the class.

Service consumers can use the statement returned by vsessionMakePingStatement to invoke this primitive.

Ownership of the resulting primitive is passed to the application, and should be treated like any other primitive that it might create.

vserviceCreatePrimitive

        vservicePrimitive * vserviceCreatePrimitive();
Allocates and initializes a new instance of a primitive and returns a pointer to it. The signature must be added with vserviceSetPrimitiveSignature, and the handler must be set with vserviceSetPrimitiveHandler before the primitive can be added to a service. The caller is responsible for eventually calling vserviceDestroyPrimitive to reclaim memory when the primitive is no longer needed. If the primitive is added to a service with vserviceAddPrimitiveOwned, it is destroyed when the service is destroyed.

        vservicePrimitive * vserviceCreatePrimitiveOfClass(
                vservicePrimitiveClass * clas);
Creates a new primitive of class clas and returns a pointer to the new primitive. Memory is allocated by the Primitive class. The new session is created with a set of default attributes, determined by the class.

The result can be passed to vserviceAddPrimitiveOwned to provide the default shutdown in a service. Service consumers can use the statement returned by vsessionMakeShutdownStatement to invoke this primitive.

vserviceDestroy

        void vserviceDestroy(
                vservice * service);
Reclaims all memory used by the service service and invalidates the pointer. If the service is enabled, it should be disabled with vserviceDisable before calling vserviceDestroy. If the service is registered it must be unregistered before disabling it. Any remaining sessions that had set their service to service with vsessionSetService no longer references any service.

vserviceDestroyPrimitive

        void vserviceDestroyPrimitive(
                vservicePrimitive * primitive);
Reclaims all memory used by the primitive primitive and invalidates the pointer. This function is not normally called directly by applications when ownership is passed to the Service class with vserviceAddPrimitiveOwned. When ownership is passed, the primitive is destroyed when the service is destroyed.

vserviceDestroyPrimitiveIterator

        void vserviceDestroyPrimitiveIterator(
                vservicePrimitiveIterator * iterator);
Reclaims all memory used by the iterator iterator and invalidates the pointer. iterator should have been previously initialized with vserviceInitPrimitiveIterator. The iterator can be destroyed before it completes sequencing over all the primitives.

vserviceDestroySessionIterator

        void vserviceDestroySessionIterator(
                vserviceSessionIterator * iterator);
Reclaims all memory used by the iterator iterator and invalidates the pointer. iterator should have been previously initialized with vserviceInitSessionIterator. The iterator can be destroyed before it completes sequencing over all the sessions.

vserviceDisable

        void vserviceDisable(
                vservice * service);
Disables the service service so it no longer generates new sessions as requested by service consumers. Disabling a service does not affect any existing active sessions. If service was registered, it must be unregistered before it is disabled. Refer to vdasservUnregisterService for more information about unregistering services.

A disabled service can be enabled again at any time by calling vserviceEnable, but must be explicitly registered again with the Service Broker. This function has no effect if the service is already disabled. This function is implimented by sending the vserviceDISABLE message to the service object.

vserviceDump

        void vserviceDump(
                vservice * service,
                FILE * fp);
Dumps information about the service service to the stream referenced by fp. If fp is NULL then vdebugGetInfoStream is used. The dump includes the service attributes and added primitives. This function is always defined, but might not produce any output when vportDEBUG is FALSE.

vserviceEnable

        void vserviceEnable(
                vservice * service);
Enables the service service so it can generate new sessions as requested by service consumers. An enabled service is not necessarily registered. If a service is enabled but not registered then service consumers must have prior knowledge of necessary service attributes and their location to begin a session with the service. This function has no effect if the service is already enabled. service can be disabled at any time by calling vserviceDisable. An enabled service can be registered by calling vdasservRegisterService.

This function is implimented by sending the vserviceENABLE message to the service object.

vserviceFindAttributeFloat

        double vserviceFindAttributeFloat(
                vservice * service,
                const vname * name);
Returns the float value of an attribute that was added to the service service with the name name. FLT_MIN is returned if there is no such float value. Float attributes can be added with vserviceSetAttributeFloat or by passing a float with the datatag vdatatagGetSingleFloat or vdatatagGetDoubleFloat to vserviceSetAttribute.

The precision is only that of a float despite the fact that a double is returned.

vserviceFindAttributeScalar

        long vserviceFindAttributeScalar(

                const vname * name);
Returns the scalar value of an attribute that was added to the service service with the name name. LONG_MIN is returned if there is no such scalar value. Scalar attributes can be added with vserviceSetAttributeScalar or by passing an integer with the datatag vdatatagGetInteger or vdatatagGetUnsignedInteger to vserviceSetAttribute.

vserviceFindAttributeScribe

        vscribe * vserviceFindAttributeScribe(
                vservice * service,
                const vname * name);
Returns a scribe containing the value of an attribute that was added to the service service with the name name. NULL is returned if there is no such attribute value. Any string, scalar, or float attribute type can be scribed.

service still retains ownership of the attribute itself, but ownership of the resulting scribe is passed to the caller.

vserviceGetPrimitiveArgs

        void vserviceGetPrimitiveArgs(
                vservicePrimitive * primitive,
                vscrap * scrapArgs,
                ... arg3);
Retrieves the argument values from the argument scrap scrapArgs that were passed to the current invocation of the primitive handler for the primitive primitive. The variable argument list contains pointers to storage for the arguments specified in the primitive's signature. Where pointers to NULL are present, the storage will be allocated. All dynamic memory ownership is passed back to the caller, regardless of whether the storage was originally passed in or allocated as a side effect. Storage for each argument is independent of each other argument.

vserviceInitPrimitive

        void vserviceInitPrimitive(
                vservicePrimitive * primitive);
Initializes the primitive primitive. You should never need to call this function directly. The signature must be added with vserviceSetPrimitiveSignature, and the handler must be set with vserviceSetPrimitiveHandler before the primitive can be added to a service. If the primitive is not added to a service with vserviceAddPrimitiveOwned, it must eventually be destroyed with vserviceDestroyPrimitive.

Since ownership of the primitive is passed to a service, this call should not be used on primitives whose scope is less than that of the service they are added to.

Messages

vserviceDISABLE

        <vserviceDISABLE>
Disables the service service so it no longer generates new sessions as requested by service consumers. Disabling a service does not affect any existing active sessions. If service was registered, it must be unregistered before it is disabled. Refer to vdasservUnregisterService for more information about unregistering services.

A disabled service can be enabled again at any time by calling vserviceEnable, but must be explicitly registered again with the Service Broker. This message has no effect if the service is already disabled. The function entry point vserviceDisable is implemented by sending this message to the service object.

vserviceENABLE

        <vserviceENABLE>
Enables the service service so it can generate new sessions as requested by service consumers. An enabled service is not necessarily registered. If a service is enabled but not registered then service consumers must have prior knowledge of necessary service attributes and their location to begin a session with the service. This function has no effect if the service is already enabled. service can be disabled at any time by calling vserviceDisable. An enabled service can be registered by calling vdasservRegisterService.

The function entry point vserviceDisable is implemented by sending this message to the service object.

vserviceTEMPLATE_SESSION_CLASS

        <vserviceTEMPLATE_SESSION_CLASS>
Returns the type of session for a service. A new session of this class is created when service consumers want to begin a new session with the service. The application sets notification function for this session.

Macros

vserviceCLASS

        #define vserviceCLASS(...) ...
Contains the method and variable declarations for service classes. This macro expands to the contents of vserviceClass. SUPER is the type identifier of the class being declared; for example, vserviceClass. This type is used by the class's subclasses to refer to it. _OBJECT is the type identifier of instances of the class being declared; for example, vservice. This macro should be used by service subclasses to declare the contents of the service class. This macro should come first in the subclass structure declaration.

This type is used by the class's subclasses to refer to it. _OBJECT is the type identifier of instances of the class being declared; for example, vservicePrimitive. This macro should be used by primitive subclasses to declare the contents of the service primitive class. This macro should come first in the subclass structure declaration.
