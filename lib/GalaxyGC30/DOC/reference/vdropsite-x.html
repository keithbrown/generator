vdropsite prefix



Types [4] Functions [35] Messages [10] Macros [1]

Types

vdropsite

        typedef ... vdropsite;
Denotes an instance of a drop site object. Drop sites interact with a drag engine during a drag operation utilizing that engine, receiving notification when a drag enters, moves within, leaves, or is dropped within the drop site.

vdropsiteClass

        typedef ... vdropsiteClass;
Contains method and variable bindings for instances of type vdropsite. Classes declared with this type contain the same set of methods and variables as the default drop site class, but the behavior of methods and the contents of variables may vary. This class type is derived from vloadable.

vdropsiteInternal

        typedef ... vdropsiteInternal;
Denotes an instance of an internal drop site. Internal drop sites are included for convenience. They are functionally the same as regular vdropsites, except that they are created pre-registered to the default internal drag engine as returned by vdragGetInternalEngine.

vdropsiteNoteProc

        typedef ... vdropsiteNoteProc;
Denotes a notification function that is called when the vdrag drag is interacting with the drop site drop. A notification procedure is set for a drop site by vdropsiteSetNotify. The state field indicates the reason for notification as follows:
vdropSTATE_ENTER
A vdrag has entered the drop site. The return value from the notification function should be one of the drop site feedback values such as vdropFEEDBACK_COPY. Also refer to the section vdropsiteHandleEnter.
vdropSTATE_DETAIL
A vdrag has moved within the drop site. The return value from the notification function should be one of the drop site feedback values such as vdropFEEDBACK_COPY. Also refer to the section vdropsiteHandleDetail.
vdropSTATE_LEAVE
A vdrag has left the drop site. The return value is ignored. Also refer to the section vdropsiteHandleLeave.
vdropSTATE_DROP
A vdrag has been dropped in the drop site. The return value is ignored. Also refer to the section vdropsiteHandleDrop.

vdropsiteClone

        vdropsite * vdropsiteClone(
                vdropsite * drop);
Creates a copy of the drop site drop and returns a pointer to its contents. Storage for the drop site instance is allocated by the Drag class, and will be reclaimed when the drop site is destroyed with vdropsiteDestroy.

vdropsiteCopy

        void vdropsiteCopy(
                vdropsite * drop,
                vdropsite * target);
Copies the information in drop into target so that the two drop sites represent equivalent objects. target must have been created or initialized by a Drag class function. drop and target must have compatible classes for this to work correctly.

vdropsiteCopyInit

        void vdropsiteCopyInit(
                vdropsite * drop,
                vdropsite * target);
Copies the information in drop into target so the memory addressed by target is a drop site equivalent to drop. target should be a pointer to uninitialized memory; the Drag class will initialize it as part of the copy.

vdropsiteDestroy

        void vdropsiteDestroy(
                vdropsite * drop);
Destroys the drop site represented by drop, which must have been created or initialized by a Drag class function. The drop site will be unregistered from all drag engines it had been previously registered with vdragRegisterEngineDropsite. Use this function to reclaim the storage occupied by a drop site.

vdropsiteHandleDetail

        int vdropsiteHandleDetail(
                vdropsite * drop,
                vdrag * drag,
                vevent * event);
Handles pointer detail events for the drag drag drag over the drop site drop. The return value communicates the desires of the drop site back to the drag engine overrefer to the sectioning the drag, and should be one of:
vdropFEEDBACK_REJECT
The drop site will not accept the drag, and does not wish to receive further notification while the drag is above the drop site.
vdropFEEDBACK_MOVE
In its current state, the drop site will accept the drag if dropped for a move operation.
vdropFEEDBACK_COPY
In its current state, the drop site will accept the drag if dropped for a copy operation.
vdropFEEDBACK_LINK
In its current state, the drop site will accept the drag if dropped for a link operation.
vdropFEEDBACK_REJECT_RETRY
The drop site will not accept the drag in its current state, but does wish to receive further notification while the drag is above the drop site.
vdrag should be used to determine what operations the drag will permit. The feedback returned by the drop site should correspond to one of the drag's valid drop actions.

This function is implemented by sending the message vdropsiteHANDLE_DETAIL to the drop site.

vdropsiteInit

        void vdropsiteInit(
                vdropsite * drop);
Initializes a drop site of the default drop site class in the memory addressed by drop. The address can be used in other calls to the Drag class as if it were created with vdropsiteCreate. A call to vdropsiteDestroy frees any internal objects allocated by the Drag class, but does not free the structure itself.

vdropsiteInitOfClass

        void vdropsiteInitOfClass(
                vdropsite * drop,
                vdropsiteClass * clas);
Initializes a drop site of the class clas in the memory addressed by drop. The address can be used in other calls to the Drag class as if it were created with vdropsiteCreateOfClass. A call to vdropsiteDestroy frees any internal objects allocated by the Drag class, but does not free the structure itself.

vdropsiteIsEnabled

        vbool vdropsiteIsEnabled(
                vdropsite * drop);
Tests if the drop site drop is enabled, indicating that drag engines should allow drags to interact with it.

vdropsiteLoad

        vdropsite * vdropsiteLoad(
                vresource resource);
Reads a drop site from the resource resource and returns a pointer to its contents. The drop site's class is determined by the value of the tag ``Class.'' It generates a vobjectUndefinedClassException exception if the name does not denote a class in the Class class registry. Storage for the drop site instance is allocated by the Drag class, and will be reclaimed when the drop site is destroyed with vdropsiteDestroy.

vdropsiteLoadInit

        void vdropsiteLoadInit(
                vdropsite * drop,
                vresource resource);
Reads a drop site from the resource resource and stores it in the uninitialized memory addressed by drop. The drop site's class is determined by the value of the tag ``Class.'' It generates a vobjectUndefinedClassException exception if the name does not denote a class in the Class class registry.

vdropsiteSetData

        void vdropsiteSetData(
                vdropsite * drop,
                const void * data);
Sets the value stored in the drop site drop's generic data pointer to data. The value is not used or modified by the drop site in any way, and may contain any pointer value, including NULL. This function is implemented by sending the message vdropsiteSET_DATA to the drop site.

vdropsiteSetNotify

        void vdropsiteSetNotify(
                vdropsite * drop,
                vdropsiteNoteProc noteProc);
Sets the drop site drop's notification function to noteProc. When set, the vdropsiteNoteProc will be called by the drop site when a drag enters, moves within, leaves, or is dropped within the drop site. This function is implemented by sending the message vdropsiteSET_NOTIFY to the drop site.

vdropsiteSetRect

                vdropsite * drop,

Sets the drop site drop's bounding rectangle to rect. Coordinates are in display units relative to the lower left-hand corner of the drop site's window, as set by vdropsiteSetWindow. This function is implemented by sending the message vdropsiteSET_RECT to the drop site.

        void vdropsiteStartup();

vdropsiteStore

Macros

vdropsiteCLASS

        #define vdropsiteCLASS(...) ...
Contains the method and variable declarations for drop site classes. This macro expands to the contents of vdropsiteClass. SUPER is the type identifier of the class being declared (for example, vdropsiteClass). This type is used by the class's subclasses to refer to it. DROP is the type identifier of instances of the class being declared (for example, vdropsite). NOTIFY is the type identifier of the instance notification function (for example, vdropsiteNoteProc). This macro should be used by drop site subclasses to declare the contents of the drop site class. This macro should come first in the subclass structure declaration.
