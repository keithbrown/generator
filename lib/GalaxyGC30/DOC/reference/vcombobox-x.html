Types

vcombobox

        typedef ... vcombobox;
Contains the information needed to represent a combobox - objects of this type have a class of vcomboboxClass. A combobox consists of a container, a list of values, an entry box, a choice box, and a set of attributes. Refer to the "Combination Box Manager" chapter of the Galaxy Application Environment Programmer's Guide for a complete description of comboboxes. All object types which derive from vcomboboxClass must declare a vcombobox as their first field so that vcombobox pointers can be interchanged with pointers to the new type.

vcomboboxClass

        typedef ... vcomboboxClass;
Contains the message and variable definitions for objects of type vcombobox. Classes declared with this type contain the same set of methods and variables as the default combobox class, but the behavior of the methods and the contents of the variables may vary. This class type is derived from vcontainerClass.

vcomboboxNoteProc

        typedef ... vcomboboxNoteProc;
Denotes a hook function which will be invoked when a combobox handles an event of importance. combo is the combobox handling the event, event is the event being handled, and message is the type of event: vcomboboxNOTIFY_SELECT for selection events, vcomboboxNOTIFY_SCROLL for scroll events, and vcomboboxNOTIFY_ACCEPT for accept events.

vcomboboxValue

        typedef ... vcomboboxValue;
Contains the information needed to represent a combobox value - objects of this type have a class of vcomboboxValueClass. A combobox value is an object which represents a thing to be selected from a range of similar things - for example, a number "5" from the range "1" to "10". A combobox value consists of an object, a combobox, and a set of attributes. Refer to the "Combination Box Manager" chapter of the Galaxy Application Environment Programmer's Guide for a complete description of combobox values. All object types which derive from vcomboboxValueClass must declare a vcomboboxValue as their first field so that vcomboboxValue pointers can be interchanged with pointers to the new type.

vcomboboxValueClass

vcomboboxACCEPT_VALUE

        <vcomboboxACCEPT_VALUE>
Sets the current selection of the combobox combo to the value closest to the combobox value value based on the currently defined accept behavior. This behavior can be one of:

vcomboboxBEHAVIOR_DO_NOTHING Accept what's in the entrybox without checking the value against the list in the choicebox. The current value of both the entrybox and the choicebox will remain unchanged. vcomboboxBEHAVIOR_MATCH_OR_ADD If the value in the entrybox matches a value in the choicebox, select that value in both the choicebox and the entrybox, otherwise add the value in the entrybox to the choicebox and select that value in both parts. vcomboboxBEHAVIOR_MATCH_OR_CLOSEST If the value in the entrybox matches a value in the choicebox select that value, otherwise select the value closest to the value in the entrybox and make that value the currently selected value of both the entrybox and the choicebox. vcomboboxBEHAVIOR_MATCH_OR_PARTIAL If the value in the entrybox matches a value in the choicebox select that value, otherwise select the value that is the best superset partial of the value in the entrybox. I.E., values 's', 'sno', and 'snorkl' in the entrybox are subset partials of 'snorklewhacker', but 'snow', 'sni', or 'soup' are not. vcomboboxBEHAVIOR_MATCH_OR_FIRST

Changes the notification procedure for the combobox combo to noteProc. noteProc is called by vcomboboxNotify when an event of importance is handled. Importance is determined by the event handlers of the class. noteProc may be NULL, in which case no notification is given.

vcomboboxSET_VALUE

        <vcomboboxSET_VALUE>
Assigns the value value as the current value of the combobox combo. value becomes the current value of both the entrybox and choicebox parts of combo, and the choicebox is scrolled so that value is visible.If value is not in combo's value list, it becomes the current value in the entrybox but not the choicebox.

vcomboboxSET_WRAP

        <vcomboboxSET_WRAP>
Controls whether or not keyboard traversal of the value list in the entrybox part of the combobox combo will wrap the values in the choicebox from the last value to the first. If onOff == TRUE, wrapping is enabled.

vcomboboxSORT_VALUES

        <vcomboboxSORT_VALUES>
Sorts the value list of the combobox combo. Comboboxes of the default class with values of the default class will have their value list sorted in the order defined by by function vcomboboxCOMPARE_VALUES (for comboboxes of the default class, this is alphabetical order).

vcomboboxValueDRAW

        <vcomboboxValueDRAW>
Draws the combobox value value in the rect rect in the current graphics state. The default combobox value class draws the title of value centered in rect.

vcomboboxValueDRAW_WITH

        <vcomboboxValueDRAW_WITH>
Draws the combobox value value in the drawing context context in the vrect rect. The function vcomboboxDrawValueWith is implemented by sending this message to the vcomboboxValue.

vcomboboxValueGET_SIZE

        <vcomboboxValueGET_SIZE>
Calculates the width and height of the part of value that will be displayed on the screen and returns the results in width and height. The default combobox value class displays just the title, so vcomboboxValueGET_SIZE will return the width and height of the title for values of the default class.

vcomboboxValueSET_COMBOBOX

        <vcomboboxValueSET_COMBOBOX>
Assigns the combobox combo to the combobox value value.

vcomboboxValueSET_DATA

        <vcomboboxValueSET_DATA>
Assigns data to the generic data pointer of the combobox value value. This may be a pointer to whatever information that you wish to attach to the combobox value - it is not used by the Combination Box Manager. data may be NULL. data is declared as const to indicate that the Combination Box Manager does not modify the contents.

vcomboboxValueSET_TITLE_SCRIBED

        <vcomboboxValueSET_TITLE_SCRIBED>
Changes the title of the combobox value value to scribe through a String Manager scribe. The manner in which the title is drawn, if at all, is defined by the class. The default combobox value class uses the title as the value of value. A copy is made of the title before it is stored in the value, and the string is freed when the title is destroyed or changed. You can inhibit copying and share titles between combobox values by using a shared scribe.
