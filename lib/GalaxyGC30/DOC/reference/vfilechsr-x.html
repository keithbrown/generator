Types

vfilechsr

        typedef ... vfilechsr;
A file chooser provides a common interface to the user for choosing files and directories. A file chooser can be created by any Galaxy application and is the recommended method for requesting a file or directory from the user. A file chooser can be customized by the application program to provide file filtering and custom sorting methods.

vfilechsrApplyFunc

        typedef ... vfilechsrApplyFunc;
Denotes an apply function that is called when a file or directory has been chosen from the file chooser. path specifies the file or directory to be operated on (opened, saved, and so on). The apply function should return TRUE if the file or directory was acceptable, in which case the file chooser closes if appropriate, or FALSE if the file or directory is not acceptable, in which case the file chooser does not close under any circumstances.

vfilechsrAddFilterByPattern

        void vfilechsrAddFilterByPattern(
                vfilechsr * fileChooser,
                const vname * id,
                const vchar * title,
                const vchar * pattern,
                int makeDefaultSelection);
Adds a filter menu item with the title title to the file chooser fileChooser's Filter menu. When the filter menu item is activated, only items whose name match the associated pattern will be displayed. pattern can be any expression accepted by the Galaxy vregexp class. title should be a natural description of the filter and should not contain filesystem specific syntax. Pass TRUE in makeDefaultSelection to make the pattern the active filter for the chooser. id is any identifier you want to attach to the filter. Note vregexp pattern filters may not be supported by all native file choosers.

For example, to add a filter that allows only files with a ``.txt'' extension, do something like the following: 


.
.
#define TAG(l) vnameInternGlobalLiteral(l)

/*
 * Create new chooser from scratch; set the apply function to
 * do whatever your application needs... 
 */ 

aChooser = vfilechsrCreate();
vfilechsrSetType(aChooser, vfilechsrGET);  /* like "open" */
vfilechsrSetApply(aChooser, DoYourThing);

/* 
 * Add filter to show all files; make it the default selection
 */

pattern = vcharScribeLiteral(".+"); 
description = vcharScribeLiteral("All Files");
vfilechsrAddFilterByPatternScribed
     (aChooser, vname_Default, description, 
      pattern, vTRUE);

/* 
 * Add filter to show only text files 
 */ 

pattern = vcharScribeLiteral(".+\.txt$");
description = vcharScribeLiteral("Text Files"); 
vfilechsrAddFilterByPatternScribed(aChooser, TAG("Text"), description, 
                                                   pattern, vFALSE);

/* 
 * Open the chooser window 
 */ 

vfilechsrPlace(aChooser, NULL, vrectPLACE_CENTER, vrectPLACE_CENTER);
vfilechsrOpen(aChooser);
.

vfilechsrAddFilterBySystemType

        void vfilechsrAddFilterBySystemType(
                vfilechsr * fileChooser,
                vresource filterTypes,
                const vname * addFilter,
                const vname * defaultFilter);
Add zero or more filter items to the chooser fileChooser's Filter menu. The filter information is stored in a filesystem dependent format in the dictionary resource filterTypes. Since the filterTypes resource can contain any number of filters, use addFilter as the name of the specific filter to be added or pass NULL if all of the filters should be installed. If all of the filters will be installed, use defaultFilter to make a specific filter the active filter for the chooser or pass NULL to let the file chooser select a default filter. The file chooser sorts the filter items in ascending order based on their titles.

If your application will be ported to different platforms (UNIX, Windows/NT, Macintosh, etc.) and needs to support native file choosers on all platforms, vfilechsrAddFilterBySystemType is the preferred method for attaching filters to a file chooser. The developer need only update the filter types dictionary resource for each new platform without any modification to the source code. Use vfilechsrMakeFilterSpecRootResource to create a skeleton filter types dictionary for all the platforms the file chooser understands; then use the Visual Resource Builder to add your filter information.

To better understand what a filter type dictionary can contain, the following example describes a filter type resource dictionary for ASCII text files on the Apple Macintosh, Microsoft Windows/NT, and UNIX platforms.

The top-level dictionary in this example is called My Filter Types. My Filter Types contains three dictionaries for the platforms supported. Each of these platform-specific dictionaries contains a Default dictionary and a Text dictionary. The contents of the Default and Text dictionary vary for each platform. The Default dictionary usually shows all items in a directory. While this item is not required, it is usually included for completeness.

The boolean field Case Insensitive, when set to TRUE, tells the file chooser that file name comparisions should be case insensitive. This flag is only used by the Galaxy file chooser and should only be included if you want to bypass the platform file system's default setting.

The string Title contains the actual string displayed in the chooser. This string indicates to the user the filters for text files.

The string Value contains the platform-specific filters for text files.

Note that the Macintosh Text dictionary also contains a delimiter character. This is because the Macintosh platform does not have a default delimiter. The delimiter is used in the string Value to separate specifications for this field.

For a more complete example of specifying filter information in a file chooser, refer to your Galaxy system resource file. The File Types dictionary, located in the the vfilechsr dictionary of the system resource file, contains filters for all platforms supported by Galaxy. It also includes a default Galaxy dictionary that can be used in cases where the File Chooser Manager can't find a platform-specific dictionary.

vfilechsrAddViewBy

        void vfilechsrAddViewBy(
                vfilechsr * fileChooser,
                const vchar * viewByName,
                vfilechsrCompareFunc compareFunc);
Adds a view by method to the view menu of the file chooser fileChooser, using the name viewByName. A view by method sorts the list of files and directories in a specified manner. When this view is used, the function compareFunc is called to sort the list of files and directories.

For example, to add a view that displays the list of files and directories so that the C source files are listed first, you first need to write the compare function: 


vbool cSourceCompare(const vfilechsrEntry *entry1,
		       const vfilechsrEntry *entry2)
{
    vbool	isCFile1, isCFile2;
    vchar	*ptr;
    size_t	length;

    length = vcharLength(entry1->name);
    ptr = entry1->name + length - 2;
    isCFile1 = (length > 1 && *(ptr++) == "." &&
                (*ptr == 'c' || *ptr == 'h'))

    length = vcharLength(entry2->name);
    ptr = entry2->name + length - 2;
    isCFile2 = (length > 1 && *(ptr++) == "." &&
                (*ptr == 'c' || *ptr == 'h'))

    if (isCFile1 && !isCFile2)
        return(1);
    else if (!isCFile1 && isCFile2)
        return(-1);
    return(0);
}

Then install it into the file chooser by adding a view: 


vfilechsrAddViewBy(fileChooser, vnameInternLiteral("C Files"), cSourceCompare);

This adds the menu item titled ``by C Files'' to the view menu. When this menu item is selected, the file/directory list is resorted using the cSourceCompare sort function.

Changes the current directory of the file chooser pointed to by fileChooser to the directory specified by the path path. vfilechsrSetDirectory can be called before or after the file chooser has been opened. The default directory the file chooser displays if a directory is not set is the current working directory. If the file chooser is open, the change to the specified directory occurs immediately.

Unlike vfilechsrSetFile, vfilechsrSetDocumentName only changes the path entry text item; it does not affect the file chooser's current directory. Note the visual effect of setting the document name is platform dependent and may not work with all native file choosers.

Once a file chooser's history list has reached its maximum, items are removed in a FIFO (first in first out) priority to make room for new entries.

vfilechsrSetMinListWidth

        void vfilechsrSetMinListWidth(

Sets the minimum file/directory list width for the file chooserfileChooser to minListWidth. The minimum list width is the smallest width in pixels that a file/directory list can be. This also determines how many file/directory lists can be displayed at one time. When the file chooser is resized, it determines the number of file/directory lists to display by taking the amount of available space (width) and dividing by the minimum list width.

Changes the file chooser fileChooser's option flags to options. Note the visual effect of setting a particular option is platform dependent and not all native file choosers support the full range of options. options should be the bitwise OR (|) of one or more of the following constants:

vfilechsrOPTION_ONLY_DIRS
Want to select a directory
vfilechsrOPTION_MULTIPLE_ABANDON

vfilechsrSetOverridePreferNative

        void vfilechsrSetOverridePreferNative(

Sets whether the File Chooser Manager should try to map the file chooser fileChooser to a native equivalent. vfilechsrSetOverridePreferNative should be used for file choosers which are subclasses of the default vfilechsr class but which should be forced to be native choosers. Normally, the File Chooser Manager will not map instances of vfilechsr subclasses to native choosers. Refer to the ``Standard Dialogs'' chapter of the Galaxy Application Environment Programmer's Guide for a list of guidelines for supporting native file choosers. Sets whether the File Chooser Manager tries to map the file chooser fileChooser to a native equivalent. vfilechsrSetType

        void vfilechsrSetType(
                vfilechsr * fileChooser,
                int type);
Sets the type for the file chooser fileChooser to type. There are two types of file chooser:

vfilechsrGET
The file chooser is used to get (choose) a file or directory to be operated on by the application.
vfilechsrPUT
The file chooser is used to put an application specified file or directory in a chosen directory with a chosen name.

The behavior of the file chooser is nearly identical for both types, with the following exception: for ``get'' choosers, files in the file/directory list are disabled if the user does not have read access to a file; for ``put'' choosers, files in the file/directory list are disabled if the user does not have write access to a file.

vfilechsrUP
Shift to the parent directory.
vfilechsrDOWN
Shift to the currently selected child of the parent directory.

Macros

vfilechsrCLASS

        #define vfilechsrCLASS(...) ...
Contains the method and variable declarations for file chooser classes. This macro expands to the contents of vfilechsrClass. SUPER is the type identifier of the class being declared (for example, vfilechsrClass). This type is used by the class's subclasses to refer to it. FILECHSR is the type identifier of instances of the class being declared (for example, vfilechsr). NOTIFY is the type identifier of the instance notification function (for example, vconfirmHookProc). This macro should be used by file chooser subclasses to declare the contents of the file chooser class. This macro should come first in the subclass structure declaration.
