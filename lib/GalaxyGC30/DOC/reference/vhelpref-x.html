Types [2] Functions [15] Messages [4] Constants [1] Macros [1]

Types

vhelpref

        typedef ... vhelpref;
A vhelpref can contain either a vurl or a help context. vhelpref's are located by vhelpreflocator and vhelphtmllocator and it is used by subclasses of vhelphypertext such as:
  • vhelpgalaxy
  • vhelpwinhelp
  • vhelpnetscape
  • vhelpexplorer
  • vhelpdefault.
vurls are useful for help displays that use HTML text. Context strings are used by displays that map text to links in proprietary help text formats (for example, vhelpsrv, WinHelp).

Context strings are arrays of NULL terminated vchar strings. Normally, they describe the tags objects in the UI hierarchy. For example, the context for a button in a dialog might be: 

   DialogButton
Where "Dialog" is the tag of the dialog and "Button" is the tag of the button in the dialog. In this case, the context's array length is 2. The first element is the button's tag and the second is the tag of the dialog. If the button were in a container in a dialog, the context would then be: 
   DialogContainerButton
A the array length would be 3. These kinds of contexts are normally constructed by a call to vobjectBuildHelpContext.

Note that the help context could contain other kinds of information. This information might be useful to custom help display subclasses.

vhelprefClass

        typedef ... vhelprefClass;
Contains method and variable bindings for objects of type vhelpref . Classes declared with this type contain the same set of methods and variables as the default help reference class, but the behavior of methods and the contents of variables may vary. This class type is derived from vinstanceClass.

Functions

vhelprefCreate

        vhelpref * vhelprefCreate();
Creates an instance of the default class of vhelpref and returns a pointer to its contents. Space for the object is allocated dynamically. The Class Manager reclaims the storage occupied by the object when it is destroyed with vhelprefDestroy. Higher-level interfaces based on the core vhelpref type can define a macro to this function for their Create function. This function essentially allocates space for a new instance and calls vhelprefInit on it.

vhelprefCreateOfClass

        vhelpref * vhelprefCreateOfClass(
                vhelprefClass * helprefClass);
Creates an instance of class helprefClass and returns a pointer to its contents. Space for the object is allocated dynamically based on the object size of the class. The Class Manager reclaims the storage occupied by the object when it is destroyed with vhelprefDestroy. Higher-level interfaces based on the core vhelpref type can define a macro to this function for their Create function. This function essentially allocates space for a new instance and calls vhelprefInitOfClass on it.

vhelprefDestroy

        vhelpref * vhelprefDestroy(
                vhelpref * helpref);
Destroys the object helpref, which must have been created or initialized by a Class Manager function. Interfaces based on the core object class can define a macro to this function for their Destroy function. This function is implemented by sending the message vinstanceDESTROY to helpref.

vhelprefGetClass

        vhelprefClass * vhelprefGetClass(
                vhelpref * helpref);
Returns the class of the object helpref. This pointer can be cast to higher-level classes if it is known that it is of the appropriate type. Class types can be verified with vclassIsKindOf.

vhelprefGetContextString

        void vhelprefGetContextString(
                vhelpref * helpref,
                const vchar *** context,
                int * count);
Sets context to point to an array of vchar * and sets count's value to the length of this array. If no help context is set, the length of the array is zero.

vhelprefGetDefaultClass

        vhelprefClass * vhelprefGetDefaultClass();
Returns the class used to create help references by the vhelprefCreate and vhelprefInit functions. This class can be modified to alter the behavior of subclasses of vhelpref. This function can be called to explicitly register the default help reference class with the Class Manager.

vhelprefGetInstance

        vinstance * vhelprefGetInstance(
                vhelpref * helpref);
Returns the instance on which the help reference helpref is built.

vhelprefGetURL

        const vurl * vhelprefGetURL(
                vhelpref * helpref);
Returns the URL, if any, stored in this help reference. If this reference contains no URL, it returns NULL.

vhelprefInit

        void vhelprefInit(
                vhelpref * helpref);
Initializes the memory addressed by helpref in such a way that the resulting structure can be used to store any objects whose classes inherit from vhelprefClass.

vhelprefInitOfClass

        vhelpref * vhelprefInitOfClass(
                vhelpref * helpref,
                vhelprefClass * clas);
Initializes an object of class clas in the memory addressed by helpref. Interfaces based on the core reference class can define a macro to this function for their Init function. This function simply stores the class in the class field of the object structure and sends it the message vinstanceINIT.

vhelprefSetContextString

        void vhelprefSetContextString(
                vhelpref * helpref,
                const vchar * context);
Sets a help context of a singular string for this help reference. This is equivalent to calling: 
   vhelpref   *ref;

   vhelprefSetNestedContextString(ref, context, 1);

vhelprefSetContextStringScribed

        void vhelprefSetContextStringScribed(
                vhelpref * helpref,
                vscribe * context);
Sets a help context of a singular string for this help reference. This is analogous to calling: 
   vhelpref   *ref;

   vhelprefSetNestedContextString(ref, context, 1);
except that ownership issues are resolved though the scribe.

This function is implemented by sending the message vhelprefSET_CONTEXT_STRING_SCRIBED to helpref.

vhelprefSetNestedContextString

        void vhelprefSetNestedContextString(
                vhelpref * helpref,
                const vchar ** context,
                int count);
Sets the help context of this help reference. The help reference does not own this context. This means that when the help reference is destroyed it will destroy the array storage but not the strings pointed to by the array.

This function is implemented by sending the message vhelprefSET_NESTED_CONTEXT_STRING to helpref.

vhelprefSetNestedContextStringOwned

        void vhelprefSetNestedContextStringOwned(
                vhelpref * helpref,
                vchar ** context,
                int count);
Sets the help context of this help reference. The help reference owns this context. This means that when the help reference is destroyed it will destroy the array storage and all of the strings pointed to by the array.

This function is implemented by sending the message vhelprefSET_NESTED_CONTEXT_STRING_OWNED to helpref.

vhelprefSetURLOwned

        void vhelprefSetURLOwned(
                vhelpref * helpref,
                vurl * url);
Sets the URL of this help reference. The help reference owns this URL so it will destroy the URL when it is destroyed.

This function is implemented by sending the message vhelprefSET_URL_OWNED to helpref.

Constants

vhelprefINCLUDED

        vhelprefINCLUDED
Use this constant to protect against multiple inclusion of the vhelpref header file. This constant is intended to be used in this way when including header files: 
#ifndef  vhelprefINCLUDED
#include vhelprefHEADER
#endif
