vform

vformNoteProc

        typedef ... vformNoteProc;
Denotes a notification function that is called when a form handles an event of some importance. form is the container handling the event, and event is the event being handled. Importance is determined by the event handlers in the form's class; the default form class gives no notification. If additional notification is required, you should override the appropriate handler.

vformAddField

        void vformAddField(
                vform * form,
                vfield * field);

vformAppendItem

        void vformAppendItem(
                vform * form,
                vdialogItem * item);
Appends the dialog item item to the item list of the form and invalidates the content rectangle of the new item if the form is open. The item is drawn behind all other items in the list, and it is given last chance at any pointer button events. The item is destroyed when the form is destroyed unless it is removed first.

Subclasses that override this method should first call the parent method to append the item to the form, then update any subclass-specific state.

vformArrange

        void vformArrange(
                vform * form,
                const vrect * rect);
Arranges the items in the form according to its spring solution, using rect as a coordinate system for the items. It has no effect if the form has no spring solution.

Subclasses that override this method should first call the parent method to arrange the form, then arrange any subclass-specific contents.

vformCalcMinSize

Subclasses that override this method should first call the parent method to calculate the form minimum size, then add in any subclass-specific requirements.

vformDisable

        void vformDisable(
                vform * form);
Makes the form form, including all its items and scroll bars, unresponsive to user events. If the form's dialog is open, the form's item rectangle is invalidated and the form is drawn in an inactive state in a subsequent update event. The items in the form are effectively disabled by this function. This function has no effect if the form is not enabled. This function is implemented by sending the message vdialogDISABLE_ITEM to form.

vformEnable

        void vformEnable(
                vform * form);

If the form's dialog is open, the form's item rectangle is invalidated and the form is drawn in an active state in a subsequent update event. This function has no effect if the form is not disabled. This function is implemented by sending the message vdialogENABLE_ITEM to form.

vformSetBounds

        void vformSetBounds(
                vform * form,
                const vrect * bounds);
Changes the virtual bounding rectangle of the form form to bounds in the coordinate system of the form's owner. The bounding rectangle is interpreted in the same coordinate system as the item rectangle of the form. Each item in the form is positioned relative to the lower left corner of the bounding rectangle. The bounding rectangle can be different from the content rectangle only if one or both dimensions are scrollable. The rectangle structure is copied. If the form's dialog is open, the form is invalidated. This function is implemented by sending the message vcontainerSET_BOUNDS to form.

vformSetFlat

        void vformSetFlat(
                vform * form,
                int onOff);
Changes the style of the border of a form and also how it responds to keyboard traversal. If flag is TRUE, the border (if any) of form container appears engraved into its dialog, and keyboard traversal behaves as if the form were part of its owner's item list. If flag is FALSE, the border appears inset and keyboard traversal behaves as if the form were its own dialog (refer to Figure). If the form's dialog is open, the form is invalidated. This function is implemented by sending the message vcontainerSET_FLAT to form.

Figure: Form border style

vformSetFocus

        void vformSetFocus(
                vform * form,
                vdialogItem * item);
Changes the current keyboard focus of the form form to the dialog item focus, which must be in the form's item list. The form focus is the item that receives all normal keystrokes and certain function keys. focus can be NULL, in which case the form is not given a keyboard focus. This function unfocuses the current focus item and focuses the new focus item if the form has the keyboard focus. If the form is open, the appearance of the current focus item and the new focus item may change.

Note that the current focus item of a form is set to the form's default focus item when it is opened, so any changes to the current focus item of a closed form is lost. This function generates vexArgException if the item is not in the item list of form. This function is implemented by sending the message vcontainerSET_FOCUS to form.

vformSetItemIndex

        void vformSetItemIndex(
                vform * form,
                vdialogItem * item,
                int index);
Changes the position of the dialog item item in the item list of form and invalidates the content rectangle of the item if the form is open. Items earlier in the item list are given preference for pointer hit detection and are drawn last. This function generates vexArgException if the item is not in the item list of form. It generates vexArgRangeException if index is less than zero or greater than or equal to the number of items in the form. This function is implemented by sending the message vcontainerSET_ITEM_INDEX to form.

vformSetNotify

        void vformSetNotify(
                vform * form,
                vformNoteProc noteProc);
Changes the instance notification procedure for the form form to noteProc. This function is called when the form handles an event of some importance. Notification is not given for every event that is handled by the form. Normal forms do not give notification, but form subclasses may choose to provide notification for events of importance to those subclasses.

The particular item giving notification can be retrieved with vcontainerGetSelection. If additional notification is required, you should override the appropriate event handler. noteProc may be NULL, in which case no instance notification is given. This function is implemented by sending the message vdialogSET_ITEM_NOTIFY to form.

vformSetSolution

        void vformSetSolution(
                vform * form,
                vspringSolution * solution);
Changes the spring solution for the form form to solution. The spring solution is used to reposition and resize the form items in form when it is resized. The object pointers of the links in solution should point to the items in the form. The minimum size of the form is computed from the minimum size determined by the solution. The solution is destroyed when the form is destroyed or the solution is changed. The solution is also cloned when the form is cloned or copied. solution can be NULL, in which case the form's items are not positioned automatically,

Refer to the ``Spring Manager'' chapter of the Galaxy Application Environment Programmer's Guide for more information about spring solutions. This function is implemented by sending the message vcontainerSET_SOLUTION to form.

vformSetTitle

Changes the title of the form form to title. The title is drawn in the upper left corner of the form, superimposed on the border with an appropriate margin (refer to Figure ). A copy is made of the title before it is stored in the form, and the string is freed when the title is destroyed or the title is changed.

vformSetTitleScribed

        void vformSetTitleScribed(
                vform * form,

Changes the title of the form form to title (through a String Manager scribe). The title is drawn in the upper left corner of the form, superimposed on the border with an appropriate margin (refer to Figure). A copy is made of the title before it is stored in the form, and the string is freed when the title is destroyed or the title is changed. You can inhibit copying and share titles between objects by using a shared scribe. The form is invalidated if the form's dialog is open. This function is implemented by sending the message vdialogSET_ITEM_TITLE_SCRIBED to form.
