Types

vobserver

        typedef ... vobserver;
Observer objects provide additional support above simple observable objects for dedicated observers. Any vobservable object can be installed as an observer on another vobservable,/code> object, but objects of this class are intended to be used more or less exclusively as observers. They can be temporarily disabled, and they maintain a patron object attribute.

The patron of an observer represents the object on whose behalf the observer acts. For example, the observer may modify or invalidate the patron object when it is enabled, and the observed object changes.

vobserverDisable

        void vobserverDisable(
                vobserver * observer);
Marks the observer object as not actively observing its targets. Disabling an observer does not uninstall it anywhere, but does affect how it responds to any invocations of vobservableWILL_OBSERVE. However, since announcing objects may not check the vobservableWILL_OBSERVE method before calling vobservableOBSERVE, a disabled observer may still receive occurrence announcements. It is up to the observer's implementation of vobservableOBSERVE to check the enabled attribute before interacting with the patron.

vobserverEnable

        void vobserverEnable(
                vobserver * observer);
Marks the observer object as actively observing its targets. An enabled observer object sees all the desired occurrences from the vobservable objects in which it is installed. Implementation of the observer's vobservableOBSERVE method should perform any interactions with its patron as normal.
