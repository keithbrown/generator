Types

vtoolbutton

        typedef ... vtoolbutton;
Contains the information needed to represent a single tool button; objects of this type have classes derived from vtoolbuttonClass.

A tool button is kind of button whose appearance and functionality have been tailored for use in a toolbar, although use within a toolbar is not required. The main differences between a normal button and a tool button are that a tool button will not draw its title if an image is set, and a tool button is customized so that it can participate in an exclusive group through the use of a tool button group field.

Tool buttons are referenced by a pointer to the vtoolbutton type. Tool buttons can be created dynamically with vtoolbuttonCreate, or initialized statically with vtoolbuttonInit. All object types with classes that directly inherit from vtoolbuttonClass must declare a vtoolbutton as their first field so that vtoolbutton pointers can be interchanged with pointers to the new data type.

vtoolbuttonClass

        typedef ... vtoolbuttonClass;
Contains method and variable bindings for objects of type vtoolbutton. Classes declared with this type contain the same set of methods and variables as the default tool button class, but the behavior of methods and the contents of variables may vary. This class type is derived from vbuttonClass.

vtoolbuttonGroupField

        typedef ... vtoolbuttonGroupField;
Contains the information needed to represent a single tool button group field. A tool button group field holds the state of an exclusive group of tool buttons and sends notification to the members of the group any time the state changes. The tool buttons in an exclusive group are typically toggle buttons which share a single group field. Also refer to the section vbutton. A tool button's group field is set by a call to vdialogItem.

In an exclusive group the "state" indicates which member of the group is currently selected. In a tool button group field, the state is stored in the field's ``Selection'' attribute and its value is the tag of the selected item. If multiple items in the group have the same tag, then they will all be selected simultaneously.

vtoolbuttonClone

        vtoolbutton * vtoolbuttonClone(
                vtoolbutton * button);
Creates a copy of the tool button button and returns a pointer to its contents. Storage for the tool button object is allocated by the Toolbar Manager. The Toolbar Manager reclaims the storage occupied by the tool button when it is destroyed with vtoolbuttonDestroy. This function copies all the copyable attributes of the tool button and clones any attribute objects that are owned by the tool button. The tool button to be cloned can be dynamic or static. This function essentially allocates space for a new tool button and calls vtoolbuttonCopyInit on it.

vtoolbuttonCopy

        void vtoolbuttonCopy(
                vtoolbutton * button,
                vtoolbutton * target);
Copies the information in button into target so that the two tool buttons represent equivalent objects. target must have been created or initialized by a Toolbar Manager function. button and target must have compatible classes for this to work correctly. This function copies all the copyable attributes of the tool button and clones any attribute objects that are owned by the tool button. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The tool buttons to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to button.

vtoolbuttonDestroy

        void vtoolbuttonDestroy(
                vtoolbutton * button);
Destroys the tool button represented by button, which must have been created or initialized by a Toolbar Manager function. Use this function to reclaim the storage occupied by a tool button. The tool button should not be referenced by any other object when it is destroyed. This function only attempts to free the storage occupied by the tool button if it was allocated by vtoolbuttonCreate, vtoolbuttonCreateOfClass, vtoolbuttonClone, or vtoolbuttonLoad. This function will also destroy any attribute objects that are owned by the tool button. This function is implemented by sending the message vobjectDESTROY to the tool button.

        void vtoolbuttonInit(
                vtoolbutton * button);

This function simply stores the default class in the class field of the tool button and sends it the message vobjectINIT.

vtoolbuttonLoad

        vtoolbutton * vtoolbuttonLoad(
                vresource resource);
Reads a tool button from the resource resource and returns a pointer to its contents. Storage for the tool button object is allocated by the Toolbar Manager. The tool button's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function creates an object based on the class of the resource, even if this class is not a tool button class. The object pointer is cast to a tool button pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default tool button class which are stored in resources are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vtoolbuttonLoadInit on it.

vtoolbuttonStore

        void vtoolbuttonStore(
                vtoolbutton * button,
                vresource resource);
Writes the contents of the tool button into the resource. The tool button can be recovered from the resource at a later time (perhaps by another program) with vtoolbuttonLoad or _typeprefixLoadInit. Note that not all of the tool button's attributes are stored in the resource---the attributes of the default tool button class which are stored in resources are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. It generates vobjectUnnamedClassException if the tool button's class is not named. This function is implemented by sending the message vobjectSTORE to the tool button.

To store a tool button in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vtoolbuttonStore to write the tool button into the resource. For example: 


vtoolbutton  *button;    /* contains a tool button */
vresource fileResource, buttonResource;

fileResource   = vresourceOpenFile( ... );
buttonResource = vresourceMake(fileResource, 
			       vnameInternGlobalLiteral("Mybutton"));

vtoolbuttonStore(button, buttonResource);
vresourceCloseFile(fileResource);

vtoolbuttonCLASS

        #define vtoolbuttonCLASS(...) ...
Contains the method and variable declarations for tool button classes. This macro expands to the contents of vtoolbuttonClass. SUPER is the type identifier of the class being declared (for example, vtoolbuttonClass). This type is used by the class's subclasses to refer to it. BUTTON is the type identifier of instances of the class being declared (for example, vtoolbutton). NOTIFY is the type identifier of the instance notification function (for example, vtoolbuttonNoteProc). This macro should be used by tool button subclasses to declare the contents of the tool button class. This macro should come first in the subclass structure declaration.
