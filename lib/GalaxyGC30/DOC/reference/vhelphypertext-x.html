Types

vhelphypertext

        typedef ... vhelphypertext;
This locator delegates the display of help to an instance of vwebbrowse. It expects to operate on help data of type vhelpref containing instances of vurls. It merely passed the URL to the web browser. Refer to the section vhelphypertext and vwebbrowse for more information.

The developer can obtain feedback about the results of an attempt to display help through the observation function. Refer to the section vhelphypertext for more information. This method is called with a vhelphypertextEvent that describes the kind of activity that took place.

Subclasses of vhelphypertext initialize themselves with specific subclasses of vwebbrowse, such as:

  • vwebbrowseNetscape,
  • vwebbrowseExplorer, and
  • vwebbrowseDefault.
The corresponding vhelphypertext subclasses are vhelpnetscape, vhelpexplorer, and vhelpdefault.

Two other subclasses of vhelphypertext do not use vwebbrowse but instead, use other hypertext browsers. These are vhelphypertext and vhelpwinhelp. vhelphypertext uses vhelpsrv as its browser and vhelpwinhelp uses WinHelp.

vhelphypertextClass

        typedef ... vhelphypertextClass;
Message and variable bindings for objects of type vhelphypertext. Classes declared with this type contain the same set of methods and variables as the default help HTML class, but the behavior of methods and the contents of variables may vary. This class type is derived from vhelpdisplay.

vhelphypertextGetSynchronous

        vbool vhelphypertextGetSynchronous(
                vhelphypertext * helpdisplay);
Currently, only synchronous operation is supported by default classes and subclasses of vhelphypertext. This means that the return value of a call to vhelphypertextLocateHelp will not return until its call to vwebbrowseGotoURL returns.

Note that vwebbrowseGotoURL returns positively if it is able to contact the browser regardless of whether the browser is able to locate the document referenced by the URL.

vhelphypertextUndisplayHelp

        void vhelphypertextUndisplayHelp(
                vhelphypertext * helpdisplay,
                vhelprequest * request);
This method is rarely called because most forms of help allow the user to abort the display of help. This method is not implemented in this base class. Subclasses should override this method to abort the display of help data in different ways.

vhelphypertextEventREQUEST_FAILED

        vhelphypertextEventREQUEST_FAILED
This message means that there was some problem fullfilling the request. Perhaps the message could not be sent or the URL could not be found. Subclasses of vhelphypertext should send this message through vhelphypertextNotify, when appropriate.

vhelphypertextEventREQUEST_SUCCEEDED

        vhelphypertextEventREQUEST_SUCCEEDED
This message usually means that a call to vwebbrowseGotoURL has returned vTRUE.

Note that this does not necessarily mean that the browser will be able to locate the URL it was sent. Subclasses of vhelphypertext should send this message through vhelphypertextNotify, when appropriate.
