Types

vclipboard

        typedef ... vclipboard;
Contains the information needed to represent a single clipboard object derived from the vclipboardClass. Clipboards are referenced by a pointer to this type. Clipboards can be created dynamically using vclipboardCreate or initialized statically using vclipboardInit.

All objects that inherit directly from vclipboardClass must declare a vclipboard as their first field so the vclipboard pointers can be exchanged with pointers of the new type.

vclipboardClass

        typedef ... vclipboardClass;
Contains method and variable bindings for objects of the class type vclipboard. Classes declared with this type contain the same set of methods and variables as the default clipboard class, but the behavior of methods and the contents of variables may vary. This class type is derived from vclassRoot.

vclipboardEngine

vclipboardCreate

        vclipboard * vclipboardCreate();
Creates a new clipboard object of the default clipboard class and returns a pointer to its contents. Storage for the clipboard object is allocated by the Clipboard class. The Clipboard class reclaims the storage occupied by the clipboard object when it is destroyed with vclipboardDestroy. The new clipboard object is created with a set of default attributes, as determined by the default clipboard class.

The default attributes of the default clipboard class are documented in the ``Clipboard Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new clipboard object and calls vclipboardInit on it.

vclipboardDestroy

        void vclipboardDestroy(
                vclipboard * clipboard);
Destroys the clipboard object represented by clipboard, which must have been created or initialized by an Clipboard class function. Use this function to reclaim the storage occupied by a clipboard. The clipboard should not be referenced by any other object when it is destroyed.

This function only attempts to free the storage occupied by clipboard if it was allocated by vclipboardCreate, vclipboardCreateOfClass, or vclipboardClone. This function is implemented by sending the message vclipboardDESTROY to clipboard.

vclipboardInit

        void vclipboardInit(
                vclipboard * clipboard);
Initializes a clipboard object of the default clipboard class in the memory addressed by clipboard. The attributes of the clipboard are set to their default values, according to the default clipboard class. The default attributes of the default clipboard class are documented in the ``Clipboard Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Clipboard class, as if it had been created with vclipboardCreate.

A call to vclipboardDestroy frees any internal objects allocated by the Clipboard class but does not free the structure itself. This function simply stores the default class in the class field of the clipboard and sends it the message vclipboardINIT.

                vclipboard * clipboard);

This function is implemented by sending the message vclipboardREQUEST_CONTENTS to the clipboard.

        <vclipboardQUERY_CONTENT_ENCODINGS>
When the request is processed, the notify procedure associated with the clipboard by vclipboardSetContentEncodingsProc is called. The default class calls the vclipboardQueryContentEncodings method of the vclipboardEngine associated with the clipboard.

vclipboardQUERY_ENGINE_CONTENT_ENCODINGS

        <vclipboardQUERY_ENGINE_CONTENT_ENCODINGS>
Requests currently available clipboard engine content encodings. Once the request is fulfilled, the vclipboardNotifyQuery method of the clipboardis called with the available encodings dictionary. The function entry point vclipboardQueryEngineContentEncodings is implemented by sending this message to the clipboard engine.

vclipboardREGISTER_FORMAT

        <vclipboardREGISTER_FORMAT>
Registers the format name and scrap encoding matching this format in the clipboard format registry. The function entry point vclipboardRegisterFormat is implemented by sending this message to the clipboard format registry.

vclipboardREQUEST_CONTENTS

        <vclipboardREQUEST_CONTENTS>
Causes the notify function of clipboard to be called when a request for the contents of clipboard is fulfilled. This procedure is called by vclipboardRequestContents. The function entry point vclipboardRequestContents is implemented by sending this message to the clipboard.

vclipboardREQUEST_ENGINE_CONTENTS

        <vclipboardREQUEST_ENGINE_CONTENTS>
Requests the clipboard engine contents. The default method converts clipboard contents into clipboard request scrap, (see vclipboardSetRequestScrap), and calls vclipboardNotify from the clipboard. The function entry point vclipboardRequestEngineContents is implemented by sending this message to the clipboard.

vclipboardSET_CONTENTS

        <vclipboardSET_CONTENTS>
Sets the contents of clipboard. This procedure is called by vclipboardSetContents. The function entry point vclipboardSetContents is implemented by sending this message to the clipboard.

vclipboardSET_DATA

        <vclipboardSET_DATA>
Sets a clipboard's generic data pointer to data. This may be a pointer to any information you want to attach to the clipboard; it is not used by the Clipboard class. data may be NULL. data is declared const to indicate that the Clipboard class will not modify its contents. The function entry point vclipboardSetData is implemented by sending this message to the clipboard.

vclipboardSET_ENGINE

        <vclipboardSET_ENGINE>
Sets a clipboard's low-level access engine to engine. This method can be used to provide implementation-independent access to custom clipboard's data. The function entry point vclipboardSetEngine is implemented by sending this message to the clipboard.

vclipboardSET_ENGINE_CONTENTS

        <vclipboardSET_ENGINE_CONTENTS>
Clears the current engine contents and sets the clipboard engine content pointer to scrap. scrap becomes owned by the clipboard engine and is destroyed when the clipboard engine is cleared or destroyed. The function entry point vclipboardSetEngineContents is implemented by sending this message to the clipboard engine.

vclipboardSET_ENGINE_REGISTRY

        <vclipboardSET_ENGINE_REGISTRY>
Sets the clipboard engine format registry to registry. Registry ownership is not transferred to the clipboard engine and the application remains responsible for keeping the registry as long as the clipboard engine exists. The function entry point vclipboardSetEngineRegistry is implemented by sending this message to the clipboard engine.

vclipboardSET_ENGINE_SCRAP_FORMAT

        <vclipboardSET_ENGINE_SCRAP_FORMAT>
Sets the format of clipboard engine scraps on the stack to format. Subclasses of vclipboardEngineScrap can map this format to application, or platform-specific clipboard format specifiers.

vclipboardSET_NOTIFY

        <vclipboardSET_NOTIFY>
Changes the instance notification procedure for the clipboard clipboard to noteProc. This procedure is called by vclipboardSetNotify. The function entry point vclipboardSetNotify is implemented by sending this message to the clipboard.

vclipboardSET_QUERY_NOTIFY

        <vclipboardSET_QUERY_NOTIFY>
Changes a clipboard's query notify function to notify. The function notify is called when a request for the current content encodings of clipboard is fulfilled. The function entry point vclipboardSetQueryNotify is implemented by sending this message to the clipboard.

vclipboardSET_REQUEST_SCRAP

        <vclipboardSET_REQUEST_SCRAP>
Specified scrap will be used to receive clipboard data when vclipboardRequestContents is called. The application gives up ownership of the scrap. The function entry point vclipboardSetRequestScrap is implemented by sending this message to the clipboard.

Constants

vclipboardDESTROY

        vclipboardDESTROY
Destroys the clipboard object represented by clipboard, which was created or initialized by a Clipboard class function. Use this function to reclaim the storage occupied by a clipboard. The clipboard should not be referenced by any other object after it is destroyed.

vclipboardINCLUDED

        vclipboardINCLUDED
Defined when the header file vclipboard.h is included.

vclipboardINIT

        vclipboardINIT
Initializes an instance of a clipboard class in the memory addressed by clipboard. clipboard must contain the class of clipboard to initialize: Call vclipboardInitOfClass to set the class pointer of a clipboard and initialize it simultaneously.

The default method for the default clipboard class clears all the attributes of the clipboard (except its class) to their default values and creates a client object for the clipboard. The default attributes of the default clipboard class are documented in the ``Clipboard Manager'' chapter of the Galaxy Application Environment Programmer's Guide.

Subclasses that override this method should first call vclassSendSuper to initialize the clipboard, then initialize each attribute that is unique to the subclass.
