vtextviewMoveCursor

        void vtextviewMoveCursor(
                vtextview * textView,
                int direction);
Moves the insertion point in the direction direction, in the textview textView. This direction can be:

vtextviewLEFT
Scroll one page left.
vtextviewUP
Scroll one page up.
vtextviewRIGHT
Scroll one page right.
vtextviewDOWN
Scroll one page down.

If a selection exists, then the following occurs, depending on the direction:

vtextviewLEFT
The insertion point is moved to the start of the selection.
vtextviewRIGHT
The insertion point is moved to the end of the selection.
vtextviewUP
The insertion point is moved to the start of the selection, and then the cursor is moved up one line.
vtextviewDOWN
The insertion point is moved to the end of the selection, and then the cursor is moved down one line.

vtextviewMoveCursorToEnd

        void vtextviewMoveCursorToEnd(
                vtextview * textview,
                int direction);
Moves the insertion point to the left, right, top, or bottom of the text view area for the text view textview, based on the value for direction, which can be:

vtextviewLEFT
The insertion point is moved to the left edge of the text view area.
vtextviewRIGHT
The insertion point is moved to the right edge of the text view area.
vtextviewUP
The insertion point is moved to the top of the text view area. If the insertion point is already at the top of the view area, the text view is scrolled one page up, and the cursor is moved to the top of the text view area (after scrolling).
vtextviewDOWN
The insertion point is moved to the bottom of the text view area. If the insertion point is already at the bottom of the view area, the text view is scrolled one page down, and the cursor is moved to the bottom of the text view area (after scrolling).

vtextviewNotifyUndo

        void vtextviewNotifyUndo(
                vtextview * textView,
                vscribe * actionName);
Removes the action list from the text view textView, names it with the given scribe, and calls the text view's undo notify procedure with the action. If the text view's text data object's action list is NULL (because either there is no undo observer or no undoable actions have been generated), this call does nothing.

You should call this routine after you make changes to the text data object that constitute a user action. The text view's undo notify routine should then install the action it receives in the application's undo log.

This routine is called internally by the Text class after the user inserts, deletes, or types, thus causing any undo actions generated by the operation to be reported to your undo notify procedure.

vtextviewSetOffscreenDrawing

        void vtextviewSetOffscreenDrawing(
                vtextview * textView,
                int useOffscreenDrawing);
Sets whether the text view textView draws text by drawing in an offscreen buffer and blitting the offscreen buffer to the textview or by drawing the text directly. Using offscreen drawing results in much smoother drawing, especially when using multi-styled, wrapped text.         void vtextviewSetUndoNotify(
                vtextview * textView,
                vtextviewUndoNoteProc undoNotifyProc);
Sets the undo notification procedure for the text view textView to undoNotifyProc. You can use this procedure to handle all undoable actions generated from the text data object. A typical response for the undo notification procedure to take would be to install the action it receives in the document's undo log.

The undo notification procedure is automatically called when the text view initiates some action that generates an undoable action through the text data object's undo observer; for example, typing or deleting. It is not automatically called when you make function calls which generate undoable actions through the text data object's undo observer. If you do that, you should call vtextviewNotifyUndo yourself after all of the changes have been made. This causes the notify procedure to be called with any actions that have been generated.

Returns the window line number of the current line number of the textview textView. vtextviewCLASS

        #define vtextviewCLASS(...) ...
Contains the method and variable declarations for the text view class. This macro expands to the contents of vtextviewClass. SUPER is the type identifier of the class being declared; for example, vtextviewClass. This type is used by the class's subclasses to refer to it. ITEM is the type identifier of instances of the class being declared; for example, vtextview.

NOTIFY is the type identifier of the instance notification function; for example, vtextviewNoteProc. This macro should be used by text view subclasses to declare the contents of the text view class. This macro should come first in the subclass structure declaration.
