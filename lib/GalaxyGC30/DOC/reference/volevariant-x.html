Types

volevariant

        typedef ... volevariant;
A class that implements a "union" of data types to be used as parameters to ActiveX Automation Invoke calls. When passing a parameter to an Automation method, you must first encode it as a volevariant, using one of the Setxxx methods on a volevariant you create. For example, to encode an integer, you would use SetInt to encode an integer parameter. volevariants are then typically added to a volevariantList by invoking the append method of the list. volevariants are also used as the return value from Automation method invocations, but in that case, they are not attached to a list but are directly accessible.

volevariantClass

        typedef ... volevariantClass;
Contains method and variable bindings for objects of type volevariant. Classes declared with this type contain the same set of methods and variables as the default volevariant class, but the behavior of methods and the contents of variables may vary. This class type is derived from vobject.

volevariantList

        typedef ... volevariantList;
An object primarily used for building parameter lists for ActiveX Automation method invocation, where each parameter is encapsulated as an volevariant object. Typically, you would invoke the list's volevariantAppendList method to add parameters (volevariant records) to it.

volevariantListClass

        typedef ... volevariantListClass;
Contains method and variable bindings for objects of type volevariantList. Classes declared with this type contain the same set of methods and variables as the default volevariantList class, but the behavior of methods and the contents of variables may vary. This class type is derived from vobjectList.

volevariantChangeType

        void volevariantChangeType(
                volevariant * variant,
                short vartype);
Converts a variant record from one type to another. This routine is useful for argument coercion. For example, if a routine expected an integer value, you could specify it in its string representation, for example, "23", and the system will coerce it to the proper integer type. This routine is essentially a wrapper function around the Win32 API VariantChangeType. Thus, you should consult the system documentation to see which coercions are provided.

volevariantClear

        void volevariantClear(
                volevariant * variant);
Clears the data in a variant record and sets its type to empty. This routine is essentially a wrapper function around the Win32 API VariantClear(). Thus, any resources associated with the record will be freed as per the system documentation.
