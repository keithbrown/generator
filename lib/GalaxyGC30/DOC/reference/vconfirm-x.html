Types

vconfirm

        typedef ... vconfirm;
Represents a confirmation dialog, that is, a dialog with standard buttons and decorations that are managed by the Confirmation class. Confirmation dialogs hide the details and behavior of look-and-feel specific items from clients that use standard dialogs.

vconfirmButton

        typedef ... vconfirmButton;
Represents a button in a confirmation dialog. The Confirmation class maintains a list of standard and generic (application defined) buttons which are of this type.

vconfirmButtonClass

        typedef ... vconfirmButtonClass;
Describes the behavior of a class of confirmation buttons. Confirmation button classes are referenced by a pointer to this type. The default confirmation button class is a subclass of the default Button class.

vconfirmClass

        typedef ... vconfirmClass;
Describes the behavior of a class of confirmation dialogs. Confirmation dialog classes are referenced by a pointer to this type. The default confirmation dialog class is a subclass of the default dialog class.

vconfirmHookProc

        typedef ... vconfirmHookProc;
Denotes a hook function that is called when a designated confirmation dialog handles an event of importance. confirm is the confirmation dialog handling the event, and event is the event being handled. Confirmation dialogs use pointers of this type to represent hook functions that are called when one of the Confirmation class's buttons has been activated.

A hook function returns a value indicating success or failure. A non-zero (TRUE) value indicates no error and the Confirmation class should continue processing as normal. A zero (FALSE) value indicates the application detected some type of error.

In this case, the Confirmation class does not close the confirmation dialog even if that is the expected behavior. This is useful for displaying a notice when the user has made such an error and leaving the confirmation dialog open for the user to correct that error.

vconfirmHandleApply

        int vconfirmHandleApply(
                vconfirm * confirm,
                vevent * event);
Override this method to provide application-specific action. The method should return a non-zero, TRUE value unless an error occurred while trying to apply the contents of the confirmation dialog. When a zero value, FALSE returns, the Confirmation class keeps the confirmation dialog open, allowing the application to raise a notice informing the user of the error.

Providing a global function or static member function by SetHandleApplyFunc is equivalent to overriding HandleApply. The default implementation of SetHandleApplyFunc for vconfirm instances calls the apply function if one has been set by SetHandleApplyFunc. If HandleApply is not overridden and there is no function attached by SetHandleApplyFunc, no notification is given when the user presses the Apply or OK buttons of the confirm.

vconfirmHandleDefaults

        int vconfirmHandleDefaults(
                vconfirm * confirm,
                vevent * event);
Override this method to provide application-specific action. The method should return a non-zero, TRUE value unless an error occurred trying to reset the contents of the confirmation dialog to defaults. When a zero value, FALSE is returned, the Confirmation class keeps the confirmation dialog open, allowing the application to raise a notice informing the user of the error.

Providing a global or static member function by SetHandleDefaultsFunc is equivalent to overriding HandleDefaults. The default implementation of SetHandleApplyFunc for vconfirm instances calls the apply function if one has been set by SetHandleApplyFunc. If HandleDefaults is not overridden and there is no function attached by SetHandleDefaultsFunc, no notification is given when the user presses the Defaults button of the confirm.

vconfirmHandleGeneric

        int vconfirmHandleGeneric(
                vconfirm * confirm,
                vevent * event);
Override this method to provide application-specific action. The function should return a non-zero, TRUE value, unless an error occurred during the attempt to act on the button press. When a zero value, FALSE is returned, the Confirmation class keeps the confirmation dialog open, allowing the application to raise a notice informing the user of the error.

Providing a global or static member function by SetHandleGenericFunc is equivalent to overriding HandleGeneric. The default implementation of HandleGeneric for vconfirm instances calls the generic function if one has been set by SetHandleGenericFunc. If HandleGeneric is not overridden, and there is no function attached by SetHandleGenericFunc, no notification is given when the user presses a generic button on the confirm. Generic buttons are to be managed by the application, so the Confirmation class does not disable or hide generic buttons. The application, however, is free to manage generic buttons as it sees fit.

vconfirmProcess

        const vname * vconfirmProcess(
                vconfirm * confirm);
Opens a blocking confirmation dialog, waits until a confirmation button is activated, and then closes the confirm. This single call is equivalent to calling:
  • vconfirmProcessStart
  • vconfirmProcessGet and
  • vconfirmProcessStop.

This routine is useful when the application does not want to use the standard confirmation Handle* overrides, and simply wants the tag of the confirmation button that was pressed by the user. vconfirmProcess could be used as shown below. 


 // assume confirm is created and
 // initialized.

 // open confirm and wait for button press

 tag = confirm->Process();

 // flow will not return here until	
 // a confirmation button is pressed.

 if ((tag == vname_Ok) || (tag == vname_Apply))
   // do OK code 
 else if (tag == vname_Help)
   // do HELP code */
 else if ((tag == vname_Cancel) || (tag == vname_Reset))
   // do RESET code

Note that when vconfirmProcess is used, the handle overrides are not called. The underlying dialog's close hook is also not called when the dialog closes since the close always happens on a button click, and the confirm is closed when control returns.

vconfirmProcessGet

        const vname * vconfirmProcessGet(
                vconfirm * confirm);
Returns the tag of a confirmation button clicked within a blocking confirmation dialog. The confirm must have been opened using vconfirmProcessStart and subsequently closed with vconfirmProcessStop. All calls to vconfirmProcessGet must be enclosed by calls to vconfirmProcessStart and vconfirmProcessStop.

A confirmation dialog can also be closed using the Window class. In this case a tag of vname_Close is returned. It is the application's responsibility to actually close the dialog in this case; the Confirmation class will not close the confirm. Valid tags are listed below.

  • vname_Ok
  • vname_Apply
  • vname_Reset
  • vname_Cancel
  • vname_Defaults
  • vname_Help
  • vname_Close

Additionally, valid tags include any application defined tags added with vconfirmSetGeneric.

vconfirmPROCESS

        <vconfirmPROCESS>
Opens a blocking confirmation dialog, waits until a confirmation button is activated, and then closes the confirm. This single call is equivalent to calling:
  • vconfirmProcessStart
  • vconfirmProcessGet and
  • vconfirmProcessStop.

This routine is useful when the application does not want to use the standard confirmation Handle* overrides, and simply wants the tag of the confirmation button that was pressed by the user. vconfirmProcess could be used as shown below. 


 // assume confirm is created and
 // initialized.

 // open confirm and wait for button press

 tag = confirm->Process();

 // flow will not return here until	
 // a confirmation button is pressed.

 if ((tag == vname_Ok) || (tag == vname_Apply))
   // do OK code 
 else if (tag == vname_Help)
   // do HELP code */
 else if ((tag == vname_Cancel) || (tag == vname_Reset))
   // do RESET code

Note that when vconfirmProcess is used, the handle overrides are not called. The underlying dialog's close hook is also not called when the dialog closes since the close always happens on a button click, and the confirm is closed when control returns.

Changes the title of the Defaults button of the confirmation dialog confirm to scribe, (through a String class scribe), if function-specific buttons are desirable for the current look-and-feel. scribe can be NULL, in which case the button uses the default for the current look-and-feel. A copy is made of the title before it is stored in the button, and the string is freed when the dialog is destroyed or the title is changed. The button title is not changed if the current look-and-feel does not permit this. Refer to the section vconfirmSetOverride to disable this look-and-feel check.

The default method sets the title unless scribe is NULL; then it resets the title to the default using the class variable vconfirmDEFAULT_DEFAULTS_TITLE and also resets the mnemonic.

vconfirmSET_GENERIC

        <vconfirmSET_GENERIC>
Controls whether the confirmation dialog confirm should have a generic button with tag tag. If flag is TRUE, the confirm has a generic button with the specified tag; if flag is FALSE, it does not and the button is removed from the confirm and deleted. By default, this method adds a button with the specified tag if flag is TRUE, or removes such a button if flag is FALSE.

vconfirmSET_GENERIC_HOOK

        <vconfirmSET_GENERIC_HOOK>
Changes the Generic hook procedure for the confirmation dialog confirm to applyProc. When the user presses a generic button of the confirm, this message is sent, with the dialog and the event as arguments.

This hook function should return a non-zero (TRUE) value unless an error occurred trying to act on the button click. When a zero value (FALSE) is returned, the Confirmation class keeps the confirmation dialog open, allowing the application to raise a notice informing the user of the error.

By default, this method simply sets the generic hook function. Generic buttons are to be managed by the application so the Confirmation class does not disable or hide generic buttons and the application is free to do this.

vconfirmSET_HELP_HOOK

        <vconfirmSET_HELP_HOOK>
Changes the help hook procedure for the confirmation dialog confirm to helpProc. When the user presses the Help button of the dialog, this message is sent, with the dialog and the event as arguments.

This hook function should return a non-zero (TRUE) value unless an error occurred trying to apply the contents of the confirmation dialog. When a zero value (FALSE) is returned, the Confirmation class keeps the confirmation dialog open, allowing the application to raise a notice informing the user of the error.

By default, this method simply sets the help hook function and if the disable buttons attribute of the confirmation dialog is set, disables the OK and Apply buttons if they exist and helpProc is NULL. When helpProc is a function and the disabled buttons attribute is set, those buttons are enabled.

vconfirmSET_OVERRIDE

        <vconfirmSET_OVERRIDE>
Controls whether the override attribute is turned on for the confirmation dialog confirm. If flag is TRUE this attribute is on, and all application-specified confirmation button titles are used, even if it violates look-and-feel standards. If flag is FALSE, look-and-feels that do not allow non-standard button titles always use the default titles.
