vdate

vdateFormat

        typedef ... vdateFormat;
Holds information on how to textually represent a date and/or time and contains information on which fields should be shown in what order and in what format. Standard internationalized vdateFormats are provided, but you can customize these or create your own if necessary.

vdateRangeException

vdateAddFormatName

        void vdateAddFormatName(
                vdateFormat * format,
                vdateFormatIterator * before,
                vdateName which,
                vbool abbrev,
                vbool required,
                vbool shown);
Adds a name to a date format. The name is added before the position indicated by before. before still points to the same element after the call, or at the end of the format if before is a null pointer.

The value passed for which determines which part of the date the name in the format represents. which can have one of the following values:

vdateMERIDIAN_NAME
AM or PM
vdateWEEK_DAY_NAME
The name of the day of the week
vdateMONTH_NAME
The name of the month
vdateTIME_ZONE_NAME
The name of the time zone

abbrev indicates whether to print the abbreviated version of the name, if there is one. Both the abbreviated and full versions of the name are recognized when parsing, regardless of the value of abbrev.

required and shown indicate whether the name is required when parsing a date and shown when printing a date, respectively.

vdateAddFormatNumber

        void vdateAddFormatNumber(
                vdateFormat * format,
                vdateFormatIterator * before,
                vdateNumber which,
                vnumFormat * number,
                vbool required,
                vbool shown);
Adds a number to a date format. The number is added before the position indicated by before, or at the end of the format if before is a null pointer. before still points to the same element after the call.

The value passed for which determines which part of the date the number in the format represents. Possible values for this parameter are described below.

vdateSECOND_NUMBER
Seconds field of the date's time (0-59)
vdateMINUTE_NUMBER
Minutes field of the date's time (0-59)
vdateHOUR_12_NUMBER
Hour field of the date's time (1-12)
vdateHOUR_24_NUMBER
Hour field of the date's time (0-23)
vdateDATE_NUMBER
Day of the month field of the date (1-31)
vdateDAY_OF_YEAR_NUMBER
Day of the year field of the date (1-366)
vdateMONTH_NUMBER
Month number field of the date (1-12)
vdateYEAR_ABBREV_NUMBER
Abbreviated year field of the date (0-99). Note that when parsing a date, the year number is converted to the closest year to the current year with the given last two digits. Also, a four digit year is recognized and used unmodified when parsing.
vdateYEAR_FULL_NUMBER
Abbreviated year field of the date (0-9999+)

number should be the number format that is used to print and parse the value of the number. number may be a null pointer if you want to use the normal number format.

Note that the number format becomes part of the date format, so copy the number format before passing it, if appropriate. required and shown indicate whether the number is required when parsing a date and shown when printing a date, respectively.

vdateAddFormatText

        void vdateAddFormatText(
                vdateFormat * format,
                vdateFormatIterator * before,
                vchar * print,
                vregexp * match);
Adds text to a date format. The text is added before the position indicated by before, or at the end of the format if before is a null pointer. before still points to the same element after the call.

The string print indicates how the text appears when the date is printed, while the regular expression match should match any variation of the text as it can be parsed. If match is a null pointer, print must be matched exactly when parsing.

Note that the space occupied by match becomes part of the date format.

vdateParse

        void vdateParse(
                const vchar * source,
                const vchar ** end,
                vdate * dest);
Parses a date from a string. source points to a null-terminated string to extract the date from. If end is non-null, the vchar * it points to is set to the character after the last character used to represent the parsed date, so parsing can continue after the date. The date is parsed in the default format. Any fields that appear in the string representation of the date are set in dest; all other fields of dest remain unchanged.

A Date/Syntax exception is generated if source does not point to a string in the appropriate format, and a Date/Range exception is generated if any numeric components of the date read are out of the legal range.

        void vdateSetSecond(
                vdate * date,
                int second);
Sets the second of the given date. It generates a Date/Range exception if second is not in the range 0-59.

vdateSetTimeZoneOffset

        void vdateSetTimeZoneOffset(
                vdate * date,
                int minutesFromGMT);
Sets the time zone of the given date, in minutes difference from Greenwich Mean Time. A positive value indicates a time zone East of Greenwich.

vdateSetTimestamp

                vtimestamp timestamp);

vdateSetYear

        void vdateSetYear(
                vdate * date,
                int year);
Sets the year of the given date.

Note that this is the full year, not just the year within the century.
