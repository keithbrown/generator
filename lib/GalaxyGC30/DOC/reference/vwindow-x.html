Types

vwindow

        typedef ... vwindow;
A window is an overlapping display region. Windows are rarely used as user interface objects in their own right; rather, vwindow is a base class for more powerful derived classes such as vdialog and vmenu.

Contains the information needed to represent a single window; objects of this type have classes derived from vwindowClass. Windows are referenced by a pointer to this type. Windows can be created dynamically with vwindowCreate or initialized statically with vwindowInit. A vwindow contains an attributed object, a set of window attributes, and a window system representation. All structure declarations that directly inherit from vwindowClass must declare a vwindow as their first field so that vwindow pointers can be interchanged with pointers to the new data type.

vwindowImage

        typedef ... vwindowImage;
Contains method and variable bindings for objects of type vwindowImage. A window image is a special type of image that allows a window's current pixel contents to be read using vimage methods, created by calling the vwindowToImage method. This image can be used to read the current display representation of the window. The image has the display's color palette, format, and depth. Behavior is undefined if the window is drawn into after the image is constructed or the image itself is modified in any way. This class type is derived from vimage.

vwindowIterator

vwindowNotOpenException

vwindowNoteProc

        typedef ... vwindowNoteProc;
Denotes a notification function that is called when a window handles an event of some importance. window is the window handling the event, and event is the event being handled. Importance is determined by the event handlers in the window's class; the default window class only gives notification for configure and state events. If additional notification is required, you should override the appropriate handler.

vwindowBlock

        void vwindowBlock(
                vwindow * window);
Changes the appearance of the window window to indicate that the window is unavailable to the user. The window will receive and process events normally once the event loop regains control, but this will not remove the blocking designation from the window. The exact implications of blocking depend on the window system and the look-and-feel; examples include changing the cursor to a watch or graying the title bar. This function is normally used by clients to indicate that a lengthy operation is about to occur, during which events will not be serviced. This function has no effect if the window is already blocked. All followers of the window are also blocked. This function is implemented by sending the message vwindowBLOCK to window.

vwindowClone

        vwindow * vwindowClone(
                vwindow * window);
Creates a copy of the window window and returns a pointer to its contents. Storage for the window object is allocated by the Window class. The Window class will reclaim the storage occupied by the window when it is destroyed with vwindowDestroy. This function copies all the attributes of the window except Open; it will clone any attribute objects that are owned by the window. The window to be cloned can be dynamic or static. This function essentially allocates space for a new window and calls vwindowCopyInit on it.

vwindowClose

        void vwindowClose(
                vwindow * window);
Closes the window window so it is no longer visible. This function has no effect on an already closed window. The window releases its input grab if its Modal attribute is TRUE. The window's instance key map, if any, is destroyed. This function may also destroy any window system resources that were allocated for the window. The window may alternatively be placed in the pop-up cache if its Popup attribute is TRUE. All followers of the window are also closed. This function is implemented by sending the message vwindowCLOSE to window.

vwindowComplementColors

        int vwindowComplementColors(
                vcolor *** colors,
                int groupCount,
                int colorCount);
Allocates window system resources, as necessary, so that corresponding colors in color groups are complemented correctly. colors is the color groups to arrange, groupCount is the number of color groups, and colorCount is the number of colors in each group.

For example, when colors[i][m] is complemented with colors[i][n], colors[j][m] will also be complemented with colors[j][n]. (i represents a color group and m represents a color in the group. n is a corresponding color in the color group j.)

This function allows you to arrange for highlighting effects before you start drawing. It should only be called once during application startup, to avoid using too many color cells. It returns TRUE if the colors were arranged successfully, or FALSE if not.

vwindowCopy

        void vwindowCopy(
                vwindow * window,
                vwindow * target);
Copies the information in window into target so that the two windows represent equivalent objects. target must have been created or initialized by a Window class function. window and target must have compatible classes for this to work correctly. If target is open, it is closed before window is copied and opened after the copy is completed. This function copies all the attributes of the window except Class, Open, and Application; it will clone any attribute objects that are owned by the window. This function will destroy any objects allocated internally by target (including its owned attributes) but will not destroy target itself. The windows to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to window.

vwindowDraw

        void vwindowDraw(
                vwindow * window);
Draws the contents of the window window into the current graphics state. The default implementation of this function does nothing, but higher-level window-based objects (dialogs and menus, for example) override it in order to draw their contents. This function is typically called in response to an update event for the given window. It should not be called to refresh the contents of the window; use vwindowInval instead. Note that window drawing functions cannot assume that they are drawing into the contents of the window itself; the current gstate may contain another device, such as an off-screen image or a printer, that is being rendered by a higher level function. If window drawing functions attempt to change the current drawing device or assume things about the device that are not explicitly documented (such as resolution), device independence will be lost. Refer to the Window chapter of the Galaxy Programmer's Guide for a more thorough treatment of window drawing. This function behaves the same way whether the window is open or not, because the current device need not be the window itself.

vwindowDrawDevice

        void vwindowDrawDevice(
                vwindow * window);
Establishes the window window as the target of drawing operations for the current gstate. Further rendering operations will result in marks on the window's interior. The transformation matrix and the clipping path of the current gstate are reset by this function. The view clip is disabled in the new device. This function sets the Flat and StrokeAdjust attributes of the current gstate to the window device defaults (normally 1.0 and TRUE, respectively). The Color, Font, and LineWidth attributes of the current gstate are set to the results of vwindowDetermineForeground, vwindowDetermineFont, and vwindowDetermineLineWidth, respectively. You should call this function to initialize the current gstate when drawing is necessary outside of an invalidation/update event sequence. This should be necessary only when drawing inside a feedback loop; in other cases, invalidation should suffice. Refer to the Window chapter of the Galaxy Programmer's Guide for a more thorough treatment of window drawing. The null device is set instead of the window device if the window is closed. Surround this function with vdrawGSave and vdrawGRestore to preserve the contents of the current gstate. This function is implemented by sending the message vwindowDRAW_DEVICE to window. Note that this function is distinguished from vwindowUpdateDevice by the device flags parameter.

        int vwindowHandleDetail(
                vwindow * window,
                vevent * event);
Handles the pointer detail event event in the window window and returns TRUE if some action was taken in response to the event. A detail event indicates that the pointer has moved after being pressed inside the window's content area. Event coordinates are display units relative to the lower left-hand corner of the window. A detail event contains the next logical pointer position according to the motion granularity of the window system, even if this position may not reflect the most recent pointer motion information available. You should use detail events for maximum accuracy and drag events for maximum responsiveness. The "pencil" of an image-editing tool, for example, would use detail events to obtain an accurate pointer motion history, while the "thumb" of a scroll bar would use drag events to obtain the most up-to-date pointer position. vwindowInvalRect

        void vwindowInvalRect(
                vwindow * window,
                const vrect * rect,
                int flags);
Invalidates the part of the content area of the window window specified by rect. Coordinates are specified in display units relative to the lower left-hand corner of the window. Invalidation indicates to the Window class that the given area no longer reflects the correct appearance of the window and that it should be redrawn as soon as possible. The Window class normally responds by accumulating the invalid area into the window's invalid region and, if this is the first invalidation since the last update event, posting an update event to redraw the invalid region. If the invalid region is empty at the time of the event, drawing will still occur, but no results will be visible. Redrawing is deferred through the use of update events and invalid regions so that multiple invalid areas can be redrawn in a single step. This minimizes flashing and improves drawing performance in most cases. It is important to recognize that extra redrawing in a graphical user interface reduces the tactility of the interface components. flags specifies hints to the Window class about the invalidation. It should be the bitwise OR (|) of zero or more of the following flags:

vwindowINVAL_OPAQUE
Do not erase before redrawing.
vwindowINVAL_IMMEDIATE
Post update event at normal priority.
vwindowINVAL_UPDATE
Update before returning.
vwindowINVAL_DISABLE
Do not invalidate.
The invalid area is erased to the background color before it is redrawn if vwindowINVAL_OPAQUE is not set. You should set this flag to avoid flashing if you know that subsequent redrawing will completely obscure the invalid area. The contents of this rectangle will be redrawn immediately by this function if vwindowINVAL_UPDATE is set; otherwise, the contents will be redrawn during a subsequent update event. The update event for the invalidation is posted at low priority unless vwindowINVAL_IMMEDIATE is set, in which case it is posted at normal priority. vwindowINVAL_DISABLE effectively disables this function; it is only useful with vwindowScrollRect. Zero can be passed for flags under most circumstances. Note that if any part of an invalid region is not opaque, the entire invalid region will be erased prior to redrawing. This behavior may be corrected in a future version of Galaxy, so you should not depend on it. This function has no effect if the window is not open, unless the window is in the pop-up cache, in which case it is removed from the cache. This function is implemented by sending the message vwindowINVAL_RECT to window.
