vobject

        typedef ... vobject;
Attributed objects have named attributes and are organized in a parent-children hierarchy.

This is the base class for most of the Galaxy user-interface classes. Attributed objects can be manipulated symbolically through the Command class rather than through the function entry points provided by the particular object's manager. The API at the vobject level is more useful to implementors of new Galaxy object classes than to users of existing Galaxy classes. All Galaxy user-interface classes (vdialog, vmenuItem, for example) inherit from this class, either as direct subclasses or subclasses of subclasses. Attribute binding methods allow an object's attributes to be accessed symbolically through the Command class. Refer to the ``Command Manager'' chapter of the Galaxy Application Environment Programmer's Guide for more information.

vobjectChildIterator

        typedef ... vobjectChildIterator;
Provides a way of iterating over all of the children of an attributed object.

This iterator is implemented as a viteratorGroup so that subclasses can introduce more child objects without knowing which of its ancestors have introduced which children. Instead, the subclass can provide a new iterator class (any class derived from vobjectPartialChildIterator) that iterates over only the new children introduced. Then instances of that new iterator class can be created and added to this group at the appropriate time (when vobjectBUILD_CHILD_ITERATOR is called).

In particular, when this iterator is started for a particular parent object, the vobjectBUILD_CHILD_ITERATOR method of that attributed object is called. The implementations of these methods may add zero, one, or many iterator parts. Finally, this iterator is advanced and finished as usual. This iterator should only return elements of the vobject class.

vobjectCommand

        typedef ... vobjectCommand;
Commands of this class, when issued or queried, call vobjectIssueSelector or vobjectQuerySelector on the ``Operand'' object (vname_Operand, in the command's context dictionary) with the command's selector as an argument. Therefore, the real work of the command is encapsulated in the Galaxy object itself and not in the command object. Currently, all class-level commands in Galaxy are instances of vobjectCommand.

Command objects are responsible for implementing the actions that an application can perform and for providing access to aspects of application state that are visible to the user-interface. Commands can be issued and queried. When a command is issued, it performs whatever action its owner has designated for it. A command is issued, for example, when a user clicks on a button in a dialog. When a command is queried, it returns information about its current state. A check box item, for example, queries a command to determine its current state when its dialog is opened. The exact mechanism used for issuing and querying depends on the class of the particular command.

vobjectList

        typedef ... vobjectList;
Attributed object lists manage an ordered list of attributed objects, which are considered hierarchical children of the list.

A vobjectList consists of an ordered collection of unique objects, indexed from zero to the number of objects in the list minus one. Object lists contain no objects by default and have no other attributes. The classes vdialogItemList and vmenuItemList are subclasses of vobjectList.

vobjectUndefinedClassException

        typedef ... vobjectUndefinedClassException;
Generated by object loading functions when an undefined class name is encountered in a resource. This usually indicates that the Galaxy module in question was not initialized properly. All Galaxy modules can be initialized at once with the function vstartup. Individual modules can be initialized by calling their GetDefaultClass function and discarding the result (for example, vwindowGetDefaultClass). This exception is also generated when a user-defined class name is encountered in a resource, and the corresponding class object has not been created and named. Refer to vclassSetName for information on how to name classes. This exception contains the name of the class that is not defined and is derived from vexException.

vobjectUnnamedClassException

        typedef ... vobjectUnnamedClassException;

This exception is generated only for user-defined classes, since all Galaxy classes are named during initialization. This exception is derived from vexException.

vobjectBuildHelpContext

        const vchar ** vobjectBuildHelpContext(
                vobject * object,
                int * count);
This method builds up the nested help context for an object. The return result is an allocated array of const vchar *s, the number of strings in the array is set into count. The list is built in reverse order of importance. It is the callers responsibility to release the memory allocated with vmemFree.

The base implemenation of this function builds an array of strings, each of which is a copy of an object's tag. The first string is a copy of the passed object's tag. The second string, if there is one, is a copy of the tag of the object's help parent (as determined by a call to vobjectGetHelpParent). The tags of the parent's parent and so on provide the subsequent strings of the array.

        void vobjectPropagateChange(

                const void * value);
Propagates the change of the attribute attr of the object whose tag is tag from the scope of the object object. The propagation extends through the object's user-interface inheritance hierarchy. The propagation reaches all lower-level objects in the hierarchy, each of which is responsible for updating its internal state, if any, to correspond with the new value. Lower-level objects in the user-interface hierarchy with a tag matching tag immediately update their internal state and visually reflect the attribute change if appropriate. This message returns NULL if the desired attribute is not defined for the object. attr must be a global name obtained from the Name class. The function entry point vobjectGetAttribute is implemented by sending this message to the object.

The default method for the core object class simply checks its arguments. Subclasses that override this method should first determine if the attribute name matches any attribute defined by the subclass and return the appropriate value if a match is found. If the attribute name is unknown, then the subclass should call vclassSendSuper to attempt to read the attribute in its superclass.

vobjectGET_CONTEXT_ATTRIBUTES

        <vobjectGET_CONTEXT_ATTRIBUTES>
Stores all the context-defined attribute bindings of the object object in the dictionary attr2value. Context attributes are characteristics of an object that cannot be changed directly (the object's parent, for example). Each key/value pair in attr2value maps a global attribute name to an attribute value. attr2value must be initialized but need not be empty. attr2value must have the hash function vnameHash. The function entry point vobjectGetContextAttributes is implemented by sending this message to the object.

The default method for the core object class simply stores the object pointer under the name ``Operator'' (vname_Operator) in the argument dictionary. Subclasses that override this method should first call vclassSendSuper to retrieve the context attributes of their parent class, then store each context attribute defined by the subclass in the dictionary.

vobjectGET_HELP_PARENT

        <vobjectGET_HELP_PARENT>
Returns the parent of the object in the object's help hierarchy, or NULL if the object has no help parent.

The default method simply returns the vobjectGetParent for the object.

vobjectGET_ID_ATTRIBUTE

        <vobjectGET_ID_ATTRIBUTE>
Reads the named attribute id from the object object. The type of object addressed by the returned value depends on the attribute being read. This function returns NULL if the desired attribute is not defined for the object. This message is sent by the default vobjectGET_ATTRIBUTE and vobjectGET_ALL_ATTRIBUTES methods: id is obtained by mapping the attribute name through the vobjectID_ATTRIBUTE_DICT of the object's class. The function entry point vobjectGetIDAttribute is implemented by sending this message to the object.

The default method for the core object class returns NULL. Subclasses that override this method should return the appropriate attribute, if it is implemented by the subclass, or call vclassSendSuper if it is not.

vobjectGET_PARENT

        <vobjectGET_PARENT>
Returns the parent of the object object in the user-interface inheritance hierarchy, or NULL if the object has no parent.

The default method for the core object class simply returns NULL. Subclasses that override this method should return a pointer to their parent object in the user interface inheritance hierarchy. The function entry point vobjectGetParent is implemented by sending this message to the object.

vobjectISSUE_SELECTOR

        <vobjectISSUE_SELECTOR>
Issues the command denoted by selector to the object object with the context attr2value. It returns TRUE if the object understood the command, FALSE if it did not. The default issue method of the default object command class sends this message as its implementation. attr2value must have the hash function vnameHash. The function entry point vobjectIssueSelector is implemented by sending this message to the object.

The default method for the core object class dispatches the selectors described in the ``Commands'' section. Subclasses that override this method should first determine if the selector matches one of their own selectors and take the appropriate action if it does. If the selector is unknown, then the subclass should call vclassSendSuper to pass the command to its superclass. Subclass selectors should start at vobjectNEXT_SELECT and increment.

vobjectKEY_MAP

        <vobjectKEY_MAP>
Contains the class key map for a given class, or NULL if the class has no key bindings. This dictionary contains mappings from command key bindings to command names. This variable must be set to the key mapping of the class command space mapping when the class command space is changed. Use vcommandMapSpaceKeys to map the key bindings of a command space. The dictionary must have the hash function vkeyHashStroke, and its keys must be the result of applying vkeyInternStroke to a keystroke.

The default key map for the attributed object class contains the key bindings described in the ``Commands'' section. Subclasses that override their class command space should simply replace the superclass key map with the mapping of the new class command space. The superclass key map does not have to be cloned first, because the command space contains the superclass key bindings as well as the subclass key bindings.

vobjectPROPAGATE_CHANGE_DICT

        <vobjectPROPAGATE_CHANGE_DICT>
Propagates the change of the attribute attr to the objects whose tags are the keys of the dictionary tag2value from the scope of the object object. The change is propagated through the object's inheritance hierarchy. The values of the dictionary tag2value should be the new attribute values. The propagation reaches all lower-level objects in the hierarchy, each of which is responsible for updating its internal state, if any, to correspond with the new values. attr and the keys of tag2value must be global names obtained from the Name Manager. tag2value must have the hash function vnameHash. The function entry point vobjectPropagateChangeDict is implemented by sending this message to the object.

The default method for the core object class attempts to load an attribute value from the change dictionary using the object's tag, and if successful, calls vobjectSetAttribute with the new value. Subclasses that override this method should first call vclassSendSuper to receive changes to their object, then call vobjectPropagateChangeDictFromParent on each of their child objects to propagate the changes down the inheritance hierarchy.

vobjectQUERY_SELECTOR

        <vobjectQUERY_SELECTOR>
Queries the state of the command denoted by selector based on the contents of the object object. attr2value should contain the context attributes of the object that issued the command. The object's non-context attributes are returned in this dictionary. Returns TRUE if the object understood the command, FALSE if it did not. attr2value must have the hash function vnameHash. The function entry point vobjectQuerySelector is implemented by sending this message to the object.

The default query method of the default object command class sends this message as its implementation. The default method for the core object class simply checks its arguments. Subclasses that override this method should first determine if the selector matches one of their own selectors and return the appropriate information if it does. If the selector is unknown, then the subclass should call vclassSendSuper to pass the command to its superclass. Subclass selectors should start at vobjectNEXT_SELECT and increment.

vobjectREMOVE_LIST_INDEX

        <vobjectREMOVE_LIST_INDEX>
Removes an object at a given position from an object list, and returns it. The default method for the default object list class removes the object from the object list. Subclasses that override this method should first call vclassSendSuper to remove the object from the list, then update any subclass-specific state.

vobjectREPLACE_LIST_AT

        <vobjectREPLACE_LIST_AT>
Replaces the object at a given position in an object list and returns the old one.

The default method for the default object list class stores the object in the object list. Subclasses that override this method should first call vclassSendSuper to store the object in the list, then update any subclass-specific state. This function generates a vexArgRangeException exception if index is invalid.

vobjectSET_ALL_ATTRIBUTES

        <vobjectSET_ALL_ATTRIBUTES>
Changes the values of a set of named attributes of the object object. attr2value maps global names into attribute values. The values should be stored as const void* types, but the type of object addressed depends on the attribute being changed. This message has no effect if the desired attribute is not defined for the object. attr2value must have the hash function vnameHash. The function entry point vobjectSetAllAttributes is implemented by sending this message to the object.

The default method for the core object class simply checks its arguments. Subclasses that override this method should first call vclassSendSuper to set the attributes in the parent object, then attempt to read each attribute that the subclass recognizes from the argument dictionary. The subclass should call the appropriate attribute modification function for each successful read.

vobjectSET_ATTRIBUTE

        <vobjectSET_ATTRIBUTE>
Changes the named attribute attr of the object object to the value value. The type of object addressed by value depends on the attribute being changed. This message has no effect if the desired attribute is not defined for the object. attr must be a global name obtained from the Name Manager. The function entry point vobjectSetAttribute is implemented by sending this message to the object.

The default method for the core object class simply checks its arguments. Subclasses that override this method should first call vclassSendSuper to set the attribute in the parent object, then determine if the attribute name matches an attribute defined by the subclass. If the attribute name is recognized, then the subclass should call the appropriate modification function to store the new value.

vobjectSET_ID_ATTRIBUTE

        <vobjectSET_ID_ATTRIBUTE>
Changes the named attribute id in the object object to value. The type of object addressed by value depends on the attribute being changed. This function does nothing if the desired attribute is not defined for the object. This message is sent by the default vobjectSET_ATTRIBUTE and vobjectSET_ALL_ATTRIBUTES methods: id is obtained by mapping the attribute name through the vobjectID_ATTRIBUTE_DICT of the object's class.

The default method for the core object class does nothing. Subclasses that override this method should change the appropriate attribute, if it is implemented by the subclass, or call vclassSendSuper if it is not.

vobjectSET_LIST_AT

        <vobjectSET_LIST_AT>
(Deprecated) Changes the object at the position index of the object list list to object. The object previously at this position is removed from the list, but it is not destroyed. This function generates a vexArgRangeException exception if index is invalid. The function entry point vobjectSetListAt is implemented by sending this message to the object list.

The default method for the default object list class stores the object in the object list. Subclasses that override this method should first call vclassSendSuper to store the object in the list, then update any subclass-specific state.

vobjectSET_LIST_INDEX_INDEX

The default method for the default object list class changes the index of the object in the object list. Subclasses that override this method should first call vclassSendSuper to change the index in the list, then update any subclass-specific state.

vobjectSPACE

        <vobjectSPACE>
Contains the class command space for a given class, or NULL if the class has no command space. This command space is used to determine class-wide command bindings.

The default command space for the attributed object class contains the command definitions described in the ``Commands'' section. Subclasses that override this variable should first clone the command space of their superclass, then add their new binding dictionaries to the clone space before storing it in the class.

vobjectUPDATE_ALL_ATTRIBUTES

        <vobjectUPDATE_ALL_ATTRIBUTES>
Sets all the named attributes of the object object to their current bindings, according to the object's scope in its inheritance hierarchy. The function entry point vobjectUpdateAllAttributes is implemented by sending this message to the object.

The default method for the core object class determines the current bindings of the attributes with vobjectAssociateAllAttributes, then sets the new value with vobjectSetAllAttributes. Subclasses that override this method should first call vclassSendSuper to update their object contents, then call vobjectUpdateAllAttributes on each of their child objects.

vobjectUPDATE_ATTRIBUTE

        <vobjectUPDATE_ATTRIBUTE>
Sets the attribute named attr of the object object to its current binding, according to the object's scope in its inheritance hierarchy. This message has no effect if no binding for the attribute exists. attr must be a global name obtained from the Name class. The function entry point vobjectUpdateAttribute is implemented by sending this message to the object.

The default method for the core object class determines the current binding of the attribute (if any) with vobjectAssociateAttribute and sets the new value with vobjectSetAttribute. Subclasses that override this method should first call vclassSendSuper to update the attributes of their object, then call vobjectUpdateAttribute on each of their child objects.
