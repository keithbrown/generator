Types

vhelpwinhelp

        typedef ... vhelpwinhelp;
This help display only operates on MS Windows platforms. Unlike other subclasses of vhelpwinhelp, this display does not use a vwebbrowse instance. Instead, it makes requests to WinHelp to display help data. Also, vhelpwinhelp does not operate using URL's, but instead uses help context strings (usually derived from vobjectBuildHelpContext). The display expects the help data of the help request to be an instance of vhelpref. It will attempt to map the context string in the vhelpref to a context id and send this id to WinHelp. Refer to the Platform Specific Notes for additional information on how context strings are mapped to context ids.

vhelpwinhelpClass

        typedef ... vhelpwinhelpClass;
Contains message and variable bindings for objects of type vhelpwinhelp . Classes declared with this type contain the same set of methods and variables as the MS Windows WinHelp display class, but the behavior of methods and the contents of variables may vary. This class type is derived from vhelpwinhelp.

vhelpwinhelpUndisplayHelp

        void vhelpwinhelpUndisplayHelp(
                vhelpwinhelp * helpdisplay,
                vhelprequest * request);
This method is rarely called because most forms of help allow the user to abort the display of help. This method is not implemented in this base class. Subclasses should override this method to abort the display of help data in different ways.

Macros

vhelpwinhelpCLASS

        #define vhelpwinhelpCLASS(...) ...
This macro contains the method declarations for the helpwinhelp class, this macro expands to the contexts of vhelpwinhelpClass. SUPER<\SAMP> is the type identifier of the class being declared. This type is used by the class subclasses to refer to it. HELPDIS is the type identifier of the instances of the class being declared. This macro should be used by helpwinhelp subclasses to declare the contents of the helpwinhelp class. This macro should come first in the subclass structure declaration.
