Types

vtimer

        typedef ... vtimer;
Defines a timer consisting of a period, a call-back procedure, and call-back data. A timer may be either one-shot or recurring. A timer may be synchronous or asynchronous with respect to other handlers.

vtimerCallbackProc

        typedef ... vtimerCallbackProc;
(Deprecated) Describes a procedure that is called by a vtimer at periodic intervals. This deprecated type exists for backward compatibility. The recommended type is vtimerNoteProc.

vtimerClass

        typedef ... vtimerClass;
Contains method and variable bindings for object of type vtimerClass. Classes declared with this type contain the same set of methods and variables as the default vtimer class, but the behavior of methods and the contents of variables can vary. This class type is derived from vobservableClass

vtimerNoteProc

        typedef ... vtimerNoteProc;
Describes a notification procedure that is called by a vtimer at periodic intervals. Every time the vtimer fires, it sends the message vtimerNOTIFY to it. For more information refer also to sections:
  • vtimerSetNotify
  • vtimerNOTFY
  • vtimerStart

Providing a notification procedure with vtimerSetNotify is equivalent to overriding the vtimerNOTIFY message handler.

vtimerSyncProc

        typedef ... vtimerSyncProc;
Denotes a procedure responsible for synchronizing the Event class event queue with the window system event queue. When synchronization is necessary, the message vtimerSYNCHRONIZE is sent to the vtimer. Synchronization may be necessary for timers with interactive visual effects, such as auto-scrolling. The default handler of this message calls the synchronization procedure. The procedure normally used for this purpose is vwindowSyncEvents. Refer also to the section vtimerSetSyncProc, and vtimerSYNCHRONIZE

Providing a synchronization procedure with vtimerSetSyncProc is equivalent to overriding the vtimerSYNCHRONIZE message handler.

vtimerAssert

        void vtimerAssert(
                const vtimer * timer);
Asserts the validity of the vtimer. Will call vdebugAbort with an appropriate message if the timer is invalid. Otherwise does nothing. This is more fine grained than vtimerValidate. Refer also to the section vtimerHandledAssert for more information.

vtimerClone

        vtimer * vtimerClone(
                const vtimer * timer);
Creates a new vtimer whose contents are identical to the vtimer timer. Cloning a running timer is inadvisable. Doing so will generate a warning and will result in a non-running vtimer.

When debugging is enabled, calls to this function are replaced with calls to vtimerCloneDebug passing it the "site" of the clone function call.

vtimerNotify

        void vtimerNotify(
                vtimer * timer);
Called when the vtimer fires. Subclasses should either override this member function or provide a function with vtimerSetNotify to add a notification procedure that will be called when the timer fires.

The default implementation calls the notification procedure set by vtimerSetNotify, if any. This function is implemented by sending the message vtimerNOTIFY to the vtimer.

vtimerRestart

        void vtimerRestart(
                vtimer * timer);
Restarts the vtimer if it is running, and starts it if it is not running. This is different from vtimerStart, which does nothing when called on a running timer.

When debugging is enabled, calls to this function are replaced with calls to vtimerRestartDebug (passing it the "site" of the restart function call.)

vtimerSetAsynchronous

        void vtimerSetAsynchronous(
                vtimer * timer);
Sets the vtimer to be asynchronous. An asynchronous timer calls its Notify procedure as soon as the timer expires instead of waiting until the currently executing Notify procedure completes. This means that this timer's Notify procedure may be called while another Notify procedure is in the middle of a statement deep inside itself. Extreme care must be taken when using asynchronous timers.

vtimerSetSynchronous

        void vtimerSetSynchronous(
                vtimer * timer);
Sets the vtimer to be synchronous. A synchronous timer will wait for a currently executing observation procedure to complete before allowing the timer to fire again. Refer also to the section vtimerSetAsynchronous.

vtimerStart

        void vtimerStart(
                vtimer * timer);
Initializes the countdown of a timer to its period and starts the countdown. When the countdown completes, the timer will fire, sending the vtimerNOTIFY message to the vtimer. Does nothing if the timer is already running. Refer also to the section vtimer.

When debugging is enabled, calls to this function are replaced with calls to vtimerStartDebug (passing it the "site" of the start function call).
