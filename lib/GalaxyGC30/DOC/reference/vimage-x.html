Types

vimage

        typedef ... vimage;
Contains the information needed to represent an image. Images are referenced by a pointer to this type. Images can be created dynamically with vimageCreate or initialized statically with vimageInit. A vimage contains a pixmap, a color palette, and an optional transparency.

An image is a two-dimensional array of mapped pixel values on top of pixmaps and color palettes. Two types of images are available: palette and true color. A palette image is an image where the pixmap values represent an index into a color palette. A true color image has no palette; the pixmap values themselves represent actual colors.

A palette image is typically used in an image where many colors are used repeatedly throughout the image and the total number of colors in the image is low. True color images are desirable when there are a large number of colors in the image.

vimageClass

        typedef ... vimageClass;
Contains the message definitions for a class of images. This class type is derived from vpixmapClass.

vimageDimension

        typedef ... vimageDimension;
Denotes an integral type that can hold a width or height of an image. Note that image dimensions are specified through the Image Library API using the type int, but they are truncated internally, if necessary, to the type vimageDimension.

vimageValue

        typedef ... vimageValue;
Denotes an integral type that can hold a pixel value of an image. The legal range of image values actually ranges from zero to the number of colors in the image minus one.

vimageClone

        vimage * vimageClone(
                vimage * image);
Creates a copy of the image image and returns a pointer to its contents. Storage for the image is allocated by the Image Library. The Image Library reclaims the storage occupied by the image when it is destroyed with vimageDestroy. This function copies all the attributes of the image, and it clones any attribute objects that are owned by the image. The image to be cloned can be dynamic or static. The map of the image is cloned also. This function essentially allocates space for a new image and calls vimageCopyInit on it.

vimageCopy

        void vimageCopy(
                vimage * image,
                vimage * target);
Copies the information in image into target so that the two images represent equivalent objects. target must have been created or initialized by an Image Library function. This function copies all the attributes of the image, and it clones any attribute objects that are owned by the image. This function destroys any objects allocated internally by target (including its owned attributes) but does not destroy target itself. The image map is automatically copied. vimageCreate

        vimage * vimageCreate();
Creates a new image and returns a pointer to its contents. Storage for the image is allocated by the Image Library. The Image Library reclaims the storage occupied by the image when it is destroyed with vimageDestroy. The new image is created with a set of default attributes, as documented in the Image chapter of the Galaxy Programmer's Guide. A map is automatically created for the image. The pixel contents of the image are initially undefined. This function essentially allocates space for a new image and calls vimageInit on it.

Space for the image is allocated dynamically based on the object size of the class. The Class Manager will reclaim the storage occupied by the image when it is destroyed with vimageDestroy. This function essentially allocates space for a new instance and calls vimageInitOfClass on it.

vimageCreateSpecific

Creates a new image of width w and height h and returns a pointer to its contents. The palette of the image is given space for numColors colors, and the depth of its map is set such that it can represent at least this many pixel values. Other attributes of the image are set to their defaults, as documented in the Image chapter of the Galaxy Programmer's Guide. Storage for the image is allocated by the Image Library. vimageDestroy

        void vimageDestroy(
                vimage * image);
Destroys the image represented by image, which must have been created or initialized by an Image Library function. Use this function to reclaim the storage occupied by an image. The image should not be referenced by any other object when it is destroyed. This function attempts to free the storage occupied by image only if it was allocated by vimageCreate, vimageCreateSpecific, or vimageClone. This function also destroys any attribute objects that are owned by the image, including its map.

vimageGetMap

        vpixmap * vimageGetMap(
                vimage * image);
Returns the pixmap that contains the pixel contents of the image image. You can read the contents of this pixmap freely and modify certain parts of it safely, but you cannot replace it entirely because it is an integral part of the image. In general, it is safe to use any Pixmap Library function on an image map that does not have an Image Library counterpart. Functions that operate on the pixel contents of the image map (scanner and bit copying functions, for example) must be called from the Pixmap Library API because there is no equivalent functionality in the Image Library.

vimageInit

        void vimageInit(

Initializes an image in the memory addressed by image. The attributes of the image are set to their default values, as documented in the Image chapter of the Galaxy Programmer's Guide. A call to vimageDestroy frees any internal objects allocated by the Image Library but does not free the structure itself. A map is automatically initialized for the image. The pixel contents of the image are initially undefined.

vimageSetDepth

        void vimageSetDepth(
                vimage * image,
                int depth);
Changes the number of legal color values for the pixel contents of the image image to 2^depth-1. This function also changes the depth of the image map. Has no effect if the image is already of depth depth. If the image is a palette image color slots from the images color palette may be added or removed to match the new number of colors. Any new color slots allocated by this function are initialized as if by vcolorInitSpec (refer to the Color chapter of the Galaxy Programmer's Guide for details). This routine is identical to vimageSetNumColors where the number of new colors is 2^depth-1.

vimageSetSize

        void vimageSetSize(
                vimage * image,
                int w,
                int h);
Changes the size of the image image to w pixels by h pixels. Any contents of the image are destroyed by this function; to preserve the image contents, use vimageSetSizePreserve. This function has no effect if the image is already of the requested size. The size of the image map is automatically changed by this function. This function generates a vexArgRangeException exception if w or h is less than one or greater than the maximum allowable pixmap dimension (at least 32767, but maybe greater).

vimageSetSizePreserve

        void vimageSetSizePreserve(
                vimage * image,
                int w,
                int h,
                int preserve,
                vimageValue background);
Changes the size of the image image to w pixels by h pixels and preserves the image contents. This function has no effect if the image is already of the requested size. The size of the image map is automatically changed by this function. This function generates a vexArgRangeException exception if w or h is less than one or greater than the maximum allowable pixmap dimension (this is at least 32767, but may be greater).

The flags in preserve specify how the existing contents of the image should be positioned in the new content area and whether or not to clear parts of the image that are exposed by the size change. preserve should be vpixmapPRESERVE_NONE or the bitwise OR (|) of two or more of the following constants:

vpixmapPRESERVE_HORIZ_LEFT
Align existing contents left.
vpixmapPRESERVE_HORIZ_CENTER
Align existing contents center.
vpixmapPRESERVE_HORIZ_RIGHT
Align existing contents right.
vpixmapPRESERVE_VERT_TOP
Align existing contents top.
vpixmapPRESERVE_VERT_CENTER
Align existing contents center.
vpixmapPRESERVE_VERT_BOTTOM
Align existing contents bottom.
vpixmapPRESERVE_CLEAR_EXPOSED
Clear exposed area to background.

Note that you can specify only one horizontal flag and one vertical flag at a time. background is ignored if vpixmapPRESERVE_CLEAR_EXPOSED is not set.

vimageSetTransparency

        void vimageSetTransparency(
                vimage * image,
                vimage * transparency);
Changes the transparency of the image image to transparency. The image transparency determines which pixels of the image are opaque and which are transparent: transparency pixels with the value one indicate that the corresponding image pixel is opaque, while a pixel value of zero indicates that the corresponding image pixel is transparent. During drawing, opaque indicates that the image map color value is painted on the target device, while transparent indicates that the color value is not painted. Use vdrawImageComposite or one of its variants to draw images with transparency taken into account.

transparency can be NULL, in which case the entire image is considered to be opaque. The transparency color map is ignored. This function generates a vexArgRangeException exception if the number of legal color values in the transparency image is not two. If transparency is the current transparency attribute of the image and the transparency is owned (refer to vimageSetTransparencyOwned), then this function changes the transparency attribute to shared without destroying the transparency (this is a special case).

vimageSetTransparencyOwned

        void vimageSetTransparencyOwned(

Changes the transparency of the image image to transparency and gives the image ownership of the transparency. The image transparency determines which pixels of the image are opaque and which are transparent: transparency pixels with the value one indicate that the corresponding image pixel is opaque, while a pixel value of zero indicates that the corresponding image pixel is transparent. During drawing, opaque indicates that the image map color value is painted on the target device, while transparent indicates that the color value is not painted. Use vdrawImageComposite or one of its variants to draw images with transparency taken into account. The transparency color map is ignored.

This function generates a vexArgRangeException exception if the number of legal color values in the transparency image is not two. The transparency is destroyed when the image is destroyed or the transparency is changed. The transparency is also cloned when the image is cloned or copied.

vimageSetTrueColor

        void vimageSetTrueColor(
                vimage * image,
                int flag);
Changes the image type to true color if flag is TRUE, the image type is changed to palette if flag is FALSE. A true color image is an image where the actual pixel values from the image's map represent a color. The pixel values from a palette image represent an index into an image's color palette---no palette is required for a true color image. True color images are only available in depths of one, twenty-four, and thirty-two bits. Use vimageSetDepth to specify the depth of an image. One bit true color images can be used as masks or drawn directly. When a one bit true color image is drawn, zero pixels are interpreted as black, and one pixels are interpreted as white. Twenty-four and thirty-two bit images both interpret pixel values the same way. Eight bits of red, green, and blue are packed into the low order bits.

xxxx xxxx rrrr rrrr gggg gggg bbbb bbbb

The eight bits of each color component allow a range of 0-255. A pixel value of 0, 0, 0 would be black and pixel value of 255, 255, 255 would be white. This is based on the RGB color model. The upper eight bits do not exist on a twenty-four bit image, though they may be used for application specific data. While it is guaranteed that the upper eight bits will always be available to the application, future releases of Galaxy may use these bits for compositing and possible other operations. However, these operations will have to be activated by the API and thus Galaxy will not touch these upper eight bits unless instructed.

vimageStore

        void vimageStore(
                vimage * image,
                vresource resource);
Writes the contents of the image image into the resource resource. The image can be recovered from the resource at a later time (perhaps by another program) with vimageLoad or vimageLoadInit. Note that not all of the images attributes are stored in the resource. (Attributes that are stored in resources are documented in the Image chapter of the Galaxy Programmer's Guide.) The image map is automatically stored in the resource.

To make an image into a resource, first call vresourceMake to create an untyped resource in the desired location, then call vimageStore to write the image into the resource. This is illustrated in Example.

Sample Code: Making an image into a resource
vimage   *image;    /* contains an image */
vresource fileResource, imageResource;

fileResource  = vresourceOpenFile( ... );
imageResource = vresourceMake(fileResource,
                              vnameInternGlobalLiteral("MyImage"));

vimageStore(image, imageResource);
vresourceCloseFile(fileResource);

Messages

vimageSET_NEVER_DITHER_FLAG

        <vimageSET_NEVER_DITHER_FLAG>
Sets whether an image should or should not be dithered when it is drawn. If the flag value is set to TRUE, the image is not dithered; conversely, if flag is FALSE the image is dithered. The function entry point vimageSetNeverDitherFlag is implemented by sending this message to the image.

The default method for the default image class changes the dither flag of the image and flushes the window system representation, if necessary. Subclasses that override this method should first call vclassSendSuper to set the new flag, then update any subclass-specific state.

Macros

vimageCLASS

        #define vimageCLASS(...) ...
This macro contains the method declarations for the image class, this macro expands to the contexts of vimageClass. SUPER<\SAMP> is the type identifier of the class being declared. This type is used by the class subclasses to refer to it. IMAGE is the type identifier of the instances of the class being declared. This macro should be used by image subclasses to declare the contents of the image class. This macro should come first in the subclass structure declaration.
