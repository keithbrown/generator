Types

vsequence

        typedef ... vsequence;
Maintains an ordered list of owned or unowned object instances.

The elements of the sequence object must be objects whose class derives from vinstance. The indexes of the elements range from zero to the number of elements minus one.

Whether an object is owned or not by the sequence object is determined by how it is added to the sequence object. The Owned entry points imply that the caller will not destroy the object and that the sequence object can and will destroy the object when it itself is destroyed. The Shared entry points imply that the caller will eventually destroy the object itself, but not before the sequence object is destroyed (or else the behavior of the sequence object is undefined).

vsequenceLoadable

        typedef ... vsequenceLoadable;
Maintains a sequence object which can be stored in and loaded from a resource.

All elements of these loadable sequence objects must be of a class derived from the vloadable class.

Creates a new copy of a sequence object.

The elements of the copied sequence object must either be vloadable objects or unowned. Owned elements are cloned and the clones are owned by the new sequence object. Unowned elements are added as unowned reference in the new sequence object.

Messages

vsequenceAPPEND_ELEMENT

        <vsequenceAPPEND_ELEMENT>
Implements the adding of new elements to the end of the sequence object, without regard to ownership.

Both Owned and Shared entry points use this method to append new elements. They then mark ownership or lack of ownership by calling vsequenceSetIndexOwned. Insert entry points call this method, too, followed by a call to vsequenceMoveIndex.

vsequenceELEMENT_CLASS

        <vsequenceELEMENT_CLASS>
The base class of all elements for the class of sequence objects.

vsequenceMOVE_INDEX

        <vsequenceMOVE_INDEX>
Implements the moving of the element found at the first specified index to the second specified index, shifting all of the intervening elements.

The ownership of all affected elements are preserved during this operation.

vsequenceREMOVE_ELEMENT_AT

        <vsequenceREMOVE_ELEMENT_AT>
Implements the removing of the element at the specified index from the sequence object.

If the element was owned by the sequence object, then it and its ownership is returned to the caller. Otherwise, NULL is returned.

vsequenceREPLACE_ELEMENT_AT

        <vsequenceREPLACE_ELEMENT_AT>
Implements the replacing of an existing element with a new element, without regard to ownership.

Both Owned and Shared entry points use this method to replace elements. They then mark ownership or lack of ownership by calling vsequenceSetIndexOwned.

The element that has been replaced if and only if it was owned by the sequence object in the first place. Otherwise, NULL is returned.

vsequenceSET_INDEX_OWNED

        <vsequenceSET_INDEX_OWNED>
Implements the setting of whether the element at the specified index is owned by the sequence object.
