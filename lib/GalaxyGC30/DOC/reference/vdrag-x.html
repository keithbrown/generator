Types

vdrag

        typedef ... vdrag;
Denotes an instance of a drag. A drag provides a graphical way to drag one or more items from one screen location to another. The drag is initiated from a button down, drag, or detail event with the call vdragStart.

The Drag class tracks the drag until the drag is completed with a button up event, or the drag is aborted with vdragStop. On completion of a drag, a notification function is called to denote the ending location of the drag.

vdragACTION_CHANGE_DRAG_STATE

vdragACTION_ABORT
Indicates that the drag has been aborted.

vdragEngine

        typedef ... vdragEngine;
Contains the information needed to represent a drag engine. A drag engine is responsible for conducting drag operating with specific vdrags, including managing the communication with all dropsites registered with the engine and the current drag. The base vdragEngine class contains the functionality necessary to set up certain state for a drag operation, but cannot perform the drag itself. The vdragInternalEngine class does contain the functionality necessary to perform a drag internal to the application.

Messages

vdragABORT_ENGINE_DRAG

        <vdragABORT_ENGINE_DRAG>
Aborts the current drag being processed by the drag engine engine. The function entry point vdragAbortEngineDrag is implemented by sending this message to the drag engine.

The default method for the default drag engine class communicates with any dropsite which may be currently interacting with the drag, notifies the drag of the abort, and cleans up the drag.

vdragCLEANUP_ENGINE_DRAG

        <vdragCLEANUP_ENGINE_DRAG>
Cleans up any application state modified by the drag engine during the course of a drag. The function entry point vdragCleanUpEngineDrag is implemented by sending this message to the engine.

The implementation in the base drag engine class restores the state of the screen and unlocks the drag engine. The internal engine class restores the application event handlers overridden in vdragSetUpEngineDrag.

vdragCONVERT_ENGINE_DRAG_SCRAP

        <vdragCONVERT_ENGINE_DRAG_SCRAP>

        <vdragENGINE_DRAG_INTERSECTS_DROPSITE>
tests for intersection between a dropsite and an event for the drag engine's current drag. During a drag operation, this message will only be sent for dropsites in the same window as the drag's event. The function entry point vdragEngineDragIntersectsDropsite is implemented by sending this message to the engine.

Notifies the drag to provide feedback to the user. Drawing should be performed into the rectangle specified in drawRect using the vdrawcontext provided in context. dragAction will be one of:

vdragDROP_ACTION_NONE
No dropsites have indicated that they will accept the drag.
vdragDROP_ACTION_MOVE
A dropsite has indicated that it will accept the drag as a move operation.
vdragDROP_ACTION_COPY
A dropsite has indicated that it will accept the drag as a copy operation.
vdragDROP_ACTION_LINK
A dropsite has indicated that it will accept the drag as a link operation.

The default implementation of vdragNOTIFY_FEEDBACK_WITH will invoke the deprecated method vdragNOTIFY_FEEDBACK to maintain backward compatibility. This is subject to change, and should not be depended on in future releases of the product.

vdragPROCESS_ENGINE_DRAG

        <vdragPROCESS_ENGINE_DRAG>
Synchronously processes a given drag using the drag engine, returning the drop action which took place. The function entry point vdragProcessEngineDrag is implemented by sending this message to the drag engine. The internal drag engine class implementation processes the drag in a sub-event loop.

vdragREGISTER_ENGINE_DROPSITE

        <vdragREGISTER_ENGINE_DROPSITE>
Registers a dropsite with the engine, thereby making that dropsite eligible to interact with drags being processed by the engine. The function entry point vdragRegisterEngineDropsite is implemented by sending this message to the drag engine.

vdragSETUP_ENGINE_DRAG

        <vdragSETUP_ENGINE_DRAG>
Initializes the pieces of state in the drag engine which correspond to a specific drag. This message is sent to the engine when a drag using that engine is started. The function entry point vdragSetUpEngineDrag is implemented by sending this message to the drag engine.

The implementation in the base engine class locks the engine for the duration of the drag, communicates with any dropsites beneath the drag, and notifies the drag to provide any feedback. The internal drag engine class overrides the application event handlers to intercept all pointer and key events for the duration of the drag.

vdragSET_ACTION_NOTIFY

        <vdragSET_ACTION_NOTIFY>
Sets the action notification procedure for the drag. When set, the action notification procedure is invoked by the base implementation of vdrag to communicate various drag actions to the drag. The function entry point vdragSetActionNotify is implemented by sending this message to the drag.

vdragSET_DATA

        <vdragSET_DATA>
Sets the drag's generic data pointer to data. This may be a pointer to any information you want to attach to the drag. data may be NULL. The function entry point vdragSetData is implemented by sending this message to the drag.

vdragSET_ENGINE

        <vdragSET_ENGINE>
Sets the drag engine to be used to coordinate the drag drag. The function entry point vdragSetEngine is implemented by sending this message to the drag.

vdragSET_ENGINE_CLIPBOARD_ENGINE

        <vdragSET_ENGINE_CLIPBOARD_ENGINE>
Sets a drag engine's clipboard engine to be used when converting drag scraps. The function entry point vdragSetEngineClipboardEngine is implemented by sending this message to the drag engine.

vdragSET_FEEDBACK_NOTIFY

        <vdragSET_FEEDBACK_NOTIFY>
(Deprecated) Sets the drag's feedback notification procedure. The function entry point vdragSetFeedbackNotify is implemented by sending this message to the drag.

This message has been deprecated. Override vdrag to receive notification when drawing is to be performed.

vdragSET_MAX_SIZE

        <vdragSET_MAX_SIZE>
Sets the maximum width and height of the area that will be obscured as a drag tracks across the screen. The width and height correspond to the bounding rectangle of the item(s) that are drawn during a drag. The function entry point vdragSetMaxSize is implemented by sending this message to the drag.

vdragSET_NATIVE_FEEDBACK

        <vdragSET_NATIVE_FEEDBACK>
Enables or disables native feedback for the drag. When native feedback is enabled, the feedback cursor is set during the drag to the correct cursor for the current look-and-feel according to the drop action. The function entry point vdragSetNativeFeedback is implemented by sending this message to the drag.

vdragSET_OFFSET

        <vdragSET_OFFSET>
Sets the offset of the mouse location in the drag drag. The offset determines the mouse location in relation to the bounding rectangle of the item(s) composing the drag. The default is 0,0 which means the item(s) will be dragged from the lower left hand corner. To drag from the center, the offset would be set to the maximum width/2, maximum height/2. The function entry point vdragSetOffset is implemented by sending this message to the drag.

vdragSET_SCRAP

        <vdragSET_SCRAP>
Sets the drag drag's scrap to scrapvdragConvertEngineDragScrap to retrieve the contents of the drag engine's clipboard engine. The function entry point vdragSetScrap is implemented by sending this message to the drag.

vdragSET_USER_UNDRAW

        <vdragSET_USER_UNDRAW>

vdragSET_VALID_DROP_ACTIONS

        <vdragSET_VALID_DROP_ACTIONS>
Sets the valid drop actions for the drag drag. actionFlags should be the bitwise OR of the desired vdragDROP_ACTION_ flags. A dropsite may use the drag's valid drop actions to determine what operation should take place if the drag were to be dropped in that dropsite. The function entry point vdragSetValidDropActions is implemented by sending this message to the drag.

vdragUNREGISTER_ENGINE_DROPSITE

        <vdragUNREGISTER_ENGINE_DROPSITE>
Removes a dropsite from the engine's dropsite registry. Drags processed by that engine will no longer be eligible to interact with that dropsite. The function entry point vdragUnregisterEngineDropsite is implemented by sending this message to the drag engine.

Macros

vdragCLASS

        #define vdragCLASS(...) ...
Contains the method and variable declarations for drag classes. This macro expands to the contents of vdragClass. SUPER is the type identifier of the class being declared (for example, vdragClass). This type is used by the class's subclasses to refer to it. DRAG is the type identifier of instances of the class being declared (for example, vdrag). This macro should be used by drag subclasses to declare the contents of the drag class. This macro should come first in the subclass structure declaration.

vdragEngineCLASS

        #define vdragEngineCLASS(...) ...
Contains the method and variable declarations for drag engine classes. This macro expands to the contents of vdragEngineClass. SUPER is the type identifier of the class being declared (for example, vdragEngineClass). This type is used by the class's subclasses to refer to it. ENGINE is the type identifier of instances of the class being declared (for example, vdragEngine). This macro should be used by drag engine subclasses to declare the contents of the drag engine class. This macro should come first in the subclass structure declaration.
