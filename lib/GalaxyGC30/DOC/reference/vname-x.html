Types [6] Functions [51] Constants [3]

Types

vname

        typedef ... vname;
A vname is a unique, atomic identifier for an arbitrary character string, implemented as a pointer to a character string that uniquely identifies that character string in the context of a given name table. A name is represented as a pointer to its character string contents so its printable representation can be easily recovered.

Note that you can pass a const vname* in place of a const vchar*, but not vice-versa.

The type vname is compatible with vchar in the same way that vstr is compatible with vchar. Under no circumstances can a const vname* be passed in place of a (non-const) vchar* or vstr*.

vnameCustomHashFunc

        typedef ... vnameCustomHashFunc;
Denotes a generic hash function that returns an identifying value for a given pointer value and a hash context pointer.

The hash context pointer passed to the hash function is the one passed to the vnameCreateCustom, vnameInitCustom, or vnameSetCustomData functions.

Whether the pointer, or the data it addresses, is used for this value depends on the context in which this type is used. For example, the context can be name tables which use the pointer contents (a string), dictionaries, or the pointer itself.

A function of this type should return a value for each pointer, or pointer contents, that is passed to it. This value must be the same for each identical pointer or contents. Values should be as distinct as possible to obtain a good distribution. vnameHashString is a good hash function for arbitrary character strings.

vnameTable

        typedef ... vnameTable;
A name table is a collection of zero or more unique names. You can pass a character string to a name table to retrieve the unique name for that character string. Given two names in the same name table, you can compare their pointer values to determine equality without comparing the characters of the names themselves. The fact that names can be compared quickly makes them particularly well suited as keys for arbitrary values. Names are often used as dictionary keys and resource tags.

vnameTableIterator

        typedef ... vnameTableIterator;
Denotes an abstract pointer variable used to enumerate the contents of a name table. Any given iterator refers to a particular name in a particular name table. An iterator behaves correctly only if it is used with the name table that it was initialized with. An iterator becomes invalid if the name table that it references is modified in any way. Iterators are normally declared as automatic variables of a function and are initialized with vnameInitTableIterator.

vnameCreate

        vnameTable * vnameCreate(
                vnameHashFunc hashFunc);
Creates an empty name table with hash function hashFunc and returns a pointer to its contents. hashFunc is used to distribute the names in the table. The hash table will be automatically resized when necessary, although it can be explicitly expanded with vnameSetHashSize. Use vnameHashString as a hash function unless you have a function that is better tuned to your application.

The new, empty name table uses the hash function hashFunc and passes the generic customData pointer to that hash function every time it is called. hashFunc is used to distribute the names in the table. The hash table will be automatically resized when necessary, although it can be explicitly expanded with vnameSetHashSize.

vnameDestroy

        void vnameDestroy(
                vnameTable * table);
Destroys the name table addressed by table, which must have been created or initialized by a Name class function. Use this function to reclaim the storage occupied by a name table. The name table should not be referenced by any other object when it is destroyed. This function only attempts to free the storage occupied by table if it was allocated by vnameCreate. All the names in the table are destroyed also.

vnameGet

        const vname * vnameGet(
                vnameTable * table,
                const vchar * str);
Returns the existing name for the character string str with respect to the name table table, or NULL if the string is not in the name table. str is not stored in the name table. You can use this function to determine if a given string is already in a name table. The pointer returned by this function is guaranteed to be the same for all strings with the same character pattern and different for all strings with different character patterns. Pointer values, instead of string characters, can thus be compared directly. However, only names in the same name table can be compared meaningfully.

The name returned by this function points to a string that is identical to the original string, except that the name is always in the vchar character set. Identifiers returned by this function must not be subsequently modified.

vnameHash

        unsigned long vport__IMPORT vnameHash(
                const void * name);
Returns an integer hash value for the name name, which must be a name object returned by the Name Manager. name is declared as a pointer to void, although it is actually treated as a pointer to vname. The address of this function should be passed to the Dictionary class when names are used exclusively as keys for a dictionary (this is the usual case). If used on other data or even regular character strings, its behavior is unpredictable. This function is significantly more efficient than vnameHashString when used in dictionaries that have names as keys.

vnameHashString

        unsigned long vnameHashString(
                const void * s);
Returns an integer hash value for the string s. s is declared as a pointer to void, although it is actually treated as a pointer to vchar. This is the standard hash function for character strings. The address of this function should be passed to the Dictionary class when character strings are used exclusively as keys for a dictionary. If used on other data, its behavior is unpredictable.

vnameIntern

        const vname * vnameIntern(
                vnameTable * table,
                const vchar * str);
Returns a name for the character string str, with respect to the name table table. str is copied into buffered storage before it is stored in the table. The pointer returned by this function is guaranteed to be the same for all strings with the same character pattern and different for all strings with different character patterns. This function may expand the hash size of the name table if there are not enough free entries. Identifiers returned by this function must not be subsequently modified.

These constants identify global names that have already been interned in the global name table by the Name Manager. Each has the type const vname*. vnameNULL is a name whose contents is the empty string. All other identifiers denote the name whose contents is the text after the vname_ part of the identifier. For example, vname_Foreground denotes a name whose character contents is Foreground: this constant is the result of vnameInternGlobalLiteral("Foreground"). Note that the global name table must be initialized before you can use these constants. You can initialize the global name table explicitly with the statement vnameInternGlobalLiteral("");.
