Types

vtextitem

        typedef ... vtextitem;
A text item is a limited text view object. It is a convenient object to use when you need to display a few lines of text, or when you need to receive a small amount of text from the user. When a text item is created, a text object (vtextitemText) is created with it; when a text item is destroyed, its text object is destroyed also. A text item is a convenience object for those who need text view functionality, but do not need to customize the text object associated with the text view.

vtextitemClone

        vtextitem * vtextitemClone(
                vtextitem * textItem);
Creates a copy of the text item textItem and returns a pointer to the copy. This function is essentially a combination of vtextitemCreate and vtextitemCopy.

vtextitemCopy

        void vtextitemCopy(
                vtextitem * textItem,
                vtextitem * targetItem);
Copies the information in textItem into targetItem so that the two text items represent equivalent text items but separate storage. Any objects created in textItem are duplicated in targetItem as well.

vtextitemCopyInit

        void vtextitemCopyInit(
                vtextitem * textItem,
                vtextitem * targetItem);
Copies the information in textItem into targetItem, so that the memory addressed by targetItem is a text item equivalent to textItem. targetItem should be a pointer to uninitialized memory; the text item initializes it as part of the copy.

vtextitemCreate

vtextitemCreateOfClass

Creates a new text item of class clas and returns a pointer to the new text item. (Memory is allocated by the Text Manager.) The new text item is created with a set of default attributes, determined by the class.

vtextitemDestroy

Destroys the text item represented by textItem and all of its sub-objects that were allocated by the Text Manager. This function attempts to free the storage directly occupied by the text item only if it was allocated by vtextitemCreate or vtextitemCreateOfClass.

vtextitemGetDefaultTextDataClass

        vtextitemTextClass * vtextitemGetDefaultTextDataClass();
Returns the text data object class used by text items when creating the text data object for the item via vtextCreateOfClass. This class can be subclassed to alter the behavior of text data objects created by text items. Text items send themselves the vtextitemCREATE_TEXT_DATA message to actually create the text data object owned by the text item. If you want to use your own text data subclass, you need to subclass vtextviewGetDefaultTextDataClass. To automatically have a text item create a text data object when it is created, you need to override vtextitemCREATE_TEXT_DATA.

vtextitemSetEditable

        void vtextitemSetEditable(
                vtextitem * textItem);
Sets the text item textItem to be editable. Editable text can be edited and selected. The editable text item has a border around it. Making a text item editable is equivalent to calling: 


vtextitemSetSelectable(textItem, TRUE);
vtextitemSetModifyable(textItem, TRUE);
vdialogSetItemBorder(vtextitemGetItem(textItem), TRUE);
vtextviewSetTopMargin(vtextitemGetView(textItem), 0);
vtextviewSetHorzLocked(vtextitemGetView(textItem), FALSE);

vtextitemSetNotify

        void vtextitemSetNotify(
                vtextitem * textItem,
                vtextitemNoteProc noteProc);
Changes the notification procedure for the text item textItem to noteProc. Whenever the text item handles an event of importance, this procedure is called with the text item, event, and text item message as arguments.

vtextviewVSCROLL_NOTIFY
The view has scrolled vertically.
vtextviewHSCROLL_NOTIFY
The view has scrolled horizontally.
vtextviewACCEPT_NOTIFY
A return was entered, or focus of the text item was lost.
vtextviewUPDATE_NOTIFY
The view has been updated.
vtextviewSELECT_NOTIFY
The selection for the view has changed.
vtextviewBUTTON_UP_NOTIFY
A button up event occurred in the text view.
vtextviewBUTTON_DOWN_NOTIFY
A button down event occurred in the text view.

noteProc may be NULL, in which case no notification is given.

vtextitemSetOneLine

        void vtextitemSetOneLine(

Sets whether or not the text item textItem accepts a return character from the keyboard. One-line text items do not allow a return to be entered from the keyboard, but instead send a vtextviewACCEPT_NOTIFY to the text view notification function.

Sets whether the text item textItem is selectable or not. An unselectable text item does not allow selection via the mouse. It does, however, display a selection if one is set programmatically or if one already exists.

vtextitemSetStatic

        void vtextitemSetStatic(
                vtextitem * textItem);
Sets the text item textItem to be static. Static text cannot be edited or selected; it does not accept button-down events. Making a text item static is equivalent to calling: 


vtextitemSetSelectable(textItem, FALSE);
vtextitemSetModifyable(textItem, FALSE);
vdialogSetItemBorder(vtextitemGetItem(textItem), FALSE);
vtextviewSetTopMargin(vtextitemGetView(textItem), 3);
vtextviewSetHorzLocked(vtextitemGetView(textItem), TRUE);

vtextitemSetText

Sets the text of the text item textItem to the text pointed to by text. text is a null-terminated string. Passing NULL for text is equivalent to replacing the text portion of the textitem with an empty string of zero-length.
