Types

vdomain

        typedef ... vdomain;
A domain represents a two and one-half dimensional space which may hold a collection of objects. The domain may have a fixed bounding rectangle specifying the domain's space, or its bounding rectangle may be determined by the positioning of any objects in the domain. A domain may also have any number of observers (derived from vdomainObserverClass) installed into it which will be notified by the domain when certain attributes of the domain and its objects change.

vdomainObjectSetIterator

        typedef ... vdomainObjectSetIterator;
Denotes an abstract pointer variable used to enumerate the contents of a vdomainObjectSet. An iterator becomes invalid if the object set that it references is modified in any way; for example, removing an element from an object set will modify the set and thus should not be done during iteration.

Iterators are normally declared as automatic variables of a function and are initialized with vdomainInitObjectSetIterator.

vdomainObserver

        typedef ... vdomainObserver;
An observer is an object which monitors the operations performed on a domain. A domain can have any number of observers installed via vdomainInstallObserver or vdomainInstallObserverOwned. The domain notifies all observers when certain aspects of the domain change.

vdomainObserverClass

        typedef ... vdomainObserverClass;
Contains method and variable bindings for objects of type vdomainObserver. Classes declared with this type contain the same set of methods and variables as the default domain observer class, but the behavior of methods and the contents of variables may vary. This class type is derived from vobjectClass.

vdomainObserverIterator

        typedef ... vdomainObserverIterator;
Denotes an abstract pointer variable used to enumerate the observers of a domain. An iterator becomes invalid if the set of observers in the domain that it references are modified in any way; for example, uninstalling an observer from a domain will modify the set of observers and thus should not be done during iteration. Iterators are normally declared as automatic variables of a function and are initialized with vdomainInitObserverIterator.

vdomainRectIntersectFunc

        typedef ... vdomainRectIntersectFunc;
Denotes a domain rectangle intersection function that is used to determine if an object and a rectangle intersect. The function returns vTRUE if the object intersects the rectangle, vFALSE if it does not.

vdomainRectIntersectFuncLong

Denotes a long domain rectangle intersection function that is used to determine if an object and a rectangle intersect. The function returns vTRUE if the object intersects the rectangle, vFALSE if it does not.

vdomainSelection

        typedef ... vdomainSelection;
Contains the information needed to represent an object selection. A domain handles creating selections for itself, and it can own multiple selections. Some object operations require a selection as a parameter to indicate what objects to operate on. Object selections are referenced by this type.

vdomainGetUndoActionList

        vundoActionList * vdomainGetUndoActionList(
                vdomain * domain);
Returns the current undo action list for the domain domain. The undo action list is the place where the undo observer posts undo actions. It returns a null pointer if there is no current action list. This either indicates that no undoable actions have been performed, or there is no undo observer to generate actions.

If you perform changes to the domain while there is an undo observer attached, you can retrieve the domain's undo action list for use as an undo action to represent the changes you just made. You can then combine this with other actions, or install it directly in your undo log. Be sure to set the name of the list if the list represents a top-level action in your application. Also, be sure to set the action list to null after you retrieve its contents. This causes the undo observer to begin a new action list when the next change occurs.

If your domain has an attached vdomainview, you do not need to call this function directly. Instead, you should call vdomainviewNotifyUndo after making changes to the domain, which causes your domain view's undo notify procedure to get called with the current action list. The action list is then automatically reset to NULL.
