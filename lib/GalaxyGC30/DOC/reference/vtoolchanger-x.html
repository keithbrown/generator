Types

vtoolchanger

        typedef ... vtoolchanger;
Contains the information needed to represent a single tool changer. Objects of this type have classes derived from vtoolchangerClass.

A tool changer is simple container which holds multiple items, but only displays one at a time. It is used primarily in toolbars where you want an item to have a completely different horizontal and vertical manifestation. The most frequently used form is a popdown combo in a horizontal toolbar that changes to a button when the toolbar becomes vertical. The Selection attribute determines which item will be shown. If two or more items are in the tool changer's item list, the first two will be treated respectively as the horizontal and vertical items. The tool changer will automatically switch between the two when its parent's Vertical attribute changes. Refer to the section ChangedParentAttribute for more information.

vtoolchangerClass

        typedef ... vtoolchangerClass;
Contains method and variable bindings for objects of type vtoolchanger. Classes declared with this type contain the same set of methods and variables as the default tool changer class, but the behavior of methods and the contents of variables may vary. This class type is derived from vcontainerClass.

Functions

vtoolchangerClone

        vtoolchanger * vtoolchangerClone(
                vtoolchanger * changer);
Creates a copy of the tool changer changer and returns a pointer to its contents. Storage for the tool changer object is allocated by the Toolbar Manager. The Toolbar Manager reclaims the storage occupied by the tool changer when it is destroyed with vtoolchangerDestroy. This function copies all the copyable attributes of the tool changer and clones any attribute objects that are owned by the tool changer. The tool changer to be cloned can be dynamic or static. This function essentially allocates space for a new tool changer and calls vtoolchangerCopyInit on it.

vtoolchangerCopy

        void vtoolchangerCopy(
                vtoolchanger * changer,
                vtoolchanger * target);
Copies the information in changer into target so that the two tool changers represent equivalent objects. target must have been created or initialized by a Toolbar Manager function. changer and target must have compatible classes for this to work correctly. This function copies all the copyable attributes of the tool changer and clones any attribute objects that are owned by the tool changer. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The tool changers to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to changer.

vtoolchangerCopyInit

        void vtoolchangerCopyInit(
                vtoolchanger * changer,
                vtoolchanger * target);
Copies the information in changer into target so that the two tool changers represent equivalent objects. target should be a pointer to uninitialized memory---the Toolbar Manager initializes it as part of the copy. This function copies all the copyable attributes of the tool changer and clones any attribute objects that are owned by the tool changer. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The tool changers to be copied can be dynamic or static. This function simply stores the class of changer in target and sends the message vobjectCOPY_INIT to changer.

vtoolchangerCreate

        vtoolchanger * vtoolchangerCreate();
Creates a new tool changer of the default tool changer class and returns a pointer to its contents. Storage for the tool changer object is allocated by the Toolbar Manager. The Toolbar Manager reclaims the storage occupied by the tool changer when it is destroyed with vtoolchangerDestroy. The new tool changer is created with a set of default attributes, as determined by the default tool changer class. The default attributes of the default tool changer class are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new tool changer and calls vtoolchangerInit on it.

vtoolchangerCreateOfClass

        vtoolchanger * vtoolchangerCreateOfClass(
                vtoolchangerClass * clas);
Creates a new tool changer of class clas and returns a pointer to its contents. Storage for the tool changer object is allocated by the Toolbar Manager. The Toolbar Manager reclaims the storage occupied by the tool changer when it is destroyed with vtoolchangerDestroy. The new tool changer is created with a set of default attributes, determined by the class. The default attributes of the default tool changer class are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new tool changer and calls vtoolchangerInitOfClass on it.

vtoolchangerDestroy

        void vtoolchangerDestroy(
                vtoolchanger * changer);
Destroys the tool changer represented by changer, which must have been created or initialized by a Toolbar Manager function. Use this function to reclaim the storage occupied by a tool changer. The tool changer should not be referenced by any other object when it is destroyed. This function only attempts to free the storage occupied by the tool changer if it was allocated by vtoolchangerCreate, vtoolchangerCreateOfClass, vtoolchangerClone, or vtoolchangerLoad. This function will also destroy any attribute objects that are owned by the tool changer. This function is implemented by sending the message vobjectDESTROY to the tool changer.

vtoolchangerGetDefaultClass

        vtoolchangerClass * vtoolchangerGetDefaultClass();
Returns the class used by the vtoolchangerCreate and vtoolchangerInit functions to create tool changers.

This class can be modified to alter the behavior of all tool changers subsequently created by these functions. Changes to this class are not visible to any other applications in the same process, or to those sharing the same display server.

vtoolchangerGetMinSize

        void vtoolchangerGetMinSize();
Returns the minimum size of the tool changer in the variables addressed by w and h. The minimum size of a tool changer is either the minimum size of its selection if non-NULL, or the minimum size according to the superclass. Refer to the sections GetSelection and vdialogGetItemMinSize for more information about the meaning of minimum size.

Subclasses that override this method should first call the parent method to get the tool changer minimum size, then add any subclass-specific minimum size to it.

vtoolchangerInit

        void vtoolchangerInit(
                vtoolchanger * changer);
Initializes a tool changer of the default tool changer class in the memory addressed by changer. The attributes of the tool changer are set to their default values, according to the default tool changer class. The default attributes of the default tool changer class are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Toolbar Manager as if it were created with vtoolchangerCreate. A call to vtoolchangerDestroy frees any internal objects allocated by the Toolbar Manager but does not free the structure itself. This function simply stores the default class in the class field of the tool changer and sends it the message vobjectINIT.

        void vtoolchangerInitOfClass(

Initializes a tool changer of class clas in the memory addressed by changer. The attributes of the tool changer are set to their default values, according to the class. The default attributes of the default tool changer class are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Toolbar Manager as if it were created with vtoolchangerCreateOfClass. A call to vtoolchangerDestroy frees any internal objects allocated by the Toolbar Manager but does not free the structure itself. This function simply stores the class in the class field of the tool changer and sends it the message vobjectINIT.

vtoolchangerLoad

        vtoolchanger * vtoolchangerLoad(
                vresource res);
Reads a tool changer from the resource resource and returns a pointer to its contents. Storage for the tool changer object is allocated by the Toolbar Manager. The tool changer's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function creates an object based on the class of the resource, even if this class is not a tool changer class. The object pointer is cast to a tool changer pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default tool changer class which are stored in resources are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vtoolchangerLoadInit on it.

vtoolchangerLoadInit

        vtoolchanger * vtoolchangerLoadInit(
                vtoolchanger * changer,
                vresource res);
Reads a tool changer from the resource resource and stores it in the uninitialized memory addressed by changer. The tool changer's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function initializes an object based on the class of the resource, even if this class is not a tool changer class. The object attempts to initialize itself in the tool changer structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attribute table in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide documents the attributes of the default tool changer class that are stored in resources. This function stores the class specified by the resource in the class field of the tool changer structure and sends it the message vobjectLOAD_INIT.

vtoolchangerSetSelection

        void vtoolchangerSetSelection();
Changes the selection of the tool changer to the dialog item specified. A tool changer's selection is the item in its item list which is shown and enabled. All other items in the tool changer's item list will be hidden and disabled, giving the illusion that only one item is "on" at a time. If selection is NULL then all items in the tool changer will be hidden and disabled.

Subclasses that override this method should first call the parent method to store the selection in the tool changer, then update any subclass-specific state.

vtoolchangerStartup

        void vtoolchangerStartup();
Initializes the tool changer module. It has no effect if the tool changer module is already initialized.

vtoolchangerStore

        vtoolchanger * vtoolchangerStore(
                vtoolchanger * changer,
                vresource res);
Writes the contents of the tool changer into the resource. The tool changer can be recovered from the resource at a later time (perhaps by another program) with vtoolchangerLoad or _typeprefixLoadInit. Note that not all of the tool changer's attributes are stored in the resource---the attributes of the default tool changer class which are stored in resources are documented in the ``Toolbar Manager'' chapter of the Galaxy Application Environment Programmer's Guide. It generates vobjectUnnamedClassException if the tool changer's class is not named. This function is implemented by sending the message vobjectSTORE to the tool changer.

To store a tool changer in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vtoolchangerStore to write the tool changer into the resource. For example: 


vtoolchanger  *changer;    /* contains a tool changer */
vresource fileResource, changerResource;

fileResource   = vresourceOpenFile( ... );
changerResource = vresourceMake(fileResource, 
			       vnameInternGlobalLiteral("Mychanger"));

vtoolchangerStore(changer, changerResource);
vresourceCloseFile(fileResource);

Macros

vtoolchangerCLASS

        #define vtoolchangerCLASS(...) ...
Contains the method and variable declarations for toolchanger classes. This macro expands to the contents of vtoolchangerClass. SUPER is the type identifier of the class being declared (for example, vtoolchangerClass). This type is used by the class's subclasses to refer to it. TOOLCHANGER is the type identifier of instances of the class being declared (for example, vtoolchanger). NOTIFY is the type identifier of the instance notification function (for example, vtoolchangerNoteProc). This macro should be used by toolchanger subclasses to declare the contents of the toolchanger class. This macro should come first in the subclass structure declaration.
