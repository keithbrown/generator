Types

vcontainer

        typedef ... vcontainer;
Contains the information needed to represent a single container---objects of this type have classes derived from vcontainerClass. A container is a dialog item that contains other dialog items. Containers are referenced by a pointer to an object of this type. Containers can be created dynamically with vcontainerCreate, or initialized statically with vcontainerInit. Containers can be created dynamically or loaded from resource files by using the appropriate constructor. A vcontainer contains a dialog item, a set of container attributes, and a dialog item list. Refer to the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of container attributes. All object types with classes that directly inherit from vcontainerClass must declare a vcontainer as their first field so that vcontainer pointers can be interchanged with pointers to the new data type.

vcontainerClass

        typedef ... vcontainerClass;
Contains method and variables bindings for objects of type vcontainer. Classes declared with this type contain the same set of methods and variables as the default container class, but the behavior of methods and the contents of variables may vary. This class type is derived from vdialogItemClass.

vcontainerExclusiveGroup

        typedef ... vcontainerExclusiveGroup;
Contains the information needed to represent a single exclusive group container---objects of this type have classes derived from vcontainerExclusiveGroupClass. An exclusive group container is a dialog item that contains controls. Exclusive group containers are referenced by a pointer to this type. Exclusive group containers can be created dynamically with vcontainerCreateExclusiveGroup, or initialized statically with vcontainerInitExclusiveGroup. Exclusive containers can be created dynamically or loaded from resource files by using the appropriate constructor. A vcontainerExclusiveGroup contains a dialog item, a set of container attributes, and a dialog item list (which must contain only controls). Refer to the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of container attributes. All object types with classes that directly inherit from vcontainerExclusiveGroupClass must declare a vcontainer as their first field so that vcontainer pointers can be interchanged with pointers to the new data type.

vcontainerNoteProc

        typedef ... vcontainerNoteProc;
Denotes a notification function that is called when a container handles an event of some importance. container is the container handling the event, and event is the event being handled. Importance is determined by the event handlers in the container's class---the default container class gives no notification, but the group and exclusive group classes pass notification from the container items on to the container's notification function. If additional notification is required, you should override the appropriate handler.

Functions

vcontainerAppendItem

        void vcontainerAppendItem(
                vcontainer * container,
                vdialogItem * item);
Appends the dialog item item to the item list of container and invalidates the content rectangle of the new item if the container is open. The item is drawn behind all other items in the list, and it is given last chance at any pointer button events. The item is destroyed when the container is destroyed unless it is removed first. If container is a group, the item's global observation function is set by the group. If container is an exclusive group, the value of the control item is set to zero. This function generates vexArgException if container is an exclusive group and the item is not a control. This function is implemented by sending the message vcontainerAPPEND_ITEM to container.

vcontainerArrange

        void vcontainerArrange(
                vcontainer * container,
                const vrect * rect);
Arranges the items in the container container according to its spring solution, using rect as a coordinate system for the items. It has no effect if the container has no spring solution, unless the container is a group, in which case the items are arranged to fit within the group's bounding rectangle. This function is implemented by sending the message vcontainerARRANGE to container.

        void vcontainerCalcMinSize(
                vcontainer * container,
                int * w,
                int * h);
Calculates the minimum size of the container container according to its spring solution and stores the result in the variables addressed by w and h. Stores zero in the variables addressed by w and h if the container has no spring solution, unless the container is a group, in which case the union of the minimum sizes of the group's items is returned. This function is implemented by sending the message vcontainerCALC_MIN_SIZE to container.

vcontainerClone

        vcontainer * vcontainerClone(
                vcontainer * container);
Creates a copy of the container container and returns a pointer to its contents. Storage for the container object is allocated by the Container Manager. The Container Manager reclaims the storage occupied by the container when it is destroyed with vcontainerDestroy. All of the container's items are cloned. This function copies all the attributes of the container except ItemList (the item list the container belongs to), Open, and Focused---it clones any attribute objects that are owned by the container. The container to be cloned can be dynamic or static. This function essentially allocates space for a new container and calls vcontainerCopyInit on it.

vcontainerCopy

        void vcontainerCopy(
                vcontainer * container,
                vcontainer * target);
Copies the information in container into target so that the two containers represent equivalent objects. target must have been created or initialized by a Container Manager function. container and target must have compatible classes for this to work correctly. The target container is invalidated, if its dialog is open. All of the container's items are cloned, and all items in the target container are destroyed. This function copies all the attributes of the container except Class, ItemList (the item list the container belongs to), Open, and Focused---it clones any attribute objects that are owned by the container. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The containers to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to container.

vcontainerCopyInit

        void vcontainerCopyInit(
                vcontainer * container,
                vcontainer * target);
Copies the information in container into target so the memory addressed by target is a container equivalent to container. target should be a pointer to uninitialized memory---the Container Manager initializes it as part of the copy. All of the container's items are cloned by this function. This function copies all the attributes of the container except ItemList (the item list the container belongs to), Open, and Focused---it clones any attribute objects that are owned by the container. This function simply stores the class of container in target and sends the message vobjectCOPY_INIT to container.

vcontainerCreate

The default attributes of the default container class are documented in the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new container and calls vcontainerInit on it.

vcontainerDeleteItem

        void vcontainerDeleteItem(
                vcontainer * container,
                vdialogItem * item);
Removes the dialog item item from the item list of container and invalidates the content rectangle of the item if the container is open. The item is not destroyed. If container is a group, the item's global observation function is set to NULL. This function generates vexArgException if the item is not in the item list of container. This function is implemented by sending the message vcontainerDELETE_ITEM to container.

vcontainerDestroy

        void vcontainerDestroy(
                vcontainer * container);
Destroys the container represented by container, which must have been created or initialized by a Container Manager function. Use this function to reclaim the storage occupied by a container. The container should not be referenced by any other object when it is destroyed. All of the container's items are destroyed by this function. This function only attempts to free the storage occupied by container if it was allocated by vcontainerCreate, vcontainerCreateGroup, vcontainerCreateExclusiveGroup, vcontainerCreateOfClass, vcontainerClone, or vcontainerLoad. This function also destroys any attribute objects that are owned by the container. This function is implemented by sending the message vobjectDESTROY to container.

vcontainerDisable

        void vcontainerDisable(
                vcontainer * container);
Makes the container container, including all its items and scroll bars, unresponsive to user events. If the container's dialog is open, the container's item rectangle is invalidated and the container is drawn in an inactive state in a subsequent update event. The items in the container are effectively disabled by this function. This function has no effect if the container is not enabled. This function is implemented by sending the message vdialogDISABLE_ITEM to container.

vcontainerEnable

        void vcontainerEnable(
                vcontainer * container);
Makes the container container, including all its items and scroll bars, responsive to user events. If the container's dialog is open, the container's item rectangle is invalidated and the container is drawn in an active state in a subsequent update event. This function has no effect if the container is not disabled. This function is implemented by sending the message vdialogENABLE_ITEM to container.

vcontainerGetDefaultClass

        vcontainerClass * vcontainerGetDefaultClass();
Returns the class used to create containers by the vcontainerCreate and vcontainerInit functions. This class can be modified to alter the behavior of all containers subsequently created by these functions. Changes to this class are not visible to any other applications in the same process or sharing the same display server. This function can be called to explicitly register the default container class with the Class Manager. The default container class can be used to construct ``sub-dialogs'' and nested hierarchies of dialog items.

vcontainerGetExclusiveGroupClass

        vcontainerExclusiveGroupClass * vcontainerGetExclusiveGroupClass();
Returns the class used to create exclusive group containers by the vcontainerCreateExclusiveGroup and vcontainerInitExclusiveGroup functions. This class can be modified to alter the behavior of all exclusive group containers subsequently created by these functions. Changes to this class are not visible to any other applications in the same process or sharing the same display server. This function can be called to explicitly register the exclusive group container class with the Class Manager. An exclusive group container differs from a standard container in that an exclusive group only accepts controls in its item list. An exclusive group container sets the Notify and Data attributes of the controls in the group so that the group gives notification when one of the items gives notification. The particular item can be retrieved by examining the Selection attribute of the exclusive group. An exclusive group also ensures that only one of the controls in the group has a non-zero value. Refer to the Control Manager chapter of the Galaxy Application Environment Programmer's Guide for more information about controls.

NOTE:

You must not change the Notify and Data attributes of items in an exclusive group. These values are assigned by the Container Manager and modifying them causes unpredictable results.

vcontainerLoad

        vcontainer * vcontainerLoad(
                vresource resource);
Reads a container from the resource resource and returns a pointer to its contents. Storage for the container is allocated by the Container Manager. The container's class is determined by the value of the ``Class'' tag. This function generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function creates an object based on the class of the resource, even if this class is not a container class. The object pointer is cast to a container pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. All of the container's items are read from the resource. The attributes of the default container class that are stored in resources are documented in the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vcontainerLoadInit on it.

vcontainerLoadInit

        void vcontainerLoadInit(
                vcontainer * container,
                vresource resource);
Reads a container from the resource resource and stores it in the uninitialized memory addressed by container. The container's class is determined by the value of the ``Class'' tag. This function generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function initializes an object based on the class of the resource, even if this class is not a container class. The object attempts to initialize itself in the container structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. All of the container's items are read from the resource. The attributes of the default container class that are stored in resources are documented in the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function stores the class specified by the resource in the class field of the container structure and sends it the message vobjectLOAD_INIT.

vcontainerRemoveItem

        vdialogItem * vcontainerRemoveItem(
                vcontainer * container,
                vdialogItem * item);
Removes the dialog item item from the item list of the container and invalidates the item's former content rectangle if the container is open. After removal, the item is no longer owned by the container and should be disposed of by the caller. This method returns a pointer to the removed item.

Subclasses that override this method should first call the parent method to remove the item from the container, then update any subclass-specific state.

This function is implemented by sending the message vcontainerREMOVE_ITEM to the container.

vcontainerSetBorder

        void vcontainerSetBorder(
                vcontainer * container,
                int flag);
Changes the container container so that a border is drawn around it if flag is TRUE. If flag is FALSE, container is given no border. The appearance of the border is determined by vcontainerSetFlat. The content rectangle of container is inset to accommodate the border within the existing item rectangle. This function is implemented by sending the message vdialogSET_ITEM_BORDER to container.

vcontainerSetBounds

        void vcontainerSetBounds(
                vcontainer * container,
                const vrect * bounds);
Changes the virtual bounding rectangle of the container container to bounds in the coordinate system of the container's owner. The bounding rectangle is interpreted in the same coordinate system as the item rectangle of the container. Each item in the container is positioned relative to the lower left corner of the bounding rectangle. The bounding rectangle can be different from the content rectangle only if one or both dimensions are scrollable. The rectangle structure is copied. If the container's dialog is open, the container is invalidated. This function is implemented by sending the message vcontainerSET_BOUNDS to container.

Sets a container's generic data pointer to data. This may be a pointer to whatever information that you want to attach to the container---it is not used by the Container Manager. data may be NULL. data is declared const to indicate that the Container Manager will not modify its contents. This function is implemented by sending the message vdialogSET_ITEM_DATA to container.

vcontainerSetDefFocus

        void vcontainerSetDefFocus(
                vcontainer * container,
                vdialogItem * defFocus);
Changes the default focus item of the container container to defFocus, which must be in the container's item list. The default focus item is the dialog item that receives the keyboard focus when the container is opened. defFocus can be NULL, in which case the focus item is not changed when the container is opened. The default focus of a container has no bearing on its current focus. This function generates vexArgException if the item is not in the item list of the container. This function is implemented by sending the message vcontainerSET_DEF_FOCUS to container.

        void vcontainerSetFlat(
                vcontainer * container,

Changes the style of the border of a container and also how it responds to keyboard traversal.If flag is TRUE, the border (if any) of container container appears engraved into its dialog, and keyboard traversal behaves as if the container were part of its owner's item list. If flag is FALSE, the border appears inset and keyboard traversal behaves as if the container were its own dialog (refer to Figure). If the container's dialog is open, the container is invalidated. This function is implemented by sending the message vcontainerSET_FLAT to container.

vcontainerSetFocus

        void vcontainerSetFocus(
                vcontainer * container,
                vdialogItem * focus);
Changes the current keyboard focus of the container container to the dialog item focus, which must be in the container's item list. The current focus is the item that receives all normal keystrokes and certain function keys. focus can be NULL, in which case the container is not given a keyboard focus. This function unfocuses the current focus item and focuses the new focus item if the container has the keyboard focus. If the container is open, the appearance of the current focus item and the new focus item may change. Note that the current focus item of a container is set to the container's default focus item when it is opened, so any changes to the current focus item of a closed container is lost. This function generates vexArgException if the item is not in the item list of container. This function is implemented by sending the message vcontainerSET_FOCUS to container.

vcontainerSetItemIndex

        void vcontainerSetItemIndex(
                vcontainer * container,
                vdialogItem * item,
                int index);
Changes the position of the dialog item item in the item list of container and invalidates the content rectangle of the item if the container is open. Items earlier in the item list are given preference for pointer hit detection and are drawn last. This function generates vexArgException if the item is not in the item list of container. It generates vexArgRangeException if index is less than zero or greater than or equal to the number of items in the container. This function is implemented by sending the message vcontainerSET_ITEM_INDEX to container.

vcontainerSetNotify

        void vcontainerSetNotify(
                vcontainer * container,
                vcontainerNoteProc noteProc);
Changes the instance notification procedure for the container container to noteProc. This function is called when the container handles an event of some importance. Notification is not given for every event that is handled by the container. Normal containers do not give notification, but a group container gives notification when any item in the group gives notification. The particular item giving notification can be retrieved with vcontainerGetSelection. If additional notification is required, you should override the appropriate event handler. noteProc may be NULL, in which case no instance notification is given. This function is implemented by sending the message vdialogSET_ITEM_NOTIFY to container.

This function adds or removes the horizontal scroll bar and adjusts the size of the content rectangle as necessary to accommodate the presence or absence of the scroll bar. If the container's dialog is open, the container is invalidated. This function is implemented by sending the message vcontainerSET_SCROLL_X to container.

Changes the spring solution for the container container to solution. The spring solution is used to reposition and resize the container items in container when it is resized. vcontainerStore

        void vcontainerStore(
                vcontainer * container,
                vresource resource);
Writes the contents of the container container into the resource resource. The container can be recovered from the resource at a later time (perhaps by another program) with vcontainerLoad or vcontainerLoadInit. Note that not all of the container's attributes are stored in the resource---the attributes of the default container class that are stored in resources are documented in the ``Container Manager'' chapter of the Galaxy Application Environment Programmer's Guide. All of the container's items are stored in the resource. This function generates vobjectUnnamedClassException if the container's class is not named. This function is implemented by sending the message vobjectSTORE to container.

To store a container in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vcontainerStore to write the container into the resource. For example: 


vcontainer *container;    /* contains a container */
vresource   fileResource, containerResource;
fileResource      = vresourceOpenFile( ... );
containerResource = vresourceMake(fileResource, 
                               
vnameInternGlobalLiteral("MyContainer"));
vcontainerStore(container, containerResource);
vresourceCloseFile(fileResource);

vcontainerTestSetFocus

        vbool vcontainerTestSetFocus(
                vcontainer * container,
                vdialogItem * focus,
                vevent * event);
Attempts to change the current keyboard focus of the container to the dialog item focus, which must be in the container's item list. It returns TRUE if both the old focus item agreed to lose focus item and focus agreed to take it, FALSE otherwise. The current focus is the item that receives all normal keystrokes and certain function keys. focus can be NULL, in which case the container only attempts to remove focus from any item which currently has it. This function attempts to unfocus the current focus item and focus the new focus item if the item list has the keyboard focus.

Note that the current focus of a container is set to the container's default focus when it is opened, so a change to the current focus of a closed container is lost. This function generates vexArgException if the container does not contain the item. This method is implemented in terms of vdialogTestSetItemListFocus.

This function is implemented by sending the message vcontainerTEST_SET_FOCUS to the container.

Messages

vcontainerAPPEND_ITEM

        <vcontainerAPPEND_ITEM>
Appends the dialog item item to the item list of container and invalidates the content rectangle of the new item if the container is open. The item is drawn behind all other items in the list, and it is given last chance at any pointer button events. The item is destroyed when the container is destroyed unless it is removed first. If container is a group, the item's Notify and Data attributes are set by the group. If container is an exclusive group, the value of the control item is set to zero. This function generates vexArgException if container is an exclusive group and the item is not a control. The function entry point vcontainerAppendItem is implemented by sending this message to the container.

The default method for the default container class appends the item to the container's item list and invalidates the item rectangle if the container is open. Subclasses that override this method should first call vclassSendSuper to append the item to the container, then update any subclass-specific state.

vcontainerARRANGE

        <vcontainerARRANGE>
Arranges the items in the container container according to its spring solution, using rect as a coordinate system for the items. Has no effect if the container has no spring solution, unless the container is a group, in which case the items are arranged to fit within the group's bounding rectangle. The function entry point vcontainerArrange is implemented by sending this message to the container.

The default method for the default container class arranges the container's item list within the coordinate system provided. Subclasses that override this method should first call vclassSendSuper to arrange the container, then arrange any subclass-specific contents.

vcontainerCALC_MIN_SIZE

        <vcontainerCALC_MIN_SIZE>
Calculates the minimum size of the container container according to its spring solution and stores the result in the variables addressed by w and h. It returns zero if the container has no spring solution, unless the container is a group, in which case the union of the minimum sizes of the group's items is returned. The function entry point vcontainerCalcMinSize is implemented by sending this message to the container.

The default method for the default container class calculates the container minimum size based on its item list and spring solution. Subclasses that override this method should first call vclassSendSuper to calculate the container minimum size, then add in any subclass-specific requirements.

vcontainerCREATE_ITEM_LIST

        <vcontainerCREATE_ITEM_LIST>
Creates an itemlist for a container when a new container is created or initialized. The default implementation of this method simply creates a new item list of the default dialog item list class. Subclasses which override this method should return a subclass of vdialogItemList.

vcontainerCREATE_SCROLL_BAR

        <vcontainerCREATE_SCROLL_BAR>
Creates a scrollbar for the container. This method is called internally anytime a container needs a new scroll bar. The default implementation of this method creates a new scroll bar which is a subclass of vsliderScrollBar that overrides the vobjectBUILD_HELP_CONTEXT method. Subclasses which override this method do not have access to the private class, so they can return a subclass of the default vsliderScrollBar class.

vcontainerDELETE_ITEM

The function entry point vcontainerDeleteItem is implemented by sending this message to the container.

The default method for the default container class removes the item from the container's item list and invalidates the item rectangle if the container is open. Subclasses that override this method should first call vclassSendSuper to delete the item from the item list, then perform any subclass-specific tasks.

vcontainerSET_BOUNDS

        <vcontainerSET_BOUNDS>
Changes the virtual bounding rectangle of the container container to bounds in the coordinate system of the container's owner. The bounding rectangle is interpreted in the same coordinate system as the item rectangle of the container. Each item in the container is positioned relative to the lower left corner of the bounding rectangle. The bounding rectangle can be different from the content rectangle only if one or both dimensions are scrollable. The rectangle structure is copied. If the container's dialog is open, the container is invalidated. The function entry point vcontainerSetBounds is implemented by sending this message to the container.

The default method for the default container class copies the rectangle into the container's structure and rearranges the items in the container if either of its dimensions are scrollable. Subclasses that override this method should first call vclassSendSuper to set the container bounding rectangle, then update any subclass-specific state.

vcontainerSET_DEF_FOCUS

        <vcontainerSET_DEF_FOCUS>
Changes the default focus item of the container container to defFocus, which must be in the container's item list. The default focus item is the dialog item that receives the keyboard focus when the container is opened. defFocus can be NULL, in which case the focus item is not changed when the container is opened. This function generates vexArgException if the item is not in the item list of container. The function entry point vcontainerSetDefFocus is implemented by sending this message to the container.

The default method for the default container class changes the default focus item in the container's item list. Subclasses that override this method should first call vclassSendSuper to store the default focus item in the container, then update any subclass-specific state.

vcontainerNEXT_ID_ATTRIBUTE

        vcontainerNEXT_ID_ATTRIBUTE

Macros

vcontainerCLASS

        #define vcontainerCLASS(...) ...
Contains the method and variable declarations for container classes. This macro expands to the contents of vcontainerClass. SUPER is the type identifier of the class being declared (for example, vcontainerClass). This type is used by the class's subclasses to refer to it. CONTAINER is the type identifier of instances of the class being declared (for example, vcontainer). This macro should be used by container subclasses to declare the contents of the container class. This macro should come first in the subclass structure declaration.
