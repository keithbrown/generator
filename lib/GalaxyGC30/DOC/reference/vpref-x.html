Types

vpref

        typedef ... vpref;
An element of the Preference class Database. Preference dictionaries use this type to load and store preference data. A vpref can be converted to a Representation class object using vprefGetScrap. All references to a vpref must be released when they are no longer being used.

vprefClass

        typedef ... vprefClass;
Contains the message definitions for a class of preferences. The default preference class only implements vprefINIT, vprefRELEASE, and vprefDESTROY. It must be subclassed to be useful.

vprefDict

        typedef ... vprefDict;
A preference dictionary. Preference dictionaries can be stored in other preference dictionaries to form a hierarchy of preferences. Preference dictionaries are referenced with a pointer of this type. Preference dictionaries can be created dynamically (with vprefDictCreate) or initialized statically (with vprefDictInit). Preferences Dictionaries contain ``key/vpref'' pairs.

vprefDictClass

        typedef ... vprefDictClass;
Contains the message definitions for a class of Preference Dictionaries. Classes declared with this type contain the same set of messages as the default preference dictionary class, but the behavior of methods may vary. This class is derived from vprefClass.

vprefDictObserver

        typedef ... vprefDictObserver;
A preference dictionary observer. Preference dictionary observers are referenced with a pointer of this type. Preference dictionary observers can be created dynamically (with vprefDictObserverCreate) or initialized statically (with vprefDictObserverInit). Preference dictionary observers are notified of changes on a preference dictionary.

vprefDictObserverClass

        typedef ... vprefDictObserverClass;
Contains the message definitions for a class of Preference Dictionary Observers. The default preference dictionary observer class only implements vprefDictObserverINIT and vprefDictObserverDESTROY. It must be subclassed to be useful.

vprefIterator

        typedef ... vprefIterator;
Denotes an abstract index variable used for enumerating the contents of a preference dictionary. Typically, a preference iterator is used by a higher-level module to determine which preferences in the preference dictionary interest it.

vprefValue

        typedef ... vprefValue;
A preference value. Preference values can be stored in preference dictionaries. Preference values are referenced with a pointer of this type. Preference values can be created dynamically (with vprefValueCreate) or initialized statically (with vprefValueInit). A Representation class object (a vscrap) can be specified for the preference value when it is created.

vprefValueClass

        typedef ... vprefValueClass;
Contains the message definitions for a class of preference values. Classes declared with this type contain the same set of messages as the default preference value class, but the behavior of methods may vary. This class is derived from vprefClass.

vprefCreateOfClass

        vpref * vprefCreateOfClass(
                vprefClass * clas);
Creates a preference of class clas. The attributes of the preference are set to their default values, according to the class. Space for the preference is dynamically allocated based on the object size for the class. The Class class will reclaim the space when all references to the new preference are released with vprefRelease. This function essentially allocates space for a new instance and calls vprefInitOfClass on it. There is no vprefCreate because the core preference class is not useful.

vprefDictRead

        void vprefDictRead(
                vprefDict * dict,
                vresource resource);
Merges a set of preferences from a dictionary resource, resource, into dict. The merge is recursive, sub-dictionaries of resource are merged into sub-dictionaries of dict. It generates a vexArgException if resource is not a vresourceDICT.

The merged preferences are considered default values and must be saved explicitly. Calling vprefDictWrite on dict will not write out the preferences loaded from resource.

vprefDictReadResource

        void vprefDictReadResource(
                vprefDict * dict,
                vresource resource);
Merges a set of preferences from a dictionary resource, resource, into the dictionary dict. The merge is recursive; sub-dictionaries of resource are merged into sub-dictionaries of the dictionary dict. It generates a vexArgException if resource is not a vresourceDICT.

This function is different from vprefDictRead because the merged preferences are not considered default values. Calling vprefDictWrite on the dictionary dict will write out the preferences loaded from resource. This function is not as space efficient as vprefDictRead.

vprefDictWritePref

        void vprefDictWritePref(
                vprefDict * dict,
                const vname * tag);
Writes the preference associated with tag in the dictionary dict into a resource file. By default, this function writes preferences into the application specific preference dictionary in the user's user resource file. However, if vprefSetPreferences has been called, vprefDictWritePref will write to that preference dictionary. This function will generate a vexModuleState if an invalid resource was passed to vprefSetPreferences. This function will write out default preferences as well (those preferences merged into dict with vprefDictRead).

This function insures that the preference associated with tag is written such that it will be loaded back into the same position in the Preference Database hierarchy if the resource dictionary it is written to is at some point merged back in to the root preferences dictionary (with vprefDictRead or vprefDictReadResource) either directly, or when the Application class starts up. (The application specific preference dictionary in the user's resource file is merged into the root preferences dictionary when the Application class starts up).

vprefDump

        void vprefDump(
                FILE * stream);
Dumps the contents of the Preferences Database to stream. Interpret the output as follows: 


N display(1 refs) : machine.name:0.0 
N laf(1 refs) : openlook
D namedcolordict(0 refs) : UNPRINTABLE 
D thistle(0 refs) : resource id 16778355 
D  vcolorComponent3(0 refs) : 55512 
D  vcolorComponent2(0 refs) : 49087 
D  vcolorComponent1(0 refs) : 55512 
D  vcolorClass(0 refs) : 0 
D  Type(0 refs) : vcolorSpec

The initial ``N'' or ``D'' indicates whether a preference is Non-default or Default.

Indentation indicates the vprefDict in which a preference is stored. A preference is indented one more space than the vprefDict in which it is stored. In the above example, thistle is stored in namedcolordict. Any preference with indented preferences immediately following it is a vprefDict.

The next word is the name under which the preference was stored.

The number of `refs' is the number of unreleased references each preference has.

The text at the end of each line is the value of the preference; ``resource id'' refers to preferences that could not be displayed, but could be converted to resources. UNPRINTABLE refers to preference dictionaries that could not be converted to a resource.

        void vprefGetArgs(

Returns the part of the argument list that the Preference class did not use for preferences during the last call to vprefReadArgs or vprefSetArgs. If all your command line options are stored as preferences, this array consists of all command line arguments that are not options. Most applications treat this array as a list of documents to open at startup time. You can use vfsGetArgs to retrieve the path name of each argument (this is usually more convenient for opening document files). By convention, argv[0] is assumed to be the application executable name.

        vkeyModifiers vprefGetCommandKey();

Reads the current command line arguments into the preference database. This function can be called multiple times, if necessary. Refer to the discussion of command line arguments in the ``Preference class'' chapter of the Galaxy Application Environment Programmer's Guide. This function leaves any command-line arguments not recognized as preferences in a separate argument list so they can be retrieved independently with vprefGetArgs or vfsGetArgs. vprefReadEnvironment

        void vprefReadEnvironment();
Reads the environment variables corresponding to each preference in the preference database and stores the value of the variable back into the database. This function queries environment variables in the same case as the preference key as well as the ``all caps'' form of the preference key. For example, the preference key ``fullpathnames'' will read the environment variables ``fullpathnames'' and ``FULLPATHNAMES.''

Refer to the discussion of environment variables in the ``Preference class'' chapter of the Galaxy Application Environment Programmer's Guide.

vprefSetArgs

        void vprefSetArgs(
                int argc,
                char ** argv);
Stores the command-line argumentswith the Preference class so that Galaxy modules can access them from a standard place, then calls vprefReadArgs to enter any command line preference settings into the preference database.

Galaxy requires knowledge of the command-line arguments to operate correctly. Since command-line arguments are only accessible from the function main, you must call this function early in your application for Galaxy to work correctly. This function is called automatically by vstartup, so you need not call it directly if you call vstartup. Refer to the Introduction chapter (0) for more information about initializing Galaxy. It generates vexArgRangeException if argc is less than one. By convention, argv[0] is assumed to be the application executable name.

Creates a preference value of class clas. The attributes of the preference value are set to their default values, according to the class. Space for the preference value is dynamically allocated based on the object size for the class. The Class class will reclaim the space when all references to the new preference value are released with vprefValueRelease. This function essentially allocates space for a new instance and calls vprefValueInitOfClass on it.

vprefValueDestroy

        void vprefValueDestroy(
                vprefValue * value);
Destroys a preference value. The user should never call this function directly. The release method (accessible by calling vprefValueRelease) will call vprefValueDestroy when all references to a preference value have been released. The user should never call this function directly.

vprefValueGetClass

        vprefValueClass * vprefValueGetClass(
                vprefValue * value);
Returns the class for the preference value value.

vprefValueGetDefaultClass

        vprefValueClass * vprefValueGetDefaultClass();
Returns the class used by vprefValueCreate and vprefValueInit. To customize preference values, subclass class returned by vprefValueGetDefaultClass.

vprefValueGetPref

        vpref * vprefValueGetPref(
                vprefValue * value);
Returns the vpref corresponding to value.

vprefValueGetScrap

        vscrap * vprefValueGetScrap(
                vprefValue * value);
Converts the data stored in value to a vscrap.

The vscrap returned by this function must be destroyed by the user. The data in the vscrap MUST NOT BE CHANGED. To change the value in a preference value, create a new preference value with a new vscrap.

vprefValueInit

        void vprefValueInit(
                vprefValue * value,
                vscrap * scrap);
Makes the preference value pointed to by value into a valid preference value of the default class. If scrap is non-NULL, then scrap is the value returned by vprefValueGetScrap. If scrap is NULL, the preference value can create a vscrap from a resource using vprefValueRead. Calling vprefValueRead on a preference value with a vscrap already assigned to it (either at creation or an earlier call to vprefValueRead) will generate an vexArgException.

vprefValueInitOfClass

        void vprefValueInitOfClass(
                vprefValue * value,
                vprefValueClass * clas,
                vscrap * scrap);
Makes the preference value pointed to by value into a valid preference value of class clas. If scrap is non-NULL, then scrap is the value returned by vprefValueGetScrap. If scrap is NULL, the preference value can create a vscrap from a resource using vprefValueRead. Calling vprefValueRead on a preference value with a vscrap already assigned to it (either at creation or an earlier call to vprefValueRead) will generate an vexArgException.

vprefValueRead

        void vprefValueRead(
                vprefValue * value,
                vresource resource);
Creates a vscrap (using vresourceGetScrap) from resource and stores it in value. This function will generate a vexArgException if value already has a vscrap assigned to it (either at creation of an earlier call to vprefValueRead).

vprefValueRelease

        void vprefValueRelease(
                vprefValue * value);
Releases a reference to a vprefValue. This function should be called when a reference to a vprefValue is no longer needed. The user should never call vprefValueDestroy. When all references to a vprefValue are released, it is destroyed.

vprefValueWrite

        void vprefValueWrite(
                vprefValue * value,
                vresource resource);
Writes the vscrap contained in value into resource. The resource parameter must be of type vresourceUNTYPED or its type must be compatible with the vscrap; otherwise this function will generate a vexArgException.

vprefWrite

        void vprefWrite(
                vpref * pref,
                vresource resource);
Writes the contents of pref into resource. This function is implemented by calling the vprefWRITE method on pref. See vprefValueWrite and vprefDictWrite for information on the semantics of their implementations of the vprefWrite method.
