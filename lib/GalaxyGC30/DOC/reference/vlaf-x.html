vlafAppear

        int vlafAppear();
Returns an identifier for the current look-and-feel appearance. This function should be called to determine the appearance of user-interface objects. The appearance can also be used to determine behavior when no better test is available. One of the following constants is returned:

vlafNONE
Indicates that no look-and-feel has been selected. This occurs only when all look-and-feel specific code is compiled out of a program. user-interface objects should not be drawn or otherwise respond to events. This look-and-feel is useful for building minimal utility programs with Galaxy libraries.
vlafMAC
Indicates that the Apple Macintosh look-and-feel has been selected. This option is only available on Macintosh platforms due to legal constraints.
vlafWINDOWS
Indicates that the Microsoft Windows look-and-feel has been selected.
vlafMOTIF
Indicates that the OSF Motif look-and-feel has been selected.
vlafCUA
Indicates that the IBM CUA look-and-feel has been selected.

Note that any symbol except vlafNONE can be undefined if you want to compile out code implementing a particular look-and-feel. You should therefore make sure that the appropriate look-and-feel symbol is defined with the #ifdef preprocessor statement before referring to a particular constant.

The following symbols are defined according to whether any one of the look-and-feels is defined for the current compilation, since user-interface code may be shared between different look-and-feels:

  • vlafMAC_WINDOWS
  • vlafMAC_WINDOWS_WIN95
  • vlafMAC_MOTIF
  • vlafMAC_CUA
  • vlafWINDOWS_MOTIF_CUA
  • vlafWINDOWS_MOTIF_CUA_CUA96
  • vlafWINDOWS_MOTIF_CUA_WIN95
  • vlafWINDOWS_MOTIF_CUA_WIN95_CUA96
  • vlafWINDOWS_MOTIF
  • vlafWINDOWS_CUA

vlafArrowHiliteFillX

        void vlafArrowHiliteFillX(
                int x,
                int y,
                int length,
                int half,
                vcolor * interior,
                vcolor * straight,
                vcolor * bent);
Draws an arrow with a three-dimensional appearance into the current graphics state, pointing in a horizontal direction. x, y is the location at which to draw the arrow and interior, straight, and bent are the shadow colors to use. length is the length of the arrow to draw and half is half the width of the arrow. These functions can be used to draw Motif-style arrows. Refer to the ``Drawing Manager'' chapter of the Galaxy Application Environment Programmer's Guide for information about drawing and graphics states.

vlafDrawGlyph

        void vlafDrawGlyph(
                int id,
                int offset);
Draws the glyph identified by id at the current position with the current color. offset is an integer representing the height of the glyph; it is used to automatically offset the font character and is ignored by the glyph images. This function draws a glyph on the screen from a glyph font if such a font exists; otherwise, an equivalent image from a Galaxy resource file is used. Currently, only the font implementation exists. Valid IDs have the prefix vlafOLG and are listed in the ``Constants'' section.
