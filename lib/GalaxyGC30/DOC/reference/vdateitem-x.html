vdateitem

        typedef ... vdateitem;
The primary means by which vdatitem carries out it mission is by the use of a vformatDate object. By default, the item is associated with a locale-specific global vformatDate object but other, non-shared, instances may be set by vdateitemSetFormat.

vdateitemClass

        typedef ... vdateitemClass;
Contains method and variable bindings for objects of type vdateitem. Classes declared with this type contain the same set of methods and variables as the default date item class, but the behavior of methods and the contents of variables may vary. This class type is derived from vtypeditemClass.

If vdateitemIsDirty returns vFALSE, then the internally stored value is returned. If the item is dirty, the current content string of the item is obtained, then validated (parsed) and the newly parsed value is returned. If the dirty content string is not parsable (invalid), a vformatParseException is thrown.

If the item has an associated vfield, as returned from vdateitemGetField, and vdateitemIsObserving] returns vFalse, the field's value is set to value.

If the item has no associated field, it uses its vformatDate object to set various characteristics of the item, such as its content string, foreground color and text justification.

vdateitemGET_VALUE

        <vdateitemGET_VALUE>
If vdateitemIsDirty returns vFALSE, then the internally stored value is returned. If the item is dirty, the current content string of the item is obtained, then validated (parsed) and the newly parsed value is returned. If the dirty content string is not parsable (invalid), a vformatParseException.

vdateitemSET_VALUE

        <vdateitemSET_VALUE>
If the item has an associated vfield, as returned from vdateitemGetField, and vdateitemIsObserving] returns vFalse, the field's value is set to value.

If the item has no associated field, it uses its vformatDate object to set various characteristics of the item, such as its content string, foreground color and text justification.
