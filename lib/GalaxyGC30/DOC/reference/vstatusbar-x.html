Types

vstatusbar

        typedef ... vstatusbar;
This subclass of vcontainer is designed to display concise and up-to-date status information. Status bars are typically located along the bottom edge of application dialogs. Usually they display short informational messages pertaining to the application state or information about the dialog item over which the pointer is currently located.

Status bars have a default item. The default item should bear the tag vstatusbarDefaultItem. (Use vstatusbar_vstatusbarDefaultItem). This item is responsible for displaying the default text messages. By default, this item is of class vstatusbarItem. To change the class of this item, set the class variable vstatusbarDEFAULT_ITEM_CLASS to the appropriate class. Note that this must be done prior to the instantiation of a vstatusbar.

Applications can control what messages the status bar displays by vstatusbarSetDefaultItemText and vstatusbarSetStatusbarText. vstatusbarSetDefaultItemText, or vstatusbarSetDefaultItemTextScribed, is used to set the text of the default text display item in a status bar. vstatusbarSetStatusbarText (or vstatusbarSetStatusbarTextScribed) is meant to be called as a static function (without necessarily instantiating a vstatusbar). It is used to set the default text display item of a status bar located above a given dialog item in the UI hierarchy.

Developers can add and delete additional dialog items to a vstatusbar (see vstatusbarAddItem and vstatusbarDeleteItem. The layout of these items is determined by an vstatusbarAdderType. While any type of vdialogItem can be added to a status bar, subclasses of vstatusbarItem are already prepared for text display (using item titles), the drawing of borders, and the proper sizing of the item based on the text to be displayed.

Status bars can be used in conjunction with status help to display help messages about the items located under the pointer. When motion events are enabled, help requests are generated bearing the name vname_Status. The application will search for a help object by that name to handle this help request. Refer to sections vhelpobject, vhelpregistry and vhelprequest for more information. The default status help object (activated by vhelpStartupStatus) uses vhelptextlocator and vhelpstatus to display help text in a status bar. By default, status bars are receptive to help text from the default status help object in the global help registry. Use vstatusbarSetObserveStatusHelp to disable this feature.

vstatusbarClass

        typedef ... vstatusbarClass;
Contains message and variable bindings for objects of type vstatusbar. Classes declared with this type contain the same set of methods and variables as the Statusbar class, but the behavior of methods and the contents of variables may vary. This class type is derived from vcontainer.

vstatusbarItem

        typedef ... vstatusbarItem;
Specifically, this subclass provides support for drawing it's own border properly inside the status bar and sizing itself based on text and font size. Refer to sections vstatusbar and vstatusbarAdderType for additional information on item placement.

vstatusbarItemClass

vstatusbarItemNoteProc

        typedef ... vstatusbarItemNoteProc;

vstatusbarNoteProc

        typedef ... vstatusbarNoteProc;
Importance is determined by the event handlers in the status bar's class: the default status bar class gives no notification. If additional notification is required, you should override the appropriate handler.

vstatusbarDestroy

        void vstatusbarDestroy(
                vstatusbar * statusbar);
Destroys the status bar represented by statusbar, which must have been created or initialized by a Statusbar class function. Use this function to reclaim the storage occupied by the status bar. The status bar should not be referenced by another object when it is destroyed. The storage is only freed when it was allocated by one of the Statusbar class functions. The status bar is destroyed by sending a vobjectDESTROY message to statusbar.

Macros

vstatusbarCLASS

        #define vstatusbarCLASS(...) ...
Contains the method and variable declarations for status bar classes. This macro expands to the contents of vstatusbarClass. SUPER is the type identifier of the class being declared (for example, vstatusbarClass). This type is used by the class's subclasses to refer to it. STATUSBAR is the type identifier of instances of the class being declared (for example, vstatusbar). NOTIFY is the type identifier of the instance notification function (for example, vstatusbarNoteProc). This macro should be used by status bar subclasses to declare the contents of the status bar class. This macro should come first in the subclass structure declaration.

vstatusbarITEM_CLASS

        #define vstatusbarITEM_CLASS(...) ...
Contains the method and variable declarations for status bar item classes. This macro expands to the contents of vstatusbarItemClass. SUPER is the type identifier of the class being declared (for example, vstatusbarItemClass). This type is used by the class's subclasses to refer to it. STATUSBARITEM is the type identifier of instances of the class being declared (for example, vstatusbarItem). NOTIFY is the type identifier of the instance notification function (for example, vstatusbarItemNoteProc). This macro should be used by status bar item subclasses to declare the contents of the status bar item class. This macro should come first in the subclass structure declaration.
