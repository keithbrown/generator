vpalette prefix



Types [5] Functions [74] Messages [7] Constants [1] Macros [2]

Types

vpalette

        typedef ... vpalette;
A palette is a dialog item that manages a rectangular array of images or icons.

vpaletteClass

        typedef ... vpaletteClass;
Describes the behavior of a class of palettes. Palette classes are referenced by a pointer to this type.

vpaletteItem

        typedef ... vpaletteItem;
Contains the information needed to represent a single palette item and its contents. A palette item is a particular item on the palette, usually represented as an image or icon.

vpaletteItemClass

        typedef ... vpaletteItemClass;
Contains method and variable bindings for objects of type vpaletteItem. Classes declared with this type contain the same set of methods and variables as the default palette item class, but the behavior of methods and the contents of variables may vary. This class type is derived from vobjectClass.

vpaletteItemNotifyProc

        typedef ... vpaletteItemNotifyProc;
Denotes a notification function that is called when a palette item is selected.

vpaletteCreateItemOfClass

        vpaletteItem * vpaletteCreateItemOfClass(
                vpaletteItemClass * clas);
Creates a new palette item of class clas and returns a pointer to its contents. Storage for the palette item is allocated by the Palette class. The Palette class reclaims the storage occupied by the item when it is destroyed with vpaletteDestroyItem. The new palette item is created with a set of default attributes, determined by the class. The default attributes of the default palette item class are documented in the ``Palette Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new item and calls vpaletteInitItemOfClass on it.

vpaletteInitItem

        void vpaletteInitItem(
                vpaletteItem * item);
Initializes a palette item of the default palette item class in the memory addressed by item. The attributes of the item are set to their default values, according to the default palette item class. The address can be used in other calls to the Palette class as if it were created with vpaletteCreateItem. A call to vpaletteDestroyItem frees any internal objects allocated by the Palette class but does not free the structure itself. This function stores the default class in the class field of the item and sends it the message vobjectINIT.

vpaletteInitItemOfClass

        void vpaletteInitItemOfClass(
                vpaletteItem * item,
                vpaletteItemClass * clas);
Initializes a palette item of class clas in the memory addressed by item. The attributes of the item are set to their default values, according to the class. The address can be used in other calls to the Palette class as if it were created with vpaletteCreateItemOfClass. A call to vpaletteDestroyItem frees any internal objects allocated by the Palette class but does not free the structure itself. This function stores the class in the class field of the item and sends it the message vobjectINIT.

vpaletteInitOfClass

        void vpaletteInitOfClass(
                vpalette * palette,
                vpaletteClass * clas);
Makes the palette addressed by palette a valid palette of class clas in the same manner as vpaletteCreateOfClass, except that storage is not allocated for the palette. If the palette is passed to vpaletteDestroy, any internally allocated storage is freed, but the palette itself is not freed. The storage addressed by palette need not be dynamically allocated.

vpaletteLoadInit

                vpalette * palette,

Reads a palette item from the resource resource and stores it in the structure addressed by palette, which should be uninitialized. An exception is generated if the resource has not been constructed properly.

vpaletteLoadItem

        vpaletteItem * vpaletteLoadItem(
                vresource resource);
Reads a palette item from the resource resource and returns a pointer to its contents. Storage for the palette item is allocated by the Palette class. The item's class is determined by the value of the ``Class'' tag. This function generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry.

Note that this function creates an object based on the class of the resource, even if this class is not a palette item class. The object pointer is cast to a palette item pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. This function essentially allocates space for a new instance and calls vpaletteLoadInitItem on it.

vpaletteNotifyItem

Messages

vpaletteAPPEND_ITEM

        <vpaletteAPPEND_ITEM>
Appends the palette item item onto the end of the palette palette.

vpaletteDELETE_ITEM

vpaletteDELETE_ITEM_INDEX

Deletes the palette item at the position index from the palette palette, destroying the item removed. All following items in the palette have their indices decremented by one.

The function vpaletteDeleteItemIndex is implemented by sending this message to the container.

vpaletteINSERT_ITEM

        <vpaletteINSERT_ITEM>
Inserts the palette item item into the palette palette at position index. An index of zero makes the item the first in the palette, while an index equal to the number of items in the palette (not including the item to be inserted) makes it the last item.

vpaletteNOTIFY_ITEM

        <vpaletteNOTIFY_ITEM>
Notifies the palette item item that the event event was handled by one of the palette item's event handlers. Typically, this means that the palette item has just been selected in the palette. The function entry point vpaletteNotifyItem is implemented by sending this message to the item.

The default method for the default palette item class calls the item's instance notification function and dispatches a command using the item's tag, if any. Subclasses that override this method should first call vclassSendSuper to notify the palette item, then notify any subclass-specific objects.

vpaletteOBSERVE_PALETTE_ITEM

        <vpaletteOBSERVE_PALETTE_ITEM>
Message sent when the application is notified of an event. Typically, this means that the palette item has just been selected in the palette.

To take direct action based on the handling of important events, either set a notification function on the application, override this message, or override vpaletteNOTIFY_ITEM.

vpaletteREMOVE_ITEM_INDEX

        <vpaletteREMOVE_ITEM_INDEX>
Removes the palette item at the position index from the palette and returns the removed palette item. All following items in the palette have their indices decremented by one. The removed palette item is not destroyed.

The function vpaletteRemoveItemIndex is implemented by sending this message to the container.

Macros

vpaletteCLASS

        #define vpaletteCLASS(...) ...
Contains the method and variable declarations for palette classes. This macro expands to the contents of vpaletteClass. SUPER is the type identifier of the class being declared (for example, vpaletteClass). This type is used by the class's subclasses to refer to it. PALETTE is the type identifier of instances of the class being declared (for example, vpalette). ITEM is the type identifier of the items which will be held by the palette class (for example, vpaletteItem). This macro should be used by palette subclasses to declare the contents of the palette class. This macro should come first in the subclass structure declaration.

vpaletteItemCLASS

        #define vpaletteItemCLASS(...) ...
Contains the method and variable declarations for palette item classes. This macro expands to the contents of vpaletteItemClass. SUPER is the type identifier of the class being declared (for example, vpaletteItemClass). This type is used by the class's subclasses to refer to it. ITEM is the type identifier of instances of the class being declared (for example, vpaletteItem). This macro should be used by palette item subclasses to declare the contents of the palette item class. This macro should come first in the subclass structure declaration.
