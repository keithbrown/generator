vlistitem prefix



Types [7] Functions [62] Messages [3] Constants [5] Macros [7]

Types

vlistitem

        typedef ... vlistitem;
Denotes an instance of the class derived from vlistitemClass. This is a subclass of the vlistview type, and displays a one column list of strings in a dialog. This can be used to create further subclasses for custom needs.

vlistitemClass

        typedef ... vlistitemClass;
Class definition for the list item object, it is derived from vlistviewClass. This class has the same basic functionality as the vlistviewClass, except that this call only works with one column of cells in the list.

vlistitemDrawCellProc

        typedef ... vlistitemDrawCellProc;
Denotes a procedure that draws the content of the specified cell in the given rectangle on the current graphic port. row and column specify which cell of the list to draw, selected is set to TRUE if the cell is in the list item's selection, FALSE otherwise; and rect is the rectangle in which to draw the cell's content.

vlistitemList

        typedef ... vlistitemList;
Denotes an instance of the class derived from vlistitemListClass. This type is a special kind of vliststr for use within a list item object.

vlistitemListClass

        typedef ... vlistitemListClass;
Class definition for the list item data object, it is derived from vliststrClass. This class has the same basic functionality as the vliststrClass, with added support for listitem operation.

vlistitemNoteProc

        typedef ... vlistitemNoteProc;
Denotes a list item notify procedure. This is called wheneverlistitem handles an interesting event (as defined by the Dialog Manager), event is the event that is being handled.

vlistitemSelectNoteProc

        typedef ... vlistitemSelectNoteProc;
Denotes a list item selection notify procedure. This is called when listitem's selection is about to change, when it is changing, and when the user commits to the selection change. state is one of the following values vlistviewSELECTION_WILL_CHANGE, vlistviewSELECTION_IS_CHANGING, vlistviewSELECTION_HAS_CHANGED.

Functions

vlistitemAppend

        void vlistitemAppend(
                vlistitem * listitem,
                const vchar * str);
Adds the string str to the end of the list being displayed by the list item listitem. If str has been dynamically allocated it is the client's responsibility to free it when finished. If str refers to a temporary buffer or a buffer that is reused for additional strings, str needs to be copied before appending it. If this is the case, use vlistitemAppendScribed instead.

vlistitemAppendScribed

        void vlistitemAppendScribed(
                vlistitem * listitem,
                vscribe * scribe);
Adds the string denoted by scribe to the end of the list being displayed by the list item listitem. The string contained in the scribe is owned by the list the list item is displaying, and is destroyed by the List Manager when the list is finished with it.

vlistitemClone

        vlistitem * vlistitemClone(
                vlistitem * listitem);
Returns a pointer to a newly allocated list item, and makes it a copy of the list item listitem. This function is essentially a combination of vlistitemCreate and vlistitemCopy.

vlistitemCopy

        void vlistitemCopy(
                vlistitem * listitem,
                vlistitem * target);
Copies the information from listitem into target, so that the two list items represent equivalent list items but separate storage. Any objects created in listitem are duplicated in target as well. The vlist object that listitem is based upon is also copied (vlistClone), that is the vchar* strings themselves are not copied, just the references to them.

vlistitemCopyInit

        void vlistitemCopyInit(
                vlistitem * listitem,
                vlistitem * target);
Copies the information in listitem into target so the memory addressed by target is a dialog item equivalent to listitem. target should be a pointer to uninitialized memory---the List Manager will initialize it as a part of the copy. This function copies all of the attributes of the item, and it will clone the list contents by calling vlistClone on the list owned by listitem. This function is implemented by sending the message vobjectCOPY_INIT to listitem.

vlistitemCreate

        vlistitem * vlistitemCreate();
Creates a new list item object of the default list item class and returns a pointer to the new object. The storage space for the new object is allocated by the List Manager. (This is the same as calling vlistitemCreateOfClass(vlistitemGetDefaultClass()).)

vlistitemCreateOfClass

        vlistitem * vlistitemCreateOfClass(
                vlistitemClass * clas);
Returns a pointer to a newly allocated list item of class clas. The new list item is created with a set of default attributes determined by the class.

vlistitemDestroy

        void vlistitemDestroy(
                vlistitem * listitem);
Destroys the list item represented by listitem and all of its sub-objects that were allocated by the List Manager. This function attempts to free the storage directly occupied by the list item only if it was allocated by vlistitemCreate or vlistitemCreateOfClass. Since a list item owns the list it is displaying the list is also destroyed, however, before the list is destroyed, any cell value that was set with a scribe (vlistitemAppendScribed, vlistitemPrependScribed, vlistitemSetValueAtScribed) is freed as needed. This function is implemented by sending the message vinstanceDESTROY to listitem.

vlistitemGetDefaultListDataClass

        vlistitemListClass * vlistitemGetDefaultListDataClass();
Returns the list data object class used by list items when creating the list data object for the item via vlistCreateOfClass. This class can be subclassed to alter the behavior of list data objects created by list items. List items send themselves the vlistitemCREATE_LIST_DATA message to actually create the list owned by the list item. If you want to use your own list data subclass, you will need to subclass vlistviewGetDefaultListDataClass. To automatically have a list item create a list data object when it is created, you will need to override vlistitemCREATE_LIST_DATA.

vlistitemGetVolatileSelectedRow

        int vlistitemGetVolatileSelectedRow(
                vlistitem * listitem);
Returns the number of the row that is currently selected in the volatile selection of the list item listitem. If the selection is empty, vlistitemNO_CELLS_SELECTED is returned. If the selection method for the list item has been changed so that more than one row may be selected, the returned row number is arbitrarily chosen from the selected rows. The row number returned is only valid during selection change operations when the selection notify type is vlistitemSELECTION_WILL_CHANGE and vlistitemSELECTION_IS_CHANGING.

vlistitemGetVolatileSelection

        vlistSelection * vlistitemGetVolatileSelection(
                vlistitem * listitem);
Returns the ``volatile'' selection, which is the selection defined while the user is changing the selection in the specified list item, before committing to an actual selection. The volatile selection information can be used to provide information about the currently selected cells; for example, by graying out selected cells for which the application has no current use, or by updating other dialog items to reflect the value of the currently selected cells. An undefined selection is returned if this function is called outside of tracking a change to the selection.

vlistitemHasAtLeastOneSelection

        int vlistitemHasAtLeastOneSelection(
                vlistitem * listitem);
Returns TRUE if the list item listitem's selection has the ``at least one'' attribute set, indicating that the selection is never empty unless the list item's list is empty, or the select method is vlistNO_SELECTION. It returns FALSE if the ``at least one'' selection attribute is not set.

vlistitemSetAutowidth

        void vlistitemSetAutowidth(
                vlistitem * listitem,
                int onOff);
Enables or disables list item column auto-sizing for the list item listitem. If onOff is TRUE, auto-sizing is enabled, if FALSE, auto-sizing is disabled. When auto-sizing is enabled, the width of column zero is determined by the contents of the cells in that column. The width of each cell is calculated by calling vlistviewDetermineCellSize. vlistitem overrides vlistviewDETERMINE_CELL_SIZE to return the width of the cell's contents by calling vfontStringWidthX using the list-item's current font. This function is implemented by sending the message vlistitemSET_AUTOWIDTH to listitem.

        void vlistitemSetData(
                vlistitem * listitem,

Sets the data field of the vdialogItem listitem is built on to be data. The List Manager does not use this field, and the value can be NULL. vlistitemSetList

        void vlistitemSetList(
                vlistitem * listitem,
                vlist * list);
Sets the list object to be displayed by list item listitem to the list object specified by list. The old list displayed by the list item is destroyed. The list item takes ownership of list, and list will be destroyed when the list item is destroyed. However any strings pointed to by the cell values that were not set with scribed list item entry points will not be freed when the list item is destroyed, that work must be done by the client. list must be an object of class vlistitemGetDefaultListDataClass, or a subclass of this. If list is not, a warning will be printed with vdebugWarn and many list item features will not work or yield undefined behavior. This function is implemented by sending the message vlistviewSET_LIST to listitem.

vlistitemSetNotify

        void vlistitemSetNotify(
                vlistitem * listitem,
                vlistitemNoteProc noteProc);
Sets the dialog item notification procedure for the list item to noteProc. All interesting events that the list item handles are passed to this function. Since user functions do not necessarily have a one-to-one correspondence to selection changes, you should use a selection-notify function to track detailed changes to the list item's selection. The events that dispatch a notification are mouse button-up events, keyboard events that change the selection, and the Enter and Return key. If vlistviewGetLastPointerClickCount returns a value greater than one, then the user has indicated that some action is to be taken on the current selection. This function is implemented by sending the vdialogSET_ITEM_NOTIFY message to listitem.

vlistitemSetRowCount

        void vlistitemSetRowCount(
                vlistitem * listitem,
                int count);
Sets the number of rows in the list being displayed by listitem to count. If count is greater than the current row count, empty rows are inserted at the end, otherwise, rows at the end of the list are removed. The idiom vlistitemSetRowCount(listitem, 0); is a good way to empty the listitem in a single operation. This function is implemented by sending the message vlistSET_ROW_COLUMN_COUNT to the list owned by listitem.

vlistitemSetRowHeight

        void vlistitemSetRowHeight(
                vlistitem * listitem,
                int height);
Sets the row height for all rows in the list item listitem to height. If the client never sets the row height, the default row height which is calculated from the list item's font is used. This function is implemented by sending the message vlistviewSET_ROW_HEIGHT to listitem.

vlistitemSetSelectMethod

        void vlistitemSetSelectMethod(
                vlistitem * listitem,
                int selectMethod);
Changes the selection method for the visible selection in the list item listitem to be selectMethod. Values for selectMethod can be:

vlistFREE_SELECTION
Any cells can be in the selection at any time.
vlistONE_ONLY_SELECTION
Zero or one cell can be in the selection at any time.
vlistRECTANGULAR_SELECTION
The selection is limited to a single rectangle.
vlistNO_SELECTION
No cells can be in the selection.

vlistitemSetSelectNotify

        void vlistitemSetSelectNotify(
                vlistitem * listitem,
                vlistitemSelectNoteProc proc);
Sets listitem's selection-notify procedure to noteProc. This procedure gets called whenever the list item's selection changes. The noteProc gets called at the start of the change with vlistitemSELECTION_WILL_CHANGE to allow any pre-selection setup, with vlistitemSELECTION_IS_CHANGING while the selection is being modified, and then vlistitemSELECTION_HAS_CHANGED when the operation is complete. During the selection change, the original selection is accessible to the noteProc by way of vlistitemGetSelection, and the selection under construction is available by way of vlistitemGetVolatileSelection. The noteProc is called when the change to the selection is a result of user action or API calls.

vlistitemSetVisibleRows

        void vlistitemSetVisibleRows(
                vlistitem * listitem,
                int rows);
Sets the number of rows to show when calculating the list item listitem's natural height. When the vdialogGET_ITEM_NATURAL_SIZE message is sent, the list item uses the value of rows multiplied by the height of the list item's font to set the natural height. If rows is zero, then the natural size is not affected. This function is implemented by sending the message vlistitemSET_VISIBLE_ROWS to listitem. vlistitem overrides vlistviewGET_CONTENT_NATURAL_SIZE to affect the natural height. The list owned by the list item does not have to actually have rows number of rows in it for the natural height be affected correctly.

vlistitemSortList

        void vlistitemSortList(
                vlistitem * listitem,
                vlistCellValueSorterFunc compar);
Sorts the items in the list displayed by the list item listitem. The compare function should return a value less than zero if string1 < string2, a value greater than zero if string1 > string2, and zero if string1 == string2. Passing NULL for compare calls a default compare function (vcharCompareBase). The sort function used by vlistitemSortList is a merge sort, which is a stable sort with good performance characteristics for sorted and unsorted data.

vlistitemStore

Messages

vlistitemCREATE_LIST_DATA

        <vlistitemCREATE_LIST_DATA>
Creates a string list data object from vlistitemGetDefaultListDataClass and returns it. Override this method to provide your own string list subclass to be used by a list item. It is not necessary to call the superclass when overriding this method.

vlistitemSET_AUTOWIDTH

        <vlistitemSET_AUTOWIDTH>
Enables or disables list item column auto-sizing for listitem. If onOff is TRUE, auto-sizing is enabled, if FALSE, auto-sizing is disabled. When auto-sizing is enabled, the width of column zero is determined by the contents of the cells in that column. The width of each cell is calculated by calling vlistviewDetermineCellSize. vlistitem overrides vlistviewDETERMINE_CELL_SIZE to return the width of the cell's contents by calling vfontStringWidthX using the list-item's current font. Subclasses overriding this method must call the superclass so that list item knows to automatically calculate the column width.

vlistitemSET_VISIBLE_ROWS

        <vlistitemSET_VISIBLE_ROWS>
Sets the number of rows to show when calculating listitem's natural height. When the vdialogGET_ITEM_NATURAL_SIZE message is sent, listitem will use the value of rows multiplied by the height of listitem's font to set the natural height. If rows is zero, then the natural size will not be affected. vlistitem overrides vlistviewGET_CONTENT_NATURAL_SIZE to affect the natural height. The list owned by listitem does not have to actually have rows number of rows in it for the natural height be affected correctly. Subclasses overriding this method need to call the superclass for the visible rows value to be stored into listitem correctly.
