Types

vmenubar

        typedef ... vmenubar;
Contains the information needed to represent a single menu bar---objects of this type have classes derived from vmenubarClass. Menu bars are referenced by a pointer to this type. Menu bars can be created dynamically with vmenubarCreate, or initialized statically with vmenubarInit. A vmenubar contains a dialog item and a set of menu bar attributes. All object types with classes that directly inherit from vmenubarClass must declare a vmenubar as their first field so that vmenubar pointers can be interchanged with pointers to the new data type.

A menu bar contains an ordered list of menus. A menu bar displays, inside the menu bar content area, the title of each of its menus. Clicking or key traversing a title opens the associated menu. A menu bar must be placed in a dialog to be operational.

vmenubarClass

        typedef ... vmenubarClass;
Contains method and variables bindings for objects of type vmenubar. Classes declared with this type contain the same set of methods and variables as the default menu bar class, but the behavior of methods and the contents of variables may vary. This class type is derived from vdialogItemClass.

vmenubarNoteProc

        typedef ... vmenubarNoteProc;
Denotes a notification function that is called when a menu bar is triggered by the user. menubar is the menu bar that was triggered, and event is the event that caused the menu bar to be triggered. Note that this function is not called for every event that is handled by the menu bar---if additional notification is required, you should override the appropriate event handler. Menu bars do not give any notification---you should attach a notification function to a menu or menu item instead. Refer to the ``Menu Manager'' chapter of the Galaxy Application Environment Programmer's Guide for more information.

Functions

vmenubarAppendMenu

        void vmenubarAppendMenu(
                vmenubar * menubar,
                vmenu * menu);
Appends the menu menu to the menu list of menubar and invalidates the content rectangle of the menu bar if the menu bar is open. The menu is drawn on the right-hand side of the menu bar. The menu is destroyed when the menu bar is destroyed unless it is removed first. This function calls vmenubarChangedMenu to ensure that the menu bar's metrics are updated correctly before it is opened. This function is implemented by sending the message vmenubarAPPEND_MENU to menubar.

vmenubarChangedMenu

        void vmenubarChangedMenu(
                vmenubar * menubar);
Indicates to the Menu Bar Manager that a menu owned by menubar has had its contents changed in some way, and that the menu bar's metrics should be recalculated to reflect the change. Functions that manipulate the menu bar's item list (such as vmenubarAppendMenu and vmenubarRemoveMenu) call this function automatically. You should call this function if a menu is changed while it is in a menu bar. The menu bar will recalculate its menu title boundaries the next time it is opened. If the menu bar's dialog is open, the menu bar's bounding rectangle is invalidated and the menu bar is redrawn in a subsequent update event. This function is implemented by sending the message vmenubarCHANGED_MENU to menubar.

vmenubarClone

        vmenubar * vmenubarClone(
                vmenubar * menubar);
Creates a copy of the menu bar menubar and returns a pointer to its contents. Storage for the menu bar object is allocated by the Menu Bar Manager. The Menu Bar Manager reclaims the storage occupied by the menu bar when it is destroyed with vmenubarDestroy. All of the menu bar's menus are cloned. This function copies all the attributes of the menu bar except Open, Focused and ItemList---it clones any attribute objects that are owned by the menu bar. The menu bar to be cloned can be dynamic or static. This function essentially allocates space for a new menu bar and calls vmenubarCopyInit on it.

vmenubarCopy

        void vmenubarCopy(
                vmenubar * menubar,
                vmenubar * target);
Copies the information in menubar into target so that the two menu bars represent equivalent objects. target must have been created or initialized by a Menu Bar Manager function. menubar and target must have compatible classes for this to work correctly. All of the menu bar's menus are cloned, and all menus in the target menu bar are destroyed. This function copies all the attributes of the menu bar except Class, Open, Focused and ItemList---it clones any attribute objects that are owned by the menu bar. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The menu bars to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to menubar.

vmenubarCopyInit

        void vmenubarCopyInit(
                vmenubar * menubar,
                vmenubar * target);
Copies the information in menubar into target so the memory addressed by target is a menu bar equivalent to menubar. target should be a pointer to uninitialized memory---the Menu Bar Manager initializes it as part of the copy. All of the menu bar's menus are cloned. This function copies all the attributes of the menu bar except Open, Focused and ItemList---it clones any attribute objects that are owned by the menu bar. This function simply stores the class of menubar in target and sends the message vobjectCOPY_INIT to menubar.

vmenubarCreate

        vmenubar * vmenubarCreate();
Creates a new menu bar of the default menu bar class and returns a pointer to its contents. The new menu bar has no menus. Storage for the menu bar object is allocated by the Menu Bar Manager. The Menu Bar Manager reclaims the storage occupied by the menu bar when it is destroyed with vmenubarDestroy. The new menu bar is created with a set of default attributes, as determined by the default menu bar class. The default attributes of the default menu bar class are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new menu bar and calls vmenubarInit on it.

vmenubarCreateOfClass

        vmenubar * vmenubarCreateOfClass(
                vmenubarClass * clas);
Creates a new menu bar of class clas and returns a pointer to its contents. The new menu bar has no menus. Storage for the menu bar object is allocated by the Menu Bar Manager. The Menu Bar Manager reclaims the storage occupied by the menu bar when it is destroyed with vmenubarDestroy. The new menu bar is created with a set of default attributes, determined by the class. The default attributes of the default menu bar class are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new menu bar and calls vmenubarInitOfClass on it.

vmenubarDeleteMenu

        void vmenubarDeleteMenu(
                vmenubar * menubar,
                vmenu * menu);
(Deprecated) Removes the menu menu from the menu list of menubar and invalidates the content rectangle of the menu bar if the menu bar is open. The menu is not destroyed. All menus following the removed menu have their indices decremented by one. This function calls vmenubarChangedMenu to insure that the menu bar's metrics are updated correctly before it is opened. It generates vexArgException if the menu is not in the menu list of menubar. This function is implemented by sending the message vmenubarDELETE_MENU to menubar.

This method has been deprecated; use vmenubarRemoveMenu instead.

vmenubarDestroy

        void vmenubarDestroy(
                vmenubar * menubar);
Destroys the menu bar represented by menubar, which must have been created or initialized by a Menu Bar Manager function. Use this function to reclaim the storage occupied by a menu bar. The menu bar should not be referenced by any other object when it is destroyed. All of the menu bar's menus are destroyed by this function. This function only attempts to free the storage occupied by menubar if it was allocated by vmenubarCreate, vmenubarCreateOfClass, vmenubarClone, or vmenubarLoad. This function will also destroy any attribute objects that are owned by the menu bar. This function is implemented by sending the message vobjectDESTROY to menubar.

vmenubarInit

        void vmenubarInit(
                vmenubar * menubar);
Initializes a menu bar of the default menu bar class in the memory addressed by menubar. The new menu bar has no menus. The attributes of the menu bar are set to their default values, according to the default menu bar class. The default attributes of the default menu bar class are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Menu Bar Manager as if it were created with vmenubarCreate. A call to vmenubarDestroy frees any internal objects allocated by the Menu Bar Manager but does not free the structure itself. This function simply stores the default class in the class field of the menu bar and sends it the message vobjectINIT.

vmenubarLoad

        vmenubar * vmenubarLoad(
                vresource resource);
Reads a menu bar from the resource resource and returns a pointer to its contents. Storage for the menu bar object is allocated by the Menu Bar Manager. The menu bar's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function creates an object based on the class of the resource, even if this class is not a menu bar class. The object pointer is cast to a menu bar pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. All of the menu bar's menus are read from the resource. The attributes of the default menu bar class which are stored in resources are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vmenubarLoadInit on it.

vmenubarLoadInit

        void vmenubarLoadInit(
                vmenubar * menubar,
                vresource resource);
Reads a menu bar from the resource resource and stores it in the uninitialized memory addressed by menubar. The menu bar's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function initializes an object based on the class of the resource, even if this class is not a menu bar class. The object attempts to initialize itself in the menu bar structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. All of the menu bar's menus are read from the resource. The attributes of the default menu bar class which are stored in resources are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. This function stores the class specified by the resource in the class field of the menu bar structure and sends it the message vobjectLOAD_INIT.

vmenubarStore

        void vmenubarStore(
                vmenubar * menubar,
                vresource resource);
Writes the contents of the menu bar menubar into the resource resource. The menu bar can be recovered from the resource at a later time (perhaps by another program) with vmenubarLoad or vmenubarLoadInit. Note that not all of the menu bar's attributes are stored in the resource---the attributes of the default menu bar class which are stored in resources are documented in the ``Menu Bar Manager'' chapter the Galaxy Application Environment Programmer's Guide. All of the menu bar's menus are stored in the resource. It generates vobjectUnnamedClassException if the menu bar's class is not named. This function is implemented by sending the message vobjectSTORE to menubar.

To store a menu bar in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vmenubarStore to write the menu bar into the resource. For example: 


vmenubar *menubar;    /* contains a menu bar */
vresource fileResource, menubarResource;

fileResource    = vresourceOpenFile( ... );
menubarResource = vresourceMake(fileResource,
                                vnameInternGlobalLiteral("MyMenuBar"));

vmenubarStore(menubar, menubarResource);
vresourceCloseFile(fileResource);

        <vmenubarAPPEND_MENU>

Constants

vmenubarINCLUDED

        vmenubarINCLUDED
Defined when the header file vmenubar.h is included.

vmenubarNEXT_ID_ATTRIBUTE

        vmenubarNEXT_ID_ATTRIBUTE
Denotes the next available attribute ID for subclasses of the default menu bar class. This constant is provided so that attribute IDs defined by subclasses do not conflict with those defined by the Menu Bar Manager. Only attributes defined by overriding the methods vobjectSET_ID_ATTRIBUTE and vobjectGET_ID_ATTRIBUTE need be concerned with the attribute IDs of their superclasses. Subclasses of vmenubarClass should begin their attribute IDs with this constant and increment by one for each attribute. Subclasses should also provide their own NEXT_ID_ATTRIBUTE constant for use by their subclasses. This constant has integral type.

vmenubarNEXT_SELECT

        vmenubarNEXT_SELECT
Denotes the first command selector available to subclasses of the default menu bar class. This constant is provided so that selector values defined by subclasses do not conflict with those defined by their superclasses. Only class-level commands (defined by overriding the methods vobjectISSUE_SELECTOR and vobjectQUERY_SELECTOR) need be concerned with the selector values of their superclasses. Subclasses of vmenubarClass should begin their command selectors with this constant and increment by one for each new command. In other words, if you add commands to the class-level command space of vmenubar, you put their implementations in your vobjectISSUE_SELECTOR override and start with the value of vmenubarNEXT_SELECT as the first selector value you handle. Subclasses should also provide their own NEXT_SELECT constant for use by their subclasses. This constant has integral type.

vmenubarCLASS

        #define vmenubarCLASS(...) ...
Contains the method and variable declarations for menu bar classes. This macro expands to the contents of vmenubarClass. SUPER is the type identifier of the class being declared (for example, vmenubarClass). This type is used by the class's subclasses to refer to it. ITEM is the type identifier of instances of the class being declared (for example, vmenubar). NOTIFY is the type identifier of the instance notification function (for example, vmenubarNoteProc). This macro should be used by menu bar subclasses to declare the contents of the menu bar class. This macro should come first in the subclass structure declaration.
