Types

vhelptextlocator

        typedef ... vhelptextlocator;
Help text locators are referenced by vhelpobjects. When vhelpobjectDisplayHelp is called, the help object will pass a help request to the help locator. The locator is responsible for finding the help data and adding it to the request, in this case the help data is the actual help text.

Help text locators first search the UI hierarchy command spaces for a dictionary bearing a certain name (refer to vhelptextlocatorSetDictionaryName). If such a dictionary is found, the locator then searches the dictionary for an entry whose key matches the tag name of the object requesting help. If a match is found in the dictionary, the locator creates a vhelptext from the dictionary value (it is assumed to be a vstr *) and sets it as the help data in the help request.

If no help data is found in the UI hierarchy, then the locator searches the application resource file for a top-level dictionary bearing the proper name. If it finds such a dictionary and a match in the dictionary, it creates a vhelptext and associates it with the help request as described above.

Alternatively, a help text locator can forgo the above search and instead create a vhelptext using a vdialogItem's title (refer to vhelptextlocatorSetUseItemTitle). If the object of the help request is not a vdialogItem or has no title, then the help text locator will continue with the search as outlined above.

vhelptextlocatorClass

        typedef ... vhelptextlocatorClass;
Contains method and variable bindings for objects of type vhelptextlocator . Classes declared with this type contain the same set of methods and variables as the default help reference class, but the behavior of methods and the contents of variables may vary. This class type is derived from vhelplocatorClass.

Functions

vhelptextlocatorCreate

        vhelptextlocator * vhelptextlocatorCreate();
Creates an instance of the default class of vhelptextlocator and returns a pointer to its contents. Space for the object is allocated dynamically. The Class Manager reclaims the storage occupied by the object when it is destroyed with vhelptextlocatorDestroy. Higher-level interfaces based on the core vhelptextlocator type can define a macro to this function for their Create function. This function essentially allocates space for a new instance and calls vhelptextlocatorInit on it.

vhelptextlocatorCreateOfClass

        vhelptextlocator * vhelptextlocatorCreateOfClass(
                vhelptextlocatorClass * helptextlocatorClass);
Creates an instance of class helptextlocatorClass and returns a pointer to its contents. Space for the object is allocated dynamically based on the object size of the class. The Class Manager reclaims the storage occupied by the object when it is destroyed with vhelptextlocatorDestroy. Higher-level interfaces based on the core vhelptextlocator type can define a macro to this function for their Create function. This function essentially allocates space for a new instance and calls vhelptextlocatorInitOfClass on it.

vhelptextlocatorDestroy

        vhelptextlocator * vhelptextlocatorDestroy(
                vhelptextlocator * helptextlocator);
Destroys the object helptextlocator, which must have been created or initialized by a Class Manager function. Interfaces based on the core object class can define a macro to this function for their Destroy function. This function is implemented by sending the message vinstanceDESTROY to helptextlocator.

        vhelptextlocatorClass * vhelptextlocatorGetClass(

Returns the class of the object helptextlocator. This pointer can be cast to higher-level classes if it is known that it is of the appropriate type. Class types can be verified with vclassIsKindOf.

vhelptextlocatorGetDefaultClass

        vhelptextlocatorClass * vhelptextlocatorGetDefaultClass();
This class can be modified to alter the behavior of subclasses of vhelptextlocator. This function can be called to explicitly register the default help text locator class with the Class Manager.

vhelptextlocatorGetDictionaryName

        const vname * vhelptextlocatorGetDictionaryName(
                vhelptextlocator * helplocator);
The locator will search the UI hierarchy and the application resource file for a dictionary bearing under this name. A newly created help text locator has no default name. When help text locators are created for use with status bars and tooltips (refer to vhelpStartupStatus and vhelpStartupTooltip), the dictionary name is set to vname_Status and vname_Phrase, respectively.

vhelptextlocatorInitOfClass

        vhelptextlocator * vhelptextlocatorInitOfClass(
                vhelptextlocator * helptextlocator,
                vhelptextlocatorClass * clas);
Initializes an object of class clas in the memory addressed by helptextlocator. Interfaces based on the core text locator class can define a macro to this function for their Init function. This function simply stores the class in the class field of the object structure and sends it the message vinstanceINIT.

vhelptextlocatorSearchResources

        vbool vhelptextlocatorSearchResources(
                vhelptextlocator * helplocator,
                vhelprequest * request);
The developer should never need to call this function directly. It is called automatically as part of vhelptextlocatorLocateHelp. It is only called if the locator is supposed to search resources (refer to vhelptextlocatorSetSearchResources).

Normally, this function searches the top level of the application resource file for a dictionary bearing a certain name (refer to vhelptextlocatorSetDictionaryName). If it finds such a dictionary, it searches the dictionary for an entry whose key matches the tag name of the object. If a match is found in the dictionary, this function constructs a vhelptext from the dictionary entry (assumed to be a vstr) and adds it to the help request. Developers should override this function in order to change the way the locator searches resources (for example, to search other resources).

This function is implemented by sending the message vhelptextlocatorSEARCH_RESOURCES to helplocator.

vhelptextlocatorSearchUIHierarchy

        vbool vhelptextlocatorSearchUIHierarchy(
                vhelptextlocator * helplocator,
                vhelprequest * request);
The developer should never need to call this function directly. It is called automatically as part of vhelptextlocatorLocateHelp. It is only called if the locator is supposed to search resources (refer to vhelptextlocatorSetSearchUIHierarchy).

Normally, this function searches the UI hierarchy "above" the object of the help request (that is, all of the containing and parent object's command spaces). In each command space, it searches for a dictionary bearing a certain name (refer to vhelptextlocatorSetDictionaryName). If it finds such a dictionary, it searches the dictionary for an entry whose key matches the tag name of the object. If a match is found in the dictionary, this function constructs a vhelptext from the dictionary entry (assumed to be a vstr) and adds it to the help request. Developers should override this function in order to change the way the locator searches the UI hierarchy.

This function is implemented by sending the message vhelptextlocatorSEARCH_UI_HIERARCHY to helplocator.

vhelptextlocatorSetUseItemTitle

        void vhelptextlocatorSetUseItemTitle(
                vhelptextlocator * helplocator,
                vbool yesNo);
Pass in vTRUE to turn on this option or vFALSE to turn off this option. By default, this option is off.

This option operates only on objects that are instances of vdialogItem and have titles. If either of these conditions are not met, the locator will continue by attempting to search the UI hierarchy and application resources (provided these options are turned on).

This function is implemented by sending the message vhelptextlocatorSET_USE_ITEM_TITLE to helplocator.
