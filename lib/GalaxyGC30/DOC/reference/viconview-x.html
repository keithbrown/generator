Types

viconview

        typedef ... viconview;
An icon view is a special version of a domain item that allows named icons to be placed and manipulated in a domain. An icon view provides convenient functionality for user interface components that allow browsing of similar things that can be represented by named icons, such as file/directory browsers, for example.

viconviewDomain

An icon view domain is a special version of a domain item domain that viconview uses to display viconviewIcons.

viconviewGeometryProc

        typedef ... viconviewGeometryProc;
Denotes the specific geometric calculation procedure used to determine the layout relationship of an icon's name and image.
  • iconView is the icon view that icon belongs to view.
  • view is the view that owns the geometry calculation procedure.
  • icon in the icon to compute the geometry for position.
  • position is the position to base the computed values on.
  • imageRect and nameRect are the computed rectangles for the image and name for the icon.

viconviewGridIterator

        typedef ... viconviewGridIterator;
Used to iterate over grid positions in an icon view. A grid iterator is an abstract index variable which may refer to a particular grid position. Each grid iterator refers to a particular icon view and only behaves correctly if used with that icon view. An iterator becomes invalid if any icons are added, removed from, or moved within the icon view to which it is attached.

viconviewIcon

        typedef ... viconviewIcon;
Contains the information needed to represent a single icon. Icons view icons are referenced by a pointer to this type.

viconviewIconClass

        typedef ... viconviewIconClass;
Contains the message definitions for an icon view Icon class. This class type is derived from vinstanceClass.

viconviewIterator

        typedef ... viconviewIterator;
Used to iterate over the icons in an icon view. An iterator is an abstract index variable which may refer to a particular icon. Each iterator refers to a particular icon view and only behaves correctly if used with that icon view. An iterator becomes invalid if any icons are added or removed from the icon view to which it is attached.

viconviewSelectNoteProc

        typedef ... viconviewSelectNoteProc;
Denotes a notification function that is called when an icon is selected or unselected in an icon view.
  • iconView is the icon view handling the event
  • icon is the icon selected or unselected
  • event is the button down event
  • selected indicates whether the icon was selected or unselected.

viconviewView

        typedef ... viconviewView;
Contains the information needed to represent an icon view view and its contents. Iconview views are referenced by a pointer to this type. An iconview view determines the appearance of the icons in an icon view by providing a default icon image, a grid which specifies icon spacing, an inset with which to pad the icons, and a geometry calculation procedure to position the components of an icon. In addition, each view has a distinct name to distinguish it from other views.

viconviewViewClass

        typedef ... viconviewViewClass;
Contains the message definitions for an icon view View class. This class type is derived from vobjectClass.

viconviewNextGridIterator

        int viconviewNextGridIterator(
                viconviewGridIterator * iterator);
Performs the next step of the iteration represented by iterator and returns vTRUE if a grid position is available, vFALSE if not. Theoretically, there are an infinite number of grid positions; there always is a next available grid position. You must break out of your interation loop yourself.

The particular grid position can be retrieved from the iterator with viconviewGetGridIteratorPosition. For example, the following code fragment prints the position of the first ten grid positions: 


viconview              *iconview;    /* contains a icon view */
viconviewGridIterator   iterator;
vpoint                  pt;
int                     count = 10;

viconviewInitGridIterator(&iterator, iconview);

while (viconviewNextGridIterator(&iterator) && count-- > 0) {
    pt = viconviewGetGridIteratorPosition(&iterator);
    vdebugInform("Grid position: (%d %d)\n", pt.x, pt.y);
}

viconviewDestroyGridIterator(&iterator);

viconviewNextIterator

        int viconviewNextIterator(
                viconviewIterator * iterator);
Performs the next step of the iteration represented by the iterator iterator and returns vTRUE if an icon is available, vFALSE if not. The particular icon can be retrieved from the iterator with viconviewGetIteratorIcon. For example, the following code fragment prints the position of all icons in an icon view. 


viconview          *iconview;    /* contains a icon view */
viconviewIterator   iterator;
vpoint		   pt;

viconviewInitIterator(&iterator, iconview);

while (viconviewNextIterator(&iterator)) {
    pt = viconviewGetIconPosition(viconviewGetIteratorIcon(&iterator));
    vdebugInform("Icon position: (%d %d)\n", pt.x, pt.y);
}

viconviewDestroyIterator(&iterator);

viconviewSetViewGeometryProc

        void viconviewSetViewGeometryProc(
                viconviewView * view,
                viconviewGeometryProc proc);
Sets the geometry calculation procedure for the icon view. The geometry calculation procedure is used to determine the layout relationship of name to image, and takes the form presented below. 


typedef void (*viconviewGeometryProc)(
    viconview		*iconView,
    viconviewView	*view,
    viconviewIcon	*icon,
    vpoint		*position,
    vrect		*iconRect,
    vrect		*nameRect
    );

The geometry calculation procedure should determine the rectangles for the name and image objects for the icon icon, based on the position position. The following code sample presents the geometry calculation procedure for the Icon view, (obtained with viconviewGetIconView), one of two built-in views. 


void viconviewIconViewGeometryCalc(
	viconview			*iconView,
	viconviewView		*view,
	viconviewIcon		*icon,
	vpoint			*position,
	vrect			*iconRect,
	vrect			*nameRect
	)
{
	int         w, h;
	vimage      *image;
	vfont	*font;
	vrect	sRect;

	image = viconviewDetermineIconImage(icon, view);

	w = vimageGetWidth(image);
	h = vimageGetHeight(image);

	vrectSet(position->x, position->y,
		 w + 2 * viconviewICON_BORDER,
		 h + 2 * viconviewICON_BORDER, iconRect);

	font   = vdialogDetermineItemFont(viconviewGetItem(iconView));

	if (icon->name != NULL) {
	    vfontStringBounds(font, icon->name, &sRect);
	    w = sRect.w + 4;
	} else
	    w = 2;
	vfontBounds(font, &sRect);
	h = sRect.h + 1;

	vrectSet(iconRect->x + (iconRect->w / 2) - (w / 2),
		 iconRect->y - h, w, h, nameRect);
}

Macros

viconviewCLASS

        #define viconviewCLASS(...) ...
Contains the declarations for icon view classes; this macro expands to the contents of viconviewCLASS. SUPER is the type identifier of the class being declared; for example, viconviewClass. This type is used by the class's subclasses to refer to it. ITEM is the type identifier of icon view instances being declared; for example, viconview. NOTIFY is the type identifier of icon view notification functions being declared; for example, vdialogItemNoteProc. DOMAIN is the type identifier of domain instances being declared; for example, vdomain. ICON is the type identifier of icon view icon instances being declared; for example, viconviewIcon. This macro should be used by higher-level class definitions to declare the icon view class definition.

viconviewIconCLASS

        #define viconviewIconCLASS(...) ...
Contains the declarations for an icon view's Icon class; this macro expands to the contents of viconviewIconCLASS. SUPER is the type identifier of the class being declared; for example, viconviewIconClass. This type is used by the class's subclasses to refer to it. OBJECT is the type identifier of icon instances being declared (for example, viconviewIcon). This macro should be used by higher-level class definitions to declare the icon class definition.
