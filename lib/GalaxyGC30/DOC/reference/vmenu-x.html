Types

vmenu

        typedef ... vmenu;
Contains the information needed to represent a single menu. Objects of this type have classes derived from vmenuClass. A menu is a top-level window that contains a list of menu items.

Menus are referenced by a pointer to this type. Menus can be created dynamically with vmenuCreate, or initialized statically with vmenuInit. A vmenu contains a window, a set of menu attributes, and a menu item list. Refer to the ``Menu Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of menu attributes.

All object types with classes that directly inherit from vmenuClass must declare a vmenu as their first field so that vmenu pointers can be interchanged with pointers to the new data type. The following code illustrates how to create a menu: 


vmenu *menu;

menu = vmenuCreate();

vmenuClass

        typedef ... vmenuClass;
Contains method and variables bindings for objects of type vmenu. Classes declared with this type contain the same set of methods and variables as the default menu class, but the behavior of methods and the contents of variables may vary. This class type is derived from vwindowClass. The following code illustrates how to create a menu of a specific class: 


vmenuClass *menuClass;
vmenu      *menu;

menuClass = vmenuGetDefaultClass();
menu      = vmenuCreateOfClass(menuClass);

vmenuItem

        typedef ... vmenuItem;
Contains the information needed to represent a single menu item. A menu item is a horizontal section of a menu. Objects of this type have classes derived from vmenuItemClass. Menu items are referenced by a pointer to this type. Menu items can be created dynamically with vmenuCreateItem, or initialized statically with vmenuInitItem. A vmenuItem contains an attributed object and a set of menu item attributes. Refer to ``Menu Item Attributes'' in the ``Menu Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of menu item attributes.

All object types with classes that directly inherit from vmenuItemClass must declare a vmenuItem as their first field so that vmenuItem pointers can be interchanged with pointers to the new data type. The following code illustrates how to create a menu item and append it to a menu: 


vmenu     *menu;
vmenuItem *menuItem;

menu = vmenuCreate();
item = vmenuCreateItem();

vmenuAppendItem(menu, item);

vmenuItemList

        typedef ... vmenuItemList;
Denotes list of zero or more objects of type vmenuItem. vmenuItemList is provided as a storage management facility to efficiently manage collections of menu items. Menu item lists are referenced by a pointer to this type. Menu item lists can be created dynamically with vmenuCreateItemList, or initialized statically with vmenuInitItemList. A vmenuItemList consists of a set of item list attributes and an ordered collection of unique menu items, indexed from zero to the number of items in the list minus one. Menu item lists contain no items by default. Item lists are used by menus to manage menu items. This data type is based on the type vobjectList. The following code illustrates how to create a menu item list: 

vmenuItemList *itemList;
itemList = vmenuCreateItemList();

vmenuItemMetrics

        typedef ... vmenuItemMetrics;
Contains measurements for the content area of a given menu item. flags indicates which members of the structure are present for future expansion---zero is returned by this release of Galaxy. width and height define the size of the item content box; this box contains the text contents of the item. left and right are the widths of the left margin and the right margin, respectively. The left margin provides the location for a selection mechanism such as a check box item; the right margin provides space for a graphical representation of the accelerator associated with the item. (The accelerator consists of the keystroke(s) that can be used to activate a command without accessing the command through a menu.) The distances specified by left and right are measured from the item bounding box (not the inner boundary of leftItem and rightItem). The width of the item list box is always the greatest width of all items in the item list. The Menu Manager computes the width of the individual menu items as follows: 


itemWidth = width + max(leftItem,left) + max(rightItem,right);

where leftItem and rightItem are defined by vmenuMetrics and left and right are defined by vmenuItemMetrics. Structures of this type are returned by vmenuCalcItemMetrics. Refer to Figure for an illustration of menu item metrics.

vmenuMetrics

        typedef ... vmenuMetrics;
Contains measurements for a given menu. flags indicates which members of the structure are present for future expansion; zero is returned by this release of Galaxy. left, right, below, and above indicate the amount of space to insert between the menu bounding box and the item list bounding box. leftItem, rightItem, belowItem, and aboveItem indicate the amount of space to insert around each item. leftItem and rightItem define a fixed minimum distance that separates the content area from the item bounding box on the left and right sides, respectively.

Note that the vmenuItemMetrics structure defined below includes metrics that can result in an additional margin beyond leftItem and rightItem. Structures of this type are returned by vmenuCalcMetrics. Refer to the figures for an illustration of how menu metrics determine item placement. metrics.above = 3;
metrics.below = 3;
metrics.leftItem = 2;
metrics.rightItem = 2;
metrics.aboveItem = 2;
metrics.belowItem = 2;

vmenuParentNoteProc

        typedef ... vmenuParentNoteProc;
Denotes a notification function that is called when a menu is closed. parent is the parent object that activated the menu. The parent is typically a menu bar or menu item in a hierarchical menu tree. event is the event that caused the menu to close. This function type is also used for providing notification to a parent object that the pointer has gone inside the menu's drag-off rectangles. Set vmenuSetParentDragRects for more information.

Note that when you set any of a menu's Parent attributes, they are only guaranteed to remain set while the menu is open. Parent attributes are transient in nature and should not be used to store persistent values.

vmenuAbandon

        int vmenuAbandon(
                vmenu * menu,
                vevent * event);
Closes the menu menu but gives no menu or item notification. If the menu's parent notification function is not NULL, it is called with the event as an argument. This is the normal way to close a menu without selecting an item.

vmenuAccelerate

        int vmenuAccelerate(
                vmenu * menu,
                vevent * event);
Sends notification of the event event to whatever item matches the keystroke in event as well as the menu itself. If the keystroke does not match the Accelerator attribute of any menu item, this function has no effect. The Selection attribute of the menu is set to the matching item by this function. If the item is a toggle item, its On attribute is inverted. If the menu's parent notification function is not NULL, it is called with the event as an argument. Call this function to invoke an accelerator key on a closed menu. Refer to vmenuSetItemAccelerator for information on how to set menu item accelerators.

vmenuAppendItem

        void vmenuAppendItem(
                vmenu * menu,
                vmenuItem * item);
Appends the menu item item to the item list of the menu menu and invalidates the content rectangle of the menu if the menu is open. The item is drawn at the bottom of the menu. The item is destroyed when the menu is destroyed unless it is removed first. This function should be used instead of vmenuAppendItemListItem so that the state of the menu can be updated correctly. This function calls vmenuChangedSize to ensure that the menu's metrics are updated correctly before it is opened. This function is implemented by sending the message vmenuAPPEND_ITEM to menu.

vmenuAppendItemListItem

        void vmenuAppendItemListItem(
                vmenuItemList * itemList,
                vmenuItem * item);
Appends the menu item item to the end of the item list itemList. The item is the new last item in the item list. The item is drawn at the bottom of the item list's menu. The item is destroyed when the item list is destroyed unless it is removed first. Use vmenuAppendItem instead of this function of the item list is part of a menu.

vmenuArrange

        void vmenuArrange(
                vmenu * menu,
                const vmenuMetrics * metrics);
Arranges the items of the menu menu and changes the menu's size to reflect its contents. These calculations are based on the item list contents and the margin measurements stored in metrics. The contents of the menu are invalidated. This function is implemented by sending the message vmenuARRANGE to menu. Refer to vmenuMetrics for more information about menu metrics.

vmenuArrangeItemList

        void vmenuArrangeItemList(
                vmenuItemList * itemList,
                const vmenuMetrics * metrics);
Recalculates the bounding rectangle of the item list itemList according to the margin measurements in metrics. The bounding rectangle of the item list is determined by the width and height of each of the list's items as well as the margins in the metrics structure. This function is normally called by the menu owning the item list.

vmenuChangedSize

        void vmenuChangedSize(
                vmenu * menu);
Indicates to the Menu class that a menu item contained by the menu menu has had its contents changed in some way, and that the menu's metrics should be recalculated to reflect the change. Functions that manipulate the menu's item list (such as vmenuAppendItem and vmenuDeleteItem) call this function automatically. Call this function if a menu item is changed while it is in a menu, or if the item list is accessed directly (for example, by vmenuInsertItemListItem). The menu recalculates its bounding rectangle the next time it is opened. This function is implemented by sending the message vmenuCHANGED_SIZE to menu.

vmenuClone

        vmenu * vmenuClone(
                vmenu * menu);
Creates a copy of the menu menu and returns a pointer to its contents. Storage for the menu object is allocated by the Menu class. The Menu class reclaims the storage occupied by the menu when it is destroyed with vmenuDestroy. All of the menu's items are cloned. This function copies all the attributes of the menu except Open; it clones any attribute objects that are owned by the menu. The menu to be cloned can be dynamic or static. This function essentially allocates space for a new menu and calls vmenuCopyInit on it.

vmenuConfirm

        int vmenuConfirm(
                vmenu * menu,
                vevent * event);
Closes the menu menu and sends notification of the event event to whatever item is currently highlighted, as well as the menu itself. The Selection attribute of the menu is set to the highlighted item by this function. If the item is a toggle item, its On attribute is inverted. If the menu's parent notification function is not NULL, it is called with the event as an argument. This is the normal way to close a menu after an item has been selected.

vmenuCopy

        void vmenuCopy(
                vmenu * menu,
                vmenu * target);
Copies the information in the menu menu into target so that the two menus represent equivalent objects. target must have been created or initialized by a Menu class function. menu and target must have compatible classes for this to work correctly. If target is open, it is closed before menu is copied and opened after the copy is completed. All of the menu's items are cloned, and all items in the target menu are destroyed. This function copies all the attributes of the menu except Class, Open, and Application; it clones any attribute objects that are owned by the menu. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The menus to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to menu.

vmenuInvalItem

        void vmenuInvalItem(
                vmenuItem * item,
                int flags);
Invalidates the content area of the menu owning the menu item. This function has no effect if the item's menu is closed unless the menu is in the window pop-up cache, in which case it is removed from the cache. Refer to the ``Window Manager'' chapter of the Galaxy Application Environment Programmer's Guide chapter for more information about the window pop-up cache. Invalidation indicates that the given area no longer reflects the correct appearance of the menu item and that it should be redrawn as soon as possible. The Window class normally responds to invalidation by accumulating the invalid area into the menu window's invalid region and, if this is the first invalidation since the last update event, posting an update event to redraw the invalid region. Redrawing is deferred through the use of update events and invalid regions so that multiple invalid areas can be redrawn in a single step---this minimizes ``flashiness'' and improves drawing performance in most cases. flags specifies hints to the Window class about the invalidation; the flags should be the bitwise OR (|) of zero or more of the following flags:

vwindowINVAL_OPAQUE
Do not erase before re-drawing.
vwindowINVAL_IMMEDIATE
Post update event at normal priority.
vwindowINVAL_UPDATE
Update before returning.
vwindowINVAL_DISABLE
Do not invalidate.

Refer to the ``Window Manager'' chapter of the Galaxy Application Environment Programmer's Guide chapter for more information about invalidation flags.

ITEM is the type identifier of instances of the class being declared (for example, vmenuItem). NOTIFY is the type identifier of the instance notification function (for example, vmenuItemNoteProc). This macro should be used by menu item subclasses to declare the contents of the menu item class. This macro should come first in the subclass structure declaration. Refer to the ``Class Manager'' chapter of the \Italic{Galaxy Application Environment Programmer's Guide} for more information about the use of class definition macros.

vmenuITEM_LIST_CLASS

        #define vmenuITEM_LIST_CLASS(...) ...
Contains the method declarations for the menu item list type. This macro expands to the contents of vmenuItemListClass. SUPER is the type identifier of the class being declared (for example, vmenuItemListClass). This type is used by the class's subclasses to refer to it. LIST is the type identifier of instances of the class being declared (for example, vmenuItemList). ITEM is the type identifier of the items of instances of the class being declared (for example, vmenuItem). This macro should be used by higher-level class definitions to declare the menu item list class definition.
