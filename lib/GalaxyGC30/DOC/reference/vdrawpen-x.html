Types

vdrawpen

        typedef ... vdrawpen;
This class manages the drawing attributes of a drawing context. A drawing context passed as a parameter to a method should be cloned before any of the pen attributes are modified.

vdrawpenClass

        typedef ... vdrawpenClass;
Contains method and variables bindings for objects of type vdrawpen.

vdrawpenClone

        vdrawpen * vdrawpenClone(
                vdrawpen * pen);
Creates a copy of the drawing pen pen and returns a pointer to its contents.The Draw Manager reclaims the storage occupied by the drawing pen when it is destroyed with vdrawpenDestroy. This function copies all the attributes of the drawing pen, and it clones any attribute objects that are owned by the pen. The draing pen to be cloned can be dynamic or static. This function essentially allocates space for a new drawing pen and calls vdrawpenCopyInit on it.

vdrawpenCopy

        void vdrawpenCopy(
                vdrawpen * pen,
                vdrawpen * target);
Copies the information in pen into target so that the two drawing pens represent equivalent objects. target must have been created or initialized by an Draw Manager function. This function copies all the attributes of the pen, and it clones any attribute objects that are owned by the drawing pen. This function destroys any objects allocated internally by target (including its owned attributes) but does not destroy target itself. The pen to be copied can be dynamic or static.

vdrawpenCopyInit

        void vdrawpenCopyInit(
                vdrawpen * pen,
                vdrawpen * target);
Copies the information in pen into target so the memory addressed by target is a drawing pen equivalent to pen. target should be a pointer to uninitialized memory; the Draw Manager initializes it as part of the copy. This function copies all the attributes of the pen and clones any attribute objects that are owned by the pen.

vdrawpenCreate

        vdrawpen * vdrawpenCreate();
Creates a new drawing pen and returns a pointer to its contents. Storage for the pen is allocated by the Draw Manager. The Draw Manager reclaims the storage occupied by the pen when it is destroyed with vdrawpenDestroy. The new pen is created with a set of default attributes, as documented in the ``Draw Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new drawing pen and calls vdrawpenInit on it.

vdrawpenDestroy

        void vdrawpenDestroy(
                vdrawpen * pen);
Destroys the drawing pen represented by pen, which must have been created or initialized by an Draw Manager function. Use this function to reclaim the storage occupied by a pen. The pen should not be referenced by any other object when it is destroyed. This function attempts to free the storage occupied by pen only if it was allocated by vdrawpenCreate, vdrawpenCreateOfClass, or vdrawpenClone. This function also destroys any attribute objects that are owned by the drawing pen.

vdrawpenGetDashPattern

        float * vdrawpenGetDashPattern(
                vdrawpen * pen,
                size_t * size);
Returns the pen's dash pattern. The array of floats that is returned contains length of each portion of the dash pattern. The odd elements of the array are "on" dashes and the odd elements are "off" dashes. The value of each element is the length of the "on" or "off" dash. Odd length arrays are concatenated with themselves to make a dash pattern; for example, {3.0} is treated as {3.0, 3.0} for computing the dash pattern. The size of the array is returned in size.

vdrawpenInit

        void vdrawpenInit(
                vdrawpen * pen);
Initializes a drawing pen in the memory addressed by pen. The attributes of the pen are set to their default values, as documented in the ``Draw Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Draw Manager as if it were created with vdrawpenCreate. A call to vdrawpenDestroy frees any internal objects allocated by the Draw Manager but does not free the structure itself. A map is automatically initialized for the pen.

The Draw Manager will not attempt to reclaim the pen's storage when it is destroyed. This function simply stores the class in the drawing pen and sends it the message vinstanceINIT.

vdrawpenSetDashPattern

        void vdrawpenSetDashPattern(
                vdrawpen * pen,
                float * pattern,
                size_t size,
                double offset);
Specifies pen's dash pattern. size is the number of elements in array pattern. pattern describes the dash pattern, and offset is the phase offset The elements of array pattern describe the segments in the dash pattern. Segment lengths are expressed in user space units. The elements of the array alternate between on and off segments; the first element of pattern is an on segment. When pattern contains an odd number of elements, the dash pattern is the concatenation of pattern with itself. For example, if pattern equals {3.0, 1.0, 3.0}, the dash pattern is {3.0, 1.0, 3.0, 3.0, 1.0, 3.0}. pattern may be NULL if size is zero (this sets a solid line). offset is the number of user space units truncated from the leading point of the dashed line. If the offset is 3 and pat is {2.0, 2.0, 3.0, 3.0}, then the first instance of the dash pattern is 1 off, 3 on, 3 off. The rest of the path is then drawn with the entire {2.0, 2.0, 3.0, 3.0} pattern. The ends of each dash have the current line cap style. If a drawing context is passed in as a parameter to a function, it should be cloned before any of its pen's attributes are modified.

vdrawpenSetDrawingFunction

        void vdrawpenSetDrawingFunction(

Sets the pen's drawing function to the function represented by the integer identifier. Changing the drawing function compromised device independents and should be avoided under most circumstances. If a drawing context is passed in as a parameter to a function, it should be cloned before any of its pen's attributes are modified. The drawing function determines how the pixels of the source and destination objects interact. Not all window systems support all drawing functions. PostScript printer devices support only vdrawFUNC_COPY. You should avoid using drawing functions other than vdrawFUNC_COPY if at all possible. The function must be one of the following:

vdrawpenSetLineJoinMiterLimit

        void vdrawpenSetLineJoinMiterLimit(
                vdrawpen * pen,
                double limit);
Sets the pen's line join miter limit. If a drawing context is passed in as a parameter to a function, it should be cloned before any of its pen's attributes are modified. When mitered ends have been specified as the style for connecting line segments, the miter limit controls the length of the mitered joins. This eliminates objectionably long miter joins that might result when segments connect at acute angles (see Figure). The limit must be less than one.

vdrawpenSetLineWidth

        void vdrawpenSetLineWidth(
                vdrawpen * pen,
                double width);
Sets the pen’s line width (thickness) to width user space units. For any given device, a line of zero width is defined as the thinnest possible width for that device. Zero width lines give better performance on some window systems, but at the expense of visual fidelity. Zero width lines should be avoided if the target device could be a printer since they are likely to be invisible to the eye. width cannot be less than zero. If a drawing context is passed in as a parameter to a function, it should be cloned before any of its pen's attributes are modified.

Macros

vdrawpenCLASS

        #define vdrawpenCLASS(...) ...
Contains the method and variable declarations for drawing pen classes. This macro expands to the contents of vdrawpenClass. CLASS is the type identifier of the class being declared (for example, vdrawpenClass). This type is used by the class's subclasses to refer to it. PEN is the type identifier of instances of the class being declared (for example, vdrawpen). This macro should be used by drawing pen subclasses to declare the contents of the drawing pen class. This macro should come first in the subclass structure declaration.
