vdomainview

        typedef ... vdomainview;
A domain view is a dialog item that handles the display and editing of a domain. A domain view provides an interface to the domain that displays the domain, selects objects, edits the domain, and so on. It uses a domain observer to monitor the operations performed on the domain. Domain objects are never added to or removed from a domain view; rather, they are added to and removed from the domain, which notifies all observers of the change. The observers then update their view of the domain.

vdomainviewTrackEvent

        typedef ... vdomainviewTrackEvent;
Contains information for a track event, one of the domain view events. This event is generated during mouse tracking (button down-drag-button up) in a domain view, and is sent to vdomainviewHANDLE_TRACK_EVENT method overrides. The fields in this event structure are defined in the following list.
event
The vevent corresponding to this track event. Can be NULL.
state
The current state of the track.
originalPosition or longOriginalPosition
The domain coordinate of the mouse when the button was pressed. Use the latter when operating on long domains.
currentPosition or longCurrentPosition
The domain coordinate of the mouse at its current position. Use the latter when operating on long domains.
dragObjects
The objects being dragged by the user, or NULL if no objects are being dragged.
overDialog
The dialog the mouse is over if external drag is turned on. NULL otherwise.
overDialogPosition
The window coordinate of the mouse in overDialog if external drag is turned on.
context
The vdrawcontext to be used to perform any drawing.

The state field will be one of the states listed below.

vdomainviewTRACK_START
Tracking has started. The button press has been processed, but no dragging has occurred yet. The track is cancelled if vFALSE is returned from the track event handler.
vdomainviewTRACK_MOVE_DRAW
Position has changed, draw graphics. Drawing should be based on originalPosition and currentPosition points.
vdomainviewTRACK_MOVE_UNDRAW
Position has changed, undraw graphics. Drawing should be based on originalPosition and currentPosition points.
vdomainviewTRACK_UPDATE
The view has become invalidated, and graphics must be redrawn. Drawing should be based on originalPosition and currentPosition points.
vdomainviewTRACK_END
Dragging has finished, and track processing has been completed.

vdomainviewNotifyUndo

        void vdomainviewNotifyUndo(
                vdomainview * domainView,
                vscribe * actionName);
Removes the action list from the domain view's domain, names it with the given scribe, and calls the domain view's undo notify procedure with the action. If the domain view's domain's action list is NULL because either there is no undo observer or no undoable actions have been generated, then this call does nothing.

You should call this routine after you make changes to the domain that constitute a user action. The domain view's undo notify routine should then install the action it receives in the application's undo log.

This routine is called internally by the Domain class after the user drags or selects objects, thus causing any undo actions generated by the drag or selection to be reported to your undo notify procedure.

vdomainviewScroll

        void vdomainviewScroll(
                vdomainview * domainView,
                int xPos,
                int yPos,
                int positioning);
Scrolls to any given location in the domain view or scrolls a given amount in any direction. The xPos and yPos parameters must be in the domain's coordinate space. The positioning parameter specifies whether the xPos and yPos parameters are an absolute location, or whether they are a relative amount to scroll. It should be one of the following combinations listed below.

vdomainRELATIVE
deltaX and deltaY are relative.
vdomainABSOLUTE
deltaY and deltaX are absolute.
vdomainX_RELATIVE | vdomainY_ABSOLUTE
deltaX is relative and deltaY is absolute.
vdomainX_ABSOLUTE | vdomainY_RELATIVE
deltaX is absolute and deltaY is relative.

This function is implemented by sending the message vdomainviewSCROLL to domainView.

vdomainviewTRACK_UPDATE
The view has become invalidated, and tracking graphics must be redrawn. Again, Drawing should be based on the originalPosition and currentPosition points found in the vdomainviewTrackEvent structure.
vdomainviewTRACK_END
The mouse button has been released, and track processing has completed.

vdomainviewNEXT_ID_ATTRIBUTE

        vdomainviewNEXT_ID_ATTRIBUTE
Denotes the next available attribute ID for subclasses of the default domain view class. This constant is provided so that attribute IDs defined by subclasses do not conflict with those defined by the Dialog Manager. Only attributes defined by overriding the methods vobjectSET_ID_ATTRIBUTE and vobjectGET_ID_ATTRIBUTE need be concerned with the attribute IDs of their superclasses. Subclasses of vdomainviewClass should begin their attributeIDs with this constant and incremement by one for each attribute. Subclasses should also provide their own NEXT_ID_ATTRIBUTE constant for use by their subclasses. This constant has integral type.

vdomainviewNEXT_SELECT

        vdomainviewNEXT_SELECT
Denotes the first command selector available to subclasses of the default domain view class. This constant is provided so that selector values defined by subclasses do not conflict with those defined by their superclasses. Only class-level commands, defined by overriding the methods vobjectISSUE_SELECTOR and vobjectQUERY_SELECTOR, need be concerned with the selector values of their superclasses. Subclasses of vdomainviewClass should begin their command selectors with this constant and increment by one for each new command.

In other words, if you add commands to the class-level command space of vdomainview, you put their implementations in your vobjectISSUE_SELECTOR override and start with the value of vdomainviewNEXT_SELECT as the first selector value you handle.

Subclasses should also provide their own NEXT_SELECT constant for use by their subclasses. This constant has integral type.

vdomainviewSELECT_MOVE_FOCUS
Moves the focus among the domain view and its scroll bars. The vname_Direction element in the command's context dictionary determines the direction the focus will move. A positive value will move the focus forward, while a zero or negative value will move it backwards.
vdomainviewSELECT_ALIGN
Aligns the objects in the domain view's selection. The vdomainview_Option element in the command's context dictionary specifies how to align the objects. Valid options are:
vdomain_Grid
For future align command issues, align the objects relative to the grid.
vdomain_Objects
For future align command issues, align the objects relative to themselves.
vname_Left
Align the left edges of the selected objects.
vname_Right
Align the right edges of the selected objects.
vname_Top
Align the top edges of the selected objects.
vname_Bottom
Align the bottom edges of the selected objects.
vdomainview_Horizontal
Align the centers of the objects along the horizontal axis.
vdomainview_Vertical
Align the centers of the objects along the vertical axis.
vdomainviewSELECT_DISPLAY_SELECTION
Scrolls the topmost selected object into view.
vdomainviewSELECT_DISTRIBUTE
Distributes the objects in the domain view's selection. The vdomainview_Option element in the command's context dictionary specifies how to distribute the objects. Valid options are:
vname_Left
Distribute the left edges of the selected objects.
vname_Right
Distribute the right edges of the selected objects.
vname_Top
Distribute the top edges of the selected objects.
vname_Bottom
Distribute the bottom edges of the selected objects.
vdomainview_Horizontal
Distribute the centers of the objects along the horizontal axis.
vdomainview_Vertical
Distribute the centers of the objects along the vertical axis.
vdomainviewSELECT_DUPLICATE
Duplicates all objects in the domain view's selection.
vdomainviewSELECT_MOVE
Changes the layering of the objects in the domain view's selection by moving them up or down according to the vdomainview_Option element in the command's context dictionary. Valid options are:
vdomainview_Front
Move each object in front of all other objects in the domain.
vdomainview_Forward
Move each object in front of the object directly above it.
vdomainview_Back
Move each object behind all other objects in the domain.
vdomainview_Backward
Move each object behind the object directly below it.
vdomainviewSELECT_OBJECT
If the vdomainview_Option element in the command's context dictionary is vname_Top, selects the topmost object in the domain.
vdomainviewSELECT_SELECT_ALL
Selects all objects in the domain.

Macros

vdomainviewCLASS

        #define vdomainviewCLASS(...) ...
Contains the method and variable declarations for domain view classes. This macro expands to the contents of vdomainviewClass. SUPER is the type identifier of the class being declared; for example, vdomainviewClass. This type is used by the class's subclasses to refer to it. ITEM is the type identifier of instances of the class being declared; for example, vdomainview.

NOTIFY is the type identifier of the instance notification function; for example, vdialogItemNoteProc. DOMAIN is the type identifier for the domain viewed by the domain view; for example, vdomain. This macro should be used by domain view subclasses to declare the contents of the domain view class. This macro should come first in the subclass structure declaration.
