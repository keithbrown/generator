Types [3] Functions [43] Messages [2] Constants [6] Macros [2]

Types

vcolorchsr

        typedef ... vcolorchsr;
Contains the information needed to represent a single color chooser and its contents. Color choosers are referenced by a pointer to this type.

vcolorchsrApplyFunc

        typedef ... vcolorchsrApplyFunc;
Denotes an apply function that is called when a color has been chosen from the color chooser. color is the color selected.

vcolorchsrClass

        typedef ... vcolorchsrClass;
Describes the behavior of a class of color choosers. The color chooser class controls color application. Color chooser classes are referenced by a pointer to this type.

Functions

vcolorchsrClose

        void vcolorchsrClose(
                vcolorchsr * chooser);
Closes the color chooser colorChooser so that it is no longer visible. This function has no effect on a color chooser that is already closed.

vcolorchsrCreate

        vcolorchsr * vcolorchsrCreate();
Creates a new color chooser of the default color chooser class and returns a pointer to the new color chooser. (Memory is allocated by the Color Chooser.)

vcolorchsrCreateOfClass

        vcolorchsr * vcolorchsrCreateOfClass(
                vcolorchsrClass * clas);
Creates a new color chooser of class clas and returns a pointer to the new color chooser. (Memory is allocated by the Color Chooser.) The new color chooser is created with a set of default attributes determined by the class.

vcolorchsrDestroy

        void vcolorchsrDestroy(
                vcolorchsr * chooser);
Destroys the color chooser colorChooser and all of its sub-objects that were allocated by the Color Chooser. This function attempts to free the storage directly occupied by color chooser only if it was allocated by vcolorchsrCreate or vcolorchsrCreateOfClass.

vcolorchsrGetApply

        vcolorchsrApplyFunc vcolorchsrGetApply(

Returns the apply procedure attached to the color chooser colorChooser, or NULL if there is no apply procedure.

vcolorchsrGetBase

        void vcolorchsrGetBase();
Returns the base vconfirm a color chooser is built upon.

vcolorchsrGetClass

        vcolorchsrClass * vcolorchsrGetClass(

Returns the color chooser class of the color chooser fontChooser.

vcolorchsrGetColor

        vcolor * vcolorchsrGetColor(
                vcolorchsr * chooser);
Returns the current value of the color being edited. The returned color is owned by the color chooser and should be copied before being used.

vcolorchsrGetColorSpec

vcolorchsrGetConfirm

Returns the confirm upon which the color chooser is built. Confirmation Manager operations on this confirm affect the color chooser it came from.

vcolorchsrGetData

                vcolorchsr * chooser);
Returns the generic data pointer attached to the color chooser chooser, which can be NULL.

vcolorchsrGetDefaultClass

        vcolorchsrClass * vcolorchsrGetDefaultClass();
Returns the class used to create a color chooser by the vcolorchsr Create function. This class can be modified to alter the behavior of all color choosers subsequently created by these functions.

vcolorchsrGetDialog

        vdialog * vcolorchsrGetDialog(
                vcolorchsr * chooser);
Returns the dialog upon which the color chooser is built. Dialog Manager operations on this dialog affect the color chooser it came from. All Dialog Manager operations have effect, but some may interfere with the color chooser's behavior.

vcolorchsrGetMethod

        int vcolorchsrGetMethod(
                vcolorchsr * chooser);
Returns the current color method being used to specify the color being edited. The returned method is one of the following values:

vcolorchsrHLS
Color specified by HLS color model.
vcolorchsrRGB
Color specified by RGB color model.
vcolorchsrCMY
Color specified by CMY color model.
vcolorchsrGRAY
Color specified using gray scales.

vcolorchsrGetNamedSortType

        int vcolorchsrGetNamedSortType(
                vcolorchsr * chooser);
Returns the sort type for the named colors list in the color chooser chooser. The return value can be vcolorchsrSORT_BY_NAME which sorts the colors alphabetically by name, or vcolorchsrSORT_BY_HUE which sorts the colors by their hue.

vcolorchsrGetPanel

        int vcolorchsrGetPanel(
                vcolorchsr * chooser);
Returns the currently active panel in the color chooser chooser. The return value can be vcolorchsrWHEEL_PANEL for the color wheel/plane panel, vcolorchsrNAMED_COLORS_PANEL for the list of named colors, or vcolorchsrTOLERANCES_PANEL for the color tolerances panel.

vcolorchsrGetPlane

        int vcolorchsrGetPlane(
                vcolorchsr * chooser);
Returns the color plane to be shown when editing RGB or CMY colors in the color chooser chooser. For the RGB color model, the return value can be vcolorchsrBG_PLANE for the blue-green plane, vcolorchsrRB_PLANE for the red-blue plane, or vcolorchsrRG_PLANE for the red-green plane. For the CMY color model, the return value can be vcolorchsrCM_PLANE for the cyan-magenta plane, vcolorchsrCY_PLANE for the cyan-yellow plane, or vcolorchsrMY_PLANE for the magenta-yellow plane.

vcolorchsrGetReformRGB_CMY

        vbool vcolorchsrGetReformRGB_CMY(
                vcolorchsr * chooser);
Indicates whether or not the chooser dynamically reforms the color ramps when in RGB or CMY color modes.

vcolorchsrGetSavedSpecAt

        vcolorSpec * vcolorchsrGetSavedSpecAt(
                vcolorchsr * chooser,
                int index);
Returns the saved color specification at the specified index index in the color chooser chooser. index should be a value between zero and vcolorchsrGetSavedSpecCount minus one.

vcolorchsrGetSavedSpecCount

        int vcolorchsrGetSavedSpecCount(
                vcolorchsr * chooser);
Returns the number of color specifications that can be saved in the color chooser chooser.

vcolorchsrHasTolerancePanel

        int vcolorchsrHasTolerancePanel(
                vcolorchsr * chooser);
Returns TRUE if the tolerance panel is shown in the color chooser chooser.

vcolorchsrOpen

        void vcolorchsrOpen(
                vcolorchsr * chooser);
Opens the color chooser colorChooser so that it is visible at its current position and size on the screen. This function has no effect if colorChooser is already open.

vcolorchsrPlace

        void vcolorchsrPlace(
                vcolorchsr * chooser,
                vdialog * base,
                int modeX,
                int modeY);
Positions the color chooser chooser relative to the dialog base. modeX and modeY specify the relation of chooser to base on each coordinate axis. (Refer to the vrectPlace routine for the legal values of modeX and modeY.) Note the visual effect of setting the chooser's position is platform dependent and not all native color choosers support this attribute. Pass NULL to position the chooser relative to the root window.

vcolorchsrPrefersNative

        int vcolorchsrPrefersNative(
                vcolorchsr * chooser);
Returns whether the Color Chooser Manager will try to map the color chooser chooser to a native equivalent. Note that a preference for the native chooser is not a guarantee that the Galaxy chooser will not be used. If, for instance, chooser is an instance of a subclass of the default vcolorchsr class, it may not be possible to use the native color chooser. Refer to the ``Standard Dialogs'' chapter of the Galaxy Application Environment Programmer's Guide for a list of guidelines for supporting native color choosers.

vcolorchsrProcess

        const vname * vcolorchsrProcess(
                vcolorchsr * chooser);
Opens a blocking color chooser chooser and waits until a confirmation button (OK or Cancel) is activated or the user closes the color chooser's window. It is important to note that the color chooser's apply function is called if the user selects the OK button. The color chooser's window is closed before vcolorchsrProcess returns.

vcolorchsrSend

vcolorchsrSetApply

        void vcolorchsrSetApply(
                vcolorchsr * chooser,
                vcolorchsrApplyFunc applyFunc);
Changes the apply notification procedure for the color chooser colorChooser to applyFunc. Whenever the color chooser handles an event of importance, this procedure is called with the color chooser and the event as arguments. applyFunc can be NULL, in which case no apply notification is given.

vcolorchsrSetColor

        void vcolorchsrSetColor(
                vcolorchsr * chooser,
                vcolor * color);
Changes the color to be edited by the color chooser to be the color color. This function can be called before or after the color chooser is opened. If the color chooser is open, the change to the color to be edited occurs immediately.

vcolorchsrSetColorSpec

        void vcolorchsrSetColorSpec(
                vcolorchsr * chooser,
                vcolorSpec * color);
Changes the color specification to be edited by the color chooser chooser to color. This function can be called before or after the color chooser is opened.

vcolorchsrSetData

        void vcolorchsrSetData(
                vcolorchsr * chooser,
                void * data);
Sets a color chooser's generic data pointer to data. This can be a pointer to whatever information the client wants to attach to the color chooser---it is not used by the Color Chooser. data can be NULL.

vcolorchsrSetLeader

        void vcolorchsrSetLeader(
                vcolorchsr * chooser,
                vwindow * leader);
Sets the leader of the color chooser chooser to the window leader. leader can be NULL in which case the root window is used as the chooser's leader. Refer to vwindowGetRoot for a description of the root window.

vcolorchsrSetMethod

        void vcolorchsrSetMethod(
                vcolorchsr * chooser,
                int method);
Sets the default color method by which the color being edited is specified. method can have one of the following values:

vcolorchsrHLS
Specify color by HLS color model.
vcolorchsrRGB
Specify color by RGB color model.
vcolorchsrCMY
Specify color by CMY color model.
vcolorchsrGRAY
Specify color using gray scales.

vcolorchsrSetNamedSortType

        void vcolorchsrSetNamedSortType(
                vcolorchsr * chooser,
                int whichSort);
Sets the sort type of the named colors list in the color chooser chooser to whichSort. whichSort can be vcolorchsrSORT_BY_NAME which sorts the colors alphabetically by name, or vcolorchsrSORT_BY_HUE which sorts the colors by their hue.

vcolorchsrSetOverridePreferNative

        void vcolorchsrSetOverridePreferNative(
                vcolorchsr * chooser,
                int doNative);
Sets whether the Color Chooser Manager should try to map the file chooser chooser to a native equivalent. vcolorchsrSetOverridePreferNative should be used for color choosers which are subclasses of the default vcolorchsr class. Normally, the Color Chooser Manager will not map instances of vcolorchsr subclasses to native choosers. Refer to the ``Standard Dialogs'' chapter of the Galaxy Application Environment Programmer's Guide for a list of guidelines for supporting native color choosers. If doNative is FALSE, the Galaxy color chooser is always displayed.

vcolorchsrSetReformRGB_CMY

        void vcolorchsrSetReformRGB_CMY(
                vcolorchsr * chooser,
                int value);
Sets the behavior the color chooser takes when the color model is either RGB or CMY. If value is FALSE, the color chooser shows static ramps from 0 to full intensity for each component of RGB or CMY. If value is TRUE, the color chooser updates the other components of RGB or CMY as one component changes to show the colors the user can get by changing the other components.

vcolorchsrSetSavedSpecAt

Saves a color specification spec at the specified index index in the color chooser chooser. index should be a value between zero and vcolorchsrGetSavedSpecCount minus one (-1).

vcolorchsrSetTitle

Changes the title of the color chooser chooser to the string title. title can be NULL, in which case the color chooser's default title is used. Note the visual effect of setting the color chooser's title is platform dependent and not all native color choosers support this feature.

vcolorchsrSetTolerancePanel

        void vcolorchsrSetTolerancePanel(
                vcolorchsr * chooser,

Adds or removes the tolerance panel from the color chooser chooser. If value is FALSE, any reference to tolerance is removed from the color chooser.

vcolorchsrStartup

        void vcolorchsrStartup();
Initializes the color chooser module. You must do this before loading any color chooser resources. This is normally called from vstartup.

Messages

vcolorchsrAPPLY_COLOR

        <vcolorchsrAPPLY_COLOR>
(Deprecated) Apply message sent when a color is chosen. This message has been deprecated. See vcolorchsrHOOK_APPLY_COLOR.

vcolorchsrHOOK_APPLY_COLOR

        <vcolorchsrHOOK_APPLY_COLOR>
Apply message sent when a color is chosen. This message should be overridden by subclasses to detect when the user has picked a color. The default implementation in the base vcolorchsr class invokes the chooser instance's vcolorchsrApplyFunc if set.

Macros

vcolorchsrCLASS

        #define vcolorchsrCLASS(...) ...
Contains the method and variable declarations for cholor chooser classes. This macro expands to the contents of vcolorchsrClass. SUPER is the type identifier of the class being declared; for example, vcolorchsrClass. This type is used by the class's subclasses to refer to it. CHOOSER is the type identifier of instances of the class being declared; for example, vcolorchsr. HOOK is the type identifier of the instance notification function; for example, vconfirmHookProc.This macro should be used by domain view subclasses to declare the contents of the domain view class. This macro should come first in the subclass structure declaration.
