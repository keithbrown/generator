Types

vfontspec

        typedef ... vfontspec;
Provides a means of specifying fonts independent of the set of fonts currently available on the system.

The font specification object keeps track of the desired attributes by value, favoring some alternatives and disfavoring others. The specification is built by adding biases for or against family, foundry, encoding, and face names; and for or against specific sizes or transformation matrixes.

The vfontspecDetermineFont function applies the instructions in the specification against the available set of system fonts. Strongest biases are applied first. Biases with equal strength are applied in the order they were added to the specificatio. When a particular bias cannot be met, either because of what fonts are available or because of what fonts have already been eliminated, it is disregarded and the remaining biases are considered in turn.

When a font specification is rendered as a single bias string, the order of the individual biases is preserved, but the specific strength values are not retained.

vfontspecAddBias

        void vfontspecAddBias(
                vfontspec * spec,
                const vchar * biasType,
                const vchar * biasValue,
                int biasToward,
                int biasStrength);
Adds a particular bias to the font specification object.

This is a low-level function used by the other functions that add biases. Since it involves some implementation details of the particular encoding of a font specification's bias string, it is best not to use this function except for more advanced manipulations of the specification object.

vfontspecDetermineFont

        vfont * vfontspecDetermineFont(
                vfontspec * spec);
Determines and returns the font which best matches the font specification.

The font returned is from the set of fonts recognized by the system at that point in time. If more fonts are installed or if the application is run on a different platform, a different font may be returned.

vfontspecGetBiasString

        const vchar * vfontspecGetBiasString(
                vfontspec * spec);
Returns the bias string that encodes the font specification.

While this bias string preserves the order of the individual biases in the font specification, it does not retain their actual strength values. Therefore, a second font specification initialized with this bias string will behave the same with respect to determining a matching font, but it may not behave the same with respect to adding new biases.

vfontspecSetDefaultBiasString

        void vfontspecSetDefaultBiasString(
                const vchar * biasString);
Sets the default font specification bias string for the application.

This specification is applied when there are more than one match after applying any specific font specification.

Different systems may start with different default specifications because of the different requirements and expectations for those platforms. For this reason, it is usually best not to change the default specification unnecessarily. The same effect can be achieved by forming a more specific font specification in the first place.

Values which indicate the relative strengths of different biases.

These constants or integral values derived from them may be passed to vfontspec or vfontspec and its more specific variants (vfontspec, etc.). Also, they may be used to gauge the strength values returned by vfontspec.
