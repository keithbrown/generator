Types

vcontrol

        typedef ... vcontrol;
Contains the information needed to represent a single control; objects of this type have classes derived from vcontrolClass. A control is a dialog item with a ``value.'' Controls are referenced by a pointer to this type. Controls can be created dynamically with vcontrolCreate, or initialized statically with vcontrolInit.

Controls can be created dynamically or loaded from resource files by using the appropriate constructor. A vcontrol contains a dialog item and a set of control attributes. Refer to the ``Control Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of control attributes.

All object types with classes that directly inherit from vcontrolClass must declare a vcontrol as their first field so that vcontrol pointers can be interchanged with pointers to the new data type.

vcontrolClass

        typedef ... vcontrolClass;
Contains method and variables bindings for objects of type vcontrol. Classes declared with this type contain the same set of methods and variables as the default control class, but the behavior of methods and the contents of variables may vary. This class type is derived from vdialogItemClass.

vcontrolNoteProc

        typedef ... vcontrolNoteProc;
Denotes a notification function that is called when a control value is changed by the user. control is the control whose value changed, and event is the event that caused the change. Notification is not given for every event that is handled by the control. If additional notification is required, override the appropriate event handler.

vcontrolPart

        typedef ... vcontrolPart;
(Deprecated) Represents a sub-part of a control that has limited behavior of its own. Higher-level objects use the part interface of the Control class to construct complex controls that can be manipulated in several different ways. For example, a slider control contains three parts: the thumb and the two page areas (the areas to either side of the thumb).

vcontrolPART_HOLD should be set in flags if the control part should track even when the mouse is not being moved (scroll bar arrows are an example of this kind of control part). The function pointed to by Hit should return TRUE if the appropriate part of control control contains the event event, FALSE if it does not. The functions pointed to by Hilite and Unhilite should add and remove highlighting from the appropriate part of the control control. The function pointed to by Track should track the appropriate part of the control control with the event event and return TRUE if the control value changed, FALSE if not.

vcontrolPART_SELECT should be set in flags if the control part defines a Select function. Since this function pointer was not available in earlier versions of the library, it is assumed absent by default. The function pointed to by Select should respond to the selection action (such as button up) and return TRUE if the control value changed. Any function pointer can be NULL, in which case default behavior is supplied by the Control class.

The vcontrolPart type uses graphics state based drawing and has therefore been deprecated. The new type vcontrolPartWith contains function pointers for tracking, selecting, and hiliting which take a drawing context. Old controls which still use vcontrolParts should work fine for the time being, but may not in future releases of Galaxy. Currently a vcontrolPart is used if no vcontrolPartWith is set.

vcontrolToggle

        typedef ... vcontrolToggle;
Contains the information needed to represent a single toggle control; objects of this type have classes derived from vcontrolToggleClass. A toggle control is a two-state switch which can be either on or off. Toggle controls are referenced by a pointer to this type. Toggle controls can be created dynamically with vcontrolCreateToggle, or initialized statically with vcontrolInitToggle. A vcontrolToggle contains a dialog item, a set of control attributes and a menu. Refer to the section ``Control Attributes'' in the ``Control Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of control attributes.

All object types with classes that directly inherit from vcontrolClass must declare a vcontrol as their first field so that vcontrol pointers can be interchanged with pointers to the new data type.

vcontrolClone

        vcontrol * vcontrolClone(
                vcontrol * control);
Creates a copy of the control control and returns a pointer to its contents. Storage for the Control class object is allocated by the Control class. The Control class reclaims the storage occupied by the control when it is destroyed with vcontrolDestroy. This function copies all the attributes of the control except ItemList, Open, and Focused; it clones any attribute objects that are owned by the control. The control to be cloned can be dynamic or static. This function essentially allocates space for a new control and calls vcontrolCopyInit on it.

vcontrolCopy

        void vcontrolCopy(
                vcontrol * control,
                vcontrol * target);
Copies the information in control into target so that the two controls represent equivalent objects. target must have been created or initialized by a Control class function. control and target must have compatible classes for this to work correctly. The target control is invalidated if its dialog is open. This function copies all the attributes of the control except Class, ItemList, Open, and Focused; it clones any attribute objects that are owned by the control. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The controls to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to control.

vcontrolCopyInit

        void vcontrolCopyInit(
                vcontrol * control,
                vcontrol * target);
Copies the information in control into target so the memory addressed by target is a control equivalent to control. target should be a pointer to uninitialized memory; the Control class initializes it as part of the copy. This function copies all the attributes of the control except ItemList, Open, and Focused; it clones any attribute objects that are owned by the control. This function simply stores the class of control in target and sends the message vobjectCOPY_INIT to control.

vcontrolDestroy

        void vcontrolDestroy(
                vcontrol * control);
Destroys the control represented by control, which must have been created or initialized by a Control class function. Use this function to reclaim the storage occupied by a control. The control should not be referenced by any other object when it is destroyed. This function only attempts to free the storage occupied by control if it was allocated by:
  • vcontrolCreate
  • vcontrolCreateToggle
  • vcontrolCreateOptionMenu
  • vcontrolCreateOfClass
  • vcontrolClone
  • vcontrolLoad.

This function also destroys any attribute objects that are owned by the control. This function is implemented by sending the message vobjectDESTROY to control.

vcontrolLoad

        vcontrol * vcontrolLoad(
                vresource resource);
Reads a control from the resource resource and returns a pointer to its contents. Storage for the control is allocated by the Control class. The control's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class class registry.

Note that this function creates an object based on the class of the resource, even if this class is not a control class.

The object pointer is cast to a control pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default control class which are stored in resources are documented in the ``Control Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vcontrolLoadInit on it.

vcontrolLoadInit

        void vcontrolLoadInit(
                vcontrol * control,
                vresource resource);
Reads a control from the resource resource and stores it in the uninitialized memory addressed by control. The control's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class class registry.

Note that this function initializes an object based on the class of the resource, even if this class is not a control class.

The object attempts to initialize itself in the control structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default control class which are stored in resources are documented in the ``Control Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function stores the class specified by the resource in the class field of the control structure and sends it the message vobjectLOAD_INIT.

vcontrolSetIndicator

        void vcontrolSetIndicator(
                vcontrol * control,
                const vrect * rect);
Changes the indicator rectangle of the control control to rect in the same coordinate system as the control's item rectangle. The exact contents of the indicator rectangle depend on the class of control being used. Typically the indicator rectangle is the sub-region of the control which is affected by value changes. The indicator rectangle is invalidated by the Control class as a result of changes to the control's value. This function is implemented by sending the message vcontrolSET_INDICATOR to control.

vcontrolSetMax

        void vcontrolSetMax(
                vcontrol * control,
                int max);
Changes the maximum value of the control control to max. If max is less than the control's minimum value, the maximum value is pinned to the minimum value. This function has no effect if the control already has the desired maximum value, after pinning. If the control's value is greater than the new maximum value minus the view size, the value is pinned to the new maximum value minus the view size. If the control's dialog is open, the control indicator rectangle is invalidated before and after the maximum value change. This function calls vcontrolRecalc to recalculate the indicator rectangle. This function is implemented by sending the message vcontrolSET_MAX to control.

vcontrolSetMenu

        void vcontrolSetMenu(
                vcontrol * control,
                vmenu * menu);
Sets the menu for the control control to menu. This menu is activated when the pointer menu button is pressed with the pointer inside the control's bounding rectangle. The menu can also be activated by the MENU key on some keyboards. An option menu control uses its menu to select among a number of mutually exclusive choices. The menu is destroyed when the control is destroyed or the Menu attribute is changed. The menu is also cloned when the control is cloned or copied. Any previous menu attribute is destroyed by this function before the new menu is set. menu can be NULL, in which case the control is given no menu. vcontrolSetView

        void vcontrolSetView(
                vcontrol * control,
                int view);
Changes the view size of the control control to view. The view size is used to calculate the length of the control's indicator (if any) and to calculate the ``effective maximum'' value of the control. For example, a text item would set the minimum value of its scroll bar to one, the maximum value to the number of lines in the document, and the view size to the size of its pane in lines.

The Control class automatically pins the control value correctly and draws the scroll bar with its indicator proportionally sized. A special case occurs when the view size is zero: this indicates that the indicator should be of fixed length and should not affect the ``effective maximum'' value of the control. This function has no effect if the control already has the desired view size. If the control's value is greater than its maximum value minus the new view size, the value is pinned to the maximum value minus the new view size.

A special case occurs when running under the Windows look-and-feel. If the view is less than the ``effective maximum'' the scrollbar is disabled. This behavior is desired since the thumb of the scrollbar under this look and feel is not proportional (the size of the thumb does not grow as the number of items outside the view associated with the scrollbar shrinks). Setting the view to zero always enables the scrollbar. If the control's dialog is open, the control indicator rectangle is invalidated before and after the view size change. This function calls vcontrolRecalc to recalculate the indicator rectangle. vexArgRangeException is generated if view is less than zero. This function is implemented by sending the message vcontrolSET_VIEW to control.

vcontrolCLASS

        #define vcontrolCLASS(...) ...
Contains the method and variable declarations for control classes. This macro expands to the contents of vcontrolClass. SUPER is the type identifier of the class being declared (for example, vcontrolClass). This type is used by the class's subclasses to refer to it.

CONTROL is the type identifier of instances of the class being declared (for example, vcontrol). NOTIFY is the type identifier of the instance notification function (for example, vcontrolNoteProc). This macro should be used by control subclasses to declare the contents of the control class. This macro should come first in the subclass structure declaration.
