Types

vspringdom

        typedef ... vspringdom;
A spring domain is a domain with a special border attribute. The size and position of this border form the dynamic bounds that are the ultimate basis for the position of all of the objects in the spring domain.

vspringdomArrangerClass

        typedef ... vspringdomArrangerClass;
Describes how to apply a given spring solution to a list of spring domain objects. The arranger class contains the functions that convert solution indexes into object pointers; that retrieve the current, minimum, and natural sizes of objects; that constrain any new sizes; and that actually set the new positions and sizes. vspringdomArrangerClass is a subclass that implements the interface described by the abstract base class, vspringArrangerClass.

vspringdomClass

        typedef ... vspringdomClass;
Describes the behavior of a class of spring domains. The spring domain class is a subclass of the default domain class (returned by vdomainGetDefaultClass). Spring domain classes are referenced by a pointer to this type.

vspringdomConnection

        typedef ... vspringdomConnection;
Contains the information needed to represent a single spring connection. This information includes whether its base length is based on a natural size or a fixed number and whether it stretches based on the dimensions of the domain. Spring connections are referenced by a pointer to this type.

vspringdomIndexFunc

        typedef ... vspringdomIndexFunc;
Specifies a function that uniquely orders all of the objects in a spring domain. The indexes returned by the function should be in the range zero to the number of spring objects minus one. No two objects should return the same index.

vspringdomObject

        typedef ... vspringdomObject;
Contains the information needed to represent a single object in a spring domain and its eight connections: a forward and backward connection for each of its left, right, bottom, and top edges. Spring domain objects are referenced by a pointer to this type.

vspringdomObjectSet

        typedef ... vspringdomObjectSet;
Contains the information needed to represent a set of spring objects. Spring object sets are referenced by a pointer to this type. You may freely cast spring object set pointers to and from domain object set pointers, allowing you to use Domain Manager object set functions to manage spring object sets.

vspringdomObserver

        typedef ... vspringdomObserver;

vspringdomFSetConnectionStretchFactor

        void vspringdomFSetConnectionStretchFactor(
                vspringdom * springdom,
                vspringdomConnection * connection,
                double stretchFactor);
Sets the specified connection's spring factor with a floating pointer number, with 1.0 being the factor for a single spring.

The stretch factor of a spring determines how much it stretches relative to the other springs in its chain. A double spring stretches half as much as a single spring, etc. If the connection does not stretch, then the stretch factor is maintained, but ignored.

The current version of Galaxy quantizes spring factors in units of one eighths (0.125) and pegs all values below 0.125 and above 8.875 to those bounds, respectively.

vspringdomSetConnectionStretchFactor

        void vspringdomSetConnectionStretchFactor(
                vspringdom * springdom,
                vspringdomConnection * connection,
                int stretchFactor);
Sets the stretch factor of the specified connection.

The stretch factor of a spring determines how much it stretches relative to the other springs in its chain. A double spring stretches half as much as a single spring, etc. If the connection does not stretch, then the stretch factor is maintained, but ignored.

The current version of Galaxy supports the following stretch factor values or any combination of them (using the bitwise or operation):

vspringdomSINGLE_SPRING
the default spring factor.
vspringdomDOUBLE_SPRING
twice as strong (therefore half as stretchy) as a single spring.
vspringdomFOUR_TIMES_SPRING
four times as strong as a single spring.
vspringdomEIGHT_TIMES_SPRING
eight times as strong as a single spring.
vspringdomHALF_SPRING
half as strong (therefore twice as stretchy) as a single spring.
vspringdomQUARTER_SPRING
one fourth as strong as a single spring.
vspringdomEIGHTH_SPRING
one eighth as strong as a single spring.

The vspringdomDEFAULT_SPRING value can be passed to denote the use of a single spring, but it should not be used in combination with the other constants. The vspringdomSTRETCH_FACTOR_MAX value represents the strongest (least stretchy) spring factor.

vspringdomSetConnectionStretches

        void vspringdomSetConnectionStretches(
                vspringdom * springdom,
                vspringdomConnection * connection,
                int stretches);
Sets whether the connection stretches based on the current size of the border. The value of any stretch ultimately depends upon the bounds of the spring domain's border. When the Spring Manager places objects, it first positions objects (actually edges) along a complete chain of connections from, say, the left border to the right border. Next, objects along complete chains from object edges already placed are placed, and so on. Within each of these chains, there must be at least one stretch segment. The stretch segments in a chain equally split the difference between the chain endpoint separation and the total of the base segments.

Sets whether the Spring Manager keeps the border of the springdom springdom to at least the minimum size as specified by the domain's internal solution. Otherwise, the border may have any size. Constraining the border size adds to the expense of recalculation. The default value of this flag is TRUE, but you may want to temporarily clear it if you are doing frequent recalculations and do not care about the intermediate border sizes.

vspringdomSetData

        void vspringdomSetData(
                vspringdom * springdom,
                void * data);
Sets the generic data pointer for springdom to data. This can be a pointer to whatever information you want to attach to the spring domain; it is not used by the Spring Manager. The value of data can be NULL. This function is implemented by calling vdomainSetData.

vspringdomSetIndexFunc

        void vspringdomSetIndexFunc(
                vspringdom * springdom,
                vspringdomIndexFunc func);
Sets the indexing function for the springdom springdom to func. This function is used to order the objects in the springdom when you compile the spring domain into a vspringSolution with vspringdomCompile. You can pass NULL as func, which means the order will be based on the layering order of the spring objects (the topmost object having an index of zero).

If this item list is set, then when objects are relocated because of spring recalculations, then the dialog item rects are updated as well.

vspringdomSetObjectData

        void vspringdomSetObjectData(
                vspringdom * springdom,
                vspringdomObjectSet * objectSet,
                void * data);
Sets the generic data pointer for the spring objects in objectSet to data. This can be a pointer to whatever information you want to attach to the spring domain object; it is not used by the Spring Manager. The value of data can be NULL. You cannot pass springdom's border in objectSet. This function is implemented by calling vdomainSetObjectData.

vspringdomSetSolution

        void vspringdomSetSolution(
                vspringdom * springdom,
                vspringSolution * solution);
Arranges the objects in the springdom springdom according to positions specified by interpreting solution with the spring domain's arranger class. The ownership of solution is given to springdom.

vspringdomStartup

Initializes the spring domain part of the Spring Manager. Has no effect if it is already initialized.

vspringdomStore

        void vspringdomStore(
                vspringdom * springdom,
                vresource resource);
Stores the spring domain pointed to by the springdom springdom and all of its objects in resource. This function is implemented by sending the message vobjectSTORE to the springdom.

vspringdomUseConnectionNaturalBase

        void vspringdomUseConnectionNaturalBase(
                vspringdom * springdom,
                vspringdomConnection * connection,
                int useNaturalBase);
Sets whether the connection uses its natural length as its base length. Typically, the natural length of an outside connection is zero, and the natural length of inside connections depend on the properties of the object.

vspringdomUsesConnectionNaturalBase

        vbool vspringdomUsesConnectionNaturalBase(
                vspringdom * springdom,
                vspringdomConnection * connection);
Returns whether the connection's base length is determined by the fixed base value or the natural base value.

Macros

vspringdomCLASS

        #define vspringdomCLASS(...) ...
Defines the necessary methods for all spring domain classes by extending the vdomainCLASS macro. This macro forms the body of the vspringdomClass data structure, and all classes that inherit from this class should have structures formed from this macro or an extension of this macro. SUPER is the type identifier of the class being declared (for example, vspringdomClass). This type will be used by the class's subclasses to refer to it. DOMAIN is the type identifier of the instances of the class being declared (namely, vspringdom for vspringdomClass). OBJECT is the type identifier for the domain's objects (namely, vspringdomObject for vspringdomClass). OBJECTSET is the type identifier for the sets of the domain's objects (namely, vspringdomObjectSet for vspringdomClass). SELECTION is the type identifier for the domain's selections (vspringdomClass still uses vdomainSelection).

vspringdomObserverCLASS

        #define vspringdomObserverCLASS(...) ...
Defines the necessary methods for all spring domain observer classes by extending the vdomainObserverCLASS macro. This macro forms the body of the vspringdomObserverClass data structure, and all classes that inherit from this class should have structures formed from this macro or an extension of this macro. SUPER is the type identifier of the class being declared (for example, vspringdomObserverClass). This type will be used by the class's subclasses to refer to it. SELECTION, OBJECT, and OBJECTSET are the same as for the vspringdomCLASS macro.
