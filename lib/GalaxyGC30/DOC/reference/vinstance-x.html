Types

vinstance

        typedef ... vinstance;
Contains the information needed to represent a single instance of an object---objects of this type have classes derived from vinstanceClass. An instance is an object that provides simple creation and destruction semantics; this class can be used instead of the vobject class for those classes that need to be instantiated but do not provide resource access semantics. Instances can be created dynamically using the vinstanceCreateOfClass function or can be initialized statically using the vinstanceInitOfClass function. All object types and classes that directly inherit from vinstanceClass must declare a vinstance as their first field so that vinstance pointers can be interchanged with pointers to the new data type. Instances inherit from vclassROOT.

vinstanceAllocFunc

vinstanceClass

        typedef ... vinstanceClass;
Contains method and variable bindings for objects of type vinstance. Classes declared with this type contain the same set of methods and variables as the default instance class, but the behavior of methods and the contents of variables may vary. This class type is derived from vclassROOT.

vinstanceCreateOfClass

        vinstance * vinstanceCreateOfClass(
                vinstanceClass * clas);
Creates a new instance of class clas and returns a pointer to its contents. Storage for the instance object is allocated by the Class Manager. The Class Manager reclaims the storage occupied by the instance when it is destroyed with vinstanceDestroy.

vinstanceDestroy

        void vinstanceDestroy(
                vinstance * instance);
Destroy the instance object represented by instance, which must have been created or initialized by a Class Manager function. Use this function to reclaim the storage occupied by instance. instance should not be referenced by any other object when it is destroyed. Any objects allocated internally by the Class Manager are freed. The function only attempts to free the storage occupied by instance if instance was allocated by vinstanceCreateOfClass. This function is implemented by sending the message vinstanceDESTROY to instance.

vinstanceDestroyLater

        void vinstanceDestroyLater(
                vinstance * instance);
Destroy the instance object represented by instance a second after the current event has finished being processed. instance must have been created or initialized by a Class Manager function. Use this function to reclaim the storage occupied by `instance' at some future time. instance should not be referenced by any other object when it is destroyed. Any objects allocated internally by the Class Manager are freed. The function only attempts to free the storage occupied by instance if instance was allocated by vinstanceCreateOfClass(). This function is implemented by posting an event to the event queue marking instance for destruction using vinstanceDestroy. Same as vobjectDestroyLater.

vinstanceGetDefaultClass

        vinstanceClass * vinstanceGetDefaultClass();
Returns the class used to create instances using the vinstanceCreateOfClass or vinstanceInitOfClass functions. This class can be modified to alter the behavior of all instances subsequently created by this function. Changes to this class are not visible to any other applications in the same process or sharing the same display server. This function can be called to explicitly register the default instance class with the Class Manager.

vinstanceInitOfClass

        void vinstanceInitOfClass(
                vinstance * instance,
                vinstanceClass * clas);
Initializes a dialog of class clas in the memory addressed by instance. A call to vinstanceDestroy frees any internal objects allocated by the Class Manager but does not free the structure itself. This function is implemented by storing clas in the class field of instance and then sending the message vinstanceINIT to instance.
