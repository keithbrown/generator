Types

vurl

        typedef ... vurl;
The URL class encapsulates URLs to make them easy to use in a Galaxy program. Several other classs including vwebbrowse and vhelpref use vurl.

The URL class provides ways of encoding and decoding URL strings, operating on the component parts of the URL, and resolving URLs relative to a base URL. The algorithms used to parse URLs are based on the specifications in RFC1808.

vurlClass

        typedef ... vurlClass;
Contains message and variable bindings for objects of type vurl. Classes declared with this type contain the same set of methods and variables as the status help display class, but the behavior of methods and the contents of variables may vary. This class type is derived from vloadable.

vurlResolve

        void vurlResolve(
                vurl * url,
                vurl * base);
This method is very useful for generating URLs to resources which are relative to some other location. A relative URL is one which does not have a scheme but which may contain either a relative or absolute path. Resolution of a relative URL involves the following steps:
  • The relative URL inherits the scheme of the base.
  • The relative URL inherits the net location of the base, if it does not already have one.
  • The path is resolved:
    • It the relative URL's path is empty, it inherits the path of the base. If the relative URL does not have a parameters component or a query component, each is inherited from the respective base components.
    • If the relative URL's path does not begin with a slash "/", it is considered relative. The base URL's path is prepended (minus any terminating file element) to that of the relative URL.
    • If the relative URL's path begins with a slash "/", it is considered absolute and is not affected by the base path.

Relative paths may contain ".." elements to effectively remove elements from the base path during resolution. See Section 4 of RFC1808 for additional information concerning this algorithm.

This function is implemented by sending the message vurlRESOLVE to url.

vurlStore

        void vurlStore(
                vurl * url,
                vresource resource);
Writes the contents of the url url into the resource resource. The url can be recovered from the resource at a later time (perhaps by another program) with vurlLoad or vurlLoadInit. Generates a vobjectUnnamedClassException exception if the url's class is not named. This function is implemented by sending the message vobjectSTORE to url.

To store a url in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vurlStore to write the url into the resource. For example: 


vurl *url;    /* contains a url */
vresource fileResource, urlResource;

fileResource   = vresourceOpenFile( ... );
urlResource = vresourceMake(fileResource, 
                               vnameInternGlobalLiteral("MyUrl"));

vurlStore(url, urlResource);
vresourceCloseFile(fileResource);

vurlCLASS

        #define vurlCLASS(...) ...
This macro contains the method declarations for the url class, this macro expands to the contexts of vurlClass. SUPER<\SAMP> is the type identifier of the class being declared. This type is used by the class subclasses to refer to it. URL is the type identifier of the instances of the class being declared. This macro should be used by url subclasses to declare the contents of the url class. This macro should come first in the subclass structure declaration.
