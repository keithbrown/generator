Types

vprint

        typedef ... vprint;
Contains the information needed to represent a printer. Objects of this type have classes derived from vprintClass. Printers can be created dynamically with vprintCreate, or initialized statically with vprintInit. A vprint contains an object and a set of printer attributes. All structure declarations that directly inherit from vprintClass must declare a vprint as their first field so that vprint pointers can be interchanged with pointers to the new data type.

Graphical applications should create an object of the vprintInteractive subclass by vprintCreateOfClass. Non-graphical applications should use vprintCreate to create a printer of an appropriate type for the current platform.

vprintCloseConnection

        void vprintCloseConnection(
                vprint * printer);
Closes the connection for the printer printer. It is only necessary to call this method if you are printing using the low-level printing methods such as vprintWrite; vprintCloseDocument will automatically close the connection if it is was opened by vprintOpenDocument. This function is implemented by sending the message vprintCLOSE_CONNECTION to the printer.

vprintCloseDocument

        void vprintCloseDocument(
                vprint * printer);
Closes the current document on the printer printer. Use this function to mark the end of each document after you have closed all of its pages. An open document must be closed before opening a new document. This function is implemented by sending the message vprintCLOSE_DOCUMENT to the printer.

vprintClosePage

        void vprintClosePage(
                vprint * printer);
Closes the current page on the printer printer. Use this function to mark the end of each page that you have drawn. You must close each page you open before opening a new page. The device attribute of the current gstate will no longer be valid after calling this function; do not attempt to draw anything until you open a new page. A page must be open for this function to work. This function is implemented by sending the message vprintCLOSE_PAGE to the printer.

vprintConfirmJob

        void vprintConfirmJob(
                vprint * printer);
For the vprintInteractive class, this method displays a printer Job dialog to confirm printer attributes, and notifies the printer. For low-level printer classes, this method simply notifies the printer directly.

Note that job notification may happen before this function returns, or it may be deferred until control returns to the event loop. A printer's Job attributes are specific to a particular print job, whereas its Style attributes are specific to a particular document. This function is implemented by sending the message vprintCONFIRM_JOB to the printer.

vprintCreate

        vprint * vprintCreate();
Creates a new printer of the preferred low-level printer class and returns a pointer to it. Non-graphical applications should use this method to create a new printer of a class appropriate to the current platform. Graphical applications should call vprintCreateOfClass(vprintGetInteractiveClass()) instead. Storage for the printer is allocated by the Print class. The Print class reclaims the storage occupied by the printer when it is destroyed with vprintDestroy. The new printer has no open documents or pages. This function essentially allocates space for a new printer and calls vprintInit on it.

vprintCreateOfClass

        vprint * vprintCreateOfClass(

Creates a new printer of class clas and returns a pointer to its contents. Storage for the printer is allocated by the Print class. The Print class will reclaim the storage occupied by the printer when it is destroyed with vprintDestroy. The new printer has no open documents or pages.

The is established with the printer according to class heuristics and user preferences. This function essentially allocates space for a new printer and calls vprintInitOfClass on it.

vprintDestroy

        void vprintDestroy(
                vprint * printer);
Destroys the printer represented by printer, which must have been created or initialized by a Print class function. Use this function to close a printer and reclaim the storage it occupies. The printer should not be referenced by any other object when it is destroyed, nor should it have any open documents or pages. This function only attempts to free the storage occupied by printer if it was allocated by vprintCreate or vprintCreateOfClass. This function is implemented by sending the message vobjectDESTROY to printer.

Messages

vprintCLOSE_CONNECTION

        <vprintCLOSE_CONNECTION>
Closes the connection for the printer printer. It is only necessary to call this method if you are printing using the low-level printing methods such as vprintWRITE; vprintCLOSE_DOCUMENT will automatically close the connection if it was opened by vprintOPEN_CONNECTION. The function entry point vprintCloseConnection is implemented by sending this message to the printer.

The default method for the default printer class does nothing. Subclasses that override this method should open a printer connection appropriate to the current printer preferences and attributes.

vprintCLOSE_DOCUMENT

        <vprintCLOSE_DOCUMENT>
Closes the current document on the printer printer. Use this message to mark the end of each document after you have closed all its pages. The function entry point vprintCloseDocument is implemented by sending this message to the printer.

The default method for the default printer class does nothing. Subclasses that override this method should take whatever action is necessary to close the current document on the printer.

vprintCLOSE_PAGE

        <vprintCLOSE_PAGE>
Closes the current page on the printer printer. Use this message to mark the end of each page that you have drawn. The function entry point vprintClosePage is implemented by sending this message to the printer.

The default method for the default printer class does nothing. Subclasses that override this method should send a printer command sequence to close the current page across the printer.

vprintCOMMENT

vprintCONFIRM_JOB

        <vprintCONFIRM_JOB>
For the vprintInteractive class, this method displays a printer Job dialog to confirm printer attributes, and notifies the printer. For low-level printer classes, this method simply notifies the printer directly. The function entry point vprintConfirmJob is implemented by sending this message to the printer.

Note that notification may happen before this function returns, or it may be deferred until control returns to the event loop. A printer's Job attributes are specific to a particular print job, whereas its Style attributes are specific to a particular document.

vprintCONFIRM_STYLE

        <vprintCONFIRM_STYLE>

For low-level printer classes, this method simply notifies the printer directly. The function entry point vprintConfirmJob is implemented by sending this message to the printer.

Note that style notification may happen before this function returns, or it may be deferred until control returns to the event loop. A printer's Job attributes are specific to a particular print job, whereas its Style attributes are specific to a particular document.

vprintOPEN_DOCUMENT

vprintOPEN_PAGE

        <vprintOPEN_PAGE>
Opens a new page on the printer printer. Use this message to mark the beginning of each page that you intend to print. This message also installs a printer device in the current gstate. The function entry point vprintOpenPage is implemented by sending this message to the printer.

The default method for the default printer class sets the device attribute of the current gstate to the null device so that subsequent drawing will have no effect. Subclasses that override this member function should take any action is necessary to open a new page on the printer, then install a subclass-specific printer device in the current gstate. The printer device should translate Drawing class drawing operations into printer command sequences. Refer to the Drawing chapter in the Galaxy Programmer's Guide for more information about drawing devices.

vprintOPEN_PAGE_CONTEXT

        <vprintOPEN_PAGE_CONTEXT>
Opens a new page on the printer and returns a vdrawcontext for drawing on the page. Use this method to obtain a drawing context for each page you intend to print. A document must be open on the printer before you call this function. Close each page before you open a new page. The drawing context returned by this method is valid until vprintClosePage is called. The function entry point vprintOpenPageContext is implemented by sending this message to the printer.

Subclasses that override this member function should take whatever action is necessary to open a new page on the printer, then return a subclass-specific printer drawing context. The printer drawing context should translate Drawing class drawing operations into printer command sequences. Refer to the Drawing chapter in the Galaxy Programmer's Guide for more information about drawing contexts.

vprintSET_PAGE_NUMBER

        <vprintSET_PAGE_NUMBER>
Sets the current page number of the printer printer to pagenum. If the current page is not in that range, drawing will be sent to the null device. Applications with basic printing requirements can simply print all their pages and rely on the Print class to discard the unwanted pages. Applications that must do time-consuming work to format a page may wish to check vprintGetFirstPageNumber and vprintGetLastPageNumber and use vprintSetPageNumber to avoid formatting discarded pages.
