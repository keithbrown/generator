Types

vrect

        typedef ... vrect;
Represents a rectangle whose sides are aligned with the coordinate axes in a given coordinate system. x and y indicate the x and y coordinates of the origin; w and h indicate the width and height of the rectangle. w and h are signed but Rectangle Library functions do not work correctly if they contain negative values. The structure members of a matrix are visible to clients of the Rectangle Library but their particular representation is not. Components of vrect have at least 16 bits of precision. A service often lets an array of one or more rectangles be specified, so that requests can be batched together for better performance.

vrectDifferenceSets

        vrect * vrectDifferenceSets(
                const vrect * rects1,
                const vrect * rects2,
                int count1,
                int count2,
                int * diffCount);
Computes the relative difference of the rectangle sets rects1 and rects2 and returns a pointer to the result set. The relative difference is defined as those points that are in the first set, but not in the second. There are count1 rectangles in the rectangle set rects1, count2 rectangles in the set rects2, and diffCount rectangles in the difference. diffCount is returned by this function. The difference set can be destroyed with vrectDestroySet. The difference set is normalized such that no two rectangles overlap, even if guaranteeing this increases the total number of rectangles in the difference. The difference set can be empty (refer to Figure).

vrectLimit

        void vrectLimit(
                const vrect * rect1,
                const vrect * rect2,
                vrect * rrect);
Translates the rect rect1 such that it lies completely inside rect2 and returns the result in rrect. If necessary, it truncates the resulting rectangle so that both its dimensions lie inside rect2. If a dimension of rrect cannot fit within the corresponding dimension of rect2, the left or bottom of that dimension is aligned with rect2 and the width or height is set to the width or height of rect2. rrect is moved as little as possible. rrect can be the same pointer as rect1 or rect2 (refer to Figure).

vrectPlace

        void vrectPlace(
                const vrect * rect,
                const vrect * base,
                int modeX,
                int modeY,
                vrect * rrect);
Positions the rectangle rect relative to the rectangle base and returns the result in rrect. modex and modey specify the relation of the rectangle to base on each coordinate axis:

vrectPLACE_NEAR
Completely outside with a small margin.
vrectPLACE_ADJACENT
Completely outside with no margin.
vrectPLACE_STRADDLE
Half inside, half outside.
vrectPLACE_ALIGN
Completely inside with no margin.
vrectPLACE_INSET
Completely inside with a small margin.
vrectPLACE_HALF
Opposite side touches center of base rectangle.
vrectPLACE_CENTER
Centered inside.

Additionally, a bitwise OR (|) of modex should be performed with either:

vrectPLACE_LEFT
Left sides of base and the rectangle rect are aligned.
or
vrectPLACE_RIGHT
Right sides of base and the rectangle rect are aligned.

An OR of modey should be performed with either:

vrectPLACE_BOTTOM
Bottom sides of base and the rectangle rect are aligned.
or
vrectPLACE_TOP
Top sides of base and the rectangle rect are aligned.

These constants specify which sides of the rectangles are used to determine the new coordinates. vrectPLACE_CENTER ignores these constants since both sides are always used to calculate centering.
