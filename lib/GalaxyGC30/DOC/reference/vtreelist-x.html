Types [8] Functions [87] Messages [55] Constants [13] Macros [2]

Types

vtreelist

        typedef ... vtreelist;
Treelist objects can be used to display hierarchical data. They allow users to expand and collapse items in the treelist. Treelist's use treelistTrees to store data.

vtreelistClass

        typedef ... vtreelistClass;
Treelist object is derived from vlistview. It contains the vlistview members and the message definitions for treelist visual properties.

vtreelistNoteProc

        typedef ... vtreelistNoteProc;
This notify procedure is called when there are certain treelist or vlistview changes. The following is a list of treelist notification messages:
  • vtreelistNOTIFY_EXPAND
  • vtreelistNOTIFY_COLLAPSE
  • vtreelistNOTIFY_SELECT_CONTENTS
  • vtreelistNOTIFY_SELECT_ICON
  • vtreelistNOTIFY_SELECT_TOGGLE

vtreelistTree

        typedef ... vtreelistTree;
Denotes an instance of the class vtreeCLASS. TreelistTree objects are used by treelist's to store the hierarchical data they display.

vtreelistTreeBreadthFirstIterator

        typedef ... vtreelistTreeBreadthFirstIterator;
TreelistTreeBreadthFirstIterator objects can be used to iterator "Breadth First" over all the expanded sub-trees.

vtreelistTreeClass

        typedef ... vtreelistTreeClass;
TreelistTree object, it is derived from vtree. It contains the vtree members the message definitions for treelistTree manipulation and traversal.

vtreelistTreeDepthFirstIterator

        typedef ... vtreelistTreeDepthFirstIterator;
TreelistTreeDepthFirstIterator objects can be used to iterate "Depth First" over all the expanded sub-trees.

vtreelistTreeOccurrence

        typedef ... vtreelistTreeOccurrence;
When an observer is notified of a change, it is sent a vtreelistTreeOccurrence object. The object contains, among other things, the behavior that caused the observer to be called.

vtreelistAppendTreeChild

        void * vtreelistAppendTreeChild(
                vtreelistTree * tree,
                vtreelistTree * child);
Every treelistTree has a list of children trees and every child tree has a list of children trees, and so on. When a child is appended, it is added to the end of the tree's child list. The tree object is also assigned to child's parent pointer. After the child tree is appended, the occurrence is announced to all observers.

Macros

vtreelistCLASS

        #define vtreelistCLASS(...) ...
This macro contains the method declarations for the tree class, this macro expands to the contexts of vtreelistClass. SUPER<\SAMP> is the type identifier of the class being declared. This type is used by the class subclasses to refer to it. TREELIST is the type identifier of the instances of the class being declared. This macro should be used by treelist subclasses to declare the contents of the treelist class. This macro should come first in the subclass structure declaration.
