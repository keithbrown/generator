vtypeditem

        typedef ... vtypeditem;
A vtypeditem is a vstritem derivative that represents some abstract value. This class provides the basis for the incrementability feature of such derivatives as vlongitem, vdateitem, and others. This class introduces an abstract method vtypeditemIncrement which is overridden by its derivatives to achieve their own type-specific increment behavior.

This class is also responsible for setting up the keystroke mapping bound to the vtypeditemIncrement method. These keymappings are loaded out of the system resource file, and therefore, can be localized in a future release. By default, CMD-I results in the item's vtypeditemIncrement method being called with its vtypeditemDirection parameter set to vtypeditemUP. CMD-SHIFT-i sends vtypeditemDOWN to the item's vtypeditemIncrement method.

Other vtypeditem derivatives, such as vdateitem, may build on this functionality to allow various portions of their underlying value to be altered independently. This class also defines another abstract method, vtypeditemSync, which is used to synchronize a possibly dirty content string. Refer to the section IsDirty for more information.

vtypeditemClass

        typedef ... vtypeditemClass;
Contains method and variable bindings for objects of type vtypeditem. Classes declared with this type contain the same set of methods and variables as the default typeditem class, but the behavior of methods and the contents of variables may vary. This class type is derived from vstritemClass.

vtypeditemIncrement

        void vtypeditemIncrement(
                vtypeditem * item,
                int direction);
Called when the user initiates an increment action and the sole parameter direction specifies the value change: vtypeditemUP or vtypeditemDOWN. It it up the particular item as to how it determines the amount its underlying value should change.

This class, and all its derivatives (such as vlongitem) respond to CMD-I by invoking this method on themselves with an argument of vtypeditemUP, and to CMD-SHIFT-I with vtypeditemDOWN. These keybindings are loaded from the system resource file so they can be localized in a future release.

vtypeditemINCREMENT

        <vtypeditemINCREMENT>
Called when the user initiates an increment action and the sole parameter direction specifies the value change: vtypeditemUP or vtypeditemDOWN.

It is up the particular item as to how it determines the amount by which its underlying value should change. This class, and all its derivatives (such as vlongitem) respond to CMD-I by invoking this method on themselves with an argument of vtypeditemUP, and to CMD-SHIFT-I with vtypeditemDOWN. These keybindings are loaded from the system resource file so they can be localized in a future release.
