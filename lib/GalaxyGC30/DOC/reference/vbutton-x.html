Types

vbutton

        typedef ... vbutton;
Contains the information needed a represent a single button; objects of this type have classes derived from vbuttonClass. Buttons are referenced by a pointer to this type.

Buttons are simple dialog items that trigger commands. To be useful, a button needs a notify function or a tag. A notify function is called when the user triggers the button. The notify function can be provided by calling vbuttonSetNotify. If a button has a tag, a command is dispatched with that tag when the button is clicked. This mechanism does not require subclassing the standard vbutton class. Buttons must be placed in a dialog to be operational.

By default buttons are stateless items, but since they are derived from vcontrol they are also able to behave like toggles if their maximum and minimum values are set appropriately. Refer to the section vbutton for more on this.

Buttons can be created dynamically with vbuttonCreate, or initialized statically with vbuttonInit.

Refer to the ``Button Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of button attributes. All object types with classes that directly inherit from vbuttonClass must declare a vbutton as their first field so that vbutton pointers can be interchanged with pointers to the new data type.

vbuttonClass

        typedef ... vbuttonClass;
Contains method and variables bindings for objects of type vbutton. Classes declared with this type contain the same set of methods and variables as the default button class, but the behavior of methods and the contents of variables may vary. This class type is derived from vcontrolClass.

vbuttonNoteProc

        typedef ... vbuttonNoteProc;
Denotes a notification function that is called when a button is triggered by the user. button is the button that was triggered, and event is the event that caused the button to be triggered (typically a pointer button up or key down event). Note that this function is not called for every event that is handled by the button; if additional notification is required, you should override the appropriate event handler.

Functions

vbuttonBlink

        void vbuttonBlink(
                vbutton * button);
Highlights and quickly unhighlights the button button, then notifies that the button was triggered. This function can be used to trigger a button and provide feedback to the user.

vbuttonClone

        vbutton * vbuttonClone(
                vbutton * button);
Creates a copy of the button button and returns a pointer to its contents. Storage for the button object is allocated by the Button class. The Button class reclaims the storage occupied by the button when it is destroyed with vbuttonDestroy. This function copies all the attributes of the button except Open, Focused, and ItemList; it clones any attribute objects that are owned by the button. The button to be cloned can be dynamic or static. This function essentially allocates space for a new button and calls vbuttonCopyInit on it.

vbuttonCopy

        void vbuttonCopy(
                vbutton * button,
                vbutton * target);
Copies the information in button into target so that the two buttons represent equivalent objects. target must have been created or initialized by a Button class function. button and target must have compatible classes for this to work correctly. This function copies all the attributes of the button except Class, Open, Focused, and ItemList; it clones any attribute objects that are owned by the button. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The buttons to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to button.

vbuttonCopyInit

        void vbuttonCopyInit(
                vbutton * button,
                vbutton * target);
Copies the information in button into target so the memory addressed by target is a button equivalent to button. target should be a pointer to uninitialized memory; the Button class initializes it as part of the copy. This function copies all the attributes of the button except Open, Focused, and ItemList; it clones any attribute objects that are owned by the button. This function simply stores the class of button in target and sends the message vobjectCOPY_INIT to button.

vbuttonCreate

        vbutton * vbuttonCreate();
Creates a new button of the default button class and returns a pointer to its contents. Storage for the button object is allocated by the Button class. The Button class reclaims the storage occupied by the button when it is destroyed with vbuttonDestroy. The new button is created with a set of default attributes, as determined by the default button class. The default attributes of the default button class are documented in the ``Button Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new button and calls vbuttonInit on it.

vbuttonCreateOfClass

        vbutton * vbuttonCreateOfClass(

Creates a new button of class clas and returns a pointer to its contents. Storage for the button object is allocated by the Button Manager. The Button Manager reclaims the storage occupied by the button when it is destroyed with vbuttonDestroy. The new button is created with a set of default attributes, determined by the class. The default attributes of the default button class are documented in the ``Button Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new button and calls vbuttonInitOfClass on it.

vbuttonDestroy

        void vbuttonDestroy(
                vbutton * button);
Destroys the button represented by button, which must have been created or initialized by a Button Manager function. Use this function to reclaim the storage occupied by a button. The button should not be referenced by any other object when it is destroyed. This function only attempts to free the storage occupied by button if it was allocated by vbuttonCreate, vbuttonCreateOfClass, vbuttonClone, or vbuttonLoad. This function will also destroy any attribute objects that are owned by the button. This function is implemented by sending the message vobjectDESTROY to button .

vbuttonDisable

This function has no effect if the button is not enabled. This function is implemented by sending the message vdialogDISABLE_ITEM to button.

vbuttonIsToggle

        int vbuttonIsToggle(
                vbutton * button);
Indicates whether the button acts as toggle by returning TRUE if it does or FALSE if not. A normal stateless button becomes a toggle when its range of values is expanded from a minimum and maximum of zero to a minimum of zero and maximum of one. The value range can be changed by calling vbuttonSetMin and vbuttonSetMax. The function vbuttonSetToggle is provided as a convenient way to easily set the value range appropriately.

vbuttonLoad

        vbutton * vbuttonLoad(
                vresource resource);
Reads a button from the resource resource and returns a pointer to its contents. Storage for the button object is allocated by the Button class. The button's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class class registry.

Note that this function creates an object based on the class of the resource, even if this class is not a button class. The object pointer is cast to a button pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default button class which are stored in resources are documented in the ``Button class'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new instance and calls vbuttonLoadInit on it.

vbuttonLoadInit

        void vbuttonLoadInit(
                vbutton * button,
                vresource resource);
Reads a button from the resource resource and stores it in the uninitialized memory addressed by button. The button's class is determined by the value of the ``Class'' tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class class registry.

Note that this function initializes an object based on the class of the resource, even if this class is not a button class. The object attempts to initialize itself in the button structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attribute table in the ``Button class'' chapter of the Galaxy Application Environment Programmer's Guide documents the attributes of the default button class that are stored in resources. This function stores the class specified by the resource in the class field of the button structure and sends it the message vobjectLOAD_INIT.

        void vbuttonSetMenu(

                vmenu * menu);
Sets the pop-up menu for the button button to menu. This menu is activated when the pointer menu button is pressed with the pointer inside the button's bounding rectangle. vbuttonSetToggle

        void vbuttonSetToggle(
                vbutton * button,
                int isToggle);
Sets whether the button acts as a toggle. If isToggle is TRUE then the button will act like a toggle that flips between two states when triggered. If isToggle is FALSE then the button will act as a stateless push button. This function works by changing the value range of the button with calls to vbuttonSetMin and vbuttonSetMax. The actual range of values for a button which toggles is -1 to 1. The -1 state is not currently used, but is intended to be used in the future to represent an indeterminate state of neither off nor on.

vbuttonStore

        void vbuttonStore(
                vbutton * button,
                vresource resource);
Writes the contents of the button button into the resource resource. The button can be recovered from the resource at a later time (perhaps by another program) with vbuttonLoad or vbuttonLoadInit.

Note that not all of the button's attributes are stored in the resource; the attributes of the default button class which are stored in resources are documented in the ``Button class'' chapter of the Galaxy Application Environment Programmer's Guide. It generates vobjectUnnamedClassException if the button's class is not named. This function is implemented by sending the message vobjectSTORE to button.

To store a button in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vbuttonStore to write the button into the resource. For example: 


vbutton *button;   /* contains a button */
vresource fileResource, buttonResource; 

fileResource  = vresourceOpenFile( ... ); 
buttonResource = vresourceMake(fileResource, 
			    vnameInternGlobalLiteral("MyButton")); 

vbuttonStore(button, buttonResource); 
vresourceCloseFile(fileResource);

vbuttonNEXT_ID_ATTRIBUTE

        vbuttonNEXT_ID_ATTRIBUTE
Denotes the next available attribute ID for subclasses of the default button class. This constant is provided so that attribute IDs defined by subclasses do not conflict with those defined by the Button class. Only attributes defined by overriding the methods vobjectSET_ID_ATTRIBUTE and vobjectGET_ID_ATTRIBUTE need be concerned with the attribute IDs of their superclasses. Subclasses of vbuttonClass should begin their attribute IDs with this constant and increment by one for each attribute. Subclasses should also provide their own NEXT_ID_ATTRIBUTE constant for use by their subclasses. This constant has integral type.

vbuttonNEXT_SELECT

        vbuttonNEXT_SELECT
Denotes the first command selector available to subclasses of the default button class. This constant is provided so that selector values defined by subclasses do not conflict with those defined by their superclasses. Only class-level commands (defined by overriding the methods vobjectISSUE_SELECTOR and vobjectQUERY_SELECTOR) need be concerned with the selector values of their superclasses. Subclasses of vbuttonClass should begin their command selectors with this constant and increment by one for each new command. In other words, if you add commands to the class-level command space of vbutton, you put their implementations in your vobjectISSUE_SELECTOR override and start with the value of vbuttonNEXT_SELECT as the first selector value you handle. Subclasses should also provide their own NEXT_SELECT constant for use by their subclasses. This constant has integral type.

vbuttonUNHILITE_WITH

        vbuttonUNHILITE_WITH
Denotes the message specifier for unhighlighting the button. The unhighlight method is now inherited from vcontrol; refer to the section vcontrolUNHILITE_WITH instead. This constant is provided for backwards compatibility with previous version of Galaxy.

Macros

vbuttonCLASS

        #define vbuttonCLASS(...) ...
Contains the method and variable declarations for button classes. This macro expands to the contents of vbuttonClass. SUPER is the type identifier of the class being declared (for example, vbuttonClass). This type is used by the class's subclasses to refer to it.

BUTTON is the type identifier of instances of the class being declared (for example, vbutton). NOTIFY is the type identifier of the instance notification function (for example, vbuttonNoteProc). This macro should be used by button subclasses to declare the contents of the button class. This macro should come first in the subclass structure declaration.
