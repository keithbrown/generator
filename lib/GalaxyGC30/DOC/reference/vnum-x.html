vnumDigitSet

        typedef ... vnumDigitSet;
Denotes an object that contains a set of digits for a particular base of number. The number of digits varies according to the base of representation, and the digits themselves may vary according to cultural variations. Default internationalizeddigit sets are provided, but you can create your own as well.

vnumFormat

        typedef ... vnumFormat;
Denotes an object that contains information about how to represent a number textually. It contains information such as how to group the digits, what the decimal point character is, and how to represent a negative or positive number. It also can specify ``decorative'' characters, such as currency symbols, to place around the numbers and padding characters. Default internationalized number formats are provided for ordinary numbers and currency values, but you can customize these or create your own for specific number printing/parsing applications.

vnumFormatIterator

        typedef ... vnumFormatIterator;
Denotes an abstract pointer to a component in a vnumFormat. The iterator can be used to examine or modify an already existing format.

vnumRangeException

        typedef ... vnumRangeException;
This is the type for a Num/Range exception, which can occur when trying to format or parse a number. When formatting a number, the exception occurs if the given number cannot be represented with the specified format (for example, it might have more digits than are specified in the format). When parsing, the exception occurs when the number parsed is outside of the range of values that can be represented by the data type specified.

vnumSyntaxException

        typedef ... vnumSyntaxException;
This is the type of a Num/Syntax exception, which can occur when trying to parse a string into a number. This occurs when the syntax of the string is incompatible with the format specified to parse the number with.

vnumParseFloat

        double vnumParseFloat(
                const vchar * source,
                vchar ** end);
Parses a floating point number (of type double) from a string. source points to a null-terminated string to extract the number from. If end is non-null, the vchar~* it points to is set to the character after the last character used to represent the parsed number, so parsing can continue after the number. The numbers are parsed as decimal numbers, in the normal number format. Scientific notation is recognized if used. A Num/Syntax exception is generated if a number cannot be parsed from the text. A Num/Range exception is generated if the magnitude of the parsed number is outside the allowable range for the given type, or if a non-integer is read for one of the integral types.

vnumParseFloatFormatted

        double vnumParseFloatFormatted(
                const vchar * source,
                vchar ** end,
                vnumDigitSet * digits,
                vnumFormat * numberFormat,
                vnumFormat * expFormat,
                int approximateOk);
Parses a number (of type double) from a string when more control is needed over the format. source specifies a null-terminated string to parse the number from. The digit set digits specifies which digits are used to make up the number. numberFormat specifies the format the mantissa appears in, while expFormat specifies the format the exponent appears in, including the exponent's prefix (typically e).

If expFormat is a null pointer,scientific notation is not recognized. approximateOK should be TRUE if it is allowable for the number to be rounded off to fit into the appropriate data type, FALSE if this is inappropriate. Finally, end, if non-null, specifies a pointer to be set to the end of the parsed number, so parsing can continue. A Num/Syntax exception is generated if the text does not contain a number of the proper format. A Num/Range exception is generated if the parsed number is out of range to fit into the specified type (or if approximateOK is FALSE and the number would have to be rounded to be fully represented).

vnumParseGeneral

        void vnumParseGeneral(
                const vchar * source,
                vchar ** end,
                vnumDigitSet * digitSet,
                vnumFormat * numberFormat,
                vnumFormat * expFormat,
                vbyte * digits,
                int * digitSize,
                int * exponent,
                vbool * negative,
                vbool * overflow);
This function is used to parse a numberthat is not one of those supported by the Number class (for example, fixed point or long double). It works the same as the vnumParseTypeFormatted calls, except that rather than returning a number of the appropriate type, it fills a buffer with the digits of the parsed number and returns other information about the number. The caller is then responsible for actually converting this information into a number of the appropriate data type below.

  • source
  • digitSet
  • numberFormat
  • exponentFormat
  • end

    These are treated the same as the vnumParseTypeFormatted calls. digits points to a vbyte array to which the numeric values of the digits of the number are written, one digit per vbyte. digitSize points to a number that should initially be set to the length of the digit buffer and on return contains the number of digits that were parsed for the number (up to the length of the buffer). exponent should point to a number that is set to the number of digits that would appear before the decimal point if the number were written in fixed place format. negative should point to a boolean variable that is set to TRUE if the number is negative, FALSE if it is positive. Finally, overflow should point to a boolean variable that is set to TRUE if the number of parsed digits overflowed the specified digit buffer, FALSE if it did not.

                vnumDigitSet * digits,
                vnumFormat * numberFormat,
                vnumFormat * expFormat,
                int approximateOk);
Parses a number (of type int) from a string when more control is needed over the format. source specifies a null-terminated string to parse the number from. The digit set digits specifies which digits are used to make up the number. numberFormat specifies the format the mantissa appears in, while expFormat specifies the format the exponent appears in, including the exponent's prefix (typically e). If expFormat is a null pointer,scientific notation is not recognized. approximateOK should be TRUE if it is allowable for the number to be rounded off to fit into the appropriate data type, FALSE if this is inappropriate. Finally, end, if non-null, specifies a pointer to be set to the end of the parsed number, so parsing can continue. A Num/Syntax exception is generated if the text does not contain a number of the proper format. A Num/Range exception is generated if the parsed number is out of range to fit into the specified type (or if approximateOK is FALSE and the number would have to be rounded to be fully represented).

vnumParseLong

        long vnumParseLong(
                const vchar * source,
                vchar ** end);
Parses a number (of type long) from a string. source points to a null-terminated string to extract the number from. If end is non-null, the vchar * it points to is set to the character after the last character used to represent the parsed number, so parsing can continue after the number. The numbers are parsed as decimal numbers, in the normal number format. Scientific notation is recognized if used. A Num/Syntax exception is generated if a number cannot be parsed from the text. A Num/Range exception is generated if the magnitude of the parsed number is outside the allowable range for the given type, or if a non-integer is read for one of the integral types.

vnumScribeFloat

        vscribe * vnumScribeFloat(
                double number);

vnumScribeFloatFormatted

        vscribe * vnumScribeFloatFormatted(
                double number,
                vnumDigitSet * digits,
                vnumFormat * numberFormat,
                vnumFormat * expFormat,
                int approximateOk,
                int precision);
This function creates a scribe that writes out the text representation of the given number, but they allow you to customize how the number is represented. digits specifies the digit set used to print the number (which allows you to use a base other than 10). numberFormat is the vnumFormat used to print the number. expFormat is the format used to print the exponent, including the exponent's prefix (typically e) if scientific notation is to be used, or NULL if it is not. approximateOK should be TRUE if you don't care if the number's full precision is printed out by the format. If it is FALSE, a Num/Range exception is generated if this occurs. Finally, precisionis the number of digits in the number that are significant and should be printed. Note that this includes all digits, not just digits after the decimal point. Pass -1 for precision if you just want to truncate all trailing zeros.

vnumScribeGeneral

        vscribe * vnumScribeGeneral(
                vbyte * digits,
                int digitCount,
                int exponent,
                int negative,
                vnumDigitSet * digitSet,
                vnumFormat * numberFormat,
                vnumFormat * exponentFormat,
                int approximateOk);
This function is used to make a scribe to write a number in a data type that is not one of those supported by the Number class (for example, fixed point or long double). It works the same as the vnumScribeTypeFormatted calls, except that rather than taking a number of the appropriate type, it takes a buffer with the digits of the number and other information about the number which must be computed by the caller.

  • digits
  • numberFormat,
  • exponentFormat
  • approximateOK

The calls above all work the same as in the vnumScribeTypeFormatted calls. digits points to an array of type vbyte which contain the numeric values of the digits in the number to be printed, one digit per vbyte. digitCount contains the number of digits in the array. exponent indicates the number of digits that would appear before the decimal point if the number were written in fixed place format. negative should be TRUE if the number is negative, FALSE if it is positive.

vnumSetScribeDigitsAfter

        void vnumSetScribeDigitsAfter(
                vscribe * scribe,
                size_t digits);
Sets the number of digits that appear after the decimal point when scribe is used. If digits is less than the current value, extra digits are truncated. If it is greater, zeroes are appended. (Note that if the TruncateZeros flag is set, these zeros will subsequently be truncated.)

If the format used to scribe the number has fixed-size digit components after the decimal, the number may be additionally padded or truncated as the format specifies.

Note that scribe must have been created by a vnumScribe call.

vnumSetScribeDigitsBefore

        void vnumSetScribeDigitsBefore(
                vscribe * scribe,
                size_t digits);
Sets the number of digits that appear before the decimal point when scribe is used. If digits is greater than the current value, leading zeros are prepended to the number. If it is less, the digits are truncated.

If the format used to scribe the number has fixed-size digit components before the decimal, the number may be additionally padded or truncated as the format specifies.

Note that scribe must have been created by a vnumScribe call.

vnumSetScribeTruncateZeroes

        void vnumSetScribeTruncateZeroes(
                vscribe * scribe,
                int value);
This flag controls whether or not the number is displayed up to its full specified precision, or if trailing zeroes are truncated. The default value is TRUE. Set it to FALSE when you want to show a number up to its full specified precision.

This truncation only applies to the precision of the number (as specified when scribing the number or by the vnumSetScribeDigitsAfter call), and will not change the behavior of any fixed-size digit components in the number format.

Note that scribe must have been created by a vnumScribe call.
