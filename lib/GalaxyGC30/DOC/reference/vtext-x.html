vtextAddTag

        void vtextAddTag(
                vtext * editRec,
                vtextTag * tag);
Adds the tag tag to the text data object editRec. The initial start and end positions of the tag are determined by the tag range. They are automatically updated as editing operations occur on the text data object. Tags can be created either with vtextCreateTag, or by allocating space for the tag yourself, and initializing the tag with vtextInitTag. For example, to keep your own markers for a text data object, you could create a marker by: 

typedef struct Marker {
        vtextTag                 tag;
        vtext                *editRec;
        vchar                 name[32];
} Marker;

void MarkSelection(vtext *editRec, vtextSelection *selection,
                   vchar *name)
{
        Marker                  *newMarker, *temp, *prev = NULL;
        vtextPosition        *position;

        newMarker = (Marker *) vmemAlloc(sizeof(Marker));
        position  = vtextGetPosition(selection);

        (void) vcharCopy(name, newMarker->name);
        vtextInitTag(&newMarker->tag);
        vtextSetTagRange(&newMarker->tag,
                           position->start, position->end);
        vtextSetTagName(&newMarker->tag,
                        vnameInternGlobalLiteral("Marker"));

        vtextAddTag(editRec, &newMarker->tag);
}

void GotoMarker(vtextview *view, Marker *marker)

vtextAppendBlock

        void vtextAppendBlock(
                vtext * editRec,
                vchar * text,
                long length,
                long textAllocSize);
Provides quick and efficient insertion for large amounts of text. You should use this routine instead of vtextInsert when initially reading in text from a file or external source. The block of text is pointed to by text and should be allocated with vmemAlloc of size textAllocSize. The allocated size should be approximately vtextTEXT_BLOCK_SIZE or smaller. length is the amount of text in the text block. This should be approximately vtextINIT_TEXT_SIZE or smaller. Blocks of text must be on whole line boundaries. Use vtextCheckEndOfLine to truncate a block to the nearest whole line.

This routine reads in blocks of text until all of the text has been inserted into the text object editRec. You should then call vtextCalculate to tell the Text class that all of the text has been inserted and to update its data structures. The initial block allocated when the text data object is created is overwritten on the first call to vtextAppendBlock.

Example illustrates how vtextAppendBlock is used to read in text from a text file. For simplicity's sake, it is assumed that the text in the text file is encoded in a single byte encoding that matches the internal single byte vchar character set.

Sample Code: Reading in text from a text file. 


/*
 * 'buffer' is a temporary buffer, used to keep around remaining
 * parts of lines, since text in a text block must be separated on
 * line boundaries.
 */
buffer = vmemAlloc(vtextTEXT_BLOCK_SIZE);

while (!done)
{
    newText = vmemAlloc(vtextTEXT_BLOCK_SIZE);

/*
 * Place any text that is in the 'buffer' into the new text handle
 */
    vmemCopy(buffer, newText, bufferSize);

    length = vtextINIT_TEXT_SIZE - bufferSize;
    length = vfsReadFile(newText + bufferSize, 1, length, fileSpec);
    if (length == 0)
        done = TRUE;                /* EOF */

    lengthThisBlock = length + bufferSize;

    if (!done)
    {
        lengthWithReturn = lengthThisBlock;
        vtextCheckEndOfLine(newText, &lengthWithReturn, TRUE);

        if (lengthWithReturn == 0)
        {
            lineToLong = TRUE;
            lengthThisBlock++;
            lengthWithReturn = lengthThisBlock;
        }

        bufferSize = lengthThisBlock - lengthWithReturn;
        vmemCopy(newText + lengthWithReturn, buffer, bufferSize);

        if (lineToLong)
            *(newText + (long) (lengthThisBlock - 1)) = vtextEOLN;
    }
    else
    {
        lengthWithReturn = lengthThisBlock;
    }
    if (done && lengthWithReturn == 0)
        break;
    vtextAppendBlock(editRec, newText, lengthWithReturn,
        vtextTEXT_BLOCK_SIZE);
}
vmemFree((void *) buffer);

vtextProcess

        void vtextProcess(
                vtext * editRec,
                vtextSelection * selection,
                vtextProcessingFunc processingFunc,
                unsigned int flags,
                unsigned int tabSettings);
Quickly performs operations on text or selected text, taking into account normal and rectangular editing modes. This function then updates the appropriate data structures efficiently and appropriately. Only the selection is processed if selection is non-NULL, otherwise all of the text is processed.

The processingFunc routine defines a processing function that is passed a pointer to a line of text at a time. The processing function consists of:

*processInfo);

The processing function should return one of the following values:

vtextSUCCESS
Processing was successful.
vtextMORE_SPACE
There is not enough room in the destination text to perform the operation for the line. vtextProcess calls your processing function again with the same text but with more space in the destination text.
vtextERROR
Processing was unsuccessful.
vtextSTOP
Stop processing.

When returning success, the destinationLength parameter should be set to the amount of text placed in the destination text.

The processInfo parameter holds processing information. This information is passed to the processing function for read-only purposes. The process information is updated appropriately for each line being processed. processInfo contains the following:

editRec
The text data object being processed.
lineNumber
Line number of the line being processed.
position
The position in the text for the line being processed.
linePosition
The position in the line for the line being processed.
linesAffected
Number of lines that are affected by the processing, computed at this point.
linesAdded
Number of new lines of text entered thus far.
textGrowth
Amount of text added/deleted thus far.
selection
The original passed in selection. The selection may change in length as text is processed.

The flags parameter affects the behavior of text processing and can have the following values:

vtextNO_PROCESS_BUFFER
The processing buffer is not needed. This should be true only when the processing operation involves copying or in-place changing and should never be set if text processing shrinks or expands the text.
vtextPROCESS_HIDDEN
Process hidden text also.
vtextOVERRIDE_RECT_MODE
When processing a selection, ignore rectangle editing mode if it is enabled.
vtextREQUEST_NUM_LINES
Call the processing function with sourceText of NULL and a value of zero for sourceLength to determine the number of lines to process. This number should be returned through destinationLength.
vtextCOPY_TO_BUFFER
Automatically copy the source text to the processing buffer and check for enough buffer space before calling the processing function. destinationText points to the processing buffer.
vtextEXPAND_TABS
Expand the tabs in the text before calling the processing function. WARNING: In this case, the sourceText pointer is actually pointing in vtextProcess's processing buffer. Be careful and use non-destructive operations with this option.
vtextENTAB
After processing is complete, entab the line.
vtextWHOLE_LINES
Process the entire line of any line that is part of the selection.
vtextNO_EOLN
Do not put an end-of-line character after processing each line.
vtextEND_WITH_EOLN
After processing all lines, insert an end-of-line character. Use only with vtextNO_EOLN.

The tabSettings parameter affects the behavior of tab conversion operations if the vtextEXPAND_TABS or vtextENTAB options are specified. The value for tabSettings can be any combination of the following:

vtextLEADING_ONLY
Convert leading tabs/spaces only. Leading spaces are tabs/spaces from the beginning of a line to the first non-tab, non-space in the line.
vtextCONVERT_IN_QUOTES
Convert tabs/spaces that are in single quotes (` ') or double quotes (`` '').
vtextCONVERT_IN_USERAREA
Convert tabs/spaces that are in the user area. The user area is defined by overriding the vtextUSER_AREA method of the text data object class.
vtextCONVERT_ALL
Convert all tabs/spaces (combination of above settings).
vtextCONVERT_ALL_LEADING
Convert all leading tabs/spaces (combination of above settings).

Note that a text processing function should always be used when it is necessary to process text directly. Getting text and inserting text is far slower.

vtextWARP_TAG

        <vtextWARP_TAG>
Determines how the range of the text tag tag should be affected by an editing operation at character position position in the text data object editRec. delta is positive for insertions and negative for deletions. The default vtextINSERT method of the default text data class calls the function that corresponds to this method for all non-fixed text tags with an endpoint at position position; the default vtextDELETE method of the default text data class calls it for all non-fixed text tags with an endpoint in the deleted text. It is assumed that other text tags will have their ranges expanded or shrunk according to the overlap of their ranges with the inserted or deleted text.

Under these circumstances, subclasses that override this method may assume that the character attributes (raw textual data, vtextLength) of the text data object editRec have already been recalculated; however, the line attributes (vtextLines, vtextLinePosition) may be out-of-date. Use vtextPositionParagraph if you need to find the start and end positions of a paragraph.

In addition, the state of other text tags is indeterminate when this method is being called for tag: some may have already had their ranges updated either implicitly or by vtextWarpTag, and others may still be in their pre-editing state. Iterate over the text tags before the editing operations start if you need to establish more context for the range update operations.

The default method for the default text data class treats the ruler tags differently than other tags, as it tries to maintain the property that text rulers apply to whole paragraphs. Characters inserted at either endpoint will become part of the ruler tag updated range. When a pararaph separator (vtextEOLN) at either end of a ruler tag range is deleted, the corresponding endpoint is pushed forward to the next paragraph start.

All other text tags with non-empty ranges are updated so that characters inserted at their start don't get included in the tag range, while characters inserted at their end do get included in the range. Empty-range text tags are updated so that characters inserted at their one endpoint get included in the tag range.

You may override this method if you want to emulate other text tag update at the boundary policies, such as the ones found in common word-processors.
