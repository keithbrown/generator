Types

voccurrence

        typedef ... voccurrence;
Occurrence objects contain the information passed during observation from the announcing object to the observing object. This class introduces an announcer attribute. The announcer of an occurrence is generally the original object that is passed to the vobservableAnnounce function.

Because the default implementation of vobservableAnnounce will not set the announcer attribute if it is already set, an intermediate object can reannounce occurrence object without losing track of the original, initiating object.

voccurrenceAttributeChanged

        typedef ... voccurrenceAttributeChanged;
Attribute-changed occurrence objects contain the information that describes a simple attribute change operation for observers. Subclasses of vobservable decide whether their instances will announce these occurrences, and for which attributes. The additional attributes of this type of occurrence are:
  • The name that identifies the attribute
  • The object that represents the old value
  • An object that represents the new value.

voccurrenceDestroyed

        typedef ... voccurrenceDestroyed;
Destroyed occurrence objects are announced when an object is destroyed by its owner. Currently, the vobservable class does not guarantee that these occurrences are sent for all instances. This class is provided for vobservable subclasses that wish to add the notification. This occurrence should be announced before the object is actually destroyed as the announcer attribute of the occurrence is typically the object being destroyed.

voccurrenceEvent

        typedef ... voccurrenceEvent;
Event occurrence objects contain the information that describes an operation initiated by an event. Subclasses of vobservable decide whether to announce these occurrences and for which events. This type of occurrence has an additional event attribute.

voccurrenceEventClass

        typedef ... voccurrenceEventClass;

voccurrenceStateChanged

        typedef ... voccurrenceStateChanged;
State-changed occurrence objects contain the information that describes a simple state-change operation for observers. Subclasses of vobservable decide whether their instance will announce these occurrences, and for which state-changes. The additional attributes of this type of occurrence are the name of the old state, and the name of the new state.
