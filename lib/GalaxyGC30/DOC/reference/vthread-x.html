vthreadAtExit

        void vthreadAtExit(
                vthreadAtExitProc proc,
                void * arg);
Register the per-thread exit handler proc with the calling thread. arg will be passed as the argument to the handler when it is invoked. Each thread maintains its own list of registered per-thread exit handlers. Per-thread exit handlers are called in the reverse order that they were registered when a thread terminates by either returning from its thread proc or calling vthreadExit. Per-thread exit handlers are not called when a thread terminates abnormally, such as with an unhandled exception, or when a thread terminates the process by calling the ANSI exit function.

vthreadExit

        void vthreadExit(
                int status);
Terminates the calling thread with status code status. The calling thread will invoke its per-thread exit handlers in reverse order, then stop executing. The thread object will be automatically destroyed if vthreadSetDestroyWhenTerminated had previously been called on it. Otherwise, the thread object will remain until explicitly destroyed by another thread. Any threads blocking in vthreadWaitForTermination on this thread will be awakened.

vthreadRequestSuspend

        void vthreadRequestSuspend(
                vthread * thread);
Requests that a thread suspend execution until another thread calls vthreadResume on it. The thread will not respond immediately to the suspension request. Rather, suspension will be honored when certain suspension points are reached within the library.

If a thread calls this function on itself, then the thread will suspend immediately, without returning from the call to vthreadRequestSuspend. The function will return when another thread calls vthreadResume on this thread.

vthreadSetConcurrencyLevel

        void vthreadSetConcurrencyLevel(
                int level);
Sets the concurrency level for a process in a two-level thread system. The concurrency level specifies how many kernel-level threads should be in the process for user-level threads to be scheduled upon. This setting is only a hint: the operating system may create more or less kernel level threads than specified with this method.

For Galaxy 3.0, the only operating systems that have two-level thread systems that support a user-setable concurrency level are Solaris 2.4 and above.

Message send to the calling thread to yield control of the processor to the kernel, possibly allowing a different thread to run. Note that there is no guarantee as to which thread will run next.

Constants

vthreadDESTROY

        vthreadDESTROY
Message sent to destroy a thread. The thread must have already terminated before calling this function.

vthreadINCLUDED

        vthreadINCLUDED
A preprocessor symbol used to indicate whether or not vthreadHEADER has been included yet.

vthreadINIT

        vthreadINIT
Message sent to initialize the memory addressed by thread as a thread object with default attributes appropriate for the current platform. The thread is not running. Call vthreadStart to start the thread.

vthreadPRIORITY_MAX

        vthreadPRIORITY_MAX
The highest scheduling priority a thread may have.

vthreadPRIORITY_MIN

        vthreadPRIORITY_MIN
The lowest scheduling priority a thread may have.

vthreadPRIORITY_NORMAL

        vthreadPRIORITY_NORMAL
The default scheduling priority.

vthreadSCOPE_KERNEL

        vthreadSCOPE_KERNEL
Specifies that a thread's scheduling scope will be kernel-level. Kernel-level threads are scheduled preemptively with other kernel-level entities.

vthreadSCOPE_PROCESS

        vthreadSCOPE_PROCESS
Specifies that a thread's scheduling scope will be process-level. Process-level threads are scheduled cooperatively across some number of kernel-level entities.

Macros

vthreadCLASS

        #define vthreadCLASS(...) ...
Contains method and variable bindings for objects of type vthread. Classes declared with this type contain the same set of methods and variables as the default thread class, but the behavior of methods and the contents of variables may vary. This class type is derived from vinstance.

vthreadGroupLOCAL

        #define vthreadGroupLOCAL(...) ...
Used to specify that a variable will be a thread group local variable. Wrap a variable that you will use to store a thread group local variable key in with vthreadGroupLOCAL. The variable you wrap should be the size of a void *. Example: 
/* a global variable to hold a thread group local variable key */

static vthreadGroupLOCAL(void *) _per_group_stuff;

/* a global variable to hold a thread group local variable key that will */
/* always hold a vdialog */

static vthreadGroupLOCAL(vdialog *) _per_group_dialog;
