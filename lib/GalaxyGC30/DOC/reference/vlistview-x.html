Types

vlistview

        typedef ... vlistview;
Denotes an instance of the class derived from vlistviewClass. This type is a derivation of vdialogItem, vlistview displays a list object in a dialog. This can also be used to create subclasses for custom needs.

vlistviewClass

        typedef ... vlistviewClass;
The class definition for the list view object, it is derived from vdialogItemClass. When an object of this class is created, it creates an observer in the list object that it is displaying. Through the observer the display of the list is maintained.

vlistviewDrawCellProc

        typedef ... vlistviewDrawCellProc;
Denotes a procedure that draws the content of the specified cell in the given rectangle on the current graphic port. row and column specify which cell of the list to draw, selected is set to TRUE if the cell is in the list view listview's displayed selection, FALSE otherwise; and rect is the rectangle in which to draw the cell's content.

vlistviewListObserver

        typedef ... vlistviewListObserver;
A specialized list observer used by list view objects to monitor their lists. See vlistObserver for information on list observers.

vlistviewNoteProc

        typedef ... vlistviewNoteProc;
Denotes a list view notify procedure. This is called whenever listview handles an interesting event (as defined by the Dialog Manager), event is the event that is being handled.

vlistviewSelectNoteProc

        typedef ... vlistviewSelectNoteProc;
Denotes a list view selection-notify procedure. This is called when listview selection is about to change, when it is changing, and when the user commits to the selection change. state is one of the following values vlistviewSELECTION_WILL_CHANGE, vlistviewSELECTION_IS_CHANGING, vlistviewSELECTION_HAS_CHANGED.

vlistviewSelectionFilterProc

        typedef ... vlistviewSelectionFilterProc;
Denotes a list view selection filter procedure that is called before selections are committed during mouse tracking. listview is the list view in which the selection is changing, and selection is the selection that is to be filtered.

                vlistview * target);
Copies the information from listview into target, so that the two list views represent equivalent list views but separate storage. Any objects created in listview are duplicated in target as well. The vlist object that listview is displaying is also copied (vlistClone).

vlistviewCopyInit

                vlistview * target);
Copies the information in listview into target so the memory addressed by target is a dialog item equivalent to listview. target should be a pointer to uninitialized memory---the List Manager will initialize it as a part of the copy. This function copies all of the attributes of the item, and it will clone the list contents by calling vlistClone on the list owned by listitem. This function is implemented by sending the message vobjectCOPY_INIT to listview.

                vlistview * listview,
                int row,

                int * width,
                int * height);
Returns the natural size of a particular cell. The width and height of the cell are returned in width and height respectively. If the cell does not have a natural size, vlistviewDONT_CARE is returned in width and/or height. vlistviewDONT_CARE is guaranteed to be less than zero. List views do not use this function, but subclasses of list view (such as list item) use this to perform automatic row and column sizing. This function is implemented by sending the message vlistviewDETERMINE_CELL_SIZE to listview.

vlistviewCOLUMN_REARRANGE
Columns have been moved.
vlistviewNO_REARRANGE
No rearrangement happened.

This function should be called in the selection notify procedure if you receive a vlistviewSELECTION_HAS_CHANGED message and you know that the list is rearrangeable.

vlistviewSetSelectNotify

        void vlistviewSetSelectNotify(
                vlistview * listview,
                vlistviewSelectNoteProc noteProc);
Sets list view's selection-notify procedure to noteProc. This procedure gets called whenever the list view's selection changes. The noteProc gets called at the start of the change with vlistviewSELECTION_WILL_CHANGE to allow any pre-selection setup, with vlistviewSELECTION_IS_CHANGING while the selection is being modified, and then vlistviewSELECTION_HAS_CHANGED when the operation is complete. During the selection change, the original selection is accessible to the noteProc by way of vlistviewGetSelection, and the selection under construction is available by way of vlistviewGetVolatileSelection. The noteProc is called when the change to the selection is a result of user action or API calls.

Sample Code: Selection-notify procedure

void exampleSelectNotify(vlistview *listview, int type)
{
int                row, column;

if ((type == vlistviewSELECTION_WILL_CHANGE) ||
    (type == vlistviewSELECTION_IS_CHANGING)) {

        /* since this is a vlistONE_ONLY_SELECTION list,
         * vlistGetSelectCell will return the only selected cell. */
        vlistGetSelectedCell(vlistviewGetList(listview),
                                 
vlistviewGetVolatileSelection(listview),
                                  &row, &column);
        if ((row != vlistNO_CELLS_SELECTED) &&
            (column != vlistNO_CELLS_SELECTED)) {
                vdebugInform("Cell (%d,%d) = %x\n", row, column,
                             vlistGetCellValue(vlistviewGetList(listview),
                             row, column))
         }
    }
} /* exampleSelectNotify */

This function is implemented by sending the message vlistviewSET_SELECT_NOTIFY to listview.

Space is added to width to accommodate selection highlighting. The default column width is equal to the width of the content area of the list view (that is, it displays one column). The cell content area will be adjusted by the mechanism set by vlistviewSetUpdateMode if column is visible. Subclasses overriding this method should call the superclass so that the column width change will be recorded in listview.

vlistviewSET_DRAWING_SELECTION

        <vlistviewSET_DRAWING_SELECTION>
Sets the DrawingSelection attribute of the list view listview to onOff. If DrawingSelection is disabled, dragging the mouse during selection sweeps out a rectangular area. If it is enabled, dragging the mouse selects only those cells the mouse pointer passes over. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_FOCUS

        <vlistviewSET_FOCUS>
Moves the keyboard focus for the list view listview to the given row and column.

vlistviewSET_GRABBING_HAND

        <vlistviewSET_GRABBING_HAND>
Designates whether clicking and dragging the mouse in the list view listview invokes the ``grabbing hand'' or whether it selects cells. The grabbing hand tracks the mouse while the mouse button is down and scrolls the list view as if you grabbed it, so that the scrolling of the list view and the movement of the mouse are synchronous. If onOff is TRUE, the grabbing hand is turned on; if FALSE, the grabbing hand is turned off. Aside from this setting there is a look-and-feel click and drag binding. This property does not affect the look-and-feel drag, which is always available. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_HORZ_BAR

        <vlistviewSET_HORZ_BAR>
Designates whether or not listview has a horizontal scroll bar. If onOff is TRUE, the scroll bar is turned on. If FALSE, the scroll bar is turned off. For a list view both scroll bars are off by default. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_HORZ_GRID

        <vlistviewSET_HORZ_GRID>
Designates whether horizontal grid lines will be drawn. If onOff is TRUE, horizontal grid lines will be drawn between every row. If FALSE, no horizontal grid lines will be drawn. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_LIST

        <vlistviewSET_LIST>
Sets the list object to be displayed by list view listview to the list object specified by list. The old list displayed by the list view is not destroyed. list can be NULL, which tells the list view that nothing is to be displayed. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_LIST_OWNED

        <vlistviewSET_LIST_OWNED>

vlistviewSET_REARRANGE

        <vlistviewSET_REARRANGE>
Tells the list view listview what kind of rearranging actions to allow. If type is vlistviewROW_REARRANGE, users will be allowed to rearrange rows. If type is vlistviewCOLUMN_REARRANGE, users will be allowed to rearrange columns. If type is vlistviewNO_REARRANGE, users will not be allowed to rearrange anything. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_ROW_HEIGHT

        <vlistviewSET_ROW_HEIGHT>
Sets the height of row row to be height. The height of all rows can be set to the same value by passing vlistviewALL_ROWS in the row parameter. This sets the usable height. Space is added to height to accommodate selection highlighting. If the client never sets the row height, the default row height which is calculated from the list view's font is used. The cell content area will be adjusted by the mechanism set by vlistviewSetUpdateMode if row is visible. Subclasses overriding this method should call the superclass so that the row height will be recorded in listview.

vlistviewSET_SCROLL_BAR_MARGIN

        <vlistviewSET_SCROLL_BAR_MARGIN>
Sets the margin between the scroll bars and the list view contents. Subclasses overriding this method should call the superclass so that the scroll bar margin will be recorded in listview.

vlistviewSET_SELECTION

        <vlistviewSET_SELECTION>
Changes the visible selection for the list view listview to the selection selection. The selection notification procedure is not notified of this change. List views automatically create a selection for you when you call vlistviewSetList, so it is unnecessary for you to create a selection and set it yourself. listview takes over ownership of selection, and will destroy it when the list view, or the list being viewed, is destroyed. Subclasses overriding this method should call the superclass so that the new selection will be stored in listview.

vlistviewSET_SELECTION_FILTER

        <vlistviewSET_SELECTION_FILTER>
Sets the list view's selection filter procedure to proc. The filter procedure is called while the selection is volatile, as the user is changing the selection before committing to an actual selection. The filter procedure can then prevent cells from becoming part of the selection or add cells to the selection. Subclasses overriding this method should call the superclass so that the selection filter will be recorded in listview.

vlistviewSET_SELECTION_SENSE

        <vlistviewSET_SELECTION_SENSE>
Sets the SelectionSense attribute of the list view listview to onOff. If SelectionSense is active and the user adjust-clicks on a selected cell, mouse tracking removes cells from the current selection instead of adding cells to the selection. When SelectionSense is off, clicking with the mouse only adds cells to the selection. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_SELECT_NOTIFY

        <vlistviewSET_SELECT_NOTIFY>
Sets list view's selection-notify procedure to noteProc. This procedure gets called whenever the list view's selection changes. The noteProc gets called at the start of the change with vlistviewSELECTION_WILL_CHANGE to allow any pre-selection setup, with vlistviewSELECTION_IS_CHANGING while the selection is being modified, and then vlistviewSELECTION_HAS_CHANGED when the operation is complete. During the selection change, the original selection is accessible to the noteProc by way of vlistviewGetSelection, and the selection under construction is available by way of vlistviewGetVolatileSelection. The noteProc is called when the change to the selection is a result of user action or API calls. Subclasses overriding this method should call the superclass so that the selection-notify procedure will be recorded in listview.

vlistviewSET_SIZES

        <vlistviewSET_SIZES>
Instructs listview to use size as its repository for row heights and column widths. This function transfers ownership of size to listview, meaning that listview will destroy size when it is destroyed. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_TRAVERSAL_WRAP

        <vlistviewSET_TRAVERSAL_WRAP>
Determines whether the keyboard focus for the list view listview wraps around the ends of the list being displayed by the list view. If onOff is FALSE, the focus is pinned at the list's extremes; if TRUE, the focus wraps. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_UPDATE_MODE

        <vlistviewSET_UPDATE_MODE>
Changes the update mode for listview. Certain list operations, such as row/column insertion and deletion, as well as adjustment to row heights and column widths, can be made more efficient by scrolling the visible list contents rather than redrawing drawing. There are times when scrolling is not appropriate and invalidation is sufficient. There are yet other times where the List Manager should not do anything, where client code will cause a redraw of the proper areas. mode should be vlistviewSCROLL if list contents should be scrolled, vlistviewINVAL if invalidations should be used instead of scrolling, and vlistviewDO_NOTHING if the List Manager should do nothing at all. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_USE_KEYBOARD_SELECTION

        <vlistviewSET_USE_KEYBOARD_SELECTION>
Sets the UseKeyboardSelection attribute of the list view listview to onOff. When disabled, all characters typed are ignored; when enabled, typing a cell value changes the list view's selection to that cell. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_USE_KEYBOARD_TRAVERSAL

        <vlistviewSET_USE_KEYBOARD_TRAVERSAL>
Determines whether keyboard traversal (using arrow keys to move the list view's keyboard focus) is active. If onOff is FALSE, all keyboard traversal commands are ignored, and the keyboard focus is not drawn. If onOff is TRUE, keyboard traversal commands are active, and the keyboard focus is drawn. Default drawing routine for the list view's keyboard focus follows the Galaxy application's look-and-feel. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_VERT_BAR

        <vlistviewSET_VERT_BAR>
Designates whether or not the list view listview has a vertical scroll bar. If onOff is TRUE, the scroll bar is turned on, if FALSE, the scroll bar is turned off. For a list view both scroll bars are off by default. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewSET_VERT_GRID

Designates whether vertical grid lines will be drawn. If onOff is TRUE, vertical grid lines will be drawn between every column. If FALSE, no vertical grid lines will be drawn. Subclasses overriding this method should call the superclass so that the attribute change will be recorded in listview.

vlistviewTRAVERSE_VECTOR

        <vlistviewTRAVERSE_VECTOR>
Called whenever the list view's keyboard focus cursor is changed. The change is expressed as a delta of rows and columns between its current position and its intended position. This method actually moves the focus to the new location, this method should follow the traversal wrap settings.

Macros

vlistviewCLASS

        #define vlistviewCLASS(...) ...
Contains the method and variable declarations for the list view class. This macro expands to the contents of vlistviewClass. SUPER is the type identifier of the class being declared; for example, vlistviewClass. This type is used by the class's subclasses to refer to it. ITEM is the type identifier of instances of the class being declared; for example, vlistview.

NOTIFY is the type identifier of the instance notification function; for example, vlistviewNoteProc. This macro should be used by text view subclasses to declare the contents of the text view class. This macro should come first in the subclass structure declaration.
