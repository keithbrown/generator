Types [8] Functions [63] Messages [12] Constants [7] Macros [4]

Types

vtree

        typedef ... vtree;
Denotes an instance of the class vtreeClass. Tree objects can be used to represent hierarchical data structures. Trees can be used within an application for the maintenance of data, or they can have views associated with them to dialog the tree in one or more places. Trees are very useful when an application needs to store and manipulate hierarchical data.

vtreeBreadthFirstIterator

        typedef ... vtreeBreadthFirstIterator;
The class definition for the treeBreadthFirstIterator. This class is used to iterate "Breadth First" over all the nodes in a tree.

vtreeClass

        typedef ... vtreeClass;
The tree object is derived from vobservable. It contains the vobservable members and the message definitions for tree manipulation and traversal.

vtreeDepthFirstIterator

        typedef ... vtreeDepthFirstIterator;
The class definition for the treeDepthFirstIterator. This class is used to iterate "Depth First" over all the nodes in a tree.

vtreeIterator

        typedef ... vtreeIterator;
Tree iterator objects are used to iterate over the sub-trees in a tree.

vtreeIteratorClass

        typedef ... vtreeIteratorClass;
The treeIterator object is derived from viteratorGROUP.

vtreeOccurrence

        typedef ... vtreeOccurrence;
A treeOccurrence is sent to all observers of a tree. The behavior of the occurrence is one of the following:
  • vtreeOccurrenceBEHAVIOR_ADD_CHILD
  • vtreeOccurrenceBEHAVIOR_CHANGE_DATA
  • vtreeOccurrenceBEHAVIOR_CHANGE_TITLE
  • vtreeOccurrenceBEHAVIOR_DESTROY
  • vtreeOccurrenceBEHAVIOR_REMOVE_CHILD
Refer to the section GetBehavior for more information.

vtreeOccurrenceClass

        typedef ... vtreeOccurrenceClass;
The treeOccurrence object is derived from voccurrence.

Functions

vtreeAppendChild

        void vtreeAppendChild(
                vtree * tree,
                vtree * child);
Every tree has a list of children trees and every child tree has a list of children trees, and so on. When a child is appended it is added to the end of the tree's child list. The tree object is also assigned to child's parent pointer. After the child tree is appended, the occurrence is announced to all observers.

This function is implemented by sending the message vtreeAPPEND_CHILD to tree.

Sample Code: Using a Tree Iterator 

vtree		*my_tree;
vtreeIterator	iterator;

...

/*
 * Iterator over all the sub-trees in "my_tree".
 */


/*
 * Initialize the iterator.
 */
vtreeInitIterator(&iterator,
		 my_tree);


/*
 * vtreeNextIterator must be called to advance the iterator
 * to the first sub-tree.
 */
while (vtreeNextIterator(&iterator))
{
  /* print the title */
  printf("%s\n", vtreeGetTitle(vtreeGetIteratorTree(&iterator)));
}

/*
 * Destroy the iterator.
 */
vtreeDestroyIterator(&iterator);

vtreeAPPEND_CHILD

        <vtreeAPPEND_CHILD>
Every tree has a list of children trees and every child tree has a list of children trees, and so on. When a child is appended it is added to the end of the tree's child list. The tree object is also assigned to child's parent pointer.

After the child tree is appended, the occurrence is announced to all observers. If you are using a custom data representation, you should always call the superclass's implementation so that the child is appended and observers are notified.

Macros

vtreeCLASS

        #define vtreeCLASS(...) ...
This macro contains the method declarations for the tree class, this macro expands to the contexts of vtreeClass. SUPER<\SAMP> is the type identifier of the class being declared. This type is used by the class subclasses to refer to it. TREE is the type identifier of the instances of the class being declared. This macro should be used by tree subclasses to declare the contents of the tree class. This macro should come first in the subclass structure declaration.
