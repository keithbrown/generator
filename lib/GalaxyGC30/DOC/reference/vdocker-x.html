Types [5] Functions [69] Messages [16] Constants [8] Macros [2]

Types

vdocker

        typedef ... vdocker;
Contains the information needed to represent a single docker. A docker is a dialog item that can added to, removed from and moved around the user-interface. Dockers can appear as floating child dialogs or they can "dock" to become part of a dialog. Users can rearrange them to suit their own user-interface preferences.

vdockerClass

        typedef ... vdockerClass;
Contains method and variables bindings for objects of type vdocker. Classes declared with this type contain the same set of methods and variables as the default docker class, but the behavior of methods and the contents of variables may vary. This class type is derived from vcontainerClass.

vdockerDrag

        typedef ... vdockerDrag;
Contains the information needed to represent a single docker drag. A docker drag provides a graphical way to drag a docker from one screen location to another. The drag is initiated from a button down, drag, or detail event in the docker.

vdockerDragClass

        typedef ... vdockerDragClass;
Contains method and variables bindings for objects of type vdockerDrag. Classes declared with this type contain the same set of methods and variables as the default docker drag class, but the behavior of methods and the contents of variables may vary. This class type is derived from vdragClass.

vdockerNoteProc

        typedef ... vdockerNoteProc;
Denotes a notification function that is called when a docker handles an event of some importance. The docker docker is the docker handling the event, and event is the event being handled. Importance is determined by the event handlers in the docker's class; the default docker class gives no notification. If additional notification is required, override the appropriate handler.

vdockerClone

        vdocker * vdockerClone(
                vdocker * docker);
Creates a copy of the docker docker and returns a pointer to its contents. Storage for the docker object is allocated by the Docker class. The Docker class reclaims the storage occupied by the docker when it is destroyed with vdockerDestroy. This function copies all the copyable attributes of the docker and clones any attribute objects that are owned by the docker. The docker to be cloned can be dynamic or static. This function essentially allocates space for a new docker and calls vdockerCopyInit on it.

vdockerClose

        void vdockerClose(
                vdocker * docker);
Closes the docker by either closing its dialog if it is floating, or by closing its dialog item if it is docked. Refer to the sections vdialogClose and vdialogCloseItem for more information about closing dialogs and dialog items.

This function is implemented by sending the message vdockerCLOSE to the docker.

vdockerCopy

        void vdockerCopy(
                vdocker * docker,
                vdocker * target);
Copies the information in docker into target so that the two dockers represent equivalent objects. target must have been created or initialized by a Docker class function. docker and target must have compatible classes for this to work correctly. This function copies all the copyable attributes of the docker and clones any attribute objects that are owned by the docker. This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The dockers to be copied can be dynamic or static. This function is implemented by sending the message vobjectCOPY to docker.

vdockerCopyInit

        void vdockerCopyInit(

This function destroys any objects allocated internally by target (including its owned attributes), but does not destroy target itself. The dockers to be copied can be dynamic or static. This function simply stores the class of docker in target and sends the message vobjectCOPY_INIT to docker.

vdockerGetSpanFillModifiers

        int vdockerGetSpanFillModifiers(
                vdocker * docker);
Returns the span filling modifier flags for the docker. This method is typically called by a docker-holding item (such as a dock item) to determine how best to arrange its dockers.

The span fill modifier flags serve as a hint as to how the docker would like to be arranged when docked, but do not enforce any particular arrangement. It is up to the item that holds the docker to adhere to the hints returned by both this method and GetSpanStyle.

The span fill modifier flags modify the span style set by SetSpanStyle; they change its behavior. For instance, in a horizontal dock item, a docker with the vdockerSPAN_BREADTH span style will grow to span all available breadth not occupied by other dockers, but a docker that also has the span fill modifier vdockerSPAN_FILL_X flag set will grow to fill up an entire row, forcing other dockers onto different rows.

Dock items that hold dockers call this method to determine if a docker should be given an entire row (or column, depending on the dock item's orientation).

The span fill modifiers only have meaning for a docker that has set one or both of the span style flags:

  • vdockerSPAN_X_WHEN_HORIZONTAL
  • vdockerSPAN_Y_WHEN_VERTICAL.

The value returned from this method is a bitwise OR combination of zero or more of the following flags:

vdockerSPAN_FILL_X
The docker should fill the entire row when it is docked horizontally and has the span style flag vdockerSPAN_X_WHEN_HORIZONTAL set.
vdockerSPAN_FILL_Y
The docker should fill the entire column when it is docked vertically and has the span style flag vdockerSPAN_Y_WHEN_VERTICAL set.

Additionally the following constants are defined for convenience:

vdockerSPAN_FILL_NONE
A constant that has neither span fill flag set.
vdockerSPAN_FILL
Equal to the bitwise OR of vdockerSPAN_FILL_X and vdockerSPAN_FILL_Y.

vdockerGetSpanStyle

        int vdockerGetSpanStyle(
                vdocker * docker);
Returns the span style flags for the docker. This method is typically called by a docker-holding item such as a dock item to determine how best to arrange its dockers. The span style flags serve as a hint as to how the docker would like to be arranged when docked, but do not enforce any particular arrangement. The span style flags do not apply to floating dockers. It is up to the item that holds the docker to adhere to the hints returned by both this method and GetSpanFillModifiers.

The term "span" refers to how a docker fills space. Two different spanning behaviors can seen in dockers docked in dock items: spanning perpendicular to the docking axis (along the y axis when docked horizontally, or along the x axis when docked vertically) means that the row or column the docker is in will be resizable perpendicular to the docking axis.

Spanning along the same axis as the docking axis (along the x axis when docked horizontally, or along the y axis when docked vertically) means that the docker will stretch out along the axis to fill up space not occupied by other dockers.

If two or more dockers that span parallel to the docking axis are on the same row, they will be resizable along that axis.

The value returned from this method is a bitwise OR combination of zero or more of the following flags.

vdockerSPAN_X_WHEN_HORIZONTAL
When the docker is docked horizontally, it should increase its width to fill up excess space in its row.
vdockerSPAN_X_WHEN_VERTICAL
When the docker is docked vertically, it should have a resizable width.
vdockerSPAN_Y_WHEN_HORIZONTAL
When the docker is docked horizontally, it should have a resizable height.
vdockerSPAN_Y_WHEN_VERTICAL
When the docker is docked vertically, it should increase its height to fill up excess space in its column.

Additionally the following constants are defined for convenience:

vdockerSPAN_NONE
A constant that has none of the span style flags set.
vdockerSPAN_HORIZONTAL
Span along the x axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_X_WHEN_HORIZONTAL and vdockerSPAN_X_WHEN_VERTICAL.
vdockerSPAN_VERTICAL
Span along the y axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_Y_WHEN_HORIZONTAL and vdockerSPAN_Y_WHEN_VERTICAL.
vdockerSPAN_BREADTH
Span along the docking axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_X_WHEN_HORIZONTAL and vdockerSPAN_Y_WHEN_VERTICAL.
vdockerSPAN_DEPTH
Span perpendicular to the docking axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_Y_WHEN_HORIZONTAL and vdockerSPAN_X_WHEN_VERTICAL.

vdockerLoad

        vdocker * vdockerLoad(
                vresource resource);
Reads a docker from the resource resource and returns a pointer to its contents. Storage for the docker object is allocated by the Docker class. The docker's class is determined by the value of the "Class" tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function creates an object based on the class of the resource, even if this class is not a docker class. The object pointer is cast to a docker pointer, even if the object represents a different data type. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attributes of the default docker class that are stored in resources are documented in the Docker chapter of the Galaxy Programmer's Guide. This function essentially allocates space for a new instance and calls vdockerLoadInit on it.

vdockerLoadInit

        void vdockerLoadInit(
                vdocker * docker,
                vresource resource);
Reads a docker from the resource resource and stores it in the uninitialized memory addressed by docker. The docker's class is determined by the value of the "Class" tag. It generates vobjectUndefinedClassException if the name does not denote a class in the Class Manager registry. Note that this function initializes an object based on the class of the resource, even if this class is not a docker class. The object attempts to initialize itself in the docker structure provided, even if it is not of the correct size. Attributes may be initialized based on the contents of the resource, or they may be set to default values, depending on the class. The attribute table in the Docker chapter of the Galaxy Programmer's Guide documents the attributes of the default docker class that are stored in resources. This function stores the class specified by the resource in the class field of the docker structure and sends it the message vobjectLOAD_INIT.

Span along the docking axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_X_WHEN_HORIZONTAL and vdockerSPAN_Y_WHEN_VERTICAL.
vdockerSPAN_DEPTH
Span perpendicular to the docking axis regardless of docking orientation. Equal to the bitwise OR of vdockerSPAN_Y_WHEN_HORIZONTAL and vdockerSPAN_X_WHEN_VERTICAL.

vdockerStore

        void vdockerStore(
                vdocker * docker,
                vresource resource);
Writes the contents of the docker into the resource. The docker can be recovered from the resource at a later time (perhaps by another program) with vdockerLoad or _typeprefixLoadInit. Note that not all of the docker's attributes are stored in the resource; the attributes of the default docker class that are stored in resources are documented in the Docker chapter of the Galaxy Programmer's Guide. It generates vobjectUnnamedClassException if the docker's class is not named. This function is implemented by sending the message vobjectSTORE to the docker.

To store a docker in a resource, first call vresourceMake to create an untyped resource in the desired location, then call vdockerStore to write the docker into the resource. For example: 


vdocker  *docker;    /* contains a docker */
vresource fileResource, dockerResource;

fileResource   = vresourceOpenFile( ... );
dockerResource = vresourceMake(fileResource, 
			       vnameInternGlobalLiteral("Mydocker"));

vdockerStore(docker, dockerResource);
vresourceCloseFile(fileResource);

Messages

vdockerCLOSE

        <vdockerCLOSE>
Closes the docker by either closing its dialog if it is floating, or by closing its dialog item if it is docked. Refer to the sections vdialogClose and vdialogCloseItem for more information about closing dialogs and dialog items.

The function vdockerCLOSE is implemented by sending this message to the docker.

vdockerCREATE_DRAG

        <vdockerCREATE_DRAG>
Creates a drag of the appropriate subclass for drag and drop with the docker. This method is called to create a drag object when a user drags the pointer inside the docker.

The default implementation of this method creates a drag of class vdockerDragClass. Subclasses may override this method to return a subclass of vdockerDrag.

vdockerDETERMINE_DRAG_DRAG_RECT

        <vdockerDETERMINE_DRAG_DRAG_RECT>
Returns the appropriate drag outline rectangle for this docker drag and the given dock. The determined drag rectangle is returned through the rrect parameter. The default implementation of this method calls vdockerDetermineDragRect on the drag's docker to do the work.

This function is implemented by sending the message vdockerDETERMINE_DRAG_DRAG_RECT to the docker drag.

The function vdockerDetermineDragDragRect is implemented by sending this message to the docker drag.

vdockerDETERMINE_DRAG_RECT

        <vdockerDETERMINE_DRAG_RECT>
Returns the drag rectangle for this docker appropriate to the given drag and dock. This rectangle approximates the dimensions the docker would have if it were to dock in the vdock specified. The drag rectangle is returned through the rrect parameter.

This method is called from vdockerDetermineDragDragRect to perform the actual job of determining the drag rectangle. This method exists so that it is not necessary to create a new vdockerDrag subclass just to modify a docker's drag rectangle.

The function vdockerDetermineDragRect is implemented by sending this message to the docker.

vdockerNOTIFY_DRAG_DROP

        <vdockerNOTIFY_DRAG_DROP>
Called when a docker drag is dropped. The dock parameter is the dock over which the docker was dropped and to which it was added, or NULL if the docker was not added to a dock. This method can be overridden if notification of the drop is desired.

The function vdockerNotifyDragDrop is implemented by sending this message to the docker drag.

vdockerOPEN

        <vdockerOPEN>
Opens a docker by either opening the docker's dialog if the docker is floating or by opening the docker's dialog item if the docker is docked. Refer to sections GetDialog and GetItem for more information.

The function vdockerOpen is implemented by sending this message to the docker.

vdockerQUERY_DRAG_ENTER_DOCK

        <vdockerQUERY_DRAG_ENTER_DOCK>
Tests whether the dock is a valid drop site for the docker drag. Returns TRUE if it is, FALSE if not. By default, any dock whose owner is a vdockitem is valid.

The function vdockerQueryDragEnterDock is implemented by sending this message to the docker drag.

vdockerSET_DOCK

        <vdockerSET_DOCK>
Sets the dock with which the docker is associated. If dock is NULL then the docker is dissociated from all docks and will be closed if it is floating. The docker's associated dock is the one in which it is currently docked, or the one in which it will dock if its floating attribute is set to FALSE. Refer to SetFloating for more information.

This method is called as a side effect of vdockAddDocker and vdockRemoveDocker, which are the recommended entry points for manipulating dockers' connections to docks. Programmers typically do not need to call this method directly.

The function vdockerSetDock is implemented by sending this message to the docker.

vdockerSET_DRAG_DOCKER

        <vdockerSET_DRAG_DOCKER>
Sets the docker that this drag is dragging.

The function vdockerSetDragDocker is implemented by sending this message to the docker drag.

vdockerSET_DRAG_RECT

        <vdockerSET_DRAG_RECT>
Sets the current rectangle of the drag.

The function vdockerSetDragRect is implemented by sending this message to the docker drag.

vdockerSET_FLOATING

        <vdockerSET_FLOATING>
Sets the floating state of the docker. If floating is TRUE then the docker will be undocked, and if floating is FALSE then the docker will be docked. This method has no effect if the docker has been locked by a call to SetLocked.

The function vdockerSetFloating is implemented by sending this message to the docker.

vdockerSET_HELD_ITEM

        <vdockerSET_HELD_ITEM>
Sets the item held by the docker. A docker does not have an item list in which it holds its contents; instead, it holds a single dialog item of arbitrary type. If more than one item is needed inside the docker, a vcontainer or other dialog item with an item list can be used as the docker's held item, and that container can hold the desired items.

Any item previously held by the docker will be removed from the docker and should be destroyed by the caller when no longer needed.

The function vdockerSetHeldItem is implemented by sending this message to the docker.

vdockerSET_LOCKED

        <vdockerSET_LOCKED>
Locks or unlocks the floating state of the docker. If lockedTRUE the docker is locked in its current floating state, if FALSE docking and undocking are enabled.

The function vdockerSetLocked is implemented by sending this message to the docker.

vdockerSET_REMEMBERED_RECT

        <vdockerSET_REMEMBERED_RECT>
Sets one of the docker's remembered rectangles. The remembered rectangles are used when a docker's floating state changes or when it is dragged about. For instance suppose a docker is moved from a horizontal dock to a vertical dock. If the docker is then dragged back over to the horizontal dock, the remembered "docked horizontal" rectangle will be used for drawing the drag outline, and will be used as the rectangle for the docker if it is dropped over the horizontal dock. Possible values for whichRect are:
  • vdockerFLOATING_RECT
  • vdockerDOCKED_HORIZONTAL_RECT
  • vdockerDOCKED_VERTICAL_RECT

The methods SetDockedHorizontalRect, SetDockedHorizontalRect, and SetFloatingRect are implemented by a sending this message with the appropriate value for whichDock to the docker.

vdockerSET_SPAN_FILL_MODIFIERS

        <vdockerSET_SPAN_FILL_MODIFIERS>
Sets the span-filling modifier flags for the docker.

The function vdockerSetSpanFillModifiers is implemented by sending this message to the docker.

vdockerSET_SPAN_STYLE

        <vdockerSET_SPAN_STYLE>
Sets the span style flags for the docker.

The function vdockerSetSpanStyle is implemented by sending this message to the docker.

Macros

vdockerCLASS

        #define vdockerCLASS(...) ...
Contains the method and variable declarations for docker classes. This macro expands to the contents of vdockerClass. SUPER is the type identifier of the class being declared (for example, vdockerClass). This type is used by the class's subclasses to refer to it. DOCKER is the type identifier of instances of the class being declared (for example, vdocker). NOTIFY is the type identifier of the instance notification function (for example, vdockerNoteProc). DRAG is the type identifier of the drag type associated with this docker class (for example, vdockerDrag). This macro should be used by docker subclasses to declare the contents of the docker class. This macro should come first in the subclass structure declaration.
