Types

vdrawpath

        typedef ... vdrawpath;
Provides an interface to create and manipulate a sequence of line and curve segments. The line and curve segments can be outlined, filled or used as a border for a clipping region.

vdrawpathClass

        typedef ... vdrawpathClass;
Contains methods and variables bindings for drawing paths.

vdrawpathAppendArcClockwise

        void vdrawpathAppendArcClockwise(
                vdrawpath * path,
                double cx,
                double cy,
                double radius,
                double startAngle,
                double endAngle);
The arc is centered around the specified point and has the specified radius. Only the clockwise segment of the arc from the start angle to the end angle is added to the path. If the current point for the path is defined, a segment connects the path's current point to the starting point of the arc, which is determined by the center point, radius and start angle. The current point of the path is moved to the end point of the arc.

vdrawpathAppendTangentArc

        void vdrawpathAppendTangentArc(
                vdrawpath * path,
                double x1,
                double y1,
                double x2,
                double y2,
                double radius);
Appends an arc segment to the path. The arc is tangent to the line segments connecting the specified points. The arc has the specified radius and is positioned tangent to the line segment connecting the path's current point to x1,y1 and to the line segment connecting x1,y1 to x2, y2. The end points of the arc are the two points that are radius in a perpendicular direction from each line segment.

A line segment connects the path's current point to the end point of the arc along the line segment connecting the path's current point to x1, y1. The current point is moved to the other end point of the arc. This function is useful for drawing rounded corners at the intersection of two lines.

vdrawpathClone

        vdrawpath * vdrawpathClone(
                vdrawpath * path);
Creates a copy of the drawing path path and returns a pointer to its contents.The Draw Manager reclaims the storage occupied by the drawing path when it is destroyed with vdrawpathDestroy. This function copies all the attributes of the drawing path, and it clones any attribute objects that are owned by the path. The draing path to be cloned can be dynamic or static. This function essentially allocates space for a new drawing path and calls vdrawpathCopyInit on it.

vdrawpathClose

        void vdrawpathClose(
                vdrawpath * path);
Adds a line segment from the current point of the path to the start of its current subpath. There is no effect if the current point is undefined.

vdrawpathCopy

        void vdrawpathCopy(
                vdrawpath * path,
                vdrawpath * target);
Copies the information in path into target so that the two drawing paths represent equivalent objects. target must have been created or initialized by an Draw Manager function. This function copies all the attributes of the path, and it clones any attribute objects that are owned by the drawing path. This function destroys any objects allocated internally by target (including its owned attributes) but does not destroy target itself. The path to be copied can be dynamic or static.

vdrawpathCopyInit

        void vdrawpathCopyInit(
                vdrawpath * path,
                vdrawpath * target);
Copies the information in path into target so the memory addressed by target is a drawing path equivalent to path. target should be a pointer to uninitialized memory; the Draw Manager initializes it as part of the copy. This function copies all the attributes of the path and clones any attribute objects that are owned by the path.

vdrawpathCreate

        vdrawpath * vdrawpathCreate();
Creates a new drawing path and returns a pointer to its contents. Storage for the path is allocated by the Draw Manager. The Draw Manager reclaims the storage occupied by the path when it is destroyed with vdrawpathDestroy. The new path is created with a set of default attributes, as documented in the ``Draw Manager'' chapter of the Galaxy Application Environment Programmer's Guide. This function essentially allocates space for a new drawing path and calls vdrawpathInit on it.

vdrawpathCreateOfClass

        vdrawpath * vdrawpathCreateOfClass(
                vdrawpathClass * clas);
Creates a draw path of class clas and returns a pointer to its contents. The attributes of the path are set to their default values, according to the class. Space for the path is allocated dynamically based on the object size of the class. The Class Manager will reclaim the storage occupied by the path when it is destroyed with vdrawpathDestroy. This function essentially allocates space for a new instance and calls vdrawpathInitOfClass on it.

vdrawpathDestroy

        void vdrawpathDestroy(
                vdrawpath * path);
Destroys the drawing path represented by path, which must have been created or initialized by an Draw Manager function. Use this function to reclaim the storage occupied by a path. The path should not be referenced by any other object when it is destroyed. This function attempts to free the storage occupied by path only if it was allocated by vdrawpathCreate, vdrawpathCreateOfClass, or vdrawpathClone. This function also destroys any attribute objects that are owned by the drawing path.

vdrawpathFlatten

        void vdrawpathFlatten(
                vdrawpath * path);
Replaces all curve segments of the path with line segment approximations. The flatness attribute of the path determines the accuracy of the flattened path. A small flatness value may produce a very large path. There is no effect if the path does not have any curve segments.

vdrawpathInit

        void vdrawpathInit(
                vdrawpath * path);
Initializes a drawing path in the memory addressed by path. The attributes of the path are set to their default values, as documented in the ``Draw Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The address can be used in other calls to the Draw Manager as if it were created with vdrawpathCreate. A call to vdrawpathDestroy frees any internal objects allocated by the Draw Manager but does not free the structure itself. A map is automatically initialized for the path.

vdrawpathInitOfClass

        void vdrawpathInitOfClass(
                vdrawpath * path,
                vdrawpathClass * clas);
Initializes the uninitialized memory addressed by path so that it becomes a dawing path of class clas. The attributes of the path are set to their default values, according to the class. The Draw Manager will not attempt to reclaim the path's storage when it is destroyed. This function simply stores the class in the drawing path and sends it the message vinstanceINIT.

Macros

vdrawpathCLASS

        #define vdrawpathCLASS(...) ...
Contains the method and variable declarations for drawing path classes. This macro expands to the contents of vdrawpathClass. CLASS is the type identifier of the class being declared (for example, vdrawpathClass). This type is used by the class's subclasses to refer to it. PATH is the type identifier of instances of the class being declared (for example, vdrawpath). This macro should be used by drawing path subclasses to declare the contents of the drawing path class. This macro should come first in the subclass structure declaration.
