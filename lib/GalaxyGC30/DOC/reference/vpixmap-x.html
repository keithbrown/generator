Types

vpixmap

        typedef ... vpixmap;
Contains the information needed to represent a pixmap. Pixmaps are referenced by a pointer to this type. Pixmaps can be created dynamically with vpixmapCreate or initialized statically with vpixmapInit. A vpixmap contains a set of size attributes, a set of format attributes, and a set of buffers.

A vpixmap implements two-dimensional arrays of unmapped pixel values on top of buffers. Pixmap pixel values can be used to represent any application data; they are unmapped in the sense that they are not given any color value by the pixmap. The color values must come from an outside source, such as an image or a color palette.

vpixmapClass

        typedef ... vpixmapClass;
Contains the message definitions for a class of pixmaps. This class type is derived from vobjectClass.

vpixmapDimension

        typedef ... vpixmapDimension;
Denotes an integral type that can hold a width or height of a pixmap. Note that pixmap dimensions are specified through the Pixmap Library API using the type int, but they are truncated internally, if necessary, to the type vpixmapDimension.

vpixmapScanner

        typedef ... vpixmapScanner;
Denotes an abstract iteration variable that can be used to access the pixels of a pixmap with a format-independent API. Pixmap scanners are not inefficient, but they are also not as efficient as writing code specific code for a particular format. Pixmap scanners are usually initialized in auto variables and referenced by a pointer to this type.

Note that moving a scanner one pixel in any given direction is generally more efficient than moving it to a specific (x, y) location. You can read or write the current pixel of a pixmap and optionally increment the pixmap pointer after the read or write operation. You can also read or write any pixel in the pixmap given its (x, y) location without moving the current pixel. Accessing a pixel in this manner is more efficient than moving the scanner to a specific (x, y) position and reading or writing the current position, but it is less efficient than reading or writing the current position and moving one pixel in a given direction. Your application dictates which interface you should use. If the access pattern of your application is linear (up, down, left, right), then you should access pixels through the scanner's current pixel. If the access pattern is non-linear or random, then you should access pixels by their (x, y) position.

vpixmapScannerClass

        typedef ... vpixmapScannerClass;
Contains the information needed to represent a pixmapScanner. PixmapScanners are referenced by a pointer to this type. PixmapScanners can be created dynamically with vpixmapScannerCreate or initialized statically with vpixmapScannerInit. A vpixmapScanner contains a set of size attributes, a set of format attributes, and a set of buffers.

A vpixmapScanner is used to iterate acrossed pixel values (so called "bits"). PixmapScanner pixel values can be used to represent any application data; they are unmapped in the sense that they are not given any color value by the pixmapScanner. The color values must come from an outside source, such as an image or a color palette.

vpixmapValue

        typedef ... vpixmapValue;
Denotes an integral type that can hold a pixel value of a pixmap. The legal range of pixel values actually ranges from zero to 2^depth-1, where depth is the depth of the pixmap. Note that if the pixmap is the map of an image, the range of legal values may be even smaller.

vpixmapClear

        void vpixmapClear(
                vpixmap * pixmap,

vpixmapCopyBits

        void vpixmapCopyBits(
                vpixmap * pixmap,
                vpixmap * target);
Copies all the pixels from the given pixmap pixmap to the pixmap target. This function does not affect any other attributes of the pixmap. It generates a vexArgException exception if the width and height of the source pixmap do not match the width and height of the target pixmap. Use vpixmapCopyRect to copy pixels between pixmaps of different sizes. The pixmap depths and formats need not match, but the copy may be more efficient if they do. If the depth of the source pixmap is greater than the depth of the target pixmap, pixel values are masked silently. This function is more efficient than copying pixel values individually using a pixmap scanner. Note that calling this function may require the window system representation of the target pixmap to be reconstructed when the pixmap is drawn.

vpixmapCopyRect

        void vpixmapCopyRect(
                vpixmap * pixmap,
                vpixmap * target,
                int x,
                int y,
                int w,
                int h,
                int targetX,
                int targetY);
Copies the pixels from rectangle (x, y, w, h) of the pixmap pixmap into the rectangle (x, y, w, h) of the pixmap target. This function does nothing if w or h is zero. Pixels outside the target rectangle are unaffected. Pixmap rectangles are similar to drawing coordinates: (0, 0, 1, 1) encloses the pixel in the lower left-hand corner of a pixmap, while (0, 0, w, h) encloses the entire pixmap. It generates a vexArgRangeException exception if either rectangle encloses an area outside the given pixmap. The pixmap depths and formats need not match, but the copy may be more efficient if they do. If the depth of the source pixmap is greater than the depth of the target pixmap, pixel values are masked silently. This function may be more efficient than copying pixel values individually using a pixmap scanner. Note that calling this function may require the window system representation of the target pixmap to be reconstructed when the pixmap is drawn.

vpixmapCreate

        vpixmap * vpixmapCreate(
                int w,
                int h,
                int depth);
Creates a pixmap of width w, height h, and depth depth and returns a pointer to its contents. The pixmap can contain w*h pixels, each of which can store a value from zero to 2^depth-1. Storage for the pixmap is allocated by the Pixmap Library. The Pixmap Library reclaims the storage occupied by the pixmap when it is destroyed with vpixmapDestroy. The new pixmap is created with a set of default attributes, as documented in the ``Image Manager'' chapter of the Galaxy Application Environment Programmer's Guide. The pixels in the pixmap are initially undefined (the pixmap contains no buffers). This function essentially allocates space for a new pixmap and calls vpixmapInit on it.

vpixmapDestroy

The scanner pointer should not be used until it is initialized again. Note if you want to destroy the pixmap a scanner is iterating over, you must destroy the scanner first, then the pixmap.

vpixmapGetBufferDevice

        const vbufferDevice * vpixmapGetBufferDevice();
Returns the buffer device for the source representation of a pixmap. This is the device the Pixmap Library uses to identify its source representation. The source representation is the contents that you specify for a pixmap, whether through the Pixmap Library API or an interactive tool like the Galaxy Resource Builder. Other pixmap representations, such as the pixmaps window system representation, may be generated from the source representation on demand. The source representation may even be regenerated from another representation if the other representation is changed directly (through drawing, for example). You only need to call this function if you are building a buffer device. Refer to ``Buffers'' in the ``Image Manager'' chapter of the Galaxy Application Environment Programmer's Guide for more information. Use vpixmapGetBits to get a pointer to a given pixmaps source contents.

vpixmapGetScannerBits

        unsigned char * vpixmapGetScannerBits(
                vpixmapScanner * scanner);
Returns the base address of the pixels being iterated by the scanner scanner. This is the base address of the first scan line if the pixmap is stored in y-increasing order; it is the base address of the last scan line if the pixmap is stored in y-decreasing order. The base address is set in this manner to ensure that the following expression evaluates to the base address of the scan line y for all pixmap formats. vpixmapGetScannerBits(scanner)+y*vpixmapGetScannerIncrementY(scanner)

vpixmapGetScannerIncrementY

        long vpixmapGetScannerIncrementY(
                vpixmapScanner * scanner);
Returns the y increment of the pixmap scanner scanner. The absolute value of the y increment of a scanner is equal to the character width of the pixmap the scanner is iterating (refer to vpixmapGetCharWidth). The y increment is positive if the pixmap is stored in y-increasing order, negative if the pixmap is stored in y-decreasing order. The y increment can be used to advance a scan line pointer independently of the format of a pixmap. Note that the following expression evaluates to the base address of the scan line y for all pixmap formats. vpixmapGetScannerBits(scanner)+y*vpixmapGetScannerIncrementY(scanner)

vpixmapMoveLeft

        void vpixmapMoveLeft(
                vpixmapScanner * scanner);
Moves the current pixel of the scanner scanner one pixel to the left. If the current pixel reaches the left edge of the pixmap, the current pixel is moved to the far right of the next scan line up. If there are no more scan lines left in the pixmap, the current pixel is moved to an indeterminate pixel. This function does not read or modify any pixels itself. This function is more efficient than calling vpixmapMoveTo with the coordinates of the pixel to the left of the current pixel.

vpixmapReadAt

        vpixmapValue vpixmapReadAt(
                vpixmapScanner * scanner,
                int x,
                int y);
Reads the value of the pixel at position (x, y) in the pixmap being iterated by the scanner scanner without moving the scanners current pixel. The pixmap value is in the range zero to 2^depth-1 where depth is the depth of the pixmap. This function is useful when you want to iterate over the contents of a pixmap, but the pattern of the iteration is not linear. This function is more efficient than calling vpixmapMoveTo and vpixmapRead in combination. For example, the code in Example averages the values of 100 random pixels from a pixmap (this code assumes that the sum of the values will not overflow a vpixmapValue).

Sample Code: Averaging the values of 100 random pixels from a pixmap

vpixmap              *pixmap;    /* contains a pixmap */
vpixmapScanner        scanner;
register vpixmapValue sum;
register int          width, height, count;

vpixmapInitScanner(&scanner, pixmap);

sum    = 0;
width  = vpixmapGetWidth(pixmap);
height = vpixmapGetHeight(pixmap);
count  = 100;

while (count-->0)
    sum += vpixmapReadAt(&scanner, rand()%width, rand()%height);

vpixmapDestroyScanner(&scanner);

return sum/100;

vpixmapReadMoveRight

        vpixmapValue vpixmapReadMoveRight(
                vpixmapScanner * scanner);
Reads the value of the current pixel of the scanner scanner and moves the current pixel one pixel to the right. If the current pixel reaches the right edge of the pixmap, the current pixel is moved to the far left of the next scan line up. If there are no more scan lines left in the pixmap, the current pixel is moved to an indeterminate pixel. The pixmap value is in the range zero to 2^depth-1 where depth is the depth of the pixmap. Calling this function repeatedly is the most efficient way to read all the pixels of a pixmap using a scanner. For example, the code in Example determines the maximum value of all the pixels in a pixmap.

Sample Code: Determining the maximum values of all the pixels in a pixmap\nopagebreak \nobreak 

vpixmap              *pixmap;    /* contains a pixmap */
vpixmapScanner        scanner;
register vpixmapValue max, value;
register int          count;

vpixmapInitScanner(&scanner, pixmap);

max   = 0;
count = vpixmapGetWidth(pixmap)*vpixmapGetHeight(pixmap);
while (count-->0)
    if ((value = vpixmapReadMoveRight(&scanner))>max)
        max = value;

vpixmapDestroyScanner(&scanner);

return max;

vpixmapSetBigEndianByte

        void vpixmapSetBigEndianByte(
                vpixmap * pixmap,
                int flag);
Changes the byte order of multi-byte pixels of the pixmap pixmap to big-endian if flag is TRUE or little-endian if flag is FALSE. Byte order affects how quickly a multi-byte pixel can be accessed in a pixmap. The pixmap's pixel values are converted to the new format. Note that calling this function may require the window system representation of the pixmap to be reconstructed when the pixmap is drawn. vpixmapSetDepth

        void vpixmapSetDepth(
                vpixmap * pixmap,
                int depth);
Changes the depth of the pixmap pixmap to depth. Pixmap values can range from zero to 2^depth-1. If depth is not a depth implemented by the Pixmap Library, then it is promoted to the next highest depth. The Pixmap Library implements depths of 1, 2, 4, 8, 16, 24, and 32 in this release of Galaxy. It generates a vexArgRangeException exception if depth is less than one or greater than 32. This function has no effect if the depth does not change after any necessary promotion. The pixmap's pixel values are converted to the new depth. If the new depth is less than the old depth, pixel values are masked silently. Note that calling this function may require the window system representation of the pixmap to be reconstructed when the pixmap is drawn.

vpixmapSetInverseY

        void vpixmapSetInverseY(
                vpixmap * pixmap,
                int flag);
Changes the scan line order of the pixmap pixmap to y-decreasing if flag is TRUE or y-increasing if flag is FALSE. Although Galaxy's rendering model is based on a lower left-hand coordinate origin, most common pixmap formats are based on an upper left-hand coordinate origin. The Pixmap Library accommodates these formats by storing their scan lines in y-decreasing order. Note that this function does not affect the logical origin of the pixmap: the logical origin is always in the lower left-hand corner. The pixmap's pixel values are converted to the new format by this function. Note that calling this function may require the window system representation of the pixmap to be reconstructed when the pixmap is drawn. Calling this function constitutes specifying a format for the pixmap: if the pixmap is stored in a resource, it may not use the most efficient format for a given host platform when it is read from the resource.

        void vpixmapSetSize(

vpixmapPRESERVE_VERT_CENTER

vpixmapPRESERVE_VERT_BOTTOM

vpixmapPRESERVE_CLEAR_EXPOSED

vpixmapStore

        void vpixmapStore(
                vpixmap * pixmap,
                vresource resource);
Writes the contents of the pixmap pixmap into the resource resource. The pixmap can be recovered from the resource at a later time (perhaps by another program) with vpixmapLoad or vpixmapLoadInit . All of the pixmaps attributes, including its pixels, are stored in the resource. The Pixmap Library also stores whether the pixmap is in the default (most efficient) format or whether a format was specified explicitly. You should use the default pixmap format unless you have a compelling reason not to do so.

To make a pixmap into a resource, first call vresourceMake to create an untyped resource in the desired location, then call vpixmapStore to write the pixmap into the resource. This is illustrated in Example.

Sample Code: Making a pixmap into a resource

vpixmap  *pixmap;    /* contains a pixmap */
vresource fileResource, pixmapResource;

fileResource   = vresourceOpenFile( ... );
pixmapResource = vresourceMake(fileResource,
                               vnameInternGlobalLiteral("MyPixmap"));

vpixmapStore(pixmap, pixmapResource);
vresourceCloseFile(fileResource);

vpixmapWrite

        void vpixmapWrite(
                vpixmapScanner * scanner,
                vpixmapValue value);
Reads the value value into the current pixel of the scanner scanner without moving the scanners current pixel. If the value is outside the legal range of values for the pixmap being iterated by the scanner, then the value is masked silently.

vpixmapWriteAt

        void vpixmapWriteAt(
                vpixmapScanner * scanner,
                int x,
                int y,
                vpixmapValue value);
Writes the value value into the pixel at position (x, y) in the pixmap being iterated by the scanner scanner without moving the scanners current pixel. If the value is outside the legal range of values for the pixmap being iterated by the scanner, the value is masked silently. This function is useful when you want to iterate over the contents of a pixmap, but the pattern of the iteration is not linear. This function is more efficient than calling vpixmapMoveTo and vpixmapWrite in combination. For example, the code in Example zeroes 100 random pixels of a pixmap.

Sample Code: Zeroing 100 random pixels in a pixmap

vpixmap       *pixmap;    /* contains a pixmap */
vpixmapScanner scanner;
register int   width, height, count;

vpixmapInitScanner(&scanner, pixmap);

width  = vpixmapGetWidth(pixmap);
height = vpixmapGetHeight(pixmap);
count  = 100;
while (count-->0)
    vpixmapWriteAt(&scanner, rand()%width, rand()%height,
                   (vpixmapValue)0);

vpixmapDestroyScanner(&scanner);

vpixmapWriteMoveRight

        void vpixmapWriteMoveRight(
                vpixmapScanner * scanner,
                vpixmapValue value);
Writes the value value at the current pixel of the scanner scanner and moves the current pixel one pixel to the right. If the current pixel reaches the right edge of the pixmap, the current pixel is moved to the far left of the next scan line up. If there are no more scan lines left in the pixmap, the current pixel is moved to an indeterminate pixel. If the value is outside the legal range of values for the pixmap being iterated by the scanner, the value is masked silently. Calling this function repeatedly is the most efficient way to write the pixels of a pixmap using a scanner. For example, the code in Example sets each pixel in a pixmap to a unique value.

Sample Code: Setting each pixel in a pixmap to a unique value

vpixmap       *pixmap;    /* contains a pixmap */
vpixmapScanner scanner;
register int   count;

vpixmapInitScanner(&scanner, pixmap);

count = vpixmapGetWidth(pixmap)*vpixmapGetHeight(pixmap);

while (count-->0)
    vpixmapWriteMoveRight(&scanner, (vpixmapValue)count);

vpixmapDestroyScanner(&scanner);
