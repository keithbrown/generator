vdialog

        typedef ... vdialog;
Contains the information needed to represent a single dialog; objects of this type have classes derived from vdialogClass. A dialog is a top-level window that contains a list of dialog items. Dialogs are referenced by a pointer to this type. Dialogs can be created dynamically with vdialogCreate, or initialized statically with vdialogInit. Dialogs can be created dynamically or loaded from resource files by using the appropriate constructor.

A vdialog contains a window, a set of dialog attributes, and a dialog item list. Refer to the ``Dialog Manager'' chapter of the Galaxy Application Environment Programmer's Guide for a complete description of dialog attributes. All object types with classes that directly inherit from vdialogClass must declare a vdialog as their first field so that vdialog pointers can be interchanged with pointers to the new data type.

vdialogItemListOwnerFuncs

        typedef ... vdialogItemListOwnerFuncs;
Contains functions that determine the interface of a dialog item list to the dialog or other object that owns it. owner is the object that owns the item list; it is set along with the function block itself in the item list with the function vdialogSetItemListOwner. The Dialog class calls these functions when it needs to determine information about an item list that is not provided by the item list itself. flags indicates which fields contain valid function pointers; it should be set to zero if DetermineShown and DetermineEnabled are not provided, or vdialogITEM_LIST_OWNER_STATE_FUNCS if all fields are provided. Most function fields implement functions that correspond to Dialog or Window Manager entry points. For example, DetermineCursor corresponds to the Window class function entry point vwindowDetermineCursor.

DetermineDialog returns the dialog that ultimately owns the item list, even if it must traverse several containing objects to do so; it returns NULL if the item list is not in any dialog. DetermineCursor returns the cursor specified for the owning object or any of its parent objects, or NULL if the window system default should be used. DetermineFont returns the font specified for the owning object or any of its parent objects; it should never return NULL. DetermineColorsFrom calculates color values based on the colors specified for the owning object or any of its parent objects; it should never return NULL.

Refer to the section vdialogDetermineItemColorsFrom for more information about color calculations. DetermineLineWidth returns the line width specified for the owning object or any of its parent objects. DetermineBounds returns the bounding box of the owning object in the coordinate system of its window in bounds. DetermineContent returns the visible portion of the owning object in the coordinate system of its window in content. GetKeyMap and GetKeyDispatch return the key map and key dispatcher of the owning object, respectively. DetermineShown returns whether or not the owning object is visible on the screen. DetermineEnabled returns whether or not the owning object responds to events.

vdialogCreateItemDrawcontext

        vdrawcontext * vdialogCreateItemDrawcontext(
                vdialogItem * item);
Creates a drawing context for drawing operations on the dialog item. Rendering operations on this drawing context result in marks on the item's interior.

Note that the coordinate system is based on the item's owner, not the item itself.

The device is automatically clipped to the visible portion of the item's owner. The view clip is disabled in the new context. This function sets the Flat and StrokeAdjust attributes of the current graphics state to the window context defaults (normally 1.0 and TRUE, respectively). The Color, Font, and LineWidth attributes of the context are set to the results of:

  • vdialogDetermineItemForeground
  • vdialogDetermineItemFont, and
  • vdialogDetermineItemLineWidth, respectively.

Call this function to create a drawing context when drawing is necessary outside of an invalidation/update event sequence. This should only be necessary when drawing inside a feedback loop; in other cases, invalidation should suffice. The drawing context based on the null device is returned if the dialog item has no owner or the owner is closed. Destroy the context created by this function when all drawing operations using it are complete.

Subclasses that override this method should first call the parent method to create the context, then perform any subclass-specific initialization. This function is implemented by sending the message vdialogCREATE_ITEM_DRAWCONTEXT to the dialog item.

vdialogInvalItemRect

        void vdialogInvalItemRect(
                vdialogItem * item,
                const vrect * rect,
                int flags);
Invalidates the part of the content area of the dialog owning the dialog item item specified by rect. This function has no effect if the item's dialog is closed. Coordinates are specified in display units relative to the lower left-hand corner of the dialog item's owner. The rectangle is clipped to the visible area of the item's owner. Invalidation indicates that the given area no longer reflects the correct appearance of the dialog item and that it should be redrawn as soon as possible.

The Window class normally responds to invalidation by accumulating the invalid area into the dialog window's invalid region and, if this is the first invalidation since the last update event, posting an update event to redraw the invalid region. Redrawing is deferred through the use of update events and invalid regions so that multiple invalid areas can be redrawn in a single step ; this minimizes ``flashiness'' and improves drawing performance in most cases. It is important to recognize that extra redrawing in a graphical user interface reduces the tactility of the interface components.

Note that you usually won't need to invalidate dialog items manually ; dialog item attribute modification functions automatically invalidate items when attributes are changed. flags specifies hints to the Window class about the invalidation; it should be the logical OR (|) of zero or one or more of the following flags:

vwindowINVAL_OPAQUE
Do not erase before re-drawing.
vwindowINVAL_IMMEDIATE
Post update event at normal priority.
vwindowINVAL_UPDATE
Update before returning.
vwindowINVAL_DISABLE
Do not invalidate.

Refer to the ``Window Manager'' chapter of the Galaxy Application Environment Programmer's Guide for more information about invalidation flags. This function is implemented by sending the message vdialogINVAL_ITEM_RECT to item.

Indicates to the dialog item item that it has lost the keyboard focus. The item is redrawn accordingly. This message has no effect if the item is already unfocused. The function entry point vdialogUnfocusItem is implemented by sending this message to the item.

The default method for the default dialog item class changes the Focused attribute of the item and calls vdialogDrawItemFocus on the item (with the current color set to the item's background color), if it is visible. Subclasses that override this method should first call vclassSendSuper to unfocus the dialog item, then update any subclass-specific state.

vdialogUNFOCUS_ITEM_LIST

        <vdialogUNFOCUS_ITEM_LIST>
Calls vdialogUnfocusItem on the current focus item in itemList. This function does nothing if there is no such item or if the item list is already unfocused. The function entry point vdialogUnfocusItemList is implemented by sending this message to the item list.

The default method for the default dialog item list class unfocuses the current focus item in the item list. Subclasses that override this method should first call vclassSendSuper to unfocus the dialog item list, then update any subclass-specific state.

vdialogCLASS

        #define vdialogCLASS(...) ...
Contains the method and variable declarations for dialog classes. This macro expands to the contents of vdialogClass. SUPER is the type identifier of the class being declared (for example, vdialogClass). This type is used by the class's subclasses to refer to it. DIALOG is the type identifier of instances of the class being declared (for example, vdialog). NOTIFY is the type identifier of the instance notification function (for example, vdialogItemNoteProc). ITEM is the type identifier of the items which will be held by instances of the dialog class (for example, vdialogItem). LIST is the type identifier of the item list which will be used to actually hold the items in instances of the dialog class (for example, vdialogItemList). This macro should be used by dialog subclasses to declare the contents of the dialog class. This macro should come first in the subclass structure declaration.
