vwebbrowse

        typedef ... vwebbrowse;
This abstract base class provides a consistent interface for controlling HTML browsers. Usually, HTML browsers exist as processes external to the application, but this need not be the case. Managing external browser processes is done through an instance of a subclass of vwebbrowseProcess.

Existing subclasses of vwebbrowse include vwebbrowseNetscape for controlling the Netscape Navigator browser,vwebbrowseExplorer for controlling the Microsoft Internet Explorer, and vwebbrowseDefault for controlling the system default browser. Note that the default browser is always Netscape Navigator on UNIX platforms.

Using vwebbrowse is typically easy because the API is very small. vwebbrowseGotoURL is the primary entry point. This call attempts to contact a running browser and request that it switch to the given URL or, if the browser is not currently running, start a new browser open to that URL.

When the application calls vwebbrowseGotoURL, the webbrowse typically calls vwebbrowseProcessGetState to determine whether or not it is necessary to start the browser process. If it is, the webbrowse will call vwebbrowseStart. If unnecessary, the webbrowse will proceed to contact the browser in a browser-specific manner.

When the webbrowse requests that the browser switch to the given URL, it also determines if the browser should also attempt to raise its window when it switches URLs. (See vwebbrowseSetAutoRaise). If possible, the message to raise the window will be encoded as part of the command to change URLs. If this cannot be done, the webbrowse calls vwebbrowseRaise.

vwebbrowseProcess

        typedef ... vwebbrowseProcess;
This abstract base subclass of vprocess adds the functionality listed below to the base class.
  • A way to associate the webbrowse with the webbrowse process.
  • A way to associate a startup URL with the webbrowse process.
  • An notification procedure.
Subclasses of vwebbrowseProcess must override vprocessGET_STATE to determine whether or not an existing browser process is running.

When a vwebbrowseProcess instance is destroyed, it does not kill the browser process. The user may be using the browser to view other resources so it is not appropriate for the application to shut down the browser.

vwebbrowseGotoURL

        vbool vwebbrowseGotoURL(
                vwebbrowse * browser,
                vurl * url);
This function is not implemented in this abstract base class. Subclasses of vwebbrowse must override this function to contact the browser in a browser-specific (and sometimes platform-specific) manner.

This call will normally result in a call to vwebbrowseProcessGetState to determine if a browser process is running. If one is not running, this function will call vwebbrowseStart; otherwise, it will attempt to communicate the command to load the given URL to the browser. If this communication is successful, this call returns vTRUE regardless of whether or not the browser can successfully load the URL.

These semantics are more consistent because it is not always possible or desirable to determine the success of loading the URL if, for instance. it takes a long time. In future releases, unsuccessful loads may be reported asynchronously via the observation procedure.

Users may prefer that the browser be raised to the top of the window when it switches URLs. This feature is controlled through the vwebbrowseSetAutoRaise function. If the webbrowse is set to automatically raise the browser, the raise command may either be encoded into the command to load the URL, or may be sent as a separate command. When sent as a separate command, this function should call vwebbrowseRaise.

This function is implemented by sending the message vwebbrowseGOTO_URL to browser.

Messages

vwebbrowseGOTO_URL

        <vwebbrowseGOTO_URL>
This message is not implemented in this abstract base class. Subclasses of vwebbrowse must override this message to contact the browser in a browser-specific (and sometimes platform-specific) manner.

This call will normally result in a call to vwebbrowseProcessGetState to determine if a browser process is running. If one is not running, this message will call vwebbrowseStart; otherwise, it will attempt to communicate the command to load the given URL to the browser. If this communication is successful, this call returns vTRUE regardless of whether or not the browser can successfully load the URL.

These semantics are more consistent because it is not always possible or desirable to determine the success of loading the URL if, for instance, it takes a long time. Unsuccessful loads may be reported asynchronously through the observation procedure.

Users may prefer that the browser to raise to the top of the window when it switches URLs. This feature is controlled through the vwebbrowseSetAutoRaise message. If the webbrowse is set to automatically raise the browser, the raise command may either be encoded into the command to load the URL, or may be sent as a separate command. When sent as a separate command, this message should call vwebbrowseRaise.
