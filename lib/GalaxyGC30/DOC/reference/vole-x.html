voleControlSite

        typedef ... voleControlSite;
Represents an OLE control site item object site for an ActiveX (OLE) control, which has its own defined behavior. The base voleControlSite is a dialog item which must be subclassed to be useful. In particular, a CLSID for a desired control type is associated with the control site object, at which point the object becomes a full OLE control container. Interaction with the OLE control happens through OLE automation method invocation, which is wrapped in the site's API, as well as through event notification, which is configurable through the API.

  • DISPATCH_METHOD
  • DISPATCH_PROPERTYGET
  • DISPATCH_PROPERTYPUT
  • DISPATCH_PROPERTYPUTREF.
The variant list is the list of parameters passed in to the invocation. Refer to the particular control's section for more details, and refer to sections for the helper functions for more information on those topics.
  • voleGetControlSiteProperty
  • volePutControlSiteProperty
  • voleInvokeControlSiteMethod
  • voleInvokeControlSiteSimpleMethod.
These are generally simpler to use. If the method invocation itself fails (as opposed to the function returning an unsuccessful result), a voleDispatchException will be thrown.

voleSetControlSiteDistributedObjectType

        void voleSetControlSiteDistributedObjectType(
                voleControlSite * site,
                voleObjectTypePtr type,
                const vchar * machineName);
Associates a particular class of ActiveX control with the control site and instantiates the control. The type parameter passed in is control's or COM/DCOM object's CLSID, as stored in the system registry, under HKEY_CLASSES_ROOT\CLSID. Setting the object type has the effect of loading the .ocx file (dll) and creating a live embedded control. If the control cannot be instantiated a voleException is thrown.

The machineName parameter is the name of machine on which to instantiate the DCOM object. Machine is named using the naming scheme of the network transport. By default, all UNC ("\\server" or "server") and DNS names ("server.com", "www.foo.com", or "135.5.33.19") names are allowed. This parameter may be NULL, in which case the object is instantiated on the current machine or at the machine specified under the class's RemoteServerName sub-key in the registry. Refer to the DCOM section documentation for more information.

Determines whether direct event notification is enabled. If it is enabled, then the voleNotifyControlSiteEventDirect method is called, if not the voleNotifyControlSiteEvent method is called instead. For performance reasons, direct notification may be useful in some cases.

With direct notification, parameters of the event are not stored into volevariantList, but passed directly in native DISPPARAMS array. Direct notification is set to FALSE by default.
